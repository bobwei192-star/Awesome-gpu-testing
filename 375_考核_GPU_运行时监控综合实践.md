# [考核] GPU 运行时监控综合实践

## 考核说明

- 本考核为 GPU 运行时监控的综合实践，总分 100 分
- 考核时间为 120 分钟
- 所有操作需在测试环境或虚拟机中完成
- 提交方式：将操作命令和输出截图整理为报告

## 第一部分：基础知识（25 分）

### 一、选择题（每题 2 分，共 10 分）

1. fdinfo 中 drm-memory-vram 字段的单位是：
   A. KB
   B. MB
   C. 字节
   D. 页

2. 以下哪个 sysfs 接口用于查看 GPU 利用率百分比：
   A. /sys/class/drm/card0/device/gpu_busy_percent
   B. /sys/class/drm/card0/device/mem_info_vram_used
   C. /sys/class/drm/card0/device/pp_dpm_sclk
   D. /sys/class/drm/card0/device/hwmon

3. umr --perf 中用于计算 L2 Cache 命中率的计数器是：
   A. gfx_BUSY 和 gfx_VALID
   B. l2_HIT 和 l2_MISS
   C. vram_READ 和 vram_WRITE
   D. cp_BUSY 和 cp_STALLED

4. 以下哪个指标指示可能存在 Draw Call 瓶颈：
   A. gfx_BUSY 高, gfx_VALID 高
   B. gfx_BUSY 低, cp_STALLED 高
   C. l2_HIT 高, vram_BW 低
   D. pcie_TX 高, pcie_RX 低

5. GEM BO 的完整信息可以通过以下哪个接口查看：
   A. /sys/class/drm/card0/device/mem_info_vram_used
   B. /sys/kernel/debug/dri/0/amdgpu_gem_info
   C. /proc/{pid}/fdinfo/{fd}
   D. /sys/kernel/debug/dri/0/amdgpu_regs

### 二、判断题（每题 2 分，共 10 分）

1. [ ] 当 GPU 利用率持续低于 50% 且 L2 Cache 命中率正常时，通常表示 GPU 是性能瓶颈。
2. [ ] VRAM 使用超过 90% 时可能导致显存内容被换出到系统内存，影响性能。
3. [ ] umr 的性能计数器采样会启用额外的 GPU 功能，因此会影响 GPU 性能。
4. [ ] 监测 GPU 内存泄漏时，应该在压力测试结束后比较 VRAM 使用量是否回落到初始水平。
5. [ ] GEM BO 的引用计数为 0 时，该对象会自动释放。

### 三、简答题（每题 5 分，共 10 分）

1. 解释以下性能计数器组合的诊断含义，并说明各自的解决方向：
   - gfx_BUSY=95%, l2_MISS=35%, vram_BW=380GB/s
   - gfx_BUSY=40%, cp_STALLED=12M/s, gfx_VALID=150MPixels/s
   - gfx_BUSY=30%, l2_MISS=5%, pcie_TX=12GB/s

2. 描述如何通过 fdinfo 和 amdgpu_gem_info 排查 VRAM 内存泄漏问题，写出完整的排查步骤。

## 第二部分：实践操作（45 分）

### 四、内存追踪实践（15 分）

1. （5 分）编写一个脚本（bash 或 Python），每隔 2 秒收集以下数据并记录到日志文件：
   - VRAM 使用量（总量、已用量）
   - GTT 使用量（总量、已用量）
   - GPU 利用率
   - GPU 温度

2. （5 分）运行 glmark2 或类似 GPU 压力测试，同时运行上述监控脚本。提供监控截图。

3. （5 分）压力测试结束后，分析日志文件中的峰值内存使用，并判断是否存在内存泄漏。

### 五、性能计数器分析（15 分）

1. （5 分）使用 umr --perf list 列出当前 GPU 支持的计数器，并选择至少 8 个有代表性的计数器。

2. （5 分）配置并运行 umr --perf 采集以下数据（采样时间 5 秒）：
   - GFX 利用率
   - L2 Cache 命中/未命中率
   - VRAM 带宽
   - CP 停顿计数
   - PCIe 传输带宽

3. （5 分）根据采集的数据分析当前系统的性能特征，判断是否存在瓶颈。

### 六、GEM 对象分析（15 分）

1. （5 分）查看 amdgpu_gem_info debugfs 输出，列出当前系统中最大的 5 个 GEM BO。

2. （5 分）选择一个正在运行 GPU 应用的进程，查看其 fdinfo，记录：
   - drm-engine-gfx 累计时间
   - drm-memory-vram 使用量
   - drm-memory-gtt 使用量

3. （5 分）解释这些 GEM BO 的可能用途（帧缓冲、纹理、顶点缓冲等）。

## 第三部分：综合案例分析（30 分）

### 七、性能问题诊断（15 分）

场景：某款游戏在 RX 7900 XTX 上的性能不佳（4K 分辨率，预期 60fps，实际 35fps）。

```bash
# 采集的性能数据:
GPU 利用率: 88%
VRAM 使用: 14.2GB / 24GB
L2 Cache 命中率: 68%
VRAM 带宽: 420 GB/s (峰值 960 GB/s)
CP 停顿: 5.2M/s
PCIE 带宽: 0.8 GB/s
功耗: 285W (峰值 355W)
温度: 72°C
```

问题：
1. （5 分）分析上述数据，指出最可能的性能瓶颈，并说明依据。
2. （5 分）如果 L2 Cache 命中率提升到 90% 以上，预计性能会提升多少？解释原因。
3. （5 分）设计一个优化方案，列出至少 3 个具体的优化方向，并说明每个方向如何验证效果。

### 八、内存泄漏排查（15 分）

场景：某 AI 训练脚本在运行 8 小时后出现 OOM (Out of Memory) 错误。

```bash
# 监控数据时间线:
时间     VRAM 使用     GPU 利用率    PVRS (Page Faults)
00:00   6.2 GB       98%           120/s
02:00   9.8 GB       97%           150/s
04:00   13.5 GB      96%           200/s
06:00   17.1 GB      95%           350/s
08:00   OOM (崩溃)    -             800/s
```

问题：
1. （5 分）分析上述数据，判断是否存在内存泄漏，并说明依据。
2. （5 分）解释为什么 PVRS（页面错误率）在 VRAM 使用增加时也在上升。
3. （5 分）设计自动化的 VRAM 泄漏检测方法，包括阈值设定和告警机制。
