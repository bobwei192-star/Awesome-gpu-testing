# 测试报告编写：结果汇总 / 趋势分析 / 建议

## 学习目标

- 理解 GPU 驱动测试报告的结构和编写规范
- 掌握测试结果汇总的不同维度和方法
- 学会通过测试数据进行趋势分析，识别质量风险
- 能够编写可执行的改进建议，推动驱动质量提升
- 了解不同受众（开发者、管理者、客户）对测试报告的不同需求

## 知识详解

### 一、概念原理

#### 1.1 测试报告的核心目标

测试报告（Test Report）是测试活动的最终交付物，其核心目标是：

```
┌─────────────────────────────────────────────────────────────┐
│                    测试报告的三大目标                         │
│                                                             │
│  1. 信息透明化                                               │
│     └── 向所有干系人呈现测试结果的全貌                        │
│                                                             │
│  2. 质量可度量                                               │
│     └── 用量化指标回答"驱动质量是否达到发布标准"              │
│                                                             │
│  3. 决策可执行                                               │
│     └── 基于数据提供明确的"发布/不发布"建议                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

在 GPU 驱动测试中，测试报告的受众和他们的关注点各不相同：

| 受众 | 关注重点 | 报告形式 |
|------|---------|---------|
| 测试经理 | 整体进度、阻塞项、资源效率 | 完整报告 |
| 开发经理 | 缺陷分布、模块质量、回归影响 | 缺陷分析附录 |
| 驱动开发者 | 具体失败用例、复现步骤、日志 | 详细失败报告 |
| 产品经理 | 发布风险评估、功能完成度 | 摘要报告 |
| 客户/社区 | 已知问题列表、兼容性状态 | 发布说明 |

#### 1.2 测试报告的标准结构

根据 IEEE 829-2008 标准，测试报告应包含以下核心部分：

```
┌─────────────────────────────────────────────────────────────┐
│              GPU 驱动测试报告标准结构                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 执行摘要 (Executive Summary)                            │
│     ├── 测试对象、范围、周期                                 │
│     ├── 总体结论（PASS/FAIL/CONDITIONAL PASS）              │
│     └── 关键数据快照（通过率、缺陷统计、风险项）              │
│                                                             │
│  2. 测试范围与覆盖                                           │
│     ├── 实际测试范围 vs 计划范围                             │
│     ├── 测试环境矩阵执行情况                                 │
│     └── 测试用例覆盖分析                                     │
│                                                             │
│  3. 测试结果汇总                                             │
│     ├── 整体通过率统计                                       │
│     ├── 按模块/功能/严重度的分类统计                          │
│     ├── 失败用例详细分析                                     │
│     └── 性能测试结果对比                                     │
│                                                             │
│  4. 缺陷分析                                                 │
│     ├── 缺陷统计（总数、按严重度/优先级/模块分布）             │
│     ├── 缺陷趋势（发现率、关闭率、遗留缺陷）                  │
│     ├── 典型缺陷案例分析                                     │
│     └── 缺陷根因分布                                         │
│                                                             │
│  5. 风险评估                                                 │
│     ├── 未修复缺陷的影响分析                                 │
│     ├── 测试覆盖盲区识别                                     │
│     └── 发布风险等级评定                                     │
│                                                             │
│  6. 改进建议                                                 │
│     ├── 对开发流程的建议                                     │
│     ├── 对测试流程的建议                                     │
│     └── 对测试工具/环境的建议                                │
│                                                             │
│  7. 附录                                                     │
│     ├── 详细测试结果数据                                     │
│     ├── 失败用例日志链接                                     │
│     └── 测试环境配置详情                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1.3 测试质量标准与决策模型

测试报告的最终输出是一个质量决策。常用的决策模型包括：

**质量门禁（Quality Gate）模型：**

```
质量门禁检查清单
┌─────────────────────────────────────────────────────────────┐
│ 门禁项                    │  通过标准       │  状态         │
├─────────────────────────────────────────────────────────────┤
│ G1: 烟雾测试通过率          │ 100%           │  ■ 通过       │
│ G2: P0 测试用例通过率       │ ≥ 98%          │  ■ 通过       │
│ G3: 无 S1 未关闭缺陷        │ 0 个           │  ■ 通过       │
│ G4: S2 缺陷关闭率           │ ≥ 90%          │  ■ 未通过 (85%)│
│ G5: 性能退化幅度            │ ≤ 5%           │  ■ 通过       │
│ G6: 关键路径测试覆盖         │ 100%           │  ■ 通过       │
│                                                            │
│ 结论：CONDITIONAL PASS                                      │
│ 条件：S2 缺陷在 1 周内修复并验证                             │
└─────────────────────────────────────────────────────────────┘
```

**质量评分卡（Quality Scorecard）模型：**

```
质量评分卡（满分 100）
┌─────────────────────────────────────────────────────────────┐
│ 维度               │  权重  │  得分  │  加权分               │
├─────────────────────────────────────────────────────────────┤
│ 功能完整性          │  25%   │  92   │  23.0                 │
│ 稳定性              │  25%   │  85   │  21.3                 │
│ 性能                │  20%   │  78   │  15.6                 │
│ 兼容性              │  15%   │  90   │  13.5                 │
│ 缺陷管理            │  15%   │  80   │  12.0                 │
├─────────────────────────────────────────────────────────────┤
│ 总分               │  100%  │       │  85.4                 │
│ 等级               │  A- (优秀)                             │
└─────────────────────────────────────────────────────────────┘
```

### 二、实践操作

#### 2.1 测试报告自动生成框架

以下是一个完整的 GPU 驱动测试报告生成框架，支持多数据源输入和多格式输出：

```python
#!/usr/bin/env python3
"""
gpu_test_report_generator.py
GPU 驱动测试报告自动生成框架
支持从 JUnit XML、JSON 和 CSV 数据源生成 Markdown/HTML 报告
"""

import json
import csv
import os
import re
from datetime import datetime, timedelta
from pathlib import Path
from collections import defaultdict, Counter
from typing import Dict, List, Optional, Tuple, Any
from xml.etree import ElementTree as ET


class TestResult:
    """单个测试结果的数据结构"""
    
    def __init__(self, name: str, status: str, duration_ms: float = 0,
                 module: str = "unknown", failure_msg: str = ""):
        self.name = name
        self.status = status          # PASS, FAIL, SKIP, ERROR, TIMEOUT
        self.duration_ms = duration_ms
        self.module = module
        self.failure_msg = failure_msg
        self.timestamp = datetime.now()
    
    def is_pass(self) -> bool:
        return self.status.upper() == "PASS"
    
    def is_fail(self) -> bool:
        return self.status.upper() in ("FAIL", "ERROR", "TIMEOUT")
    
    def is_skip(self) -> bool:
        return self.status.upper() == "SKIP"


class DefectRecord:
    """缺陷记录数据结构"""
    
    def __init__(self, defect_id: str, title: str, severity: str,
                 module: str, status: str, created_at: str):
        self.defect_id = defect_id
        self.title = title
        self.severity = severity
        self.module = module
        self.status = status
        self.created_at = created_at
        self.closed_at: Optional[str] = None
        self.resolution_days: Optional[float] = None


class GPUTestReportGenerator:
    """GPU 驱动测试报告生成器"""
    
    def __init__(self, project_name: str, version: str):
        self.project_name = project_name
        self.version = version
        self.test_results: List[TestResult] = []
        self.defects: List[DefectRecord] = []
        self.performance_data: Dict[str, Dict] = {}
        self.env_info: Dict[str, str] = {}
        
    def load_junit_xml(self, xml_path: str) -> int:
        """从 JUnit XML 加载测试结果"""
        count = 0
        tree = ET.parse(xml_path)
        root = tree.getroot()
        
        for suite in root.findall("testsuite"):
            suite_name = suite.get("name", "unknown")
            for case in suite.findall("testcase"):
                class_name = case.get("classname", "")
                name = case.get("name", "")
                full_name = f"{class_name}.{name}" if class_name else name
                duration = float(case.get("time", 0)) * 1000
                
                failure = case.find("failure")
                error = case.find("error")
                
                if failure is not None:
                    status = "FAIL"
                    msg = failure.get("message", "")
                elif error is not None:
                    status = "ERROR"
                    msg = error.get("message", "")
                elif case.find("skipped") is not None:
                    status = "SKIP"
                    msg = ""
                else:
                    status = "PASS"
                    msg = ""
                
                self.test_results.append(TestResult(
                    name=full_name, status=status,
                    duration_ms=duration, module=suite_name,
                    failure_msg=msg
                ))
                count += 1
        
        return count
    
    def load_json_results(self, json_path: str) -> int:
        """从 JSON 文件加载测试结果"""
        with open(json_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        
        count = 0
        for item in data:
            self.test_results.append(TestResult(
                name=item.get("name", "unknown"),
                status=item.get("status", "PASS"),
                duration_ms=item.get("duration_ms", 0),
                module=item.get("module", "unknown"),
                failure_msg=item.get("failure_msg", ""),
            ))
            count += 1
        
        return count
    
    def load_defects_from_csv(self, csv_path: str) -> int:
        """从 CSV 加载缺陷数据"""
        count = 0
        with open(csv_path, 'r', encoding='utf-8') as f:
            reader = csv.DictReader(f)
            for row in reader:
                self.defects.append(DefectRecord(
                    defect_id=row.get("id", f"BUG-{count}"),
                    title=row.get("title", ""),
                    severity=row.get("severity", "S3"),
                    module=row.get("module", "unknown"),
                    status=row.get("status", "OPEN"),
                    created_at=row.get("created_at", ""),
                ))
                count += 1
        return count
    
    def set_environment_info(self, info: Dict[str, str]):
        """设置测试环境信息"""
        self.env_info = info
    
    def add_performance_data(self, test_name: str, metrics: Dict):
        """添加性能测试数据"""
        self.performance_data[test_name] = metrics
    
    def compute_summary_statistics(self) -> Dict:
        """计算汇总统计数据"""
        total = len(self.test_results)
        if total == 0:
            return {"error": "无测试数据"}
        
        passed = sum(1 for t in self.test_results if t.is_pass())
        failed = sum(1 for t in self.test_results if t.is_fail())
        skipped = sum(1 for t in self.test_results if t.is_skip())
        
        # 按模块统计
        module_stats = defaultdict(lambda: {"total": 0, "pass": 0, "fail": 0, "skip": 0})
        for t in self.test_results:
            ms = module_stats[t.module]
            ms["total"] += 1
            if t.is_pass():
                ms["pass"] += 1
            elif t.is_fail():
                ms["fail"] += 1
            else:
                ms["skip"] += 1
        
        # 耗时统计
        durations = [t.duration_ms for t in self.test_results if t.duration_ms > 0]
        avg_duration = sum(durations) / len(durations) if durations else 0
        max_duration = max(durations) if durations else 0
        
        # 失败用例详情
        failures = [
            {"name": t.name, "module": t.module, "msg": t.failure_msg}
            for t in self.test_results if t.is_fail()
        ]
        
        return {
            "total": total,
            "passed": passed,
            "failed": failed,
            "skipped": skipped,
            "pass_rate": round(passed / total * 100, 1) if total > 0 else 0,
            "module_stats": dict(module_stats),
            "avg_duration_ms": round(avg_duration, 1),
            "max_duration_ms": round(max_duration, 1),
            "failures": failures,
        }
    
    def compute_defect_statistics(self) -> Dict:
        """计算缺陷统计数据"""
        total = len(self.defects)
        if total == 0:
            return {"total": 0}
        
        open_defects = [d for d in self.defects if d.status.upper() in ("OPEN", "IN_PROGRESS")]
        closed_defects = [d for d in self.defects if d.status.upper() == "CLOSED"]
        
        severity_dist = Counter(d.severity for d in self.defects)
        module_dist = Counter(d.module for d in self.defects)
        
        return {
            "total": total,
            "open": len(open_defects),
            "closed": len(closed_defects),
            "close_rate": round(len(closed_defects) / total * 100, 1) if total > 0 else 0,
            "severity_distribution": dict(severity_dist),
            "module_distribution": dict(module_dist),
        }
    
    def generate_markdown_report(self, output_path: str = "test_report.md"):
        """生成 Markdown 格式测试报告"""
        stats = self.compute_summary_statistics()
        defect_stats = self.compute_defect_statistics()
        
        if "error" in stats:
            print(f"无法生成报告: {stats['error']}")
            return
        
        now = datetime.now().strftime("%Y-%m-%d %H:%M")
        env_table = "\n".join(
            f"| {k} | {v} |" for k, v in self.env_info.items()
        )
        
        # 模块统计表
        module_table = []
        for mod, s in sorted(stats["module_stats"].items()):
            pass_rate = round(s["pass"] / s["total"] * 100, 1) if s["total"] > 0 else 0
            module_table.append(
                f"| {mod} | {s['total']} | {s['pass']} | {s['fail']} | "
                f"{s['skip']} | {pass_rate}% |"
            )
        
        # 缺陷统计表
        defect_table = []
        if defect_stats["total"] > 0:
            for sev, count in sorted(defect_stats["severity_distribution"].items()):
                defect_table.append(f"| {sev} | {count} |")
        
        # 失败用例表
        failure_table = []
        for f in stats["failures"][:20]:  # 最多显示 20 个
            failure_table.append(f"| {f['name']} | {f['module']} | {f['msg'][:80]} |")
        
        report = f"""# GPU 驱动测试报告

**项目**: {self.project_name}
**版本**: {self.version}
**报告生成时间**: {now}

---

## 1. 执行摘要

### 总体结论

| 度量项 | 数值 | 状态 |
|--------|------|------|
| 总测试用例 | {stats['total']} | - |
| 通过 | {stats['passed']} | - |
| 失败 | {stats['failed']} | - |
| 跳过 | {stats['skipped']} | - |
| 通过率 | {stats['pass_rate']}% | {'✅ PASS' if stats['pass_rate'] >= 98 else '❌ FAIL'} |
| 平均执行时长 | {stats['avg_duration_ms']}ms | - |
| 最长执行时长 | {stats['max_duration_ms']}ms | - |

### 缺陷快照

| 度量项 | 数值 |
|--------|------|
| 缺陷总数 | {defect_stats['total']} |
| 开放缺陷 | {defect_stats['open']} |
| 已关闭缺陷 | {defect_stats['closed']} |
| 关闭率 | {defect_stats['close_rate']}% |

### 环境信息

| 项目 | 值 |
|------|-----|
{env_table}

---

## 2. 测试覆盖

### 按模块统计

| 模块 | 总数 | 通过 | 失败 | 跳过 | 通过率 |
|------|------|------|------|------|--------|
{chr(10).join(module_table)}

---

## 3. 测试结果分析

### 失败用例详情

| 用例名称 | 模块 | 失败原因 |
|----------|------|---------|
{chr(10).join(failure_table) if failure_table else '无失败用例'}

"""

        if self.performance_data:
            report += """
## 4. 性能测试结果

| 测试项 | 指标 | 当前值 | 基线值 | 变化率 |
|--------|------|--------|--------|--------|
"""
            for test_name, metrics in self.performance_data.items():
                current = metrics.get("current", "N/A")
                baseline = metrics.get("baseline", "N/A")
                change = metrics.get("change_percent", "N/A")
                metric_name = metrics.get("metric", "score")
                report += f"| {test_name} | {metric_name} | {current} | {baseline} | {change}% |\n"

        report += f"""
---

## 5. 缺陷分析

### 严重度分布

| 严重度 | 数量 |
|--------|------|
{chr(10).join(defect_table)}

### 模块分布

| 模块 | 缺陷数 |
|------|--------|
"""
        for mod, count in sorted(defect_stats["module_distribution"].items()):
            report += f"| {mod} | {count} |\n"

        report += """
---

## 6. 风险评估

| 风险项 | 等级 | 说明 |
|--------|------|------|
"""
        if stats["failed"] > 0:
            report += f"| 存在 {stats['failed']} 个失败用例 | 高 | 需要人工分析每项失败原因 |\n"
        else:
            report += "| 无测试失败 | 低 | 所有测试通过 |\n"

        report += """
---

## 7. 改进建议

1. **测试覆盖**: [基于数据分析的测试覆盖改进建议]
2. **缺陷管理**: [基于缺陷趋势的管理流程改进建议]
3. **测试效率**: [基于执行时长的测试优化建议]
4. **工具/环境**: [基于测试环境的改进建议]

---

*本报告由 GPU 测试报告自动生成框架生成*
"""
        with open(output_path, 'w', encoding='utf-8') as f:
            f.write(report)
        
        print(f"Markdown 报告已生成: {output_path}")
```

#### 2.2 测试结果趋势分析工具

趋势分析是测试报告中最有价值的部分之一，能够揭示质量随时间的变化：

```bash
#!/bin/bash
# test_trend_analyzer.sh
# 测试结果趋势分析工具

RESULTS_DIR="${1:-test_results}"
TREND_REPORT="trend_report.md"

echo "执行测试趋势分析..."
echo "数据来源: ${RESULTS_DIR}"

# 收集历史数据
declare -A DAILY_PASS_RATE
declare -A DAILY_TOTAL
declare -A DAILY_FAIL
declare -A DEFECT_OPEN_TREND

for result_file in "${RESULTS_DIR}"/daily_*.json; do
    if [ ! -f "${result_file}" ]; then
        continue
    fi
    
    date_str=$(basename "${result_file}" | sed 's/daily_//;s/\.json//')
    
    total=$(python3 -c "import json; d=json.load(open('${result_file}')); print(d.get('total',0))")
    passed=$(python3 -c "import json; d=json.load(open('${result_file}')); print(d.get('passed',0))")
    failed=$(python3 -c "import json; d=json.load(open('${result_file}')); print(d.get('failed',0))")
    
    if [ "${total}" -gt 0 ]; then
        pass_rate=$(echo "scale=1; ${passed} * 100 / ${total}" | bc)
        DAILY_PASS_RATE["${date_str}"]="${pass_rate}"
        DAILY_TOTAL["${date_str}"]="${total}"
        DAILY_FAIL["${date_str}"]="${failed}"
    fi
done

# 计算移动平均线（5 日）
echo "计算趋势指标..."
declare -A MOVING_AVG
dates=($(for d in "${!DAILY_PASS_RATE[@]}"; do echo "$d"; done | sort))

for i in "${!dates[@]}"; do
    if [ "${i}" -ge 4 ]; then
        sum=0
        for j in $(seq $((i - 4)) "${i}"); do
            sum=$(echo "${sum} + ${DAILY_PASS_RATE[${dates[$j]}]}" | bc)
        done
        MOVING_AVG["${dates[$i]}"]=$(echo "scale=1; ${sum} / 5" | bc)
    fi
done

# 生成趋势报告
cat > "${TREND_REPORT}" << TREND_HEADER
# 测试结果趋势分析报告

**分析时间**: $(date '+%Y-%m-%d %H:%M')
**数据范围**: ${dates[0]:-N/A} ~ ${dates[-1]:-N/A}
**数据点数**: ${#dates[@]}

---

## 通过率趋势

| 日期 | 总用例 | 失败数 | 通过率(%) | 5日移动平均(%) |
|------|--------|--------|-----------|---------------|
TREND_HEADER

for date in "${dates[@]}"; do
    rate="${DAILY_PASS_RATE[$date]}"
    total="${DAILY_TOTAL[$date]}"
    fail="${DAILY_FAIL[$date]}"
    ma="${MOVING_AVG[$date]:----}"
    echo "| ${date} | ${total} | ${fail} | ${rate} | ${ma} |" >> "${TREND_REPORT}"
done

cat >> "${TREND_REPORT}" << TREND_ANALYSIS

## 趋势分析结论

### 通过率趋势
TREND_ANALYSIS

# 判断趋势方向
first_rate="${DAILY_PASS_RATE[${dates[0]}]}"
last_rate="${DAILY_PASS_RATE[${dates[-1]}]}"

if [ -n "${first_rate}" ] && [ -n "${last_rate}" ]; then
    trend=$(echo "${last_rate} - ${first_rate}" | bc)
    if [ "$(echo "${trend} > 0" | bc)" -eq 1 ]; then
        echo "- **趋势方向**: 通过率上升中 ✅" >> "${TREND_REPORT}"
    elif [ "$(echo "${trend} < 0" | bc)" -eq 1 ]; then
        echo "- **趋势方向**: 通过率下降中 ❌" >> "${TREND_REPORT}"
    else
        echo "- **趋势方向**: 通过率稳定 ➖" >> "${TREND_REPORT}"
    fi
    
    echo "- **起始通过率**: ${first_rate}%" >> "${TREND_REPORT}"
    echo "- **最新通过率**: ${last_rate}%" >> "${TREND_REPORT}"
    echo "- **变化幅度**: ${trend} 个百分点" >> "${TREND_REPORT}"
    
    # 异常检测
    if [ "$(echo "${last_rate} < 95" | bc)" -eq 1 ]; then
        echo "" >> "${TREND_REPORT}"
        echo "### ⚠️ 异常告警" >> "${TREND_REPORT}"
        echo "通过率低于 95% 阈值，请立即关注！" >> "${TREND_REPORT}"
    fi
fi

echo ""
echo "趋势报告已生成: ${TREND_REPORT}"
```

#### 2.3 缺陷趋势分析

```python
#!/usr/bin/env python3
"""
defect_trend_analysis.py
缺陷趋势分析工具——基于历史数据预测缺陷发现趋势
"""

import json
import math
from datetime import datetime, timedelta
from typing import Dict, List, Tuple
from collections import defaultdict


class DefectTrendAnalyzer:
    """缺陷趋势分析器"""
    
    def __init__(self):
        self.daily_defects: Dict[str, int] = defaultdict(int)
        self.daily_closed: Dict[str, int] = defaultdict(int)
        self.cumulative_found: List[int] = []
        self.cumulative_closed: List[int] = []
        self.dates: List[str] = []
    
    def load_from_json(self, json_path: str):
        """从 JSON 文件加载缺陷数据"""
        with open(json_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
        
        for record in data:
            created = record.get("created_at", "")[:10]
            closed = record.get("closed_at", "")[:10] if record.get("closed_at") else ""
            
            if created:
                self.daily_defects[created] += 1
            if closed:
                self.daily_closed[closed] += 1
    
    def compute_cumulative(self):
        """计算累计数据"""
        all_dates = sorted(set(list(self.daily_defects.keys()) + 
                               list(self.daily_closed.keys())))
        
        found_so_far = 0
        closed_so_far = 0
        
        for date in all_dates:
            found_so_far += self.daily_defects.get(date, 0)
            closed_so_far += self.daily_closed.get(date, 0)
            self.cumulative_found.append(found_so_far)
            self.cumulative_closed.append(closed_so_far)
            self.dates.append(date)
    
    def predict_remaining_defects(self) -> Dict:
        """使用 Rayleigh 模型预测剩余缺陷数"""
        if len(self.cumulative_found) < 5:
            return {"error": "数据点不足，需要至少 5 个数据点"}
        
        # Rayleigh 模型: E(t) = K * (1 - exp(-a * t^2))
        # 使用最小二乘法拟合参数
        n = len(self.cumulative_found)
        
        # 简化估算：使用最近 3 天的平均发现率
        recent_rate = []
        for i in range(max(0, n - 3), n):
            if self.daily_defects.get(self.dates[i], 0) > 0:
                recent_rate.append(self.daily_defects[self.dates[i]])
        
        avg_rate = sum(recent_rate) / len(recent_rate) if recent_rate else 0
        
        # 如果发现率趋于 0，认为缺陷接近发现完毕
        if avg_rate < 0.5:
            remaining = 0
            confidence = "高"
        else:
            # 简单估算: 假设发现率线性下降
            remaining = int(avg_rate * 5)  # 预估未来 5 天
            confidence = "中"
        
        return {
            "total_found": self.cumulative_found[-1] if self.cumulative_found else 0,
            "total_closed": self.cumulative_closed[-1] if self.cumulative_closed else 0,
            "estimated_remaining": remaining,
            "confidence": confidence,
            "recent_avg_daily_rate": round(avg_rate, 2),
        }
    
    def generate_trend_section(self) -> str:
        """生成趋势分析章节"""
        self.compute_cumulative()
        prediction = self.predict_remaining_defects()
        
        lines = [
            "## 缺陷趋势分析",
            "",
            "| 日期 | 新增缺陷 | 关闭缺陷 | 累计发现 | 累计关闭 |",
            "|------|----------|----------|----------|----------|",
        ]
        
        for i, date in enumerate(self.dates):
            lines.append(
                f"| {date} | {self.daily_defects.get(date, 0)} | "
                f"{self.daily_closed.get(date, 0)} | "
                f"{self.cumulative_found[i]} | {self.cumulative_closed[i]} |"
            )
        
        if "error" not in prediction:
            lines.extend([
                "",
                "### 缺陷预测",
                "",
                f"| 指标 | 数值 |",
                f"|------|------|",
                f"| 已发现缺陷总数 | {prediction['total_found']} |",
                f"| 已关闭缺陷总数 | {prediction['total_closed']} |",
                f"| 预估剩余缺陷 | {prediction['estimated_remaining']} |",
                f"| 预测置信度 | {prediction['confidence']} |",
                f"| 近期日均发现率 | {prediction['recent_avg_daily_rate']} |",
            ])
        
        return '\n'.join(lines)


# 使用示例
if __name__ == "__main__":
    analyzer = DefectTrendAnalyzer()
    if os.path.exists("defects_history.json"):
        analyzer.load_from_json("defects_history.json")
        print(analyzer.generate_trend_section())
```

#### 2.4 改进建议生成器

基于测试数据自动生成改进建议，使报告更具可操作性：

```python
#!/usr/bin/env python3
"""
improvement_suggestion_engine.py
基于测试数据的改进建议生成引擎
"""

from typing import Dict, List


class ImprovementSuggestionEngine:
    """改进建议生成引擎"""
    
    def __init__(self, stats: Dict, defect_stats: Dict):
        self.stats = stats
        self.defect_stats = defect_stats
        self.suggestions: List[Dict] = []
    
    def analyze_test_coverage(self) -> List[str]:
        """分析测试覆盖并提出建议"""
        suggestions = []
        module_stats = self.stats.get("module_stats", {})
        
        # 检查是否有模块通过率过低
        low_coverage_modules = []
        for mod, s in module_stats.items():
            pass_rate = s["pass"] / s["total"] * 100 if s["total"] > 0 else 0
            if pass_rate < 80 and s["total"] > 10:
                low_coverage_modules.append(f"{mod}({pass_rate:.0f}%)")
        
        if low_coverage_modules:
            suggestions.append(
                f"模块 {', '.join(low_coverage_modules)} 通过率低于 80%，"
                f"建议对这些模块进行代码审查和专项测试"
            )
        
        return suggestions
    
    def analyze_defect_patterns(self) -> List[str]:
        """分析缺陷模式并提出建议"""
        suggestions = []
        sev_dist = self.defect_stats.get("severity_distribution", {})
        
        # 检查 S1/S2 缺陷比例
        high_sev = sev_dist.get("S1", 0) + sev_dist.get("S2", 0)
        total = self.defect_stats.get("total", 0)
        
        if total > 0 and high_sev / total > 0.3:
            suggestions.append(
                f"S1/S2 严重缺陷占比 {high_sev/total*100:.0f}%，"
                f"建议在开发阶段增加静态分析和代码审查"
            )
        
        mod_dist = self.defect_stats.get("module_distribution", {})
        if mod_dist:
            worst_module = max(mod_dist, key=mod_dist.get)
            suggestions.append(
                f"模块 '{worst_module}' 缺陷最多（{mod_dist[worst_module]} 个），"
                f"建议对该模块进行重构或增加单元测试覆盖"
            )
        
        return suggestions
    
    def analyze_performance(self, perf_data: Dict) -> List[str]:
        """分析性能数据并提出建议"""
        suggestions = []
        for test_name, metrics in perf_data.items():
            change = metrics.get("change_percent", 0)
            if isinstance(change, (int, float)) and change < -5:
                suggestions.append(
                    f"性能测试 '{test_name}' 退化 {change:.1f}%（超过 5% 阈值），"
                    f"建议分析性能回归原因"
                )
        return suggestions
    
    def generate_all(self, perf_data: Dict = None) -> List[Dict]:
        """生成所有改进建议"""
        categories = [
            ("测试覆盖", self.analyze_test_coverage()),
            ("缺陷管理", self.analyze_defect_patterns()),
        ]
        
        if perf_data:
            categories.append(("性能", self.analyze_performance(perf_data)))
        
        for category, items in categories:
            for item in items:
                self.suggestions.append({
                    "category": category,
                    "suggestion": item,
                })
        
        return self.suggestions
    
    def format_as_markdown(self) -> str:
        """格式化为 Markdown"""
        if not self.suggestions:
            return "## 改进建议\n\n当前未发现需要改进的关键问题。\n"
        
        lines = ["## 改进建议", ""]
        
        by_category = {}
        for s in self.suggestions:
            by_category.setdefault(s["category"], []).append(s["suggestion"])
        
        for category, items in by_category.items():
            lines.append(f"### {category}")
            for item in items:
                lines.append(f"- {item}")
            lines.append("")
        
        return '\n'.join(lines)
```

### 五、今日小结

今天学习了 GPU 驱动测试报告的完整编写方法：

- **报告结构**：执行摘要、覆盖分析、结果汇总、缺陷分析、风险评估、改进建议
- **受众分析**：不同角色（管理层、开发、测试、客户）关注不同的报告维度
- **统计分析**：通过率、缺陷分布、趋势分析、质量评分卡
- **自动生成**：Markdown/HTML 报告自动生成框架、趋势分析工具
- **改进建议**：基于数据分析的量化改进建议生成
- **典型案例**：AMDGPU 6.8-rc1 版本测试报告和跨版本质量趋势报告

### 六、扩展思考

1. **报告自动化程度**：测试报告中的哪些部分可以完全自动化？哪些需要人工分析和判断？如何在自动化和人工之间取得平衡？

2. **数据可视化**：除了表格和柱状图，还有哪些可视化方式能更直观地呈现 GPU 驱动质量？（如热力图展示硬件配置覆盖矩阵）

3. **AI 生成报告摘要**：能否利用大语言模型自动生成测试报告的执行摘要和关键发现？

4. **实时质量仪表盘**：相比定期测试报告，实时质量仪表盘（如 Grafana 集成 CI 数据）能否提供更及时的质量反馈？如何设计有效的 GPU 驱动质量仪表盘？
