#### 错误类型和严重级别

```ascii
RAS 错误分类:

┌──────────────────────────────────────────────────────────────┐
│  错误类型                                                    │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  CE (Correctable Error) - 可纠正错误                    │  │
│  │  ──────────────────────────────────────────────────    │  │
│  │  ● 单比特 ECC 错误 (Single-bit ECC Error)               │  │
│  │  ● CRC 校验错误 (可重试恢复)                             │  │
│  │  ● 不会导致数据丢失                                      │  │
│  │  ● 驱动记录错误计数并继续运行                             │  │
│  │                                                         │  │
│  │  ● 处理方式: 自动纠正 + 计数累加                         │  │
│  │  ● 告警阈值: 超过阈值时触发中断通知                      │  │
│  │  ● 用户响应: 监控错误增长趋势                            │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  UE (Uncorrectable Error) - 不可纠正错误                │  │
│  │  ──────────────────────────────────────────────────    │  │
│  │  ● 多比特 ECC 错误 (Multi-bit ECC Error)                │  │
│  │  ● 数据已损坏，无法自动恢复                               │  │
│  │  ● 需要人为干预                                           │  │
│  │                                                         │  │
│  │  ● 处理方式: 记录错误 + 通知 + 可能的页面隔离             │  │
│  │  ● 中断: 触发 RAS 中断                                   │  │
│  │  ● 用户响应: 分析根因，考虑 RMA                          │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  DE (Deferred Error) - 延迟错误                         │  │
│  │  ──────────────────────────────────────────────────    │  │
│  │  ● 错误已检测但未立即纠正                                 │  │
│  │  ● 在后续访问时可能升级为 UE                              │  │
│  │  ● 用于预测性故障分析                                     │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │  Fatal (致命错误)                                       │  │
│  │  ──────────────────────────────────────────────────    │  │
│  │  ● 导致 GPU 完全不可用                                   │  │
│  │  ● 需要 GPU reset 或重启系统                              │  │
│  │  ● 例如: XGMI 链路断裂, PCIe 致命错误                    │  │
│  └────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘

严重级别:
┌────────────┬──────────┬──────────────┬───────────────────────┐
│ 级别       │ 类型     │ 系统影响      │ 驱动响应               │
├────────────┼──────────┼──────────────┼───────────────────────┤
│ INFO       │ CE       │ 无           │ 计数 + 继续运行        │
│ WARN       │ CE 阈值  │ 无           │ 记录 + 通知 sysfs      │
│ ERROR      │ UE       │ 可能丢数据    │ 隔离 + 通知用户空间     │
│ CRIT       │ Fatal    │ GPU 不可用    │ GPU reset + 日志       │
│ PANIC      │ 多重 fatal│ 系统不稳定    │ 内核 panic (可配置)    │
└────────────┴──────────┴──────────────┴───────────────────────┘
```

#### RAS 中断处理机制

```ascii
RAS 中断处理流程:

┌──────────────────────────────────────────────────────────────┐
│  硬件错误检测                                                    │
│                                                               │
│  ┌──────────┐                                                  │
│  │ UMC ECC  │  ──→  CE: 自动纠正, 更新计数器                     │
│  │ 检测错误  │  ──→  UE: 触发 RAS 中断                           │
│  └──────────┘                                                   │
│       │                                                         │
│       ▼                                                         │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  RAS 中断处理流程                                         │  │
│  │                                                           │  │
│  │  1. 硬件触发中断 (MSI-X)                                   │  │
│  │     │                                                      │  │
│  │  2. amdgpu_ras_interrupt_handler()                         │  │
│  │     │  - 读取错误状态寄存器                                 │  │
│  │     │  - 识别错误源 (UMC/SDMA/GFX/...)                    │  │
│  │     │  - 清除中断标志                                      │  │
│  │     ▼                                                      │  │
│  │  3. amdgpu_ras_log_error()                                 │  │
│  │     │  - 记录错误信息到环形缓冲区                            │  │
│  │     │  - 更新错误计数                                       │  │
│  │     │  - 写入 EEPROM (如果使能)                             │  │
│  │     ▼                                                      │  │
│  │  4. 通知用户空间                                            │  │
│  │     │  - sysfs 错误计数更新                                  │  │
│  │     │  - uevent 广播                                        │  │
│  │     │  - trace event 输出                                   │  │
│  │     ▼                                                      │  │
│  │  5. 高级响应 (如果配置)                                     │  │
│  │     │  - UE → 页面隔离                                      │  │
│  │     │  - CE 阈值 → 预替告                                    │  │
│  │     │  - Fatal → GPU reset                                 │  │
│  └──────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

### 实践操作

#### 查看 RAS 功能支持

```bash
# 1. 检查 RAS 是否可用
cat /sys/class/drm/card0/device/ras_features

# 输出:
# ras feature mask: 0x3f
# bit0: UMC (umc) - supported
# bit1: SDMA (sdma) - supported
# bit2: GFX (gfx) - supported
# bit3: MMHUB (mmhub) - supported
# bit4: PCIE (pcie) - supported
# bit5: HDP (hdp) - supported

# 2. 查看 RAS 控制
cat /sys/class/drm/card0/device/ras_ctrl

# 输出:
# ras control: inject, disable, enable
# inject: error injection control
# disable: disable RAS
# enable: enable RAS

# 3. 查看当前 RAS 状态
cat /sys/class/drm/card0/device/ras_status

# 输出:
# ras status: enabled
# UMC ECC: enabled
# SDMA ECC: enabled
# GFX ECC: disabled (not supported)
```

#### RAS 配置和控制

```bash
# 1. 禁用 RAS
echo disable > /sys/class/drm/card0/device/ras_ctrl

# 2. 启用 RAS (如果支持)
echo enable > /sys/class/drm/card0/device/ras_ctrl

# 3. 配置 CE 告警阈值
# 当 CE 错误超过阈值时触发告警
echo 100 > /sys/class/drm/card0/device/ras_ce_threshold

# 4. 配置 UE 自动处理
# 0: 仅记录, 1: 隔离页面, 2: GPU reset
echo 1 > /sys/class/drm/card0/device/ras_ue_action

# 5. 查看 EEPROM 存储的错误记录
cat /sys/kernel/debug/dri/0/amdgpu_ras_eeprom_test
```

#### Python RAS 监控脚本

```python
#!/usr/bin/env python3
# gpu_ras_monitor.py - GPU RAS 监控工具

import json
import time
import signal
from pathlib import Path
from datetime import datetime
from typing import Dict, Optional, List


class RASMonitor:
    """GPU RAS 子系统监控器"""
    
    def __init__(self, gpu_index: int = 0):
        self.gpu_index = gpu_index
        self.base = f"/sys/class/drm/card{gpu_index}/device"
        self.running = False
        self.events: List[Dict] = []
    
    def read_file(self, path: str) -> Optional[str]:
        """读取 sysfs 文件"""
        try:
            with open(path) as f:
                return f.read().strip()
        except (FileNotFoundError, PermissionError):
            return None
    
    def get_ras_features(self) -> List[str]:
        """获取 RAS 特性列表"""
        content = self.read_file(f"{self.base}/ras_features")
        if not content:
            return []
        
        features = []
        for line in content.split("\n"):
            if "supported" in line and "bit" in line:
                name = line.split("(")[1].split(")")[0]
                features.append(name)
        return features
    
    def get_ras_status(self) -> Dict:
        """获取 RAS 状态"""
        content = self.read_file(f"{self.base}/ras_status")
        if not content:
            return {}
        
        status = {}
        for line in content.split("\n"):
            if ":" in line:
                key, value = line.split(":", 1)
                status[key.strip()] = value.strip()
        return status
    
    def get_ecc_counts(self) -> List[Dict]:
        """获取 ECC 错误计数"""
        content = self.read_file(f"{self.base}/ras_ecc_cnt")
        if not content:
            return []
        
        channels = []
        lines = content.split("\n")
        for line in lines[2:]:  # 跳过表头
            if "│" in line:
                parts = [p.strip() for p in line.split("│")[1:-1]]
                if len(parts) >= 4:
                    channel = {
                        "block": parts[0],
                        "ce_count": self._parse_count(parts[1]),
                        "ue_count": self._parse_count(parts[2]),
                        "last_error_addr": parts[3],
                    }
                    channels.append(channel)
        return channels
    
    def _parse_count(self, s: str) -> int:
        """解析计数值"""
        try:
            return int(s.replace(",", ""))
        except ValueError:
            return 0
    
    def check_ras_health(self) -> Dict:
        """检查 RAS 健康状态"""
        ecc_counts = self.get_ecc_counts()
        status = self.get_ras_status()
        features = self.get_ras_features()
        
        total_ce = sum(ec["ce_count"] for ec in ecc_counts)
        total_ue = sum(ec["ue_count"] for ec in ecc_counts)
        
        # 检查是否有 UE 错误
        has_ue = total_ue > 0
        
        # 检查 CE 错误增长
        ce_threshold = 50  # 可配置
        ce_excessive = total_ce > ce_threshold
        
        # 检查 RAS 是否启用
        ras_enabled = status.get("ras status") == "enabled"
        
        health = {
            "timestamp": datetime.now().isoformat(),
            "ras_enabled": ras_enabled,
            "features": features,
            "total_ce": total_ce,
            "total_ue": total_ue,
            "channels": ecc_counts,
            "has_ue": has_ue,
            "ce_excessive": ce_excessive,
            "status": "healthy"
        }
        
        if has_ue:
            health["status"] = "critical"
            health["issues"] = [f"Uncorrectable errors detected: {total_ue}"]
        
        if ce_excessive:
            if health["status"] == "healthy":
                health["status"] = "warning"
            health.setdefault("issues", [])
            health["issues"].append(
                f"Excessive CE errors: {total_ce} (threshold: {ce_threshold})"
            )
        
        return health
    
    def start_monitoring(self, interval: float = 10.0):
        """开始监控"""
        self.running = True
        
        def signal_handler(signum, frame):
            self.running = False
        
        signal.signal(signal.SIGINT, signal_handler)
        
        print(f"RAS Monitor (interval: {interval}s)")
        print(f"{'时间':20s} {'CE总数':8s} {'UE总数':8s} {'状态':12s} {'详情'}")
        print("-" * 70)
        
        try:
            while self.running:
                health = self.check_ras_health()
                self.events.append(health)
                
                now = datetime.now().strftime("%H:%M:%S")
                detail = ""
                if health.get("issues"):
                    detail = health["issues"][0][:30]
                
                print(f"{now:20s} {health['total_ce']:^6d}  {health['total_ue']:^6d}  "
                      f"{health['status']:12s} {detail}")
                
                if health["has_ue"]:
                    print(f"  ⚠ UE 错误详情:")
                    for ch in health["channels"]:
                        if ch["ue_count"] > 0:
                            print(f"    {ch['block']}: {ch['ue_count']} UE @ {ch['last_error_addr']}")
                
                time.sleep(interval)
        except KeyboardInterrupt:
            pass
        
        print(f"\n监控完成, 共 {len(self.events)} 次采样")
    
    def generate_report(self) -> str:
        """生成 RAS 健康报告"""
        if not self.events:
            return "No data collected"
        
        report = []
        report.append("GPU RAS 健康报告")
        report.append("=" * 50)
        report.append(f"采样时段: {self.events[0]['timestamp']} - {self.events[-1]['timestamp']}")
        report.append(f"采样次数: {len(self.events)}")
        
        # 统计 CE/UE 总数
        total_ce = sum(e["total_ce"] for e in self.events)
        total_ue = sum(e["total_ue"] for e in self.events)
        report.append(f"\n错误统计:")
        report.append(f"  CE 总数: {total_ce}")
        report.append(f"  UE 总数: {total_ue}")
        
        # 检查增长趋势
        first_ce = self.events[0]["total_ce"]
        last_ce = self.events[-1]["total_ce"]
        ce_growth = last_ce - first_ce
        report.append(f"  CE 增长: {ce_growth}")
        
        # 各通道统计
        report.append(f"\n各通道错误:")
        last_channels = self.events[-1]["channels"]
        for ch in last_channels:
            if ch["ce_count"] > 0 or ch["ue_count"] > 0:
                report.append(f"  {ch['block']}: CE={ch['ce_count']} UE={ch['ue_count']} "
                              f"Last={ch['last_error_addr']}")
        
        # 评估
        report.append(f"\n健康评估:")
        if total_ue > 0:
            report.append(f"  ⚠ 存在 UE 错误 - 需要人工分析")
        if ce_growth > 50:
            report.append(f"  ⚠ CE 错误增长过快 - 可能预示硬件退化")
        if total_ce == 0 and total_ue == 0:
            report.append(f"  ✓ 正常 - 无错误记录")
        
        return "\n".join(report)
    
    def export_json(self, path: str):
        """导出 JSON 报告"""
        report = {
            "gpu_index": self.gpu_index,
            "samples": self.events,
            "summary": {
                "total_samples": len(self.events),
                "errors": {}
            }
        }
        
        if self.events:
            report["summary"]["latest_health"] = self.events[-1]
        
        with open(path, "w") as f:
            json.dump(report, f, indent=2)

class RASDiagnostics:
    """RAS 诊断工具"""
    
    @staticmethod
    def analyze_ecc_trend(ecc_history: List[List[Dict]]) -> str:
        """分析 ECC 趋势"""
        if len(ecc_history) < 2:
            return "数据不足"
        
        first = ecc_history[0]
        last = ecc_history[-1]
        
        findings = []
        
        for f_ch, l_ch in zip(first, last):
            if f_ch["block"] != l_ch["block"]:
                continue
            
            ce_diff = l_ch["ce_count"] - f_ch["ce_count"]
            ue_diff = l_ch["ue_count"] - f_ch["ue_count"]
            
            if ce_diff > 10:
                findings.append(
                    f"{f_ch['block']}: CE 快速增长 (+{ce_diff})"
                )
            if ue_diff > 0:
                findings.append(
                    f"{f_ch['block']}: 出现 UE 错误 (+{ue_diff})"
                )
        
        if not findings:
            return "ECC 错误无明显增长"
        
        return "\n".join(findings)
    
    @staticmethod
    def check_hardware_degradation(ce_rate: float) -> str:
        """检查硬件退化"""
        # CE 错误率 (每小时)
        if ce_rate < 1:
            return "正常"
        elif ce_rate < 10:
            return "监控 (CE 轻度增长)"
        elif ce_rate < 100:
            return "警告 (CE 中速增长, 建议更换)"
        else:
            return "严重 (CE 高速增长, 必须更换)"
```

### 今日小结

1. **RAS 架构**：AMDGPU RAS 子系统包括 UMC（内存控制器）、SDMA、GFX、MMHUB、PCIE、XGMI 等多个错误源，通过统一的管理框架进行错误检测、记录和处理。

2. **错误类型**：CE（可纠正错误，单比特 ECC）不影响数据正确性；UE（不可纠正错误，多比特 ECC）导致数据损坏，需要人工干预。

3. **ECC 原理**：每 128 字节数据附加 8 字节 ECC 码（SECDED），硬件自动纠正单比特错误。约 6% 的 HBM 容量用于 ECC 保护。

4. **监控接口**：sysfs（ras_ecc_cnt、ras_status、ras_features）、debugfs（amdgpu_ras_eeprom_test）、rasdaemon 守护进程提供多层次的 RAS 监控能力。

5. **错误响应**：CE 错误自动纠正并计数；UE 错误触发中断、记录地址、隔离页面；Fatal 错误触发 GPU reset。

### 扩展思考

1. **ECC 与性能**：启用 ECC 会占用部分 HBM 容量（~6%）并在读写路径上引入额外延迟。这对 HPC/数据中心场景是可以接受的，但对游戏场景是否值得启用？

2. **预测性故障分析**：CE 错误率的增长趋势能否用于预测 GPU 寿命？如何区分正常老化导致的错误率和预示即将故障的错误率？

3. **多 GPU 系统的 RAS**：在 MI250/MI300 等多 GPU 系统中，RAS 信息如何聚合？XGMI 互连的 RAS 如何与 UMC RAS 协同工作？
