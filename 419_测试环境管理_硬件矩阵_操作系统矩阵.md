# 测试环境管理：硬件矩阵 / 操作系统矩阵

## 学习目标

- 理解 GPU 驱动测试环境管理的复杂性和关键维度
- 掌握硬件矩阵设计方法，包括 GPU 型号、显存、PCIe 版本等维度
- 学会操作系统矩阵设计，覆盖不同发行版和内核版本
- 了解测试环境版本锁定和基线管理策略
- 能够设计可维护的测试环境矩阵并编写环境管理脚本

## 知识详解

### 一、概念原理

#### 1.1 GPU 驱动测试环境的多维复杂性

GPU 驱动测试需要在多个维度上进行组合覆盖，形成复杂的测试矩阵：

```
┌─────────────────────────────────────────────────────────────────────┐
│              GPU 驱动测试环境多维矩阵                                │
│                                                                     │
│   维度 1: GPU 硬件                                                   │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐              │
│   │ 架构代际 │ 型号    │ 显存    │ PCIe    │ 显示接口│              │
│   ├─────────┼─────────┼─────────┼─────────┼─────────┤              │
│   │ RDNA3   │ RX 7900 │ 24GB    │ Gen4    │ DP 2.1  │              │
│   │ RDNA2   │ RX 6900 │ 16GB    │ Gen4    │ DP 1.4  │              │
│   │ RDNA1   │ RX 5700 │ 8GB     │ Gen4    │ DP 1.4  │              │
│   │ CDNA3   │ MI300X  │ 192GB   │ Gen5    │ -       │              │
│   │ CDNA2   │ MI250X  │ 128GB   │ Gen4    │ -       │              │
│   │ GCN 5   │ Vega 64 │ 8GB     │ Gen3    │ DP 1.4  │              │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘              │
│                                                                     │
│   维度 2: 软件栈                                                     │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐              │
│   │ 内核版本 │ Mesa版本│ 固件版本│ ROCm版本│ 显示服务│              │
│   ├─────────┼─────────┼─────────┼─────────┼─────────┤              │
│   │ 6.8     │ 24.0    │ 2024Q1  │ 6.0     │ Wayland │              │
│   │ 6.7     │ 23.3    │ 2023Q4  │ 5.7     │ X11     │              │
│   │ 6.6 LTS │ 23.2    │ 2023Q3  │ 5.6     │ 两者    │              │
│   │ 5.15 LTS│ 22.3    │ 2023Q1  │ 5.4     │ X11     │              │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘              │
│                                                                     │
│   维度 3: 系统配置                                                   │
│   ┌─────────┬─────────┬─────────┬─────────┬─────────┐              │
│   │ CPU平台 │ 内存    │ 主板    │ 电源    │ 散热    │              │
│   ├─────────┼─────────┼─────────┼─────────┼─────────┤              │
│   │ Zen4    │ 64GB    │ X670E   │ 1000W   │ 水冷    │              │
│   │ Zen3    │ 32GB    │ B550    │ 750W    │ 风冷    │              │
│   │ EPYC    │ 512GB   │ SP5     │ 2000W   │ 风冷    │              │
│   └─────────┴─────────┴─────────┴─────────┴─────────┘              │
│                                                                     │
│   组合爆炸：6 GPU × 4 内核 × 4 Mesa × 3 CPU = 288 种组合           │
```

#### 1.2 矩阵精简策略

由于完全覆盖所有组合不现实，需要采用科学的精简策略：

```
┌─────────────────────────────────────────────────────────────────────┐
│              测试矩阵精简策略                                        │
│                                                                     │
│  策略 1: 正交数组 (Orthogonal Array)                                │
│  ─────────────────────────────────────                              │
│  不是测试所有组合，而是确保每对参数值至少出现一次                    │
│                                                                     │
│  示例：3 参数 × 3 值 = 9 组合 → 精简为 4 组合                      │
│  ┌─────────┬─────────┬─────────┐                                   │
│  │ GPU     │ 内核    │ Mesa    │                                   │
│  ├─────────┼─────────┼─────────┤                                   │
│  │ RDNA3   │ 6.8     │ 24.0    │  ← 最新组合                       │
│  │ RDNA3   │ 6.6 LTS │ 23.2    │  ← 新硬件 + 稳定软件              │
│  │ RDNA2   │ 6.8     │ 23.3    │  ← 旧硬件 + 新软件                │
│  │ CDNA2   │ 6.6 LTS │ 24.0    │  ← 计算卡组合                     │
│  └─────────┴─────────┴─────────┘                                   │
│                                                                     │
│  策略 2: 风险驱动选择                                                │
│  ─────────────────────────────────────                              │
│  优先覆盖高风险组合：                                                │
│  ● 新硬件 + 新软件（回归风险高）                                    │
│  ● 旧硬件 + 新软件（兼容性风险）                                    │
│  ● 客户报告问题的特定组合                                           │
│                                                                     │
│  策略 3: 等价类划分                                                  │
│  ─────────────────────────────────────                              │
│  将相似硬件/软件归为等价类：                                        │
│  ● RDNA3 系列：RX 7900 XTX / XT（同一驱动代码路径）                 │
│  ● 6.x 内核系列：6.6 / 6.7 / 6.8（DRM 子系统相似）                  │
│  ● LTS 内核：6.6 LTS / 5.15 LTS（长期支持分支）                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 二、实践操作

#### 2.1 硬件矩阵管理脚本

```bash
#!/bin/bash
# hardware_matrix_manager.sh - 硬件测试矩阵管理
# 参考：AMD 内部测试实验室管理实践

set -e

# 硬件矩阵定义
# 格式: "设备ID:GPU型号:架构:显存:PCIe版本:显示接口:标签"
declare -a HARDWARE_MATRIX=(
    "gpu-01:RX_7900_XTX:RDNA3:24GB:Gen4:DP2.1:rdna3,high-end,gaming"
    "gpu-02:RX_7900_XT:RDNA3:20GB:Gen4:DP2.1:rdna3,high-end,gaming"
    "gpu-03:RX_7800_XT:RDNA3:16GB:Gen4:DP2.1:rdna3,mid-range,gaming"
    "gpu-04:RX_6950_XT:RDNA2:16GB:Gen4:DP1.4:rdna2,high-end,gaming"
    "gpu-05:RX_6750_XT:RDNA2:12GB:Gen4:DP1.4:rdna2,mid-range,gaming"
    "gpu-06:RX_6650_XT:RDNA2:8GB:Gen4:DP1.4:rdna2,entry-level,gaming"
    "gpu-07:RX_5700_XT:RDNA1:8GB:Gen4:DP1.4:rdna1,mid-range,gaming"
    "gpu-08:MI300X:CDNA3:192GB:Gen5:none:cdna3,data-center,compute"
    "gpu-09:MI250X:CDNA2:128GB:Gen4:none:cdna2,data-center,compute"
    "gpu-10:MI100:CDNA1:32GB:Gen4:none:cdna1,data-center,compute"
    "gpu-11:RX_Vega_64:GCN5:8GB:Gen3:DP1.4:gcn5,legacy,gaming"
    "gpu-12:Ryzen_9_7945HX:iGPU:RDNA3:共享:Gen4:eDP:apu,mobile,integrated"
)

# 获取设备信息
get_device_info() {
    local device_id=$1
    for entry in "${HARDWARE_MATRIX[@]}"; do
        if [[ $entry == "$device_id:"* ]]; then
            echo "$entry"
            return 0
        fi
    done
    return 1
}

# 按标签筛选设备
filter_devices_by_tag() {
    local tag=$1
    for entry in "${HARDWARE_MATRIX[@]}"; do
        IFS=':' read -r dev_id gpu_model arch vram pcie display tags <<< "$entry"
        if [[ ",$tags," == *,"$tag",* ]]; then
            echo "$dev_id:$gpu_model:$arch"
        fi
    done
}

# 检测设备实际状态
check_device_health() {
    local device_id=$1
    echo "=== 检查设备 $device_id 健康状态 ==="
    
    # 检查 GPU 是否可见
    if ! lspci -nn | grep -q "1002:"; then
        echo "[FAIL] 未检测到 AMD GPU"
        return 1
    fi
    
    # 获取 GPU 信息
    local gpu_info=$(lspci -nn -d 1002: | head -1)
    echo "[INFO] GPU 信息: $gpu_info"
    
    # 检查 DRM 设备
    if [ -d /sys/class/drm/card0 ]; then
        echo "[PASS] DRM 设备存在"
        
        # 读取温度
        if [ -f /sys/class/drm/card0/device/hwmon/hwmon0/temp1_input ]; then
            local temp=$(cat /sys/class/drm/card0/device/hwmon/hwmon0/temp1_input)
            local temp_c=$((temp / 1000))
            echo "[INFO] GPU 温度: ${temp_c}°C"
            
            if [ "$temp_c" -gt 90 ]; then
                echo "[WARN] GPU 温度过高"
            fi
        fi
        
        # 检查显存
        if [ -f /sys/class/drm/card0/device/mem_info_vram_total ]; then
            local vram=$(cat /sys/class/drm/card0/device/mem_info_vram_total)
            local vram_gb=$((vram / 1024 / 1024 / 1024))
            echo "[INFO] 显存总量: ${vram_gb}GB"
        fi
        
        # 检查功耗
        if [ -f /sys/class/drm/card0/device/hwmon/hwmon0/power1_average ]; then
            local power=$(cat /sys/class/drm/card0/device/hwmon/hwmon0/power1_average)
            local power_w=$(echo "scale=2; $power / 1000000" | bc)
            echo "[INFO] 当前功耗: ${power_w}W"
        fi
    else
        echo "[FAIL] DRM 设备不存在"
        return 1
    fi
    
    return 0
}

# 生成测试矩阵报告
generate_matrix_report() {
    echo "# GPU 硬件测试矩阵报告"
    echo "生成时间: $(date)"
    echo ""
    echo "| 设备ID | GPU型号 | 架构 | 显存 | PCIe | 显示接口 | 标签 | 状态 |"
    echo "|--------|---------|------|------|------|----------|------|------|"
    
    for entry in "${HARDWARE_MATRIX[@]}"; do
        IFS=':' read -r dev_id gpu_model arch vram pcie display tags <<< "$entry"
        
        # 简化状态检查（实际应远程检查）
        local status="unknown"
        if [ -f "/var/run/gpu-status/$dev_id.status" ]; then
            status=$(cat "/var/run/gpu-status/$dev_id.status")
        fi
        
        echo "| $dev_id | $gpu_model | $arch | $vram | $pcie | $display | $tags | $status |"
    done
}

# 主函数
case "${1:-report}" in
    info)
        get_device_info "$2"
        ;;
    filter)
        filter_devices_by_tag "$2"
        ;;
    health)
        check_device_health "$2"
        ;;
    report)
        generate_matrix_report
        ;;
    *)
        echo "用法: $0 {info|filter|health|report} [参数]"
        exit 1
        ;;
esac
```

#### 2.2 操作系统矩阵管理

```bash
#!/bin/bash
# os_matrix_manager.sh - 操作系统测试矩阵管理
# 参考：Linux 发行版 GPU 驱动测试实践

set -e

# 操作系统矩阵定义
# 格式: "发行版:版本:内核版本:Mesa版本:ROCm版本:显示服务:包管理器"
declare -a OS_MATRIX=(
    "Ubuntu:24.04_LTS:6.8:24.0:6.0:Wayland:apt"
    "Ubuntu:22.04_LTS:6.5:23.2:5.7:X11:apt"
    "Fedora:40:6.8:24.0:6.0:Wayland:dnf"
    "Fedora:39:6.7:23.3:5.7:Wayland:dnf"
    "Debian:12:6.1:22.3:none:X11:apt"
    "openSUSE:Tumbleweed:6.8:24.0:6.0:Wayland:zypper"
    "Arch:rolling:6.8:24.0:6.0:Wayland:pacman"
    "RHEL:9.3:5.14:23.1:5.5:X11:dnf"
    "Rocky:9.3:5.14:23.1:5.5:X11:dnf"
)

# 获取当前系统信息
get_current_os() {
    if [ -f /etc/os-release ]; then
        . /etc/os-release
        echo "$NAME:$VERSION_ID"
    else
        echo "unknown:unknown"
    fi
}

get_current_kernel() {
    uname -r | cut -d'-' -f1-2
}

get_current_mesa() {
    if command -v glxinfo &> /dev/null; then
        glxinfo | grep "OpenGL version string" | awk '{print $4}'
    else
        echo "unknown"
    fi
}

# 安装 GPU 驱动依赖
install_gpu_dependencies() {
    local distro=$1
    local pkg_manager=$2
    
    echo "=== 在 $distro 上安装 GPU 驱动依赖 ==="
    
    case "$pkg_manager" in
        apt)
            sudo apt-get update
            sudo apt-get install -y \
                build-essential \
                git \
                meson \
                ninja-build \
                libdrm-dev \
                libegl1-mesa-dev \
                libgl1-mesa-dev \
                libgbm-dev \
                libvulkan-dev \
                vulkan-tools \
                mesa-utils \
                xorg-dev \
                wayland-protocols \
                libwayland-dev
            ;;
        dnf)
            sudo dnf install -y \
                gcc \
                gcc-c++ \
                git \
                meson \
                ninja-build \
                libdrm-devel \
                mesa-libEGL-devel \
                mesa-libGL-devel \
                mesa-libgbm-devel \
                vulkan-devel \
                vulkan-tools \
                mesa-dri-drivers \
                libX11-devel \
                wayland-devel
            ;;
        zypper)
            sudo zypper install -y \
                gcc \
                gcc-c++ \
                git \
                meson \
                ninja \
                libdrm-devel \
                Mesa-libEGL-devel \
                Mesa-libGL-devel \
                libgbm-devel \
                vulkan-devel \
                vulkan-tools \
                Mesa-dri \
                libX11-devel \
                wayland-devel
            ;;
        pacman)
            sudo pacman -S --needed \
                base-devel \
                git \
                meson \
                ninja \
                libdrm \
                mesa \
                vulkan-tools \
                xorg-server-devel \
                wayland
            ;;
        *)
            echo "不支持的包管理器: $pkg_manager"
            return 1
            ;;
    esac
}

# 验证 ROCm 安装
verify_rocm() {
    local rocm_version=$1
    
    echo "=== 验证 ROCm $rocm_version 安装 ==="
    
    if command -v rocminfo &> /dev/null; then
        echo "[PASS] rocminfo 可用"
        rocminfo | head -20
    else
        echo "[FAIL] rocminfo 不可用"
        return 1
    fi
    
    if command -v rocm-smi &> /dev/null; then
        echo "[PASS] rocm-smi 可用"
        rocm-smi --showproductname
    else
        echo "[FAIL] rocm-smi 不可用"
        return 1
    fi
    
    return 0
}

# 生成环境报告
generate_env_report() {
    echo "# GPU 驱动测试环境报告"
    echo "生成时间: $(date)"
    echo ""
    
    echo "## 当前系统信息"
    echo "- 操作系统: $(get_current_os)"
    echo "- 内核版本: $(get_current_kernel)"
    echo "- Mesa 版本: $(get_current_mesa)"
    echo ""
    
    echo "## 已安装 GPU 工具"
    for tool in glxinfo vkcube rocminfo rocm-smi amdgpu_top; do
        if command -v "$tool" &> /dev/null; then
            echo "- $tool: $(which $tool)"
        else
            echo "- $tool: [未安装]"
        fi
    done
    echo ""
    
    echo "## GPU 硬件信息"
    if [ -d /sys/class/drm/card0 ]; then
        echo "- DRM 设备: 存在"
        if [ -f /sys/class/drm/card0/device/vendor ]; then
            local vendor=$(cat /sys/class/drm/card0/device/vendor)
            echo "- 厂商 ID: $vendor"
        fi
        if [ -f /sys/class/drm/card0/device/device ]; then
            local device=$(cat /sys/class/drm/card0/device/device)
            echo "- 设备 ID: $device"
        fi
    else
        echo "- DRM 设备: [不存在]"
    fi
}

# 主函数
case "${1:-report}" in
    install)
        install_gpu_dependencies "$2" "$3"
        ;;
    verify-rocm)
        verify_rocm "$2"
        ;;
    report)
        generate_env_report
        ;;
    *)
        echo "用法: $0 {install|verify-rocm|report} [参数]"
        exit 1
        ;;
esac
```

#### 2.3 环境快照与恢复

```bash
#!/bin/bash
# env_snapshot.sh - 测试环境快照与恢复
# 参考：Phoronix Test Suite 环境管理实践

set -e

SNAPSHOT_DIR="/var/lib/gpu-test-snapshots"

# 创建环境快照
create_snapshot() {
    local snapshot_name=$1
    local snapshot_path="$SNAPSHOT_DIR/$snapshot_name"
    
    echo "=== 创建环境快照: $snapshot_name ==="
    mkdir -p "$snapshot_path"
    
    # 保存系统信息
    {
        echo "# 环境快照: $snapshot_name"
        echo "创建时间: $(date -Iseconds)"
        echo ""
        echo "## 操作系统"
        cat /etc/os-release
        echo ""
        echo "## 内核版本"
        uname -a
        echo ""
        echo "## 已安装内核模块"
        lsmod | grep -E "amdgpu|drm" || true
        echo ""
        echo "## Mesa 版本"
        glxinfo 2>/dev/null | grep "OpenGL version string" || echo "glxinfo 不可用"
        echo ""
        echo "## 固件版本"
        cat /sys/class/drm/card0/device/fw_version 2>/dev/null || echo "固件版本不可用"
        echo ""
        echo "## 已安装包列表 (GPU 相关)"
        dpkg -l | grep -E "mesa|amdgpu|rocm" 2>/dev/null || \
        rpm -qa | grep -E "mesa|amdgpu|rocm" 2>/dev/null || \
        pacman -Q | grep -E "mesa|amdgpu|rocm" 2>/dev/null || \
        echo "包信息不可用"
    } > "$snapshot_path/system-info.txt"
    
    # 保存内核配置
    if [ -f /boot/config-$(uname -r) ]; then
        cp "/boot/config-$(uname -r)" "$snapshot_path/kernel-config"
    fi
    
    # 保存 modprobe 配置
    if [ -d /etc/modprobe.d ]; then
        cp -r /etc/modprobe.d "$snapshot_path/"
    fi
    
    # 保存 X11/Wayland 配置
    if [ -d /etc/X11/xorg.conf.d ]; then
        cp -r /etc/X11/xorg.conf.d "$snapshot_path/" 2>/dev/null || true
    fi
    
    echo "[PASS] 快照已保存到: $snapshot_path"
}

# 恢复快照
restore_snapshot() {
    local snapshot_name=$1
    local snapshot_path="$SNAPSHOT_DIR/$snapshot_name"
    
    if [ ! -d "$snapshot_path" ]; then
        echo "[FAIL] 快照不存在: $snapshot_name"
        return 1
    fi
    
    echo "=== 恢复环境快照: $snapshot_name ==="
    
    # 显示快照信息
    if [ -f "$snapshot_path/system-info.txt" ]; then
        echo "快照信息:"
        head -20 "$snapshot_path/system-info.txt"
    fi
    
    # 恢复 modprobe 配置
    if [ -d "$snapshot_path/modprobe.d" ]; then
        echo "恢复 modprobe 配置..."
        sudo cp -r "$snapshot_path/modprobe.d"/* /etc/modprobe.d/ 2>/dev/null || true
    fi
    
    # 恢复 X11 配置
    if [ -d "$snapshot_path/xorg.conf.d" ]; then
        echo "恢复 X11 配置..."
        sudo cp -r "$snapshot_path/xorg.conf.d"/* /etc/X11/xorg.conf.d/ 2>/dev/null || true
    fi
    
    echo "[PASS] 快照已恢复"
    echo "注意：内核版本和驱动版本需要手动恢复"
}

# 列出所有快照
list_snapshots() {
    echo "=== 可用环境快照 ==="
    if [ ! -d "$SNAPSHOT_DIR" ] || [ -z "$(ls -A "$SNAPSHOT_DIR" 2>/dev/null)" ]; then
        echo "无可用快照"
        return
    fi
    
    for snapshot in "$SNAPSHOT_DIR"/*; do
        if [ -d "$snapshot" ]; then
            local name=$(basename "$snapshot")
            local created=$(stat -c %y "$snapshot" 2>/dev/null || stat -f %Sm "$snapshot" 2>/dev/null)
            echo "- $name (创建: $created)"
        fi
    done
}

# 主函数
case "${1:-list}" in
    create)
        create_snapshot "$2"
        ;;
    restore)
        restore_snapshot "$2"
        ;;
    list)
        list_snapshots
        ;;
    *)
        echo "用法: $0 {create|restore|list} [快照名称]"
        exit 1
        ;;
esac
```

### 三、案例分析

#### 3.1 AMD ROCm 平台支持矩阵

ROCm 平台有明确的硬件和操作系统支持矩阵，这是 GPU 驱动测试环境设计的重要参考：

```
┌─────────────────────────────────────────────────────────────────────┐
│              AMD ROCm 6.0 官方支持矩阵                               │
│                                                                     │
```

#### 3.2 内核版本兼容性测试案例

**背景**：测试 AMDGPU 驱动在不同内核版本上的兼容性。

**测试矩阵设计**：

| 内核版本 | AMDGPU 版本 | 测试重点 | 预期结果 |
|---------|------------|---------|---------|
| 6.8 (最新) | 主线 | 新特性、性能 | 完全通过 |
| 6.6 LTS | 主线 | 稳定性、长期支持 | 完全通过 |
| 5.15 LTS | backport | 向后兼容 | 核心功能通过 |
| 5.10 LTS | backport | 向后兼容 | 基础功能通过 |

**自动化脚本**：

```bash
#!/bin/bash
# kernel_compat_test.sh - 内核兼容性测试
# 参考：drm-misc 内核兼容性测试实践

KERNEL_VERSIONS=("6.8" "6.7" "6.6" "5.15")
TEST_CASES=("load" "modeset" "render" "compute")

for kernel in "${KERNEL_VERSIONS[@]}"; do
    echo "=== 测试内核 $kernel ==="
    
    # 启动对应内核的测试环境（通过 PXE 或容器）
    boot_test_environment "$kernel"
    
    for test in "${TEST_CASES[@]}"; do
        echo "  执行测试: $test"
        
        case "$test" in
            load)
                sudo modprobe amdgpu
                lsmod | grep amdgpu && echo "  [PASS] 驱动加载" || echo "  [FAIL] 驱动加载"
                ;;
            modeset)
                kmscube --drm-device=/dev/dri/card0 && echo "  [PASS] 模式设置" || echo "  [FAIL] 模式设置"
                ;;
            render)
                glmark2 --off-screen && echo "  [PASS] 渲染测试" || echo "  [FAIL] 渲染测试"
                ;;
            compute)
                rocminfo && echo "  [PASS] 计算测试" || echo "  [SKIP] 计算测试 (ROCm 未安装)"
                ;;
        esac
    done
done
```

## 今日小结

- GPU 驱动测试环境管理面临硬件矩阵、软件栈矩阵、系统配置矩阵的多维组合爆炸问题
- 测试矩阵精简策略包括正交数组、风险驱动选择和等价类划分
- 硬件矩阵应覆盖不同架构代际、市场定位和使用场景
- 操作系统矩阵应覆盖主流发行版和关键内核版本
- 环境快照与恢复机制是保障测试可重复性的重要手段
