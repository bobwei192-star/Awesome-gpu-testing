### 第三部分：简答题

**1. amdgpu_pci_probe() 的 8 个阶段：**

Phase 1：pcim_enable_device — 启用 PCI 设备，使能总线主控和内存空间
Phase 2：pcim_iomap_regions — 映射 BAR 空间，建立 MMIO 访问通道
Phase 3：dma_set_mask — 配置 DMA 掩码（64-bit/32-bit），确定 DMA 地址范围
Phase 4：pci_enable_msi/msix — 配置 MSI/MSI-X 中断，建立中断传递通道
Phase 5：drm_dev_alloc — 分配 DRM 设备结构体，初始化 DRM 子系统
Phase 6：amdgpu_detect_ip_versions — 检测 IP 版本，解析 IP Discovery 表
Phase 7：amdgpu_device_init — 初始化 AMDGPU 设备，执行 IP 块初始化序列
Phase 8：drm_dev_register — 注册 DRM 设备到系统，暴露用户空间接口

**2. DKMS 完整工作流程：**

add 阶段：将源码目录 /usr/src/<module>-<version> 注册到 DKMS，在 /var/lib/dkms/ 下创建符号链接，记录模块元数据
build 阶段：将源码复制到 /var/lib/dkms/<module>/<version>/build/，调用 `make -C /lib/modules/<kernel>/build M=<build_dir> modules` 编译，生成 .ko 文件
install 阶段：将编译好的 .ko 复制到 /lib/modules/<kernel>/kernel/drivers/extra/ 或 dkms.conf 指定的位置，运行 depmod 更新模块依赖，可选执行 initramfs 重建

**3. GPU CHIP 类型和 IP 版本确定流程：**

第一步：PCI 子系统枚举 PCI 总线，发现 GPU 设备后读取 Vendor ID（0x1002=AMD）和 Device ID（如 0x73BF=Navi21、0x744C=Navi31、0x740C=Aldebaran）
第二步：amdgpu 驱动的 pciidlist[] 表将 Device ID 映射到 CHIP 枚举值（CHIP_NAVI21/CHIP_NAVI31/CHIP_ALDEBARAN），确定 GPU 架构世代
第三步：amdgpu_detect_ip_versions() 读取 GPU 的 IP Discovery 表（存储在显存或 VBIOS 中），解析出每个 IP 块的类型（GFX/SDMA/GMC/VCN 等）、Major/Minor/Revision 版本号
第四步：根据 CHIP 类型和 IP 版本，amdgpu_device_init() 选择对应的 IP 初始化函数指针（如 gfx_v10_0_init 用于 Navi2x、gfx_v11_0_init 用于 Navi3x），执行完整的设备初始化序列
