# 322 自适应功耗管理算法分析

## 1. 概述

### 1.1 什么是自适应功耗管理

```text
自适应功耗管理 (Adaptive Power Management, APM) 是 GPU 驱动中
根据实时工作负载特征、温度、功耗预算等动态调整电源状态的技术体系。

与传统静态电源管理相比, 自适应方案的核心特点:

  传统方案:
    - 基于固定阈值 (如温度 > 85°C 降频)
    - 响应延迟大 (秒级)
    - 缺乏负载感知
    - 能效优化粗放

  自适应方案:
    - 基于连续观测和预测
    - 响应延迟小 (微秒级)
    - 深度感知负载特征
    - 精细化的能效优化

  ┌─────────────────────────────────────────────────────────────┐
  │  自适应功耗管理的闭环控制流程:                                │
  │                                                             │
  │  ┌──────────┐    ┌───────────┐    ┌───────────┐             │
  │  │ 传感器   │───▶│ 状态估计  │───▶│ 策略决策  │             │
  │  │ (功耗/   │    │ (当前状态 │    │ (目标频率 │             │
  │  │  温度/   │    │  和趋势)  │    │  和电压)  │             │
  │  │  负载)   │    │           │    │           │             │
  │  └──────────┘    └───────────┘    └─────┬─────┘             │
  │       ▲                                  │                  │
  │       │                                  ▼                  │
  │  ┌────┴─────┐    ┌───────────┐    ┌───────────┐             │
  │  │ 执行器   │◀───│ 策略映射  │◀───│ 预测模型  │             │
  │  │ (P-state │    │ (频率→    │    │ (负载/    │             │
  │  │  切换)   │    │  P-state) │    │  功耗预测)│             │
  │  └──────────┘    └───────────┘    └───────────┘             │
  └─────────────────────────────────────────────────────────────┘
```

### 1.2 自适应算法的分类

```text
自适应功耗管理算法按策略类型可分为以下几类:

  ┌─────────────────────────────────────────────────────────────┐
  │  算法类别          │ 典型算法               │ 适用场景      │
  ├───────────────────┼──────────────────────┼───────────────┤
  │ 基于阈值          │ 滞回比较器            │ 简单负载场景   │
  │  (Threshold)      │ 多级阈值              │ 快速响应要求   │
  ├───────────────────┼──────────────────────┼───────────────┤
  │ 基于预测          │ 指数平滑              │ 周期性负载     │
  │  (Prediction)     │ 自回归 (AR)           │ 可预测负载     │
  │                   │ 卡尔曼滤波            │ 噪声环境       │
  ├───────────────────┼──────────────────────┼───────────────┤
  │ 基于控制理论      │ PID 控制              │ 稳定控制       │
  │  (Control)        │ 模型预测控制 (MPC)    │ 约束优化       │
  │                   │ 自适应控制            │ 时变系统       │
  ├───────────────────┼──────────────────────┼───────────────┤
  │ 基于学习          │ Q-learning            │ 复杂负载       │
  │  (Learning)       │ DQN                   │ 长期优化       │
  │                   │ 监督学习              │ 负载分类       │
  └─────────────────────────────────────────────────────────────┘

  实际 GPU 驱动中通常采用混合策略:
    - 短期: PID + 阈值控制 (微秒级响应)
    - 中期: 预测算法 (毫秒级调整)
    - 长期: 学习算法 (秒级优化)
```

### 1.3 自适应管理的目标

```text
自适应功耗管理的多目标优化:

  目标 1: 性能最大化 (Performance)
    在功耗预算内, 最大化 FPS 或吞吐量
    度量: FPS, TFLOPS, 作业完成时间

  目标 2: 能效优化 (Energy Efficiency)
    最小化每帧或每操作的能耗
    度量: FPS/W, Joules/Frame

  目标 3: 温度控制 (Thermal Management)
    将温度控制在安全范围内
    度量: T_junction, dT/dt

  目标 4: 公平性 (Fairness)
    多进程/多用户场景下公平分配 GPU 资源
    度量: Jain's Fairness Index

  目标 5: 稳定性 (Stability)
    避免频率和电压的剧烈波动
    度量: 频率标准差, P-state 切换频率

  多目标优化函数:

  minimize:  J = w1 * (1 - Perf/P_max) +
                  w2 * Power/P_maxPower +
                  w3 * max(0, Temp - T_safe)/T_margin +
                  w4 * (1 - Fairness)

  其中 w1..w4 是权重系数, 可根据使用场景动态调整

  约束条件:
    - 温度 < T_junction_max
    - 功耗 < Power_budget
    - 频率 ∈ [f_min, f_max]
    - 电压 ∈ [v_min, v_max]
    - dF/dt < ramp_rate_max
```

## 2. DVFS 算法基础

### 2.1 DVFS 基本原理

```text
DVFS (Dynamic Voltage and Frequency Scaling) 是自适应功耗管理的核心机制:

  功率与频率/电压的关系:

  P_dynamic = α × C × V² × f

  其中:
    α: 活动因子 (0~1), 表示电路翻转比例
    C: 等效电容
    V: 核心电压
    f: 工作频率

  关键洞察:
    - 功耗与电压平方成正比 (V²)
    - 功耗与频率线性相关 (f)
    - 频率增加通常需要同时增加电压
    - 降低频率可以大幅降压, 从而显著降低功耗

  频率-电压关系 (F-V Curve):

  电压(V)
    │
  1.2├────●────●────●────●──
    │         P0  P1  P2  P3
  1.0├────●───────────────
    │      P4
  0.8├────●───────────────
    │      P5
  0.6├────●───────────────
    │      P6
    └─────────────────────── 频率(GHz)
        0.5 1.0 1.5 2.0 2.5

  P-state 表 (示例):

  P-state  频率(GHz)  电压(V)  功耗(W)  能效(FPS/W)
  P0        2.5        1.20     250      0.32
  P1        2.2        1.10     200      0.38
  P2        2.0        1.05     170      0.42
  P3        1.7        0.95     130      0.49
  P4        1.5        0.88     105      0.54
  P5        1.2        0.78     80       0.60
  P6        0.8        0.65     50       0.68
  P7        0.5        0.55     30       0.72
```

### 2.3 频率缩放策略

```text
常见的频率缩放策略:

  ┌─────────────────────────────────────────────────────────────┐
  │  策略 1: Ondemand (按需)                                     │
  │                                                             │
  │  原理: 根据当前负载立即调整频率                                │
  │                                                             │
  │  if (load > up_threshold)                                    │
  │      freq = max_freq                                         │
  │  else if (load < down_threshold)                             │
  │      freq = max_freq * load / up_threshold                   │
  │  else                                                        │
  │      hold current freq                                       │
  │                                                             │
  │  优点: 响应快, 实现简单                                      │
  │  缺点: 频率波动大, 可能导致性能抖动                           │
  │                                                             │
  │  策略 2: Conservative (保守)                                 │
  │                                                             │
  │  原理: 逐步调整频率, 避免大幅波动                              │
  │                                                             │
  │  if (load > up_threshold)                                    │
  │      freq = next_higher(freq)   /* 每次只升一级 */            │
  │  else if (load < down_threshold)                             │
  │      freq = next_lower(freq)    /* 每次只降一级 */            │
  │                                                             │
  │  优点: 频率平滑, 稳定性好                                     │
  │  缺点: 响应慢, 可能跟不上负载变化                             │
  │                                                             │
  │  策略 3: Performance (性能)                                  │
  │                                                             │
  │  原理: 始终运行在最高频率                                     │
  │                                                             │
  │  freq = max_freq                                             │
  │                                                             │
  │  优点: 最低延迟, 最佳性能                                     │
  │  缺点: 功耗最大                                              │
  │                                                             │
  │  策略 4: Powersave (节能)                                    │
  │                                                             │
  │  原理: 始终运行在最低频率                                     │
  │                                                             │
  │  freq = min_freq                                             │
  │                                                             │
  │  优点: 功耗最小                                              │
  │  缺点: 性能最差                                              │
  │                                                             │
  │  策略 5: Adaptive (自适应)                                   │
  │                                                             │
  │  原理: 根据负载预测和能效模型动态调整                           │
  │                                                             │
  │  target_freq = predict_optimal_freq(                         │
  │      load_history, power_budget, temp)                      │
  │  freq = clamp(target_freq, min_freq, max_freq)              │
  │                                                             │
  │  优点: 能效优化, 兼顾性能和功耗                               │
  │  缺点: 实现复杂, 需要精准的预测模型                           │
  └─────────────────────────────────────────────────────────────┘
```

### 2.4 电压缩放与 AVS

```text
AVS (Adaptive Voltage Scaling) 是 DVFS 的高级形式:

  传统 DVFS 的局限性:
    - 使用固定的 F-V 表, 针对最差工艺角 (Worst-Case Corner)
    - 实际芯片可能比最差角好很多 (频率余量 10-30%)
    - 固定电压导致不必要的功耗

  AVS 原理:

  ┌─────────────────────────────────────────────────────────────┐
  │  每个 GPU 芯片在制造时具有不同的电气特性:                     │
  │                                                             │
  │  芯片 A (Fast):  2.0GHz @ 0.95V                            │
  │  芯片 B (Typical): 2.0GHz @ 1.00V                          │
  │  芯片 C (Slow):  2.0GHz @ 1.10V                            │
  │                                                             │
  │  Fuse 中存储了每个芯片的速度 bin 信息:                       │
  │    - Speed bin: [0x3A] 指示该芯片所属的速度等级              │
  │    - F-V offset: 每个 P-state 的电压偏移量                  │
  │                                                             │
  │  AVS 硬件闭环控制:                                          │
  │                                                             │
  │  ┌────────┐    ┌──────────┐    ┌────────┐                   │
  │  │ Ring   │───▶│ 比较器   │───▶│ 电压   │                   │
  │  │ Osc.   │    │ (vs ref) │    │ 调节器 │                   │
  │  └────────┘    └──────────┘    └────┬───┘                   │
  │       ▲                             │                       │
  │       │                             ▼                       │
  │       └─────────────────────────────────┘                   │
  │                                                             │
  │  工作原理:                                                   │
  │    1. 芯片上集成环形振荡器 (Ring Oscillator)                  │
  │    2. 振荡频率反映当前硅片的电气特性                          │
  │    3. 与参考频率比较, 动态调整电压                            │
  │    4. 目标: 在给定频率下找到最低稳定电压                       │
  └─────────────────────────────────────────────────────────────┘

  AVS 实现代码示意:

  /* 读取速度 bin 信息 */
  static u32 amdgpu_avs_read_speed_bin(struct amdgpu_device *adev)
  {
      u32 fuse_val;
      /* 从 efuse 寄存器读取速度等级 */
      fuse_val = RREG32_SMC(mmFUSE_SPEED_BIN);
      return (fuse_val & FUSE_SPEED_BIN_MASK) >> FUSE_SPEED_BIN_SHIFT;
  }

  /* 计算优化的 F-V 表 */
  static int amdgpu_avs_calc_optimal_fv(
      struct amdgpu_device *adev,
      struct amdgpu_pstate *pstate)
  {
      u32 speed_bin = amdgpu_avs_read_speed_bin(adev);
      s32 voltage_offset;

      /* 根据速度 bin 计算电压偏移 */
      switch (speed_bin) {
      case SPEED_BIN_FAST:
          voltage_offset = -50;  /* 降低 50mV */
          break;
      case SPEED_BIN_TYPICAL:
          voltage_offset = 0;    /* 标准电压 */
          break;
      case SPEED_BIN_SLOW:
          voltage_offset = 50;   /* 增加 50mV */
          break;
      default:
          voltage_offset = 25;   /* 保守偏移 */
      }

      pstate->voltage_mv = pstate->base_voltage_mv + voltage_offset;
      return 0;
  }

  AVS 带来的功耗节省:

  场景                  AVS 关闭     AVS 开启     节省
  P0 (2.5GHz, Fast)     250W        220W       12%
  P4 (1.5GHz, Typical)  105W        100W       4.8%
  P7 (0.5GHz, Slow)     30W         32W        -6.7%*

  * 慢芯片在低频下可能需要更高电压才能在频率切换后稳定
```

## 3. 负载预测算法

### 3.1 指数平滑预测

```text
指数平滑 (Exponential Smoothing) 是最常用的负载预测方法:

  基本原理:

  pred_load(t+1) = α * actual_load(t) + (1-α) * pred_load(t)

  其中:
    α: 平滑因子 (0 < α < 1)
    α 越大, 对最近观测值的响应越快
    α 越小, 预测越平滑

  在 GPU 功耗管理中的应用:

  struct gpu_load_predictor {
      /* 预测器状态 */
      double      smooth_load;         /* 平滑后的负载 */
      double      trend;               /* 趋势分量 */
      double      seasonal[MAX_SEASON]; /* 周期性分量 */

      /* 参数 */
      double      alpha;               /* 水平平滑因子 */
      double      beta;                /* 趋势平滑因子 */
      double      gamma;               /* 季节性平滑因子 */
      u32         season_period;       /* 周期长度 (帧数) */

      /* 统计 */
      u64         samples;             /* 样本数 */
      double      mse;                 /* 均方误差 */
  };

  三重指数平滑 (Holt-Winters):

  /* 水平分量: 当前负载的基础水平 */
  level(t) = α * (actual(t) - seasonal(t - period))
           + (1-α) * (level(t-1) + trend(t-1))

  /* 趋势分量: 负载的变化趋势 */
  trend(t) = β * (level(t) - level(t-1))
           + (1-β) * trend(t-1)

  /* 季节性分量: 周期性模式 */
  seasonal(t) = γ * (actual(t) - level(t))
              + (1-γ) * seasonal(t - period)

  /* 预测: 提前 k 步 */
  pred(t+k) = level(t) + k * trend(t) + seasonal(t + k - period)

  /* 代码实现 */
  static double gpu_load_predict(struct gpu_load_predictor *pred,
                                  double actual_load)
  {
      double last_level, last_trend, season, pred_load;

      if (pred->samples < pred->season_period) {
          /* 冷启动: 使用简单指数平滑 */
          if (pred->samples == 0) {
              pred->smooth_load = actual_load;
              return actual_load;
          }
          pred->smooth_load = pred->alpha * actual_load
              + (1 - pred->alpha) * pred->smooth_load;
          return pred->smooth_load;
      }

      /* Holt-Winters 预测 */
      last_level = pred->smooth_load;
      last_trend = pred->trend;
      season = pred->seasonal[pred->samples % pred->season_period];

      /* 更新水平分量 */
      pred->smooth_load = pred->alpha * (actual_load - season)
          + (1 - pred->alpha) * (last_level + last_trend);

      /* 更新趋势分量 */
      pred->trend = pred->beta * (pred->smooth_load - last_level)
          + (1 - pred->beta) * last_trend;

      /* 更新季节性分量 */
      pred->seasonal[pred->samples % pred->season_period] =
          pred->gamma * (actual_load - pred->smooth_load)
          + (1 - pred->gamma) * season;

      /* 一步预测 */
      pred_load = pred->smooth_load + pred->trend
          + pred->seasonal[(pred->samples + 1) % pred->season_period];

      pred->samples++;
      return pred_load;
  }

  预测效果分析:

  α 值        响应速度     噪声抑制    适用场景
  0.1          慢           好         稳定负载
  0.3          中等         中等       一般用途
  0.5          快           差         快速变化负载
  0.8          极快         极差       突发负载
```

### 3.2 自回归模型 (AR)

```text
自回归 (Autoregressive, AR) 模型使用历史值的线性组合预测未来值:

  AR(p) 模型定义:

  y(t) = c + φ1*y(t-1) + φ2*y(t-2) + ... + φp*y(t-p) + ε(t)

  其中:
    y(t): 当前负载值
    c: 常数项
    φi: 自回归系数
    p: 模型阶数
    ε(t): 白噪声

  在 GPU 负载预测中的应用:

  struct ar_predictor {
      double  coefficients[MAX_AR_ORDER]; /* AR 系数 φ */
      double  history[MAX_AR_ORDER];      /* 历史负载 */
      int     order;                       /* 模型阶数 p */
      double  constant;                    /* 常数项 c */
      double  residual;                    /* 当前残差 */
  };

  /* 使用 Yule-Walker 方程估计 AR 系数 */
  static void ar_estimate_coefficients(
      struct ar_predictor *ar,
      double *samples, int n)
  {
      /* 1. 计算自协方差 */
      double acvf[MAX_AR_ORDER + 1];
      double mean = 0;

      for (int i = 0; i < n; i++)
          mean += samples[i];
      mean /= n;

      for (int k = 0; k <= ar->order; k++) {
          acvf[k] = 0;
          for (int i = k; i < n; i++) {
              acvf[k] += (samples[i] - mean)
                       * (samples[i - k] - mean);
          }
          acvf[k] /= n;
      }

      /* 2. 求解 Yule-Walker 方程 (Levinson-Durbin 递归) */
      double reflection[MAX_AR_ORDER];
      double pe = acvf[0]; /* 预测误差功率 */

      for (int i = 1; i <= ar->order; i++) {
          /* 计算反射系数 */
          double sum = acvf[i];
          for (int j = 1; j < i; j++)
              sum += ar->coefficients[j - 1] * acvf[i - j];
          reflection[i - 1] = -sum / pe;

          /* 更新 AR 系数 */
          for (int j = 1; j < i; j++) {
              ar->coefficients[ar->order - j] +=
                  reflection[i - 1] * ar->coefficients[ar->order - i + j];
          }
          ar->coefficients[ar->order - i] = reflection[i - 1];

          /* 更新预测误差功率 */
          pe *= (1 - reflection[i - 1] * reflection[i - 1]);
      }

      /* 3. 计算常数项 (假设零均值) */
      ar->constant = mean * (1 - 0);
      for (int i = 0; i < ar->order; i++)
          ar->constant -= ar->coefficients[i] * mean;
  }

  /* AR 模型一步预测 */
  static double ar_predict(struct ar_predictor *ar)
  {
      double pred = ar->constant;

      for (int i = 0; i < ar->order; i++)
          pred += ar->coefficients[i] * ar->history[i];

      return pred;
  }

  AR 模型阶数的选择:

  阶数 p  模型复杂度  预测精度    适用场景
  1        低          一般       平稳负载
  2        低          中等       线性趋势负载
  4        中等        好         周期性负载
  8        高          很好       复杂负载模式
  16       很高        可能过拟合  噪声数据

  AIC (Akaike Information Criterion) 定阶:

  AIC(p) = n * ln(RSS/n) + 2p

  其中:
    RSS: 残差平方和
    n: 样本数
    p: 模型阶数
    选择使 AIC 最小的 p 值
```

### 3.3 卡尔曼滤波

```text
卡尔曼滤波 (Kalman Filter) 在 GPU 负载预测中的应用:

  卡尔曼滤波是一种递归的状态估计方法,
  在存在测量噪声的情况下提供最优估计。

  系统模型:

  状态方程:  x(t+1) = A * x(t) + B * u(t) + w(t)
  观测方程:  z(t) = H * x(t) + v(t)

  其中:
    x(t): 系统状态 (GPU 负载, 功耗, 温度)
    A: 状态转移矩阵
    B: 控制输入矩阵
    u(t): 控制输入 (频率设置)
    w(t): 过程噪声 ~ N(0, Q)
    z(t): 观测值 (传感器读数)
    H: 观测矩阵
    v(t): 观测噪声 ~ N(0, R)

  应用于 GPU 功耗管理:

  ┌─────────────────────────────────────────────────────────────┐
  │  状态向量:                                                   │
  │                                                             │
  │  x(t) = [load(t), dload(t), power(t), temp(t)]^T            │
  │                                                             │
  │  其中:                                                      │
  │    load(t): 当前 GPU 负载 (0~1)                             │
  │    dload(t): 负载变化率                                      │
  │    power(t): 当前功耗 (W)                                   │
  │    temp(t): 当前温度 (°C)                                   │
  │                                                             │
  │  状态转移矩阵:                                               │
  │                                                             │
  │      ┌ 1  dt  0   0 ┐                                       │
  │      │ 0  1   0   0 │                                       │
  │  A = │ 0  0   1   0 │                                       │
  │      └ 0  0   0   1 ┘                                       │
  │                                                             │
  │  观测矩阵:                                                   │
  │                                                             │
  │  H = ┌ 1  0  0  0 ┐  (观测 GPU 利用率)                      │
  │      └ 0  0  1  0 ┘  (观测功耗)                              │
  └─────────────────────────────────────────────────────────────┘

  卡尔曼滤波的递推过程:

  /* 预测步骤 */
  static void kalman_predict(struct kalman_filter *kf)
  {
      /* 状态预测 */
      kf->x_pred = kf->A * kf->x_est + kf->B * kf->u;

      /* 协方差预测 */
      kf->P_pred = kf->A * kf->P_est * kf->A^T + kf->Q;
  }

  /* 更新步骤 */
  static void kalman_update(struct kalman_filter *kf,
                             struct measurement *z)
  {
      /* 卡尔曼增益 */
      Matrix S = kf->H * kf->P_pred * kf->H^T + kf->R;
      Matrix K = kf->P_pred * kf->H^T * S^(-1);

      /* 状态更新 */
      kf->x_est = kf->x_pred + K * (z - kf->H * kf->x_pred);

      /* 协方差更新 */
      kf->P_est = (I - K * kf->H) * kf->P_pred;
  }

  /* C 语言简化实现 */
  typedef struct {
      double x[4];      /* 状态向量 [load, dload, power, temp] */
      double P[4][4];   /* 状态协方差矩阵 */
      double Q[4][4];   /* 过程噪声协方差 */
      double R[2][2];   /* 观测噪声协方差 */
      double A[4][4];   /* 状态转移矩阵 */
      double H[2][4];   /* 观测矩阵 */
  } kalman_filter_t;

  void kalman_step(kalman_filter_t *kf,
                   double measured_load,
                   double measured_power)
  {
      int i, j;
      double x_pred[4], P_pred[4][4];
      double y[2], S[2][2], K[4][2];
      double tmp[4][4];

      /* === 预测 === */
      for (i = 0; i < 4; i++) {
          x_pred[i] = 0;
          for (j = 0; j < 4; j++)
              x_pred[i] += kf->A[i][j] * kf->x[j];
      }

      /* P_pred = A * P * A^T + Q */
      for (i = 0; i < 4; i++)
          for (j = 0; j < 4; j++) {
              tmp[i][j] = 0;
              for (int k = 0; k < 4; k++)
                  tmp[i][j] += kf->A[i][k] * kf->P[k][j];
          }
      for (i = 0; i < 4; i++)
          for (j = 0; j < 4; j++) {
              P_pred[i][j] = kf->Q[i][j];
              for (int k = 0; k < 4; k++)
                  P_pred[i][j] += tmp[i][k] * kf->A[j][k];
          }

      /* === 更新 === */
      /* y = z - H * x_pred */
      y[0] = measured_load - (kf->H[0][0] * x_pred[0]
                            + kf->H[0][1] * x_pred[1]
                            + kf->H[0][2] * x_pred[2]
                            + kf->H[0][3] * x_pred[3]);
      y[1] = measured_power - (kf->H[1][0] * x_pred[0]
                             + kf->H[1][1] * x_pred[1]
                             + kf->H[1][2] * x_pred[2]
                             + kf->H[1][3] * x_pred[3]);

      /* S = H * P_pred * H^T + R */
      for (i = 0; i < 2; i++)
          for (j = 0; j < 2; j++) {
              S[i][j] = kf->R[i][j];
              for (int k = 0; k < 4; k++)
                  for (int l = 0; l < 4; l++)
                      S[i][j] += kf->H[i][k] * P_pred[k][l] * kf->H[j][l];
          }

      /* K = P_pred * H^T * S^(-1) */
      double det = S[0][0] * S[1][1] - S[0][1] * S[1][0];
      double S_inv[2][2] = {
          {S[1][1]/det, -S[0][1]/det},
          {-S[1][0]/det, S[0][0]/det}
      };
      for (i = 0; i < 4; i++)
          for (j = 0; j < 2; j++) {
              K[i][j] = 0;
              for (int k = 0; k < 4; k++)
                  for (int l = 0; l < 2; l++)
                      K[i][j] += P_pred[i][k] * kf->H[l][k] * S_inv[l][j];
              /* 简化: K = P_pred * H^T * S_inv */
          }

      /* x = x_pred + K * y */
      for (i = 0; i < 4; i++)
          kf->x[i] = x_pred[i] + K[i][0] * y[0] + K[i][1] * y[1];

      /* P = (I - K * H) * P_pred */
      double KH[4][4];
      for (i = 0; i < 4; i++)
          for (j = 0; j < 4; j++) {
              KH[i][j] = 0;
              for (int k = 0; k < 2; k++)
                  KH[i][j] += K[i][k] * kf->H[k][j];
          }
      for (i = 0; i < 4; i++)
          for (j = 0; j < 4; j++) {
              kf->P[i][j] = P_pred[i][j];
              for (int k = 0; k < 4; k++)
                  kf->P[i][j] -= KH[i][k] * P_pred[k][j];
          }
  }

  卡尔曼滤波的优势:
    - 融合多传感器数据 (负载, 功耗, 温度)
    - 提供状态估计的置信度 (协方差矩阵)
    - 自适应调整 (Q 和 R 可自适应)
    - 实时性好 (O(n³) 复杂度, n=4 很小)
```

## 4. PID 控制算法

### 4.1 PID 控制基础

```text
PID (Proportional-Integral-Derivative) 控制在 GPU 功耗管理中的应用:

  基本原理:

  ┌─────────────────────────────────────────────────────────────┐
  │                    ┌──────────┐                             │
  │  Setpoint ──┬──▶  │ 误差 e   │──▶ PID 控制器 ──▶ GPU ──┐   │
  │  (目标温度)  │     │ = SP-PV  │                     │   │   │
  │              │     └──────────┘                     │   │   │
  │              │                                      │   │   │
  │              └──────────────────────────────────────┘   │   │
  │                                              PV (当前值) │   │
  │                                                         │   │
  │  PID 输出:                                               │   │
  │                                                         │   │
  │  u(t) = Kp * e(t) + Ki * ∫e(τ)dτ + Kd * de(t)/dt       │   │
  └─────────────────────────────────────────────────────────────┘

  离散化 PID (位置式):

  u[k] = Kp * e[k] + Ki * Σe[i]*Ts + Kd * (e[k] - e[k-1])/Ts

  离散化 PID (增量式):

  Δu[k] = Kp*(e[k]-e[k-1]) + Ki*e[k]*Ts + Kd*(e[k]-2e[k-1]+e[k-2])/Ts
  u[k] = u[k-1] + Δu[k]

  增量式优于位置式:
    - 计算量小
    - 积分饱和更容易处理
    - 手动/自动切换更平滑

  PID 系数的作用:

  参数    作用                    增大影响            减小影响
  Kp     加快响应速度             超调增加, 稳定变差   响应变慢
  Ki     消除稳态误差             积分饱和风险         存在稳态误差
  Kd     抑制超调, 改善动态       噪声放大, 不稳定     超调增加
```

### 4.2 GPU 温度 PID 控制

```text
GPU 温度 PID 控制器的完整实现:

  #define PID_SAMPLE_RATE_HZ   1000   /* 1kHz 控制频率 */
  #define PID_SAMPLE_INTERVAL  1      /* 1ms */

  struct gpu_temp_controller {
      /* PID 参数 */
      double Kp;           /* 比例系数 */
      double Ki;           /* 积分系数 */
      double Kd;           /* 微分系数 */

      /* 控制目标 */
      double setpoint;     /* 目标温度 (°C) */
      double max_temp;     /* 最高允许温度 (°C) */

      /* 状态 */
      double prev_error;   /* 上一次误差 */
      double integral;     /* 积分项 */
      double output;       /* 当前输出 (0~1) */

      /* 限制 */
      double out_min;      /* 最小输出 */
      double out_max;      /* 最大输出 */
      double integral_min; /* 积分下限 (抗饱和) */
      double integral_max; /* 积分上限 (抗饱和) */

      /* 滤波器 */
      double d_filter;     /* 微分项低通滤波系数 */
      double filtered_deriv; /* 滤波后的微分项 */

      /* 统计 */
      u64    samples;
      double mse;
  };

  /* PID 控制器初始化 */
  static void temp_controller_init(struct gpu_temp_controller *tc)
  {
      /* RDNA3 GPU 典型温度 PID 参数 */
      tc->Kp = 0.8;
      tc->Ki = 0.05;
      tc->Kd = 0.3;

      tc->setpoint = 85.0;     /* 目标温度 85°C */
      tc->max_temp = 95.0;     /* 最高 95°C (junction) */

      tc->prev_error = 0;
      tc->integral = 0;
      tc->output = 1.0;         /* 初始满频 */

      tc->out_min = 0.2;       /* 最低 20% 性能 */
      tc->out_max = 1.0;       /* 最高 100% 性能 */
      tc->integral_min = -0.5;
      tc->integral_max = 0.5;

      tc->d_filter = 0.8;       /* 滤波系数 */
      tc->filtered_deriv = 0;
  }

  /* PID 步进计算 */
  static double temp_controller_update(
      struct gpu_temp_controller *tc,
      double current_temp)
  {
      double error, p_term, i_term, d_term, derivative;
      double dt = PID_SAMPLE_INTERVAL / 1000.0; /* 秒 */

      /* 计算误差 */
      error = tc->setpoint - current_temp;

      /* 比例项 */
      p_term = tc->Kp * error;

      /* 积分项 (带抗饱和) */
      tc->integral += error * dt;
      tc->integral = clamp(tc->integral,
                          tc->integral_min, tc->integral_max);
      i_term = tc->Ki * tc->integral;

      /* 微分项 (带低通滤波) */
      derivative = (error - tc->prev_error) / dt;
      tc->filtered_deriv = tc->d_filter * tc->filtered_deriv
                         + (1 - tc->d_filter) * derivative;
      d_term = tc->Kd * tc->filtered_deriv;

      /* PID 输出 */
      tc->output = p_term + i_term + d_term;
      tc->output = clamp(tc->output, tc->out_min, tc->out_max);

      /* 更新状态 */
      tc->prev_error = error;
      tc->samples++;

      /* 统计 MSE */
      tc->mse += (error * error - tc->mse) / tc->samples;

      return tc->output;
  }

  /* PID 输出到 P-state 映射 */
  static int pid_to_pstate(struct gpu_temp_controller *tc,
                           int num_pstates)
  {
      /* 将 PID 输出 (0~1) 映射到 P-state 索引 */
      int pstate = (int)((1.0 - tc->output) * (num_pstates - 1));
      return clamp(pstate, 0, num_pstates - 1);
  }

  /* 过温保护: 当温度超过 max_temp 时, 强制降频 */
  static bool temp_controller_emergency(
      struct gpu_temp_controller *tc,
      double current_temp,
      int *emergency_pstate)
  {
      if (current_temp >= tc->max_temp) {
          /* 紧急降频到最低 P-state */
          *emergency_pstate = PSTATE_LOWEST;
          tc->integral = tc->integral_min; /* 重置积分 */
          return true;
      }
      return false;
  }

  温度 (°C)
   95│     ┌────────────────────────────────────────────
      │     │  无 PID: 温度振荡剧烈
   90│     │  ┌──┐    ┌──┐    ┌──┐    ┌──

      │     │  目标温度 85°C

      │     │
      └─────┴──────────────────────────────────────── 时间(s)
            0    5    10   15   20   25   30

    1. 设置 Kp=0.5, Ki=0, Kd=0

      /* 低负载时增加 Kp, 减少 Kd 以快速响应 */

      /* 中等负载使用默认参数 */
      else {
          tc->Kp = 0.8;
          tc->Kd = 0.3;
          tc->Ki = 0.05;
      }
  }

## 5. 功耗预算管理算法

### 5.1 功耗预算分配

```text
功耗预算 (Power Budget) 管理是自适应算法的关键组成部分:

  ┌─────────────────────────────────────────────────────────────┐
  │  GPU 功耗预算模型:                                          │
  │                                                             │
  │  Power_total = P_gfx + P_mem + P_dcn + P_soc + P_leak      │
  │                                                             │
  │  其中:                                                      │
  │    P_gfx: GFX 引擎功耗 (可变, 最大占比 ~60%)                │
  │    P_mem: 显存功耗 (可变, ~20%)                             │
  │    P_dcn: 显示引擎功耗 (相对固定, ~10%)                     │
  │    P_soc: SoC 功耗 (相对固定, ~5%)                          │
  │    P_leak: 漏电流功耗 (与温度相关, ~5%)                     │
  │                                                             │
  │  功耗预算约束:                                              │
  │                                                             │
  │  Power_total ≤ TDP (Thermal Design Power)                   │
  │                                                             │
  │  TDP 是 GPU 的热设计功耗上限, 通常由以下因素决定:             │
  │    - 散热器冷却能力                                          │
  │    - PCB 供电能力                                            │
  │    - VRM (电压调节模块) 额定电流                              │
  │    - 工作环境温度范围                                        │
  └─────────────────────────────────────────────────────────────┘

  动态功耗预算分配算法:

  struct power_budget_manager {
      /* 预算配置 */
      u32 tdp;                   /* TDP (mW) */
      u32 sustained_limit;       /* 持续功耗限制 (mW) */
      u32 burst_limit;           /* 突发功耗限制 (mW, 短时) */
      u32 burst_duration_ms;     /* 突发持续时间 (ms) */

      /* 各引擎分配 */
      u32 gfx_budget;            /* GFX 引擎预算 (mW) */
      u32 mem_budget;            /* 显存预算 (mW) */
      u32 dcn_budget;            /* 显示引擎预算 (mW) */
      u32 soc_budget;            /* SoC 预算 (mW) */

      /* 当前测量 */
      u32 current_power;         /* 当前总功耗 (mW) */
      u32 gfx_power;             /* GFX 功耗 (mW) */
      u32 mem_power;             /* 显存功耗 (mW) */
      u32 temp;                  /* 当前温度 (°C) */

      /* 滑动窗口 */
      u32 power_history[POWER_HISTORY_LEN]; /* 功耗历史 */
      u8  history_idx;
      u32 power_avg_1s;          /* 1 秒平均功耗 */
      u32 power_avg_100ms;       /* 100ms 平均功耗 */
  };

  /* 功耗预算动态分配 */
  static void power_budget_alloc(
      struct power_budget_manager *pbm)
  {
      u32 available;
      u32 gfx_min, gfx_max;
      u32 workload_type;

      /* 计算可用预算 */
      available = pbm->tdp;
      available -= pbm->dcn_budget;   /* 显示引擎固定分配 */
      available -= pbm->soc_budget;   /* SoC 固定分配 */
      available -= pbm->mem_budget;   /* 显存基础分配 */

      /* 确定 GFX 工作负载类型 */
      workload_type = detect_workload_type();

      /* 根据负载类型分配剩余预算 */
      switch (workload_type) {
      case WORKLOAD_GAMING:
          /* 游戏负载: GFX 70%, MEM 30% */
          gfx_min = available * 50 / 100;
          gfx_max = available * 70 / 100;
          break;
      case WORKLOAD_COMPUTE:
          /* 计算负载: GFX 80%, MEM 20% */
          gfx_min = available * 60 / 100;
          gfx_max = available * 80 / 100;
          break;
      case WORKLOAD_VIDEO:
          /* 视频负载: GFX 40%, MEM 60% (VCE/SDMA 使用显存) */
          gfx_min = available * 30 / 100;
          gfx_max = available * 40 / 100;
          break;
      default:
          gfx_min = available * 40 / 100;
          gfx_max = available * 60 / 100;
      }

      /* 基于当前温度调整 */
      if (pbm->temp > 90) {
          /* 高温: 降低 20% 预算 */
          gfx_max = gfx_max * 80 / 100;
      } else if (pbm->temp < 60) {
          /* 低温: 可适当增加 */
          gfx_max = min(gfx_max * 105 / 100, available);
      }

      /* 应用预算 */
      pbm->gfx_budget = clamp(
          pbm->gfx_power + 50, gfx_min, gfx_max);
      pbm->mem_budget = available - pbm->gfx_budget;
  }

```text

  │  超过 cap 时, 算法通过降频使功耗回到限制以下                  │

  Power Capping 算法实现:

  #define POWER_CAP_HYSTERESIS 5000   /* 5W 滞回 */

  struct power_cap_controller {
      u32 cap_mw;              /* 功耗上限 (mW) */
      u32 current_power;       /* 当前功耗 (mW) */

      /* 控制参数 */
      u32 reduction_step;      /* 每次降频步长 (MHz) */
      u32 recovery_step;       /* 每次恢复步长 (MHz) */
      u32 min_freq;            /* 最低频率 */

      /* 状态 */
      bool throttling;         /* 是否正在节流 */
      u32 throttle_count;      /* 节流次数 */
      ktime_t throttle_start;  /* 节流开始时间 */
  };

  /* Power Capping 步进控制 */
  static int power_cap_step(
      struct power_cap_controller *pcc,
      u32 current_power_mw,
      int current_freq_mhz)
  {
      int new_freq = current_freq_mhz;

      if (current_power_mw > pcc->cap_mw + POWER_CAP_HYSTERESIS) {
          /* 超过上限: 降频 */
          u32 excess = current_power_mw - pcc->cap_mw;
          u32 steps = (excess / 10000) + 1; /* 每 10W 降一步 */

          new_freq -= pcc->reduction_step * steps;
          new_freq = max(new_freq, (int)pcc->min_freq);

          if (!pcc->throttling) {
              pcc->throttling = true;
              pcc->throttle_start = ktime_get();
          }
          pcc->throttle_count++;
      }
      else if (pcc->throttling &&
               current_power_mw < pcc->cap_mw - POWER_CAP_HYSTERESIS) {
          /* 低于上限: 缓慢恢复 */
          new_freq += pcc->recovery_step;
          new_freq = min(new_freq, current_freq_mhz +
                         pcc->recovery_step);
          pcc->throttling = false;
      }

      return new_freq;
  }

  /* 多级 Power Capping (Intel/AMD 通用) */
  static int power_cap_multilevel(
      struct power_cap_controller *pcc,
      u32 power, int freq)
  {
      int freq_adjust = 0;

      /* Level 1: 软限制 (慢速 PID) */
      if (power > pcc->cap_mw) {
          freq_adjust -= 5; /* 缓慢降频 */
      }

      /* Level 2: 硬限制 (快速响应) */
      if (power > pcc->cap_mw * 110 / 100) {
          freq_adjust -= 50; /* 快速降频 */
      }

      /* Level 3: 紧急限制 */
      if (power > pcc->cap_mw * 120 / 100) {
          /* 立即切换到最低 P-state */
          return pcc->min_freq;
      }

      return freq + freq_adjust;
  }

  Power Capping 的常见问题:

  问题 1: 频率振荡 (Power Hunting)
    现象: 频率在 cap 附近来回调整
    原因: 响应过快或步进过大
    解决: 增加滞回区间, 使用 PID 平滑

  问题 2: 性能悬崖 (Performance Cliff)
    现象: 微小的 cap 降低导致性能大幅下降
    原因: P-state 间隔大, 非线性功耗-频率关系
    解决: 使用更细粒度的频率调节

  问题 3: 瞬时过冲 (Transient Overshoot)
    现象: 功耗短暂超过 cap
    原因: 传感器读数延迟, 控制响应延迟
    解决: 使用预测算法预先降频
```

  /* 快速节流中断处理 (在 SMU 固件中) */
  void smu_fast_throttle_isr(void)
  {
      u32 power = read_power_sensor(RAIL_GFX);
      u32 threshold = read_power_threshold();

      if (power > threshold) {
          /* 计算需要降低的频率量 */
          u32 reduction = (power - threshold) * FREQ_PER_WATT;
          u32 current_freq = get_gfx_freq();

          u32 target_freq = current_freq - reduction;
          target_freq = max(target_freq, SMU_MIN_FREQ);

          /* 立即设置新频率 (硬件层面) */
          set_gfx_freq_hw(target_freq);

          /* 通知驱动 */
          send_smu_interrupt(IRQ_FAST_THROTTLE);
      }
  }

  /* 驱动层处理快速节流事件 */
  static void amdgpu_fast_throttle_handler(struct work_struct *work)
  {
      struct amdgpu_device *adev = container_of(work,
          struct amdgpu_device, fast_throttle_work);

      u32 current_freq = amdgpu_dpm_get_current_freq(adev);
      u32 target_freq;

      /* 驱动层评估是否需要进一步降频 */
      if (adev->pm.current_power > adev->pm.fast_throttle_threshold) {
          target_freq = current_freq * 80 / 100; /* 降低 20% */
          amdgpu_dpm_set_freq(adev, target_freq);
          adev->pm.fast_throttle_count++;

          /* 调度恢复检查 */
          schedule_delayed_work(
              &adev->pm.throttle_recovery_work,
              msecs_to_jiffies(100)); /* 100ms 后检查恢复 */
      }
  }

  /* 节流恢复检查 */
  static void amdgpu_throttle_recovery(struct work_struct *work)
  {
      struct amdgpu_device *adev = container_of(work,
          struct amdgpu_device, throttle_recovery_work.work);

      u32 power = amdgpu_dpm_get_current_power(adev);

      if (power < adev->pm.fast_throttle_threshold * 70 / 100) {
          /* 功耗安全, 逐步恢复频率 */
          u32 current_freq = amdgpu_dpm_get_current_freq(adev);
          u32 target_freq = current_freq * 110 / 100; /* 增加 10% */
          amdgpu_dpm_set_freq(adev, target_freq);
      }

      /* 如果还未达到目标频率, 继续检查 */
      u32 target = adev->pm.requested_freq;
      u32 current = amdgpu_dpm_get_current_freq(adev);
      if (current < target) {
          schedule_delayed_work(
              &adev->pm.throttle_recovery_work,
              msecs_to_jiffies(100));
      }
  }

  负载分类器的实现:

  #define NUM_FEATURES    10
  #define NUM_CLASSES     8
  #define NUM_TREES       100

  struct workload_classifier {
      /* 随机森林模型 */
      struct decision_tree *trees[NUM_TREES];
      int num_trees;

      /* 特征归一化参数 */
      float feature_mean[NUM_FEATURES];
      float feature_std[NUM_FEATURES];

      /* 混淆矩阵 (用于评估) */
      u32 confusion_matrix[NUM_CLASSES][NUM_CLASSES];
  };

  /* 决策树节点 */
  struct decision_tree {
      int feature_idx;           /* 分裂特征索引 */
      float threshold;           /* 分裂阈值 */
      int class_label;           /* 叶节点的类别 (仅叶子) */
      struct decision_tree *left;   /* 左子树 */
      struct decision_tree *right;  /* 右子树 */
  };

  /* 前向推理 */
  static int classifier_predict(
      struct decision_tree *tree,
      float *features)
  {
      /* 遍历决策树 */
      while (tree->left && tree->right) {
          if (features[tree->feature_idx] <= tree->threshold)
              tree = tree->left;
          else
              tree = tree->right;
      }
      return tree->class_label;
  }

  /* 随机森林投票 */
  static int rf_predict(
      struct workload_classifier *wc,
      float *features)
  {
      int votes[NUM_CLASSES] = {0};
      int i;

      for (i = 0; i < wc->num_trees; i++) {
          int pred = classifier_predict(wc->trees[i], features);
          votes[pred]++;
      }

      /* 选择票数最多的类别 */
      int best_class = 0;
      int best_votes = 0;
      for (i = 0; i < NUM_CLASSES; i++) {
          if (votes[i] > best_votes) {
              best_votes = votes[i];
              best_class = i;
          }
      }

      return best_class;
  }

  /* 特征归一化 */
  static void normalize_features(
      float *features,
      float *mean, float *std)
  {
      for (int i = 0; i < NUM_FEATURES; i++) {
          if (std[i] > 1e-6) {
              features[i] = (features[i] - mean[i]) / std[i];
          }
      }
  }

  /* 在线更新混淆矩阵 */
  static void update_confusion_matrix(
      struct workload_classifier *wc,
      int predicted, int actual)
  {
      if (predicted >= 0 && predicted < NUM_CLASSES &&
          actual >= 0 && actual < NUM_CLASSES) {
          wc->confusion_matrix[actual][predicted]++;
      }
  }

  /* 在线预测更新：结合历史反馈修正分类结果 */
  static int update_prediction_model(struct work_classifier *clf,
                                     uint32_t actual_class,
                                     uint32_t predicted_class,
                                     struct workload_features *features)
  {
      if (actual_class == predicted_class)
          clf->consecutive_hits++;
      else
          clf->consecutive_hits = 0;

      /* 连续误分类超过阈值时触发模型更新 */
      if (clf->consecutive_misses > 5) {
          retrain_model(clf, features, actual_class);
          clf->consecutive_misses = 0;
          return 1; /* 模型已更新 */
      }

      clf->total_predictions++;
      clf->correct_predictions += (actual_class == predicted_class) ? 1 : 0;
      clf->accuracy = (float)clf->correct_predictions /
                      (float)clf->total_predictions * 100.0f;

      return 0;
  }

  /* 特征重要性分析（基于随机森林的Gini重要性） */
  static void update_feature_importance(struct work_classifier *clf,
                                        uint32_t *feature_importances)
  {
      /* 特征索引: 0=GFX占用, 1=MEM带宽, 2=ACE利用率,
       * 3=VCN占用, 4=DCN带宽, 5=PCIe吞吐, 6=温度梯度 */
      static const char *feature_names[] = {
          "GFX_UTIL", "MEM_BW", "ACE_UTIL",
          "VCN_UTIL", "DCN_BW", "PCIE_THR", "TEMP_GRAD"
      };

      for (int i = 0; i < NUM_FEATURES; i++) {
          clf->feature_importance[i] = feature_importances[i];
      }

      /* 按重要性排序 */
      for (int i = 0; i < NUM_FEATURES - 1; i++) {
          for (int j = i + 1; j < NUM_FEATURES; j++) {
              if (clf->feature_importance[j] >
                  clf->feature_importance[i]) {
                  uint32_t tmp = clf->feature_importance[i];
                  clf->feature_importance[i] =
                      clf->feature_importance[j];
                  clf->feature_importance[j] = tmp;
              }
          }
      }
  }

  int dpm_serialize_model(struct work_classifier *clf,
                          struct smu_context *smu)
  {
      struct dpm_model_params params = {0};

      params.version = MODEL_VERSION;
      params.num_trees = clf->num_trees;
      params.max_depth = clf->max_depth;
      params.accuracy = clf->accuracy;
      params.total_samples = clf->total_predictions;

      for (int i = 0; i < NUM_CLASSES; i++) {
          params.class_weights[i] = clf->class_weights[i];
      }

      return smu_send_params(smu, SMU_MSG_SetModelParams,
                             (uint32_t *)&params,
                             sizeof(params) / sizeof(uint32_t));
  }

  6.2 强化学习DQN的GPU功耗控制

  深度Q网络(DQN)是一种无模型的强化学习算法，适合GPU功耗管理的序贯决策问题。
  GPU功耗管理的MDP建模如下：

  - 状态空间S：{f_GFX, f_MEM, V_domain, T_junction, P_pkg, workload_type, IPC, mem_stall}
  - 动作空间A：{Δf_GFX(-100~+100MHz), ΔV_domain(-12.5~+12.5mV), pstate_switch, cg_mask}
  - 奖励函数R：r = w_1 * perf_gain - w_2 * power_penalty - w_3 * temp_penalty - w_4 * latency_penalty
  - 折扣因子γ：0.95（偏重长期收益）

  DQN网络架构：

  ```
  输入层(8维状态)
    ↓
  全连接层(256神经元, ReLU)
    ↓
  BatchNorm层
    ↓
  全连接层(128神经元, ReLU)
    ↓
  Dropout(0.2)
    ↓
  全连接层(64神经元, ReLU)
    ↓
  输出层(动作数, Linear)
    ↓
  ε-greedy动作选择
  ```

  DQN训练流程：

  ```c
  /* DQN经验回放缓冲区 */
  struct replay_buffer {
      struct transition *buffer;
      int capacity;
      int size;
      int head;
      spinlock_t lock;
  };

  struct transition {
      float state[STATE_DIM];
      int action;
      float reward;
      float next_state[STATE_DIM];
      bool done;
  };

  /* 经验回放采样（优先经验回放） */
  static void sample_prioritized(struct replay_buffer *rb,
                                 struct transition *batch,
                                 int batch_size,
                                 float *weights)
  {
      /* 基于TD-error的优先级采样 */
      float total_priority = 0.0f;
      float priorities[REPLAY_CAPACITY];

      for (int i = 0; i < rb->size; i++) {
          priorities[i] = rb->buffer[i].td_error + 1e-6;
          total_priority += priorities[i];
      }

      for (int i = 0; i < batch_size; i++) {
          float r = (float)rand() / RAND_MAX * total_priority;
          float cumsum = 0.0f;

          for (int j = 0; j < rb->size; j++) {
              cumsum += priorities[j];
              if (cumsum >= r) {
                  batch[i] = rb->buffer[j];
                  weights[i] = 1.0f / (rb->size * priorities[j] /
                                       total_priority);
                  break;
              }
          }
      }
  }

  /* DQN损失函数计算（Double DQN） */
  static float compute_ddqn_loss(struct dqn_network *online,
                                 struct dqn_network *target,
                                 struct transition *batch,
                                 int batch_size)
  {
      float total_loss = 0.0f;

      for (int i = 0; i < batch_size; i++) {
          /* 使用online网络选择动作 */
          float q_values[ACTION_DIM];
          online->forward(online, batch[i].next_state, q_values);
          int best_action = argmax(q_values, ACTION_DIM);

          /* 使用target网络评估动作 */
          float target_q[ACTION_DIM];
          target->forward(target, batch[i].next_state, target_q);

          /* Double DQN目标值 */
          float target_val = batch[i].reward;
          if (!batch[i].done) {
              target_val += GAMMA * target_q[best_action];
          }

          /* 当前Q值 */
          float current_q[ACTION_DIM];
          online->forward(online, batch[i].state, current_q);

          /* Huber损失（对异常值更鲁棒） */
          float diff = current_q[batch[i].action] - target_val;
          if (fabs(diff) < 1.0f) {
              total_loss += 0.5f * diff * diff;
          } else {
              total_loss += fabs(diff) - 0.5f;
          }
      }

      return total_loss / batch_size;
  }

  /* 目标网络软更新 */
  static void soft_update_target(struct dqn_network *online,
                                 struct dqn_network *target,
                                 float tau)
  {
      for (int i = 0; i < online->num_params; i++) {
          target->params[i] = tau * online->params[i] +
                              (1.0f - tau) * target->params[i];
      }
  }
  ```

  DQN在GPU功耗控制中的关键挑战：
  1. 训练收敛速度：GPU工作负载变化快，DQN需要快速适应
  2. 安全约束：探索阶段可能选择不安全动作（如过度升压）
  3. 计算开销：神经网络推理必须在微秒级完成
  4. 多目标平衡：性能、功耗、温度三者难以同时最优

  解决上述挑战的工程方案：

  | 挑战 | 解决方案 | 实现细节 |
  |------|---------|---------|
  | 收敛速度慢 | 结合专家演示(DQfD) | 预填充回放缓冲区20%专家轨迹 |
  | 安全探索 | 安全层约束 | 在动作输出后应用安全过滤器 |
  | 计算开销 | 模型量化(INT8) | 权重8bit量化，推理速度提升4x |
  | 多目标平衡 | 加权和奖励整形 | 权重随温度动态调整 |

  6.3 在线学习与模型自适应

  在线学习允许功耗管理模型在GPU运行过程中持续更新，适应工作负载变化和芯片老化。

  在线学习框架设计：

  ```
  初始化(基线模型)
    ↓
  while GPU 运行:
    采集特征向量 x_t
    模型推理 → 电源决策 a_t
    执行决策, 观察结果 (功耗, 性能, 温度)
    构建反馈样本 (x_t, a_t, r_t, x_{t+1})
    更新模型参数
    监控模型漂移指标
    如果漂移 > 阈值: 触发模型重置或集成
  ```

  在线梯度下降实现：

  ```c
  /* 在线线性模型（用于快速自适应） */
  struct online_linear_model {
      float weights[NUM_FEATURES];
      float bias;
      float learning_rate;
      float decay;           /* 学习率衰减 */
      uint64_t update_count;
  };

  /* 在线梯度更新（单样本） */
  static void online_gd_update(struct online_linear_model *model,
                               float *features,
                               float target)
  {
      /* 前向计算 */
      float prediction = model->bias;
      for (int i = 0; i < NUM_FEATURES; i++) {
          prediction += model->weights[i] * features[i];
      }

      /* 误差计算 */
      float error = prediction - target;

      /* 学习率衰减 */
      float lr = model->learning_rate /
                 (1.0f + model->decay * model->update_count);

      /* 权重更新（带动量） */
      static float momentum[NUM_FEATURES] = {0};
      float mu = 0.9f; /* 动量系数 */

      for (int i = 0; i < NUM_FEATURES; i++) {
          float grad = error * features[i];
          momentum[i] = mu * momentum[i] + lr * grad;
          model->weights[i] -= momentum[i];
      }

      model->bias -= lr * error;
      model->update_count++;
  }

  /* 概念漂移检测（Page-Hinkley检验） */
  struct drift_detector {
      float cumulative_sum;
      float min_value;
      float threshold;
      float drift_level;
      uint64_t samples_seen;
  };

  static int detect_concept_drift(struct drift_detector *detector,
                                  float prediction_error)
  {
      detector->samples_seen++;

      /* 累积偏差 */
      float mean_error = detector->cumulative_sum /
                         (float)detector->samples_seen;
      float deviation = prediction_error - mean_error -
                        detector->min_value;

      detector->cumulative_sum += deviation;
      detector->drift_level = detector->cumulative_sum;

      if (detector->drift_level > detector->threshold) {
          /* 检测到概念漂移 */
          detector->cumulative_sum = 0;
          detector->samples_seen = 0;
          return 1;
      }

      return 0;
  }

  /* 自适应模型集成 */
  struct model_ensemble {
      struct online_linear_model base_model;
      struct online_linear_model *adaptive_models;
      float *model_weights;
      int num_adaptive;
      int max_adaptive;
  };

  static float ensemble_predict(struct model_ensemble *ensemble,
                                float *features)
  {
      float prediction = 0.0f;
      float total_weight = 0.0f;

      /* 基础模型预测 */
      float base_pred = 0;
      for (int i = 0; i < NUM_FEATURES; i++) {
          base_pred += ensemble->base_model.weights[i] * features[i];
      }
      base_pred += ensemble->base_model.bias;

      prediction += base_pred * 0.5f;
      total_weight += 0.5f;

      /* 自适应模型加权预测 */
      for (int i = 0; i < ensemble->num_adaptive; i++) {
          float adapt_pred = 0;
          for (int j = 0; j < NUM_FEATURES; j++) {
              adapt_pred += ensemble->adaptive_models[i].weights[j] *
                            features[j];
          }
          adapt_pred += ensemble->adaptive_models[i].bias;

          prediction += adapt_pred * ensemble->model_weights[i];
          total_weight += ensemble->model_weights[i];
      }

      if (total_weight > 0)
          prediction /= total_weight;

      return prediction;
  }
  ```

  7. 模型预测控制(MPC)在GPU功耗管理中的应用

  模型预测控制是一种基于系统模型的前瞻控制方法，通过在有限时域内求解优化问题来确定最优控制序列。

  7.1 MPC基本原理

  MPC在GPU功耗管理中的框架：

  ```
          ┌─────────────┐   未来控制序列    ┌─────────┐
  参考值 ─→│  优化求解器  │───────────────→│  GPU    │
  状态  ─→│ (约束QP)   │                │  系统   │─→ 输出
          └─────────────┘                └─────────┘
                ↑                              │
                │  预测模型                     │
                │  ┌──────────────────┐        │
                └──│ f(x,u) → x_next  │←───────┘
                   │ 功耗/温度/性能    │
                   └──────────────────┘
  ```

  MPC控制公式：

  预测模型：x_{k+1} = A·x_k + B·u_k + w_k

  其中：
  - x_k = [T_junction, P_pkg, f_GFX, utilization]^T  （状态向量）
  - u_k = [Δf_GFX, ΔV_domain, cg_mask]^T             （控制输入）
  - A = 系统状态转移矩阵（由系统辨识获得）
  - B = 控制输入矩阵
  - w_k = 过程噪声

  优化目标函数：

  J = Σ_{i=0}^{N-1} (||x_{k+i+1} - x_ref||²_Q + ||u_{k+i}||²_R) + ||x_{k+N} - x_ref||²_P

  约束条件：
  - 频率约束：f_min ≤ f_GFX ≤ f_max
  - 电压约束：V_min ≤ V_domain ≤ V_max
  - 温度约束：T_junction ≤ T_max (105°C)
  - 功耗约束：P_pkg ≤ TDP
  - 压摆率约束：|Δf_GFX| ≤ slew_rate_max
  - 电压变化率：|ΔV_domain| ≤ 12.5mV/step

  7.2 MPC求解器实现

  ```c
  /* MPC参数结构 */
  struct mpc_params {
      int prediction_horizon;    /* 预测时域 N */
      int control_horizon;       /* 控制时域 M */
      float Q[STATE_DIM];        /* 状态权重 */
      float R[CTRL_DIM];         /* 控制权重 */
      float P[STATE_DIM];        /* 终端权重 */
      float dt;                  /* 采样时间(ms) */
  };

  /* GPU热模型参数 */
  struct thermal_model {
      float R_th;       /* 热阻 (°C/W) */
      float C_th;       /* 热容 (J/°C) */
      float T_ambient;  /* 环境温度 (°C) */
      float tau;        /* 热时间常数 (s) */
  };

  /* 四阶RC热模型：ΨJT, ΨJB, ΨPC, Ψ ambient */
  static void thermal_state_transition(float *x, float *u,
                                       struct thermal_model *thermal,
                                       float dt)
  {
      /* x[0] = T_junction (结温)
       * x[1] = T_hotspot  (热点温度)
       * x[2] = T_memory   (显存温度)
       * x[3] = T_vrm      (供电温度)
       *
       * u[0] = P_GFX (图形功耗)
       * u[1] = P_MEM (显存功耗)
       * u[2] = P_SOC (SoC功耗)
       * u[3] = fan_rpm (风扇转速)
       */

      /* 简化的四阶RC热模型 */
      float R[4] = {thermal->R_th * 0.4f,
                    thermal->R_th * 0.3f,
                    thermal->R_th * 0.2f,
                    thermal->R_th * 0.1f};

      float C[4] = {thermal->C_th * 0.35f,
                    thermal->C_th * 0.25f,
                    thermal->C_th * 0.25f,
                    thermal->C_th * 0.15f};

      for (int i = 0; i < 4; i++) {
          float p_in = u[i] + (i > 0 ? x[i-1] / R[i-1] : 0);
          float p_out = x[i] / R[i];
          float dT_dt = (p_in - p_out) / C[i];
          x[i] += dT_dt * dt;
      }
  }

  /* QP求解器（梯度投影法） */
  static int solve_mpc_qp(float *x0, float *u_seq,
                          struct mpc_params *params,
                          float *constraints)
  {
      int N = params->prediction_horizon;
      int M = params->control_horizon;
      float dt = params->dt;

      /* 状态预测矩阵 */
      float Phi[STATE_DIM * (N+1)];
      float Gamma[STATE_DIM * (N+1) * CTRL_DIM];

      /* 构建预测矩阵 */
      build_prediction_matrices(Phi, Gamma, x0,
                                params->A, params->B, N, M);

      /* 构建QP问题 */
      /* min 0.5 * u' * H * u + f' * u */
      /* s.t. lb <= u <= ub */

      float H[CTRL_DIM * M * CTRL_DIM * M];
      float f[CTRL_DIM * M];

      /* H = 2 * (Gamma' * Q_bar * Gamma + R_bar) */
      compute_hessian(H, Gamma, params->Q, params->R, N, M);

      /* f = 2 * (Phi * x0 - x_ref)' * Q_bar * Gamma */
      compute_gradient(f, Phi, x0, Gamma, params->Q,
                       params->ref, N, M);

      /* 梯度投影迭代求解 */
      float u[CTRL_DIM * M];
      memset(u, 0, sizeof(u));

      for (int iter = 0; iter < MAX_QP_ITER; iter++) {
          /* 计算梯度 */
          float grad[CTRL_DIM * M];
          mat_vec_mul(H, u, grad, CTRL_DIM * M);
          for (int i = 0; i < CTRL_DIM * M; i++) {
              grad[i] += f[i];
          }

          /* 线搜索 */
          float alpha = 1.0f;
          float step[CTRL_DIM * M];

          for (int i = 0; i < CTRL_DIM * M; i++) {
              step[i] = u[i] - alpha * grad[i];
          }

          /* 投影到可行域 */
          project_constraints(step, constraints,
                              CTRL_DIM * M);

          /* Armijo条件检查 */
          float cost_current = compute_cost(u, H, f,
                                            CTRL_DIM * M);
          float cost_new = compute_cost(step, H, f,
                                        CTRL_DIM * M);

          if (cost_new < cost_current - 1e-4 * alpha *
              dot_product(grad, step, CTRL_DIM * M)) {
              memcpy(u, step, sizeof(u));
              break;
          }

          alpha *= 0.5f;
      }

      memcpy(u_seq, u, CTRL_DIM * sizeof(float));
      return 0;
  }
  ```

  7.3 MPC在AMDGPU驱动中的集成

  ```c
  /* MPC控制器主循环 */
  struct mpc_controller {
      struct mpc_params params;
      struct thermal_model thermal;
      float state[STATE_DIM];
      float control[CTRL_DIM];
      struct work_struct mpc_work;
      struct mutex lock;
      bool enabled;
  };

  static void mpc_control_loop(struct work_struct *work)
  {
      struct mpc_controller *mpc =
          container_of(work, struct mpc_controller, mpc_work);

      mutex_lock(&mpc->lock);

      /* 1. 采集当前状态 */
      mpc->state[0] = amdgpu_get_temperature();       /* T_junction */
      mpc->state[1] = amdgpu_get_hotspot_temp();       /* T_hotspot */
      mpc->state[2] = amdgpu_get_memory_temp();        /* T_memory */
      mpc->state[3] = amdgpu_get_vrm_temp();           /* T_vrm */

      /* 2. 更新控制周期 */
      update_thermal_model(&mpc->thermal, mpc->state);

      /* 3. 求解MPC优化问题 */
      float constraints[] = {
          300,  2200,    /* f_min, f_max (MHz) */
          800,  1200,    /* V_min, V_max (mV) */
          0,    105,     /* T_min, T_max (°C) */
          0,    250      /* P_min, P_max (W) */
      };

      float u_opt[CTRL_DIM] = {0};
      solve_mpc_qp(mpc->state, u_opt, &mpc->params,
                   constraints);

      /* 4. 应用第一个控制动作（滚动优化） */
      mpc->control[0] = u_opt[0];  /* GFX频率 */
      mpc->control[1] = u_opt[1];  /* 电压 */
      mpc->control[2] = u_opt[2];  /* 时钟门控掩码 */

      amdgpu_set_gfx_freq(mpc->control[0]);
      amdgpu_set_voltage(mpc->control[1]);
      amdgpu_set_cg_mask((uint32_t)mpc->control[2]);

      mutex_unlock(&mpc->lock);

      /* 5. 调度下一次控制 */
      schedule_delayed_work(&mpc->mpc_work,
                            msecs_to_jiffies(mpc->params.dt));
  }

  9.2 能效基准测试方法

  ```c
  /* 能效基准测试框架 */
  struct ef_benchmark {
      const char *name;
      const char *workload;
      float fps;
      float avg_power;
      float max_power;
      float avg_temp;
      float efficiency;
      uint32_t pstate_residency[14];
      struct timespec duration;
  };

  /* 能效测试执行 */
  static int run_efficiency_benchmark(
      struct ef_benchmark *bench,
      struct amdgpu_device *adev,
      int duration_ms)
  {
      struct timespec start, now;
      uint64_t total_energy = 0;
      uint32_t samples = 0;

      /* 重置计数器 */
      bench->fps = 0;
      bench->avg_power = 0;
      bench->max_power = 0;
      memset(bench->pstate_residency, 0,
             sizeof(bench->pstate_residency));

      /* 开始监控 */
      ktime_get_ts(&start);

      while (1) {
          ktime_get_ts(&now);
          int elapsed = (now.tv_sec - start.tv_sec) * 1000 +
                        (now.tv_nsec - start.tv_nsec) / 1000000;

          if (elapsed >= duration_ms)
              break;

          /* 采样功耗 */
          uint32_t power = amdgpu_get_power(adev);
          total_energy += power;
          samples++;

          if (power > bench->max_power)
              bench->max_power = power;

          /* 采样P-state驻留 */
          uint8_t pstate = amdgpu_get_pstate(adev);
          if (pstate < 14)
              bench->pstate_residency[pstate]++;

          msleep(10); /* 10ms采样周期 */
      }

      /* 计算结果 */
      bench->avg_power = total_energy / samples;
      bench->efficiency = bench->fps / bench->avg_power;

      return 0;
  }
  ```

  /* PID控制器单元测试 */
  static int test_pid_controller(void)
  {
      struct pid_controller pid;
      int failures = 0;

      /* Test 1: 基本PID响应 */
      pid_init(&pid, 1.0f, 0.1f, 0.05f, 100, 0, 100);
      float output = pid_update(&pid, 80, 75);
      if (output <= 0)
          failures++;

      /* Test 2: 积分饱和测试 */
      pid_init(&pid, 2.0f, 0.5f, 0.1f, 100, 0, 100);
      for (int i = 0; i < 100; i++) {
          output = pid_update(&pid, 90, 75);
      }
      if (output > 100 || output < 0)
          failures++;

      /* Test 3: 设定点追踪 */
      pid_init(&pid, 0.8f, 0.05f, 0.02f, 100, 0, 100);
      float setpoint = 80;
      float temp = 50;
      float mae = 0;

      for (int i = 0; i < 200; i++) {
          float control = pid_update(&pid, setpoint, temp);
          temp += 0.1f * control;
          mae += fabs(setpoint - temp);
      }

      mae /= 200;
      if (mae > 5.0f)
          failures++;

      return failures;
  }

  static int test_load_predictor(void)
  {

      int failures = 0;

      /* 测试指数平滑预测 */
      pred_init(&pred, 0.3f, 100);
      float actual[] = {50, 55, 53, 58, 60, 62, 65, 63, 68, 70};
      float predictions[10];

      for (int i = 0; i < 10; i++) {
          predictions[i] = predict_next(&pred);
          pred_update(&pred, actual[i]);
      }

      /* 验证预测误差逐步减小 */
      float errors[9];
      for (int i = 0; i < 9; i++) {
          errors[i] = fabs(predictions[i+1] - actual[i+1]);
      }

      if (errors[0] < errors[5])
          failures++;

      return failures;
  }

  12. 总结

  自适应功耗管理算法是现代GPU驱动中最为复杂和关键的子系统之一。本文从DVFS基础算法出发，深入分析了负载预测、PID控制、功耗预算管理、机器学习驱动以及模型预测控制等多种算法方法，并结合AMD和NVIDIA的具体实现进行了对比分析。

  关键要点总结：

  1. **多种算法协同**：实际GPU功耗管理不是单一算法，而是PID、预测、分类、优化等多种算法的有机组合

  2. **实时性要求**：所有算法必须在微秒到毫秒级完成推理，需要精心设计计算路径

  3. **安全第一**：任何功耗管理决策都不能导致硬件损坏，安全层是必备组件

  4. **自适应能力**：GPU工作负载多样性要求算法具备在线学习和自适应能力

  5. **硬件-固件-驱动协同**：高效的功耗管理需要三层紧密协作，缺一不可

  6. **测试验证**：功耗管理算法的测试需要覆盖功能、性能、压力、长时间等多个维度

  7. **持续演进**：随着GPU架构复杂化和制程进步，功耗管理算法将持续向智能化方向发展
