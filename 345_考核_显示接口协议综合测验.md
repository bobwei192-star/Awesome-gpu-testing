### 23. 多显示器的带宽规划与模式配置（10 分）

  - 显示器 A：4K (3840×2160) @ 144Hz, 10-bit HDR, 通过 DP 2.1 连接
  - 显示器 B：1440p (2560×1440) @ 165Hz, 8-bit SDR, 通过 HDMI 2.1 连接

c) 如果显示器 C 改为 4K @ 60Hz HDR（10-bit），是否需要开启 DSC（Display Stream Compression）？计算开启 DSC 3:1 压缩后的带宽需求。

## 第四部分：综合案例分析（20 分）

### 24. 显示接口故障综合排查案例

**场景描述：**

你收到一个用户报告：使用 AMD Radeon RX 7900 XTX 显卡，通过 USB-C 连接一个 Dell U3224KB（6K 分辨率 6144×3456 @ 60Hz）显示器，同时通过 DP 1.4 连接一个 Samsung Odyssey G7（4K @ 240Hz）显示器。显示器的连接拓扑如下：

```
[AMD GPU]
    ├── USB-C (DP Alt Mode, Pin Assignment D) → Dell U3224KB 6K (需 DSC)
    ├── DisplayPort 1.4 → Samsung G7 4K @ 240Hz (需 DSC)
    └── HDMI 2.1 → LG C2 4K @ 120Hz (HDR)
```

用户反馈的问题：

1. 当三台显示器同时连接时，Samsung G7 只能达到 120Hz 而非 240Hz
2. Dell U3224KB 间歇性黑屏（每次持续 1-3 秒）
3. LG C2 无法开启 HDR 模式，虽然之前单屏使用时支持

**要求：**

a) **问题分析（8 分）**：
   - 分析问题 1：Samsung G7 只能达到 120Hz 的原因。考虑 DSC 带宽分配与 DP 1.4 的限制。
   - 分析问题 2：Dell U3224KB 间歇性黑屏的可能原因。考虑 USB-C 连接的稳定性与 Pin Assignment 的影响。
   - 分析问题 3：LG C2 无法开启 HDR 的原因。考虑 HDMI 带宽限制和 EDID 冲突。

b) **排查方案（6 分）**：
   - 写出使用 `dmesg`、`edid-decode`、`modetest` 等工具的具体排查命令和预期输出。
   - 描述如何检查 DSC 是否已启用以及当前带宽使用情况。
   - 描述如何检查 USB-C Alt Mode 的 Pin Assignment 状态。

c) **解决方案（6 分）**：
   - 针对三个问题分别提出至少一种可行的解决方案。
   - 如果必须减少一台显示器才能正常工作，说明选择保留哪两台以及理由。
   - 给出一个最优的完整配置方案，使三台显示器都能正常工作（可能包括降低某些参数）。

### 第二部分：简答题评分要点

**第 16 题**（5 分）：
- TMDS：3 对数据通道 + 1 对时钟通道，8B/10B 编码，最大 18 Gbps（每通道 6 Gbps）
- FRL：4 对数据通道（无独立时钟），16B/18B 编码，每通道最高 12 Gbps，最大 48 Gbps
- FRL 使用数据时钟恢复（CDR）而非独立时钟
- FRL 支持 VRR、DSC 和更高带宽
- FRL 固定延迟（Fixed Rate Link 特性）

**第 17 题**（5 分）：
- CR（Clock Recovery）：接收端锁定发送端时钟频率
- EQ（Equalization）：调整均衡器参数补偿信号损耗
- Symbol Lock：确保字符边界对齐
- Lane Alignment：多 lane 间对齐（去偏斜）
- Training Complete：验证链路状态，进入正常传输

**第 18 题**（5 分）：
- CEA-861 扩展块包含：视频时序列表（VICs）、音频格式、色彩格式支持、HDR 静态元数据、3D 结构等
- 驱动解析：提取支持的视频模式列表与 DisplayID 合并建立 mode list
- 音频能力决定 Audio InfoFrame 发送
- HDR 元数据决定是否启用 HDR 输出

**第 19 题**（5 分）：
- MST 通过将 DP 链路划分为多个虚通道（VC Payload）
- 每个 VC Payload 有唯一 ID 和带宽分配
- 核心数据结构：dc_mst_stream 包含 vcpi、sink、link 等信息
- 带宽管理：总带宽不超过 Main Link 带宽，动态分配
- 拓扑变化触发全路径重新检测

**第 20 题**（5 分）：
- Mode C：2 lane DP + USB 3.x，适合常规扩展坞
- Mode D：4 lane DP（无 USB 3.x），适合高分辨率高刷新率单屏
- Mode E：2 lane DP + USB 3.x（与 C 同），DP 引脚不同
- Mode F：4 lane DP（无 USB 3.x），与 D 不同厂家实现
- 选择依据：带宽需求 vs. USB 功能取舍

**第 21 题**（5 分）：
- 显示器通过 EDID 报告 VRR 范围（min Hz ~ max Hz）
- 驱动根据渲染帧率动态调整 VBlank 时长
- 帧率低于最低范围时触发 LFC：以整数倍频率刷新（如 30fps → 60Hz）
- LFC 消除了帧率低于 VRR 下限时的撕裂问题

### 第四部分：综合案例分析评分要点

**a) 问题分析（8 分）**：
1. DP 1.4 带宽限制：4K@240Hz 10-bit HDR 需要约 64 Gbps，DP 1.4 HBR3 仅 32.4 Gbps，DSC 3:1 后约 11 Gbps，但多显示器共享带宽时可能不足
2. USB-C 间歇性黑屏：Pin Assignment 切换问题、线缆质量、电磁干扰
3. HDMI HDR 失效：多显示器 EDID 冲突导致 HDMI 带宽分配不足

**b) 排查方案（6 分）**：
- `edid-decode < /sys/class/drm/card0-DP-1/edid`
- `cat /sys/kernel/debug/dri/0/amdgpu_dm_visual_confirm`
- `dmesg | grep -i "dsc\|link rate\|lane count"`
- `modetest -M amdgpu -p` 查看 connector 属性

**c) 解决方案（6 分）**：
- DP 1.4 在 DSC 下足够支持 4K@240Hz，需检查 DSC 是否启用
- USB-C 换用高质量线缆或改用原生 DP 口
- HDMI HDR 需检查 EDID 是否正确传递
- 最优方案：USB-C 接 6K（DSC），DP 接 4K@240Hz（DSC），HDMI 接 4K@120Hz（FRL 4-lane）
