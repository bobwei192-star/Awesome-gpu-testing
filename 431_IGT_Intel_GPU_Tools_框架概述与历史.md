# IGT（Intel GPU Tools）框架概述与历史

## 学习目标

- 理解 IGT GPU Tools 框架的起源、发展历程和设计目标
- 掌握 IGT 在 Linux DRM 驱动测试中的核心地位和作用
- 了解 IGT 与 Intel、AMD 等 GPU 厂商的关系
- 理解 IGT 的架构设计哲学和关键技术特点
- 能够评估 IGT 在 AMDGPU 驱动测试中的适用性

## 知识详解

### 一、概念原理

#### 1.1 IGT 发展历程

IGT GPU Tools（Intel Graphics Test Tools）最初由 Intel 开发，现已成为 Linux DRM 子系统的标准测试框架：

```
┌─────────────────────────────────────────────────────────────────────┐
│              IGT GPU Tools 发展历程                                  │
│                                                                     │
│  2008 年    Intel 内部开发图形驱动测试工具                           │
│     │       最初用于 Intel i915 驱动验证                             │
│     ▼                                                               │
│  2010 年    开源发布为 intel-gpu-tools                               │
│     │       迁移到 freedesktop.org                                   │
│     ▼                                                               │
│  2012 年    更名为 igt-gpu-tools                                     │
│     │       开始支持非 Intel GPU（通用 DRM 测试）                    │
│     ▼                                                               │
│  2015 年    成为 DRM 子系统官方测试框架                              │
│     │       集成到 drm-misc CI 流水线                                │
│     ▼                                                               │
│  2018 年    支持 AMDGPU 专用测试                                     │
│     │       添加 amdgpu/ 测试子目录                                  │
│     ▼                                                               │
│  2020+ 年   持续扩展：Chamelium、VKMS、多 GPU 支持                   │
│     │       成为 GPU 驱动测试的事实标准                              │
│     ▼                                                               │
│  现在       每月活跃更新，社区贡献者来自 Intel/AMD/Red Hat 等         │
│                                                                     │
│  数据来源：IGT Git 提交历史                                          │
│  https://gitlab.freedesktop.org/drm/igt-gpu-tools/-/commits/master   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 二、实践操作

#### 2.1 IGT 源码获取

```bash
#!/bin/bash
# get_igt.sh - IGT 源码获取脚本

set -e

IGT_DIR="${1:-$HOME/igt-gpu-tools}"

echo "=== IGT GPU Tools 源码获取 ==="
echo ""

# 官方仓库
IGT_REPO="https://gitlab.freedesktop.org/drm/igt-gpu-tools.git"

if [ -d "$IGT_DIR" ]; then
    echo "更新现有仓库..."
    cd "$IGT_DIR"
    git pull origin master
else
    echo "克隆仓库..."
    git clone --depth 1 "$IGT_REPO" "$IGT_DIR"
    cd "$IGT_DIR"
fi

echo ""
echo "仓库信息:"
echo "  位置: $IGT_DIR"
echo "  最新提交: $(git log --oneline -1)"
echo "  提交日期: $(git log -1 --format=%cd)"
echo ""

# 查看标签（稳定版本）
echo "最近标签:"
git tag --sort=-creatordate | head -5
echo ""

# 切换到最新稳定版本（可选）
# git checkout $(git describe --tags --abbrev=0)
```

## 今日小结

- IGT GPU Tools 最初由 Intel 开发，现已成为 Linux DRM 子系统的标准测试框架
- IGT 设计哲学包括贴近硬件、可重复性、可移植性、可扩展性和可调试性
- IGT 核心组件包括 libigt（核心库）、测试程序（tests/）、工具程序（tools/）和执行器（runner/）
- IGT 支持多种 GPU 厂商（Intel、AMD、ARM），通过条件编译和运行时检测实现硬件抽象
- IGT 是 DRM CI 流水线的核心测试组件，覆盖快速回归、功能、性能和压力测试
