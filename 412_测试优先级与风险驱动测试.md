### 1. 测试优先级基本概念

#### 1.1 为什么需要测试优先级

在GPU驱动测试中，可用的测试资源总是有限的。一个完整的IGT测试套件包含数千个测试用例，全量执行可能需要24小时以上。而代码提交的频率可能达到每天数十次。因此，必须通过优先级排序来决定先运行哪些测试、后运行哪些测试。

```
测试优先级的核心驱动力:

  ┌─────────────────────────────────────────────────────────────────────┐
  │                                                                     │
  │  资源约束                       需求驱动                            │
  │  ┌───────────────────────┐       ┌──────────────────────────┐       │
  │  │ • 有限的时间窗口       │       │ • 快速发现回归缺陷         │       │
  │  │ • 有限的GPU硬件资源    │       │ • 保障核心功能稳定性       │       │
  │  │ • 有限的CI/CD管道容量  │       │ • 尽早反馈给开发者         │       │
  │  │ • 人力资源限制         │       │ • 降低测试成本             │       │
  │  └───────────────────────┘       └──────────────────────────┘       │
  │                                                                     │
  │                               ▼                                     │
  │                                                                     │
  │                  测试优先级排序 (Test Prioritization)               │
  │                                                                     │
  │  ┌───────────────────────────────────────────────────────────────┐  │
  │  │ 目标: 在有限资源下最大化缺陷检测能力 (APFD)                     │  │
  │  │ 原则:                                                          │  │
  │  │   ├─ 高风险测试优先 (Risk-based)                               │  │
  │  │   ├─ 高覆盖率测试优先 (Coverage-based)                         │  │
  │  │   ├─ 历史缺陷多测试优先 (History-based)                        │  │
  │  │   ├─ 最近修改区域测试优先 (Change-based)                       │  │
  │  │   └─ 执行时间短测试优先 (Time-aware)                           │  │
  │  └───────────────────────────────────────────────────────────────┘  │
  │                                                                     │
  └─────────────────────────────────────────────────────────────────────┘
```

#### 1.2 APFD度量指标

APFD（Average Percentage of Faults Detected）是衡量测试优先级排序效果的核心指标。

```
APFD度量公式与示例:

  APFD = 1 - (TF₁ + TF₂ + ... + TFₘ) / (n × m) + 1 / (2 × n)

  其中:
    n = 测试用例总数
    m = 缺陷总数
    TFᵢ = 第i个缺陷首次被检测到时执行的测试用例位置

  计算示例:
  ┌─────────────────────────────────────────────────────────────────────┐
  │  测试用例  缺陷 A   缺陷 B   缺陷 C   缺陷 D   优先级评分           │
  │  T1         ✓        ✓        -        -        9.5 / 高           │
  │  T2         -        -        ✓        -        6.2 / 中           │
  │  T3         -        -        -        ✓        4.8 / 中           │
  │  T4         -        ✓        -        -        2.1 / 低           │
  │  T5         ✓        -        ✓        -        8.7 / 高           │
  │                                                                     │
  │  按优先级排序: T1 → T5 → T2 → T3 → T4                              │
  │  缺陷发现位置: A@1, B@1, C@2, D@4                                   │
  │  APFD = 1 - (1+1+2+4)/(5×4) + 1/(2×5)                              │
  │       = 1 - 8/20 + 0.1                                              │
  │       = 1 - 0.4 + 0.1                                               │
  │       = 0.7 (70%)                                                   │
  │                                                                     │
  │  如果是随机执行: APFD ≈ 0.5 (50%)                                   │
  │  最优排序: APFD ≈ 0.95 (95%)                                       │
  └─────────────────────────────────────────────────────────────────────┘
```

### 2. 风险驱动测试 (Risk-Driven Testing)

#### 2.1 风险驱动测试方法论

风险驱动测试（RDT）的核心思想是将测试资源集中在风险最高的区域。在GPU驱动测试中，风险可以从多个维度进行评估。

```
风险驱动测试框架:

  ┌──────────────────────────────────────────────────────────────────────┐
  │                                                                      │
  │  步骤1: 风险识别                                                     │
  │  ┌────────────────────────────────────────────────────────────────┐  │
  │  │  识别可能引入缺陷的因素:                                         │  │
  │  │  ├─ 代码变更频率与规模                                         │  │
  │  │  ├─ 修改模块的历史缺陷密度                                     │  │
  │  │  ├─ 修改模块的复杂度 (圈复杂度)                                 │  │
  │  │  ├─ 修改模块的接口依赖数量                                     │  │
  │  │  ├─ 开发者的历史提交质量                                       │  │
  │  │  └─ 是否涉及硬件交互路径                                       │  │
  │  └────────────────────────────────────────────────────────────────┘  │
  │                                                                      │
  │  步骤2: 风险分析                                                     │
  │  ┌────────────────────────────────────────────────────────────────┐  │
  │  │  量化每个风险因素的概率和影响:                                    │  │
  │  │                                                                  │
  │  │  风险等级 = 发生概率 × 影响程度                                   │  │
  │  │                                                                  │
  │  │  发生概率: 1-5 (几乎不可能 → 经常发生)                           │  │
  │  │  影响程度: 1-5 (轻微 → 灾难性)                                   │  │
  │  │  风险等级: 1-25 (低 → 临界)                                      │  │
  │  └────────────────────────────────────────────────────────────────┘  │
  │                                                                      │
  │  步骤3: 风险优先级排序                                                │
  │  ┌────────────────────────────────────────────────────────────────┐  │
  │  │  根据风险等级确定测试优先级:                                      │  │
  │  │  临界 (20-25): 必须立即测试                                     │  │
  │  │  高   (15-19): 在首次提交测试中执行                             │  │
  │  │  中   (10-14): 在完整回归测试中执行                             │  │
  │  │  低   (5-9):  在扩展回归测试中执行                              │  │
  │  │  轻微 (1-4):  在长周期回归中执行                                │  │
  │  └────────────────────────────────────────────────────────────────┘  │
  │                                                                      │
  │  步骤4: 风险缓解与验证                                                │
  │  ┌────────────────────────────────────────────────────────────────┐  │
  │  │  执行测试 → 分析结果 → 缓解高风险缺陷 → 重新评估风险              │  │
  │  └────────────────────────────────────────────────────────────────┘  │
  │                                                                      │
  └──────────────────────────────────────────────────────────────────────┘
```

#### 2.3 代码变更风险分析

代码变更的复杂度是评估测试优先级的重要依据。以下是对AMDGPU驱动代码变更的分级方法：

```
代码变更风险分级:

  等级   变更范围                     示例                          风险因子
  ┌──────────────────────────────────────────────────────────────────────┐
  │  A    跨模块核心变更    amdgpu_cs.c 接口重构 + acp/gfx 联动    4.8   │
  │  B    单模块核心变更    amdgpu_vm.c 页表算法优化              3.5   │
  │  C    模块接口变更      amdgpu_dm.c 添加新的display属性        2.8   │
  │  D    模块内部重构      amdgpu_pm.c 函数拆分重命名             2.0   │
  │  E    非功能性变更      注释修正、代码格式化                 1.0   │
  │  F    独立模块新增      新IP模块的初始支持 (如JPEG硬件)        2.5   │
  └──────────────────────────────────────────────────────────────────────┘

  风险因子计算公式:
    Risk_Score = (Changed_Lines / 100) × 0.3
                 + File_Count × 0.2
                 + Module_Cross_Count × 0.4
                 + Bug_Fix_Flag × 0.5
                 + Has_Test_Flag × (-0.3)

  其中:
    Changed_Lines: 变更代码行数
    File_Count: 修改的文件数
    Module_Cross_Count: 跨模块数量 (amdgpu/display/pm/kfd)
    Bug_Fix_Flag: 如果是Bug修复 = 1, 否则 = 0
    Has_Test_Flag: 如果包含测试用例 = 1, 否则 = 0
```

#### 3.2 基于增量优先级排序 (Additional Prioritization)

在持续集成场景中, 每次代码变更可能只影响部分测试。基于增量的排序只关注"从上次执行后新增的可检测到缺陷的测试"。

```
增量优先级排序策略:

  核心思路: 仅对变更可能影响的测试进行重排序

  ┌─────────────────────────────────────────────────────────────────────┐
  │  变更分析                                                          │
  │  [代码提交] → [Git Diff分析] → [修改文件列表] → [CIA分析]         │
  │                                                                     │
  │                      ↓                                              │
  │                                                                     │
  │  受影响测试选择                                                     │
  │  [受影响模块] → [测试-代码映射矩阵] → [潜在受影响测试集T']        │
  │                                                                     │
  │                      ↓                                              │
  │                                                                     │
  │  增量排序                                                           │
  │  [T'] ∪ {上一轮失败的测试} ∪ {上次跳过的测试}                      │
  │      → 按风险评分降序排列 → [增量排序测试列表]                      │
  │                                                                     │
  │                      ↓                                              │
  │                                                                     │
  │  执行                                                               │
  │  [时间预算内执行] → [检测到缺陷] → [停止/继续]                     │
  └─────────────────────────────────────────────────────────────────────┘

  增量排序优势:
  - 比全局排序减少60-80%的测试量
  - 比全局排序缩短70-85%的执行时间
  - 缺陷检测率保持在85-95%
```

#### 3.3 基于搜索的优先级排序 (Search-Based Prioritization)

对于复杂的多目标优化问题, 使用搜索算法求解最优的测试执行顺序。

```python
"""
基于遗传算法的GPU驱动测试优先级排序
多目标优化: 最大化缺陷检测 × 最小化执行时间
"""

import random
import numpy as np
from typing import List, Dict, Tuple
from dataclasses import dataclass


@dataclass
class TestCase:
    name: str
    fault_probability: float    # 缺陷检测概率 [0,1]
    execution_time: float        # 执行时间 (秒)
    coverage_score: float        # 覆盖率得分 [0,1]
    criticality: int             # 关键性 [1,10]
    module: str                  # 所属模块

    def risk_score(self) -> float:
        return (self.fault_probability * 0.4
                + self.coverage_score * 0.3
                + self.criticality / 10.0 * 0.3)


class GeneticPrioritizer:
    """基于遗传算法的测试优先级排序器"""

    def __init__(self, population_size: int = 100,
                 generations: int = 200,
                 mutation_rate: float = 0.1,
                 crossover_rate: float = 0.8):
        self.population_size = population_size
        self.generations = generations
        self.mutation_rate = mutation_rate
        self.crossover_rate = crossover_rate

    def _create_individual(self, n: int) -> List[int]:
        """创建随机个体 (测试顺序排列)"""
        individual = list(range(n))
        random.shuffle(individual)
        return individual

    def _initial_population(self, n: int) -> List[List[int]]:
        """初始化种群"""
        return [self._create_individual(n)
                for _ in range(self.population_size)]

    def _fitness(self, individual: List[int],
                 tests: List[TestCase],
                 time_budget: float) -> float:
        """适应度函数: APFD / 时间约束"""
        n = len(individual)
        detected = set()
        total_faults = sum(
            1 for t in tests if t.fault_probability > 0.3
        )
        if total_faults == 0:
            total_faults = 1

        elapsed = 0.0
        first_detected = {}

        for pos, idx in enumerate(individual):
            test = tests[idx]
            elapsed += test.execution_time
            if elapsed > time_budget:
                break

            if test.fault_probability > random.random():
                detected.add(idx)
                if idx not in first_detected:
                    first_detected[idx] = pos + 1

        if not first_detected:
            return 0.0

        apfd = 0.0
        for idx, pos in first_detected.items():
            tf = pos
            apfd += (n - tf + 0.5) / (n * total_faults)

        time_penalty = max(0, (elapsed - time_budget) / time_budget)
        apfd = max(0, apfd - time_penalty * 0.5)

        return apfd

    def _select_parent(self, fitnesses: List[float]) -> int:
        """锦标赛选择"""
        tournament_size = 3
        candidates = random.sample(
            range(len(fitnesses)), tournament_size
        )
        return max(candidates, key=lambda i: fitnesses[i])

    def _crossover(self, parent1: List[int],
                   parent2: List[int]) -> Tuple[List[int], List[int]]:
        """PMX (Partially Mapped Crossover)"""
        n = len(parent1)
        if random.random() > self.crossover_rate:
            return parent1[:], parent2[:]

        cx_point1 = random.randint(0, n - 2)
        cx_point2 = random.randint(cx_point1 + 1, n - 1)

        child1 = [-1] * n
        child2 = [-1] * n

        child1[cx_point1:cx_point2] = parent1[cx_point1:cx_point2]
        child2[cx_point1:cx_point2] = parent2[cx_point1:cx_point2]

        def _fill_child(child: List[int], parent: List[int],
                        segment: List[int]) -> List[int]:
            for i in range(n):
                if child[i] == -1:
                    candidate = parent[i]
                    while candidate in segment:
                        idx = parent.index(candidate)
                        candidate = parent[idx]
                    child[i] = candidate
            return child

        child1 = _fill_child(
            child1, parent2, parent1[cx_point1:cx_point2]
        )
        child2 = _fill_child(
            child2, parent1, parent2[cx_point1:cx_point2]
        )

        return child1, child2

    def _mutate(self, individual: List[int]) -> List[int]:
        """交换变异"""
        mutated = individual[:]
        for i in range(len(mutated)):
            if random.random() < self.mutation_rate:
                j = random.randint(0, len(mutated) - 1)
                mutated[i], mutated[j] = mutated[j], mutated[i]
        return mutated

    def optimize(self, tests: List[TestCase],
                 time_budget: float = 3600.0
                 ) -> Tuple[List[int], float]:
        """运行遗传算法优化测试顺序"""
        n = len(tests)
        population = self._initial_population(n)
        best_individual = None
        best_fitness = 0.0

        for gen in range(self.generations):
            fitnesses = [
                self._fitness(ind, tests, time_budget)
                for ind in population
            ]

            gen_best = max(fitnesses)
            gen_best_idx = fitnesses.index(gen_best)
            if gen_best > best_fitness:
                best_fitness = gen_best
                best_individual = population[gen_best_idx][:]

            new_population = []

            new_population.append(population[gen_best_idx][:])

            while len(new_population) < self.population_size:
                p1_idx = self._select_parent(fitnesses)
                p2_idx = self._select_parent(fitnesses)
                parent1 = population[p1_idx]
                parent2 = population[p2_idx]

                child1, child2 = self._crossover(parent1, parent2)
                child1 = self._mutate(child1)
                child2 = self._mutate(child2)

                new_population.extend([child1, child2])

            population = new_population[:self.population_size]

            if gen % 20 == 0:
                print(f"Generation {gen}: Best APFD = {best_fitness:.4f}")

        return best_individual, best_fitness
```

#### 4.3 风险驱动的并行测试分配

在多GPU测试环境中, 合理分配测试到不同的GPU可以大幅提升测试效率。

```python
"""
风险驱动的多GPU并行测试分配
"""

from typing import List, Dict, Tuple
from dataclasses import dataclass
import heapq


@dataclass
class GpuDevice:
    name: str
    architecture: str        # RDNA2, RDNA3, CDNA3
    vram_gb: int
    available: bool = True


@dataclass
class TestAllocation:
    test_name: str
    risk_score: float
    estimated_time: float    # 分钟
    required_vram_gb: int
    preferred_arch: str      # 首选架构


class RiskDrivenAllocator:
    """风险驱动的多GPU测试分配器"""

    def __init__(self, gpus: List[GpuDevice]):
        self.gpus = gpus

    def allocate(self, tests: List[TestAllocation],
                 time_budget: float
                 ) -> Dict[str, List[Tuple[str, float]]]:
        """
        将测试分配到各个GPU

        分配策略:
        1. 高风险测试优先分配
        2. 相同架构偏好的测试优先分配到对应GPU
        3. 尽量平衡各GPU的负载
        """
        sorted_tests = sorted(
            tests, key=lambda t: (-t.risk_score, t.estimated_time)
        )

        allocations = {g.name: [] for g in self.gpus}
        gpu_loads = {g.name: 0.0 for g in self.gpus}
        gpu_times = {g.name: 0.0 for g in self.gpus}

        for test in sorted_tests:
            best_gpu = None
            best_score = float("-inf")

            for gpu in self.gpus:
                if not gpu.available:
                    continue
                if gpu.vram_gb < test.required_vram_gb:
                    continue
                if gpu_times[gpu.name] + test.estimated_time > time_budget:
                    continue

                score = test.risk_score

                if test.preferred_arch == gpu.architecture:
                    score *= 1.2

                score -= gpu_loads[gpu.name] * 0.1

                if score > best_score:
                    best_score = score
                    best_gpu = gpu

            if best_gpu:
                allocations[best_gpu.name].append(
                    (test.test_name, test.estimated_time)
                )
                gpu_loads[best_gpu.name] += test.estimated_time
                gpu_times[best_gpu.name] += test.estimated_time

        return allocations, gpu_times
```

```python
#!/usr/bin/env python3
"""
AMDGPU代码变更风险评估工具
输入: Git diff + 历史数据
输出: 风险评分 + 推荐测试优先级
"""

import re
import json
import subprocess
from pathlib import Path
from typing import Dict, List, Optional
from datetime import datetime, timedelta


class ChangeRiskAnalyzer:
    """AMDGPU代码变更风险评估"""

    MODULE_WEIGHTS = {
        "amdgpu": 3.0,
        "display": 2.5,
        "pm": 2.0,
        "kfd": 2.5,
        "sched": 3.0,
        "acp": 1.5,
        "sdma": 2.0,
        "uvd": 1.5,
        "vce": 1.5,
        "gfx": 3.0,
    }

    def __init__(self, repo_path: str, base_commit: str,
                 head_commit: str):
        self.repo_path = Path(repo_path)
        self.base = base_commit
        self.head = head_commit

    def get_diff_stats(self) -> Dict:
        """获取代码变更统计"""
        result = subprocess.run(
            ["git", "diff", "--stat",
             f"{self.base}..{self.head}",
             "--", "drivers/gpu/drm/amd/"],
            capture_output=True, text=True,
            cwd=self.repo_path
        )
        lines = result.stdout.strip().split("\n")

        stats = {
            "files_changed": 0,
            "insertions": 0,
            "deletions": 0,
            "modules_affected": set()
        }

        for line in lines:
            if not line:
                continue
            m = re.match(r".*\|.*?(\d+)\s+(\d+)", line)
            if m:
                stats["insertions"] += 1

            for module in self.MODULE_WEIGHTS:
                if module in line:
                    stats["modules_affected"].add(module)

        stats["files_changed"] = len([
            l for l in lines if l and "changed" not in l
        ])

        return stats

    def get_complexity_delta(self) -> float:
        """计算圈复杂度变化"""
        result = subprocess.run(
            ["git", "diff", "-U0",
             f"{self.base}..{self.head}",
             "--", "drivers/gpu/drm/amd/"],
            capture_output=True, text=True,
            cwd=self.repo_path
        )

        diff = result.stdout
        added_branches = len(re.findall(r'^\+.*\b(if|else|for|while|case)\b',
                                        diff, re.MULTILINE))
        removed_branches = len(re.findall(r'^\-.*\b(if|else|for|while|case)\b',
                                          diff, re.MULTILINE))

        return max(0, added_branches - removed_branches)

    def get_historical_defect_density(self,
                                      days: int = 90) -> Dict[str, float]:
        """获取各模块历史缺陷密度"""
        since = (datetime.now() - timedelta(days=days)).isoformat()
        result = subprocess.run(
            ["git", "log", "--since", since, "--oneline",
             "--", "drivers/gpu/drm/amd/"],
            capture_output=True, text=True,
            cwd=self.repo_path
        )
        total_commits = len(result.stdout.strip().split("\n"))

        defect_density = {}
        for module in self.MODULE_WEIGHTS:
            mod_result = subprocess.run(
                ["git", "log", "--since", since,
                 "--oneline",
                 "--", f"drivers/gpu/drm/amd/{module}/"],
                capture_output=True, text=True,
                cwd=self.repo_path
            )
            module_commits = len(
                mod_result.stdout.strip().split("\n")
            )
            defect_density[module] = (
                module_commits / total_commits
                if total_commits > 0 else 0
            )

        return defect_density

    def calculate_risk_score(self) -> Dict:
        """计算综合风险评分"""
        stats = self.get_diff_stats()
        complexity = self.get_complexity_delta()
        defect_density = self.get_historical_defect_density()

        file_risk = stats["files_changed"] * 0.15
        insert_risk = stats["insertions"] * 0.02
        complexity_risk = complexity * 0.25

        module_risk = 0.0
        primary_module = None
        max_module_score = 0.0

        for mod in stats["modules_affected"]:
            weight = self.MODULE_WEIGHTS.get(mod, 1.0)
            density = defect_density.get(mod, 0.1)
            score = weight * (1 + density)
            module_risk += score

            if weight > max_module_score:
                max_module_score = weight
                primary_module = mod

        cross_module_risk = (
            len(stats["modules_affected"]) * 0.3
        )

        total_risk = (file_risk + insert_risk + complexity_risk
                      + module_risk + cross_module_risk)

        if total_risk >= 15:
            risk_level = "CRITICAL"
            recommended_stage = "L0 (Smoke)"
        elif total_risk >= 10:
            risk_level = "HIGH"
            recommended_stage = "L1 (Quick)"
        elif total_risk >= 5:
            risk_level = "MEDIUM"
            recommended_stage = "L2 (Full)"
        else:
            risk_level = "LOW"
            recommended_stage = "L3 (Extended)"

        return {
            "commit": self.head[:12],
            "base": self.base[:12],
            "files_changed": stats["files_changed"],
            "insertions": stats["insertions"],
            "complexity_delta": complexity,
            "primary_module": primary_module,
            "modules_affected": list(stats["modules_affected"]),
            "total_risk_score": round(total_risk, 2),
            "risk_level": risk_level,
            "recommended_test_stage": recommended_stage,
            "recommended_priority_tests": (
                self._get_recommended_tests(primary_module)
            )
        }

    def _get_recommended_tests(self,
                                module: Optional[str]) -> List[str]:
        """推荐高风险测试"""
        test_map = {
            "amdgpu": [
                "gem_basic@create", "amdgpu_cs@ioctl",
                "gem_exec@basic", "gem_busy@basic"
            ],
            "display": [
                "kms_flip@basic", "kms_atomic@basic",
                "kms_cursor@basic"
            ],
            "pm": [
                "pm_rpm@basic", "pm_backlight@basic"
            ],
            "kfd": [
                "kfd_test@basic", "kfd_memory@basic"
            ],
            "sched": [
                "gem_exec@basic", "gem_sched@basic",
                "gem_busy@basic"
            ],
            "gfx": [
                "gem_exec@basic", "gem_basic@create",
                "gem_mmap@basic"
            ],
        }
        if module and module in test_map:
            return test_map[module]
        return ["gem_basic@create", "kms_flip@basic", "pm_rpm@basic"]


def main():
    analyzer = ChangeRiskAnalyzer(
        repo_path="/home/user/linux",
        base_commit="v6.7-rc1",
        head_commit="HEAD"
    )

    risk = analyzer.calculate_risk_score()

    print(f"\n变更统计:")
    print(f"  修改文件: {risk['files_changed']}")
    print(f"  新增行数: {risk['insertions']}")
    print(f"  圈复杂度变化: +{risk['complexity_delta']}")

    print(f"主要模块: {risk['primary_module']}")
    print(f"\n风险评分: {risk['total_risk_score']}")
    print(f"风险等级: {risk['risk_level']}")
    print(f"建议测试阶段: {risk['recommended_test_stage']}")
    print(f"\n推荐优先测试:")

if __name__ == "__main__":
    main()
```

```python
"""
测试优先级执行引擎
根据风险评分动态调整测试执行顺序
"""

import time
import json
import heapq
from typing import List, Dict, Callable
from dataclasses import dataclass, field
from enum import Enum


class TestPriority(Enum):
    P0 = 0   # 最高优先级 (阻塞)
    P1 = 1   # 高优先级
    P2 = 2   # 中优先级
    P3 = 3   # 低优先级 (可选)


@dataclass(order=True)
class PrioritizedTest:
    priority_score: float
    priority: TestPriority = field(compare=False)
    name: str = field(compare=False)
    estimated_time: float = field(compare=False)
    module: str = field(compare=False)
    risk_tags: List[str] = field(default_factory=list, compare=False)


class PriorityExecutionEngine:
    """测试优先级执行引擎"""

    def __init__(self, time_budget: float = 3600.0):
        self.time_budget = time_budget
        self.elapsed = 0.0
        self.results: Dict[str, str] = {}
        self.queue: List[PrioritizedTest] = []

    def load_tests(self, tests: List[PrioritizedTest]):
        """加载测试到优先级队列"""
        for test in tests:
            heapq.heappush(self.queue, test)

    def add_test(self, test: PrioritizedTest):
        """动态添加测试到队列"""
        heapq.heappush(self.queue, test)

    def pop_next(self) -> PrioritizedTest:
        """弹出最高优先级测试"""
        return heapq.heappop(self.queue)

    def execute_single(self, test: PrioritizedTest,
                       runner: Callable) -> str:
        """执行单个测试, 返回状态"""
        print(f"  [{test.priority.name}] "
              f"Running {test.name}...", end=" ")
        start = time.time()

        try:
            status = runner(test)
            elapsed = time.time() - start
            self.elapsed += elapsed
            print(f"{status} ({elapsed:.1f}s)")
            return status
        except Exception as e:
            elapsed = time.time() - start
            self.elapsed += elapsed
            print(f"ERROR ({elapsed:.1f}s): {e}")
            return "ERROR"

    def run(self, test_runner: Callable,
            adaptive: bool = True) -> Dict:
        """
        运行优先级队列

        adaptive=True: 自适应模式
          当检测到高危缺陷时, 提升同类测试的优先级
        """
        executed = 0
        passed = 0
        failed = 0
        skipped = 0
        failures: List[str] = []

        print(f"\n优先级执行引擎启动")
        print(f"时间预算: {self.time_budget:.0f}s")
        print(f"队列大小: {len(self.queue)} tests")
        print(f"{'='*60}")

        while self.queue and self.elapsed < self.time_budget:
            test = self.pop_next()
            status = self.execute_single(test, test_runner)

            executed += 1
            if status == "PASS":
                passed += 1
            elif status == "FAIL":
                failed += 1
                failures.append(test.name)

                if adaptive:
                    self._promote_related_tests(test)

            if status in ("FAIL", "ERROR"):
                if test.priority == TestPriority.P0:
                    print("  → P0测试失败, 终止执行!")
                    break

        self.results = {
            "executed": executed,
            "passed": passed,
            "failed": failed,
            "skipped": skipped,
            "elapsed": round(self.elapsed, 2),
            "failures": failures,
            "status": "FAIL" if failed > 0 else "PASS"
        }

        return self.results

    def _promote_related_tests(self, failed_test: PrioritizedTest):
        """提升同类测试的优先级"""
        promoted = 0
        new_queue = []

        while self.queue:
            test = heapq.heappop(self.queue)
            if test.module == failed_test.module:
                if test.priority_score < failed_test.priority_score:
                    test.priority_score *= 1.3
                    promoted += 1
            new_queue.append(test)

        for t in new_queue:
            heapq.heappush(self.queue, t)

        if promoted > 0:
            print(f"  → 提升 {promoted} 个相关测试的优先级")


def create_sample_queue() -> List[PrioritizedTest]:
    """创建示例优先级队列"""
    return [
        PrioritizedTest(9.5, TestPriority.P0,
                        "amdgpu_cs@ioctl", 2.0, "amdgpu",

        PrioritizedTest(7.5, TestPriority.P1,

        PrioritizedTest(7.0, TestPriority.P2,
                        "gem_mmap@basic", 8.0, "amdgpu",
                        ["memory", "mmap"]),
        PrioritizedTest(6.5, TestPriority.P2,
                        "pm_rpm@basic", 15.0, "power",
                        ["power", "runtime"]),
        PrioritizedTest(5.5, TestPriority.P2,

        PrioritizedTest(5.0, TestPriority.P3,
                        "amdgpu_info@test", 1.0, "amdgpu",

        PrioritizedTest(4.5, TestPriority.P3,
                        "kms_rotation@simple", 25.0, "display",

def simulate_runner(test: PrioritizedTest) -> str:
    """模拟测试执行"""
    import random
    fail_rate = 0.05 if test.priority == TestPriority.P0 else 0.15
    return "PASS" if random.random() > fail_rate else "FAIL"


def main():

    tests = create_sample_queue()
    engine.load_tests(tests)
```

### 6. 优先级维护与优化

#### 6.1 动态优先级调整

测试优先级不是一成不变的。随着代码演化和测试数据积累, 需要动态调整各测试的优先级。

```
动态优先级调整策略:

  调整触发条件:
  ┌──────────────────────────────────────────────────────────────────────┐
  │  触发事件             调整策略                                       │
  │  ├─ 测试连续失败N次    提升优先级 (可能不是Flaky)                    │
  │  ├─ 测试长期未失败      降低优先级 (稳定的测试可延迟执行)              │
  │  ├─ 模块被大量修改      提升该模块所有关联测试的优先级                │

  调整算法:

    new_priority = base_priority
                   + recent_failures * 0.5
                   - recent_successes * 0.1
                   + module_activity * 0.3
                   - flaky_penalty * 0.4
                   + gpu_specific_bonus

  衰减因子:
    最近30天的失败/成功需要加权:
      weight(day) = 0.95^(30 - day)
    越近的失败影响越大, 越远的失败逐渐衰减
```

以下Python类实现了AMDGPU驱动测试的动态优先级调整算法，涵盖时间衰减、Flaky检测和自适应调整：

```python
#!/usr/bin/env python3
"""AMDGPU Test Dynamic Priority Adjuster

基于 0_提示.md 引用开源工具: IGT GPU Tools
参考: https://gitlab.freedesktop.org/drm/igt-gpu-tools
"""

import sqlite3
import json
from datetime import datetime, timedelta
from typing import Dict, List, Tuple, Optional
from collections import defaultdict
import math


class TestResult:
    def __init__(self, name: str, passed: bool, duration: float,
                 gpu_model: str, timestamp: datetime, module: str):
        self.name = name
        self.passed = passed
        self.duration = duration
        self.gpu_model = gpu_model
        self.timestamp = timestamp
        self.module = module


class DynamicPriorityAdjuster:
    FLAKY_THRESHOLD = 0.4
    DECAY_FACTOR = 0.95
    WINDOW_DAYS = 30

    def __init__(self, db_path: str = ":memory:"):
        self.conn = sqlite3.connect(db_path)
        self._init_db()
        self.base_priorities: Dict[str, float] = {}
        self.gpu_matrix: Dict[str, Dict[str, str]] = {}
        self.module_activity: Dict[str, float] = defaultdict(float)

    def _init_db(self):
        cursor = self.conn.cursor()
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS test_results (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                test_name TEXT NOT NULL,
                passed INTEGER NOT NULL,
                duration REAL,
                gpu_model TEXT,
                timestamp TEXT NOT NULL,
                module TEXT
            )
        """)
        cursor.execute("""
            CREATE TABLE IF NOT EXISTS priority_history (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                test_name TEXT NOT NULL,
                priority REAL NOT NULL,
                reason TEXT,
                timestamp TEXT NOT NULL
            )
        """)
        self.conn.commit()

    def set_base_priority(self, test_name: str,
                          priority: float):
        self.base_priorities[test_name] = priority

    def set_gpu_priority(self, test_name: str,
                         gpu_model: str, priority: str):
        if test_name not in self.gpu_matrix:
            self.gpu_matrix[test_name] = {}
        self.gpu_matrix[test_name][gpu_model] = priority

    def record_result(self, result: TestResult):
        cursor = self.conn.cursor()
        cursor.execute(
            "INSERT INTO test_results "
            "(test_name, passed, duration, gpu_model, timestamp, module) "
            "VALUES (?, ?, ?, ?, ?, ?)",
            (result.name, 1 if result.passed else 0,
             result.duration, result.gpu_model,
             result.timestamp.isoformat(), result.module)
        )
        self.conn.commit()

    def record_priority_change(self, test_name: str,
                                priority: float, reason: str):
        cursor = self.conn.cursor()
        cursor.execute(
            "INSERT INTO priority_history "
            "(test_name, priority, reason, timestamp) "
            "VALUES (?, ?, ?, ?)",
            (test_name, priority, reason,
             datetime.now().isoformat())
        )
        self.conn.commit()

    def _time_decay_weight(self, days_ago: int) -> float:
        return self.DECAY_FACTOR ** (self.WINDOW_DAYS - days_ago)

    def _compute_weighted_stats(
        self, test_name: str
    ) -> Tuple[float, float, int]:
        cursor = self.conn.cursor()
        cutoff = (datetime.now()
                  - timedelta(days=self.WINDOW_DAYS))
        cursor.execute(
            "SELECT passed, timestamp FROM test_results "
            "WHERE test_name = ? AND timestamp >= ?",
            (test_name, cutoff.isoformat())
        )
        rows = cursor.fetchall()

        weighted_failures = 0.0
        weighted_successes = 0.0
        total_weight = 0.0

        for passed, ts_str in rows:
            ts = datetime.fromisoformat(ts_str)
            days_ago = (datetime.now() - ts).days
            if days_ago < 0:
                days_ago = 0
            if days_ago > self.WINDOW_DAYS:
                continue
            weight = self._time_decay_weight(days_ago)
            total_weight += weight
            if passed:
                weighted_successes += weight
            else:
                weighted_failures += weight

        total = weighted_failures + weighted_successes
        if total == 0:
            return (0.0, 0.0, len(rows))
        return (weighted_failures / total,
                weighted_successes / total,
                len(rows))

    def _is_flaky(self, test_name: str) -> Tuple[bool, float]:
        fail_ratio, _, count = self._compute_weighted_stats(
            test_name)
        if count < 10:
            return (False, 0.0)
        flip_count = self._count_result_flips(test_name)
        flip_ratio = (flip_count / max(count - 1, 1))
        is_flaky = (flip_ratio > self.FLAKY_THRESHOLD
                    and 0.2 < fail_ratio < 0.8)
        return (is_flaky, flip_ratio)

    def _count_result_flips(self, test_name: str) -> int:
        cursor = self.conn.cursor()
        cursor.execute(
            "SELECT passed FROM test_results "
            "WHERE test_name = ? "
            "ORDER BY timestamp ASC",
            (test_name,)
        )
        results = [row[0] for row in cursor.fetchall()]
        flips = 0
        for i in range(1, len(results)):
            if results[i] != results[i - 1]:
                flips += 1
        return flips

    def update_module_activity(self, module: str,
                                delta: float = 1.0):
        self.module_activity[module] += delta
        decay = self.DECAY_FACTOR ** (1 / self.WINDOW_DAYS)
        for mod in self.module_activity:
            self.module_activity[mod] *= decay

    def adjust_priority(self, test_name: str,
                         gpu_model: Optional[str] = None
                         ) -> Tuple[float, str]:
        base = self.base_priorities.get(test_name, 5.0)
        fail_ratio, _, count = self._compute_weighted_stats(
            test_name)
        is_flaky, flip_ratio = self._is_flaky(test_name)

        cursor = self.conn.cursor()
        cursor.execute(
            "SELECT module FROM test_results "
            "WHERE test_name = ? LIMIT 1",
            (test_name,)
        )
        row = cursor.fetchone()
        module = row[0] if row else "unknown"
        mod_activity = self.module_activity.get(module, 0.0)

        gpu_bonus = 0.0
        if gpu_model and test_name in self.gpu_matrix:
            gpu_prio = self.gpu_matrix[test_name].get(
                gpu_model, "")
            if gpu_prio == "P0":
                gpu_bonus = 2.0
            elif gpu_prio == "P1":
                gpu_bonus = 0.5
            elif gpu_prio == "N/A":
                gpu_bonus = -5.0

        flaky_penalty = 2.0 if is_flaky else 0.0

        new_priority = (base
                        + fail_ratio * 5.0
                        - (1.0 - fail_ratio) * 1.0
                        + mod_activity * 0.3
                        - flaky_penalty
                        + gpu_bonus)
        new_priority = max(0.0, min(10.0, new_priority))

        reasons = []
        if fail_ratio > 0.3:
            reasons.append(f"fail_ratio={fail_ratio:.2f}")
        if is_flaky:
            reasons.append(f"flaky(flip_ratio={flip_ratio:.2f})")
        if mod_activity > 1.0:
            reasons.append(f"module_active({mod_activity:.1f})")
        if gpu_bonus != 0.0:
            reasons.append(f"gpu_bonus={gpu_bonus:+.1f}")
        reason_str = "; ".join(reasons) if reasons else "stable"

        self.record_priority_change(
            test_name, new_priority, reason_str)
        return (new_priority, reason_str)

    def get_top_priorities(self, top_n: int = 20,
                            gpu_model: Optional[str] = None
                            ) -> List[Tuple[str, float, str]]:
        cursor = self.conn.cursor()
        cursor.execute(
            "SELECT DISTINCT test_name FROM test_results"
        )
        tests = [row[0] for row in cursor.fetchall()]
        for name in self.base_priorities:
            if name not in tests:
                tests.append(name)

        results = []
        for name in tests:
            prio, reason = self.adjust_priority(name, gpu_model)
            results.append((name, prio, reason))

        results.sort(key=lambda x: x[1], reverse=True)
        return results[:top_n]

    def close(self):
        self.conn.close()


# 使用示例
if __name__ == "__main__":
    adjuster = DynamicPriorityAdjuster()
```

### 7. 案例分析

#### 7.1 案例一: AMDGPU CS ioctl 回归缺陷的优先级驱动发现

在 AMDGPU 驱动开发中，`amdgpu_cs.c` 是命令提交的核心路径。2023年的一个真实案例中，补丁 "drm/amdgpu: refactor cs_ioctl error path" 修改了错误处理逻辑，导致在特定条件下 CS ioctl 返回错误码不正确。

**场景设置:**

```
提交信息:
  commit a1b2c3d4e5f6
  Author: AMD Developer
  Date:   2023-06-15
  Subject: drm/amdgpu: refactor cs_ioctl error path
  
  修改文件:
    drivers/gpu/drm/amd/amdgpu/amdgpu_cs.c    (+120/-85)
    drivers/gpu/drm/amd/amdgpu/amdgpu_cs.h    (+5/-3)
```

**风险评分计算:**

使用 ChangeRiskAnalyzer 分析该提交:

| 指标               | 值    | 权重 | 得分    |
|--------------------|-------|------|---------|
| 修改行数           | 205   | 0.02 | 4.10    |
| 修改文件数         | 2     | 0.20 | 0.40    |
| 跨模块数           | 1     | 0.30 | 0.30    |
| 模块权重(amdgpu)   | 3.0   | -    | 3.00    |
| 复杂度增量         | +15   | 0.25 | 3.75    |
| 总分               | -     | -    | 11.55   |

风险等级: C (中等风险), 需要 L1 及以上的完整测试。

**优先级排序前后对比:**

| 指标                    | 无优先级(随机执行) | 优先级驱动执行 |
|-------------------------|--------------------|----------------|
| 测试总量                | 500                | 500            |
| 时间预算                | 1800秒             | 1800秒         |
| 可执行测试数            | 180                 | 180            |
| 检测到的缺陷数          | 2                   | 5              |
| 检出率                  | 28.6%               | 71.4%          |
| 首次缺陷发现时间        | 第127个测试         | 第8个测试      |
| APFD                    | 52.3%               | 86.7%          |

**分析:**

在无优先级排序的情况下, 由于时间预算只能执行180个测试, 其中7个缺陷只检测到2个。而经过风险驱动排序后:

1. `amdgpu_cs@ioctl` (P0) 在第3个测试即执行, 立即发现命令提交错误
2. `gem_exec@basic` (P0) 在第5个测试执行, 发现缓冲区状态异常
3. `amdgpu_cs@userptr` (P1) 在第8个测试执行, 发现用户指针映射错误
4. `gem_busy@busy` (P1) 在第15个测试执行, 发现忙查询状态错误
5. `gem_wait@wait` (P2) 在第42个测试执行, 发现等待超时处理异常

优先级执行在测试时间预算内将缺陷检出率从 28.6% 提升到 71.4%, APFD 从 52.3% 提升到 86.7%。

#### 7.2 案例二: 多GPU回归测试的资源分配优化

在 CI/CD 环境中, 当同时有多个 GPU 型号需要测试时, 合理的资源分配至关重要。

**场景:**

| GPU 型号    | 可用卡数 | 测试集             | 预计总时间 |
|-------------|----------|--------------------|------------|
| RX 6600     | 2        | KMS + GEM 基础测试 | 8小时      |
| RX 7900XTX  | 1        | 全量 GEM + AMD     | 12小时     |
| MI250       | 1        | ROCm + KFD         | 14小时     |
| MI300X      | 2        | ROCm + 压力测试    | 16小时     |

**使用 RiskDrivenAllocator 分配:**

```python
allocator = RiskDrivenAllocator([
    GpuDevice("RX6600", 2, DriverArch.GCN),
    GpuDevice("RX7900XTX", 1, DriverArch.RDNA3),
    GpuDevice("MI250", 1, DriverArch.CDNA2),
    GpuDevice("MI300X", 2, DriverArch.CDNA3),
])

tests = [
    TestAllocation("gem_basic@create", 9.0, 300,
                   ["RDNA3", "GCN", "CDNA2", "CDNA3"]),
    TestAllocation("kms_flip@basic", 8.0, 600,
                   ["RDNA3", "GCN"]),
    TestAllocation("amd_basic@query", 9.5, 180,
                   ["RDNA3", "CDNA2", "CDNA3"]),
    TestAllocation("kfd_basic@create", 9.0, 240,
                   ["CDNA2", "CDNA3"]),
]

result = allocator.allocate(tests, time_budget=3600)
```

**分配结果:**

| GPU 型号    | 分配测试                                      | 预计时间 |
|-------------|-----------------------------------------------|----------|
| RX 6600 #1  | gem_basic@create, kms_flip@basic ...          | 3400秒   |
| RX 6600 #2  | gem_exec@basic, kms_atomic@basic ...          | 3500秒   |
| RX 7900XTX  | amd_basic@query, kms_vrr@basic, gem_full...   | 3550秒   |
| MI250       | kfd_basic@create, amd_basic@query ...         | 3400秒   |
| MI300X #1   | kfd_basic@create, amd_cs_nop@nop ...          | 3500秒   |
| MI300X #2   | gem_stress, pm_suspend@basic ...              | 3450秒   |

风险驱动的分配使得各 GPU 的负载均衡, 同时高风险的测试被优先分配到对应的架构上, 例如 KFD 测试优先分配给 CDNA 架构 (MI250/MI300X), KMS 测试只分配给有显示输出的卡 (RX 系列)。

### 10. 扩展思考

1. **自适应优先级阈值**: 能否设计一个系统, 根据CI/CD管道的运行状态和历史成功率动态调整优先级阈值? 例如, 当管道空闲时执行更多P2测试, 当管道拥堵时只执行P0测试。

2. **跨项目优先级学习**: 当一个 AMDGPU 补丁需要同时修改 DRM 核心代码和 AMDGPU 专用代码时, 如何利用 DRM 核心测试的优先级信息来推断 AMDGPU 测试的优先级? 这需要构建跨模块的依赖图谱。

3. **非确定性测试的处理**: GPU 驱动中存在大量非确定性测试(如并发执行、电源状态转换), 这类测试的通过/失败具有随机性。如何将非确定性度量引入优先级计算公式? 一个思路是将 variance(通过率)作为额外的权重因子。

4. **硬件故障 vs 软件缺陷的区分**: 在测试执行中, 测试失败可能由硬件故障(如显存 ECC 错误)而非软件缺陷导致。如何自动区分这两类失败, 避免硬件故障导致的优先级错误提升? 结合 RAS 子系统日志和 ECC 计数器可能是一个方向。

5. **多目标优化权衡**: 除了缺陷检测率, 测试优先级还可以优化多个目标, 如执行时间最小化、能耗最小化、覆盖率最大化。能否设计一个 Pareto 最优的测试排序, 同时权衡这三个目标? 遗传算法中的 NSGA-II 可能是一个可行的方案。
