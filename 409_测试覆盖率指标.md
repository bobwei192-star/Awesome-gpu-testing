# 测试覆盖率指标

## 1. 学习目标

- 理解测试覆盖率的核心概念和分类
- 掌握代码覆盖率的不同度量方法（语句、分支、条件、路径等）
- 学会使用gcov/lcov收集C代码覆盖率
- 理解覆盖率在GPU驱动测试中的应用和局限性
- 掌握覆盖率驱动的测试改进方法
- 了解覆盖率阈值设定和验收标准

## 2. 测试覆盖率概述

### 2.1 什么是测试覆盖率

测试覆盖率（Test Coverage）是衡量测试完整性的一种度量指标，反映被测代码被测试用例执行的程度。

```
覆盖率在测试中的位置：

软件测试活动
  │
  ├── 测试设计
  │   ├── 需求覆盖率（Requirement Coverage）
  │   ├── 功能覆盖率（Functional Coverage）  ← 黑盒视角
  │   └── 风险覆盖率（Risk Coverage）
  │
  └── 测试执行
      └── 代码覆盖率（Code Coverage）        ← 白盒视角
          ├── 语句覆盖率
          ├── 分支覆盖率
          ├── 条件覆盖率
          ├── 路径覆盖率
          └── 函数/方法覆盖率

覆盖率的两个维度：
  广度：覆盖了多少功能特性
  深度：每个功能特性被测试到了什么程度
```

### 2.2 覆盖率的分类体系

```
按测试阶段分类：

  ┌─────────────────────────────────────────────────────────────┐
  │  覆盖率类型   │  适用阶段    │  度量对象          │  典型工具  │
  ├─────────────────────────────────────────────────────────────┤
  │  需求覆盖率    │  测试设计    │  需求规格          │  DOORS/RQM │
  │  功能覆盖率    │  功能测试    │  功能列表          │  手动追踪   │
  │  代码覆盖率    │  单元/集成   │  源代码            │  gcov/lcov │
  │  分支覆盖率    │  白盒测试    │  控制流分支        │  gcov/lcov │
  │  MC/DC覆盖率  │  安全关键    │  条件组合          │  LDRA/Vector│
  │  接口覆盖率    │  集成测试    │  API/函数调用      │  手动/自动  │
  │  场景覆盖率    │  系统测试    │  用户场景          │  TestRail  │
  └─────────────────────────────────────────────────────────────┘

按度量粒度分类：

  ┌─────────────────────────────────────────────────────────────┐
  │  粒度层级   │  定义                  │  GPU驱动中的示例       │
  ├─────────────────────────────────────────────────────────────┤
  │  函数级     │  函数被调用的百分比      │  是否覆盖了           │
  │             │                        │  amdgpu_bo_create()   │
  │  语句级     │  语句被执行的比例        │  if体内的代码是否执行  │
  │  分支级     │  分支真假都被覆盖        │  if/else两个分支      │
  │  条件级     │  每个条件取真和假        │  (a>0 && b<0)的两个条件│
  │  路径级     │  所有可能路径            │  多重if的组合路径     │
  └─────────────────────────────────────────────────────────────┘
```

### 2.3 为什么GPU驱动测试需要覆盖率分析

```
GPU驱动测试的特殊性：

1. 硬件依赖性强
   无法在所有硬件组合上运行
   覆盖率分析帮助识别未测试的硬件路径

2. 错误路径难以触发
   GPU驱动的错误处理代码通常只在罕见条件下执行
   覆盖率分析检查这些错误路径是否被覆盖

3. 并发和非确定性
   竞态条件导致的行为不确定性
   多次运行可能产生不同的覆盖率结果

4. 庞大的代码基
   AMDGPU驱动超过200万行代码
   手动判断覆盖范围几乎不可能

5. 安全关键性
   内核驱动中的安全漏洞可能影响整个系统
   需要高覆盖率的测试保证
```

## 3. 代码覆盖率度量方法详解

### 3.1 语句覆盖率（Statement Coverage）

```
定义：被执行的语句数 / 总语句数 * 100%

计算方式：
  覆盖率 = 被执行的语句数 / 可执行语句总数 * 100%

示例代码：
  01: int func(int a, int b) {
  02:     int c = 0;                    // 语句1
  03:     if (a > 0) {                  // 语句2
  04:         c = a + b;                // 语句3
  05:     }                             //
  06:     return c;                     // 语句4
  07: }

测试用例: func(1, 5)
  执行路径: 1 → 2(真) → 3 → 4 → 6
  覆盖语句: 1, 2, 3, 4
  语句覆盖率: 4/4 = 100% ✓

测试用例: func(0, 5)
  执行路径: 1 → 2(假) → 4 → 6
  覆盖语句: 1, 2, 4
  语句覆盖率: 3/4 = 75%

局限性：
  ┌─────────────────────────────────────────────────────────────┐
  │  100%语句覆盖率 ≠ 100%正确性                                │
  │                                                            │
  │  示例:                                                      │
  │    if (a > 0) {                                             │
  │        c = a / b;    ← 如果b=0，这里会除零错误              │
  │    }                                                        │
  │  测试 func(1, 1) 覆盖了所有语句，但未发现 b=0 的缺陷        │
  │  需要条件覆盖或分支覆盖来发现这类问题                        │
  └─────────────────────────────────────────────────────────────┘

GPU驱动中的语句覆盖示例：
  amdgpu_cs_ioctl() 的语句覆盖分析：
    总语句数: 156
    已覆盖语句: 134
    未覆盖语句: 22
    语句覆盖率: 85.9%

  未覆盖语句分析：
    - 8条错误处理（内存不足路径）
    - 6条退化的硬件路径（旧GPU型号）
    - 5条调试断言（需要特殊条件触发）
    - 3条死代码（应清理）
```

### 3.2 分支覆盖率（Branch Coverage）

```
定义：执行的分支数 / 总分支数 * 100%

分支类型：
  ┌─────────────────────────────────────────────────────────────┐
  │  分支类型      │  示例            │  分支数量               │
  ├─────────────────────────────────────────────────────────────┤
  │  简单if        │  if (x) {...}     │  2 (真/假)             │
  │  if-else       │  if (x) A else B │  2 (A/B)               │
  │  switch/case   │  switch(x){...}  │  N+1 (N个case+default) │
  │  三元操作符    │  x ? A : B       │  2                     │
  │  循环条件      │  for(;x;) {...}  │  2 (进入/退出)         │
  │  逻辑操作符    │  a && b          │  短路场景              │
  └─────────────────────────────────────────────────────────────┘

示例代码：
  01: int validate_params(int a, int b, int c) {
  02:     int result = 0;
  03:     if (a > 0 && b < 100) {         // 分支点1 (复合条件)
  04:         if (c == 0) {              // 分支点2
  05:             result = 1;
  06:         } else {
  07:             result = 2;
  08:         }
  09:     } else {
  10:         result = -1;               // 分支点1的else
  11:     }
  12:     return result;
  13: }

分支识别：
  分支点1: (a > 0 && b < 100) → 真/假 → 2个分支
  分支点2: (c == 0) → 真/假 → 2个分支
  总分支数: 4

测试用例与分支覆盖：
  ┌───────────┬──────────┬──────────┬──────┬──────┬───────────┐
  │ 测试用例   │ a,b,c    │ 分支点1  │分支点2│覆盖  │ 分支覆盖率 │
  ├───────────┼──────────┼──────────┼──────┼──────┼───────────┤
  │ TC1       │ 1,50,0   │ 真       │ 真   │ 2/4  │ 50%       │
  │ TC2       │ 0,50,0   │ 假       │ -    │ 3/4  │ 75%       │
  │ TC3       │ 1,50,1   │ 真       │ 假   │ 4/4  │ 100% ✓   │
  └───────────┴──────────┴──────────┴──────┴──────┴───────────┘

GPU驱动示例 - amdgpu_job_submit()的分支覆盖：

int amdgpu_job_submit(struct amdgpu_job *job) {
    int r;

    if (!job)                           // 分支1: job是否为NULL
        return -EINVAL;

    if (job->ctx->sched == NULL) {      // 分支2: 调度器是否初始化
        job->ctx->sched = amdgpu_sched_get(job->ctx->ring);
        if (!job->ctx->sched) {         // 分支3: 获取调度器是否成功
            return -ENOMEM;
        }
    }

    r = amdgpu_ib_schedule(job);        // 核心路径
    if (r) {                            // 分支4: IB调度是否成功
        return r;
    }

    return 0;
}

分支覆盖矩阵：
  ┌──────┬────────────────────────┬────────┬────────┬──────────┐
  │ 分支  │ 条件                   │ 真覆盖  │ 假覆盖  │ 覆盖状态 │
  ├──────┼────────────────────────┼────────┼────────┼──────────┤
  │ B1   │ job == NULL            │ TC-ERR1│ 大多数  │ ✓        │
  │ B2   │ sched == NULL          │ TC-ERR2│ TC-NORM│ ✓        │
  │ B3   │ 获取sched失败          │ 未覆盖  │ TC-ERR2│ ✗ 缺失! │
  │ B4   │ ib_schedule失败        │ TC-ERR3│ TC-NORM│ ✓        │
  └──────┴────────────────────────┴────────┴────────┴──────────┘

  发现: 分支B3（get_sched失败路径）未被任何测试覆盖
  修复: 增加测试用例模拟sched创建失败
```

### 3.3 条件覆盖率（Condition Coverage）

```
定义：每个布尔子条件取真和取假的比例

与分支覆盖的区别：
  分支覆盖：关注整个条件表达式的结果（真/假）
  条件覆盖：关注每个子条件的取值（真/假）

示例：
  if (A && B)         ← 分支点
     │     │
     │     └── 条件B: 子条件
     └──────── 条件A: 子条件

  ┌─────────────────────────────────────────────────────────────┐
  │  覆盖类型       │  结果                  │ 覆盖度量          │
  ├─────────────────────────────────────────────────────────────┤
  │  分支覆盖       │  (A&&B)=真 或 假       │ 2/2分支           │
  │  条件覆盖       │  A=真, A=假            │ 4/4子条件取值     │
  │                │  B=真, B=假             │                   │
  └─────────────────────────────────────────────────────────────┘

示例代码：
  01: int check_gpu_state(struct amdgpu_device *adev) {
  02:     if (adev->in_reset && adev->fence_available
  03:         && adev->num_rings > 0) {
  04:         amdgpu_ring_restart(adev);
  05:         return 0;
  06:     }
  07:     return -EAGAIN;
  08: }

条件分解：
  C1: adev->in_reset
  C2: adev->fence_available
  C3: adev->num_rings > 0

覆盖矩阵：
  ┌────────┬──────┬──────┬──────┬────────────────────────────────┐
  │ 用例   │ C1   │ C2   │ C3   │ (C1 && C2 && C3)              │
  ├────────┼──────┼──────┼──────┼────────────────────────────────┤
  │ TC1    │ T    │ T    │ T    │ T (进入if)                     │
  │ TC2    │ T    │ F    │ T    │ F (短路, 未进入)               │
  │ TC3    │ F    │ T    │ T    │ F (短路, 未进入)               │
  │ TC4    │ T    │ T    │ F    │ F (不短路, 但条件为假)         │
  ├────────┼──────┼──────┼──────┼────────────────────────────────┤
  │覆盖    │ T/F  │ T/F  │ T/F  │ 分支真/假 = 2/2                │
  │结果    │ ✓    │ ✓    │ ✓    │ ✓                              │
  └────────┴──────┴──────┴──────┴────────────────────────────────┘

MC/DC覆盖（Modified Condition / Decision Coverage）：
  MC/DC要求：每个子条件独立影响决策结果

  验证C1的独立性：
    C1=T时: C2=T, C3=T → 结果=T
    C1=F时: C2=T, C3=T → 结果=F
    → C1独立影响结果 ✓

  验证C2的独立性：
    C2=T时: C1=T, C3=T → 结果=T
    C2=F时: C1=T, C3=T → 结果=F
    → C2独立影响结果 ✓

  验证C3的独立性：
    C3=T时: C1=T, C2=T → 结果=T
    C3=F时: C1=T, C2=T → 结果=F
    → C3独立影响结果 ✓

  MC/DC在GPU驱动中的重要性：
    - DO-178C航空标准要求MC/DC覆盖率
    - GPU驱动的安全关键功能（如GPU重置、电源管理）应达到MC/DC
    - 示例：amdgpu_device_gpu_recover()的重置条件
```

### 3.4 路径覆盖率（Path Coverage）

```
定义：执行的路径数 / 所有可能的路径数 * 100%

路径覆盖的特点：
  ┌─────────────────────────────────────────────────────────────┐
  │                                                             │
  │  路径数随分支数指数增长                                      │
  │  N个分支 → 最多2^N条路径                                     │
  │                                                             │
  │  示例:                                                      │
  │  10个连续if → 1024条路径                                     │
  │  20个连续if → 1,048,576条路径                                │
  │                                                             │
  │  实际中无法追求100%路径覆盖                                  │
  │  使用路径覆盖的简化形式：                                     │
  │    - 函数级路径覆盖（控制流图）                                │
  │    - 关键路径覆盖（高风险路径）                                │
  │    - 循环边界路径（0次、1次、多次）                            │
  └─────────────────────────────────────────────────────────────┘

控制流图示例：

amdgpu_gem_create_ioctl()的简化控制流图：

    ┌──────────────┐
    │  入口         │
    │  validate_args│
    └──────┬───────┘
           │
    ┌──────▼───────┐     假
    │  args->size   │──────────→ ┌──────────┐
    │  >= PAGE_SIZE │            │ return   │
    └──────┬───────┘            │ -EINVAL  │
           │ 真                  └──────────┘
    ┌──────▼───────┐
    │  ttm_bo_init │
    └──────┬───────┘
           │
    ┌──────▼───────┐     假
    │  bo != NULL   │──────────→ ┌──────────┐
    └──────┬───────┘            │ return   │
           │ 真                  │ -ENOMEM  │
    ┌──────▼───────┐            └──────────┘
    │  gem_object   │
    │  init         │
    └──────┬───────┘
           │
    ┌──────▼───────┐     假
    │  is_userptr   │──────────→ ┌──────────┐
    └──────┬───────┘            │ 正常返回  │
           │ 真                  │ 0        │
    ┌──────▼───────┐            └──────────┘
    │  amdgpu_bo   │
    │  _userptr    │
    └──────┬───────┘
           │
    ┌──────▼───────┐
    │  返回 0      │
    └──────────────┘

路径分析：
  路径1: 真 → 真 → 真 → 正常返回          [正常分配]
  路径2: 假 → ... → 返回 -EINVAL           [无效大小]
  路径3: 真 → 假 → 返回 -ENOMEM            [内存不足]
  路径4: 真 → 真 → 假 → 返回 0             [非userptr分配]
  路径5: 真 → 真 → 真 → userptr → 返回 0   [userptr分配]

路径覆盖状态：
  ┌────────┬──────────────────────┬─────────────┬──────────────┐
  │ 路径ID  │ 路径描述              │ 是否覆盖     │ 未覆盖原因    │
  ├────────┼──────────────────────┼─────────────┼──────────────┤
  │ P1     │ 正常分配路径          │ ✓           │              │
  │ P2     │ 无效大小路径          │ ✓           │              │
  │ P3     │ 内存不足路径          │ ✗           │ 需要注入      │
  │        │                      │             │ 内存分配失败  │
  │ P4     │ 非userptr路径        │ ✓           │              │
  │ P5     │ userptr路径          │ ✗           │ 需要特殊      │
  │        │                      │             │ flags参数    │
  └────────┴──────────────────────┴─────────────┴──────────────┘
```

### 3.5 函数覆盖率（Function Coverage）

```
定义：被调用的函数数 / 总函数数 * 100%

函数覆盖在GPU驱动中的意义：

  ┌─────────────────────────────────────────────────────────────┐
  │  函数覆盖率的三个层次：                                      │
  │                                                             │
  │  1. 导出函数覆盖（Driver API）                               │
  │     所有对外暴露的ioctl接口函数是否都被测试                   │
  │     amdgpu_driver_open(), amdgpu_gem_create_ioctl(), ...    │

  │  2. 内部函数覆盖（Internal Helpers）                          │

  │                                                             │
  │  3. 回调函数覆盖（Callbacks）                                 │
  │     DRM框架回调函数是否被触发                                 │
  │     amdgpu_gem_object_free(), amdgpu_gem_vm_fault(), ...    │
  └─────────────────────────────────────────────────────────────┘
```

## 4. GPU驱动覆盖率分析工具链

### 4.1 gcov —— 基础代码覆盖率工具

```
gcov是GCC内置的代码覆盖率分析工具，是Linux内核测试的标准选择。

工作原理：
  源代码
    │
    ▼
  GCC编译（-coverage标志）
    │
    ├── .gcno 文件（编译时生成，包含代码块信息）
    │
    ▼
  执行被测程序
    │
    └── .gcda 文件（运行时生成，包含执行计数）
         │
         ▼
  gcov命令分析
    │
    └── .gcov 文件（可读的覆盖率报告）

内核中的gcov使用：
  ┌─────────────────────────────────────────────────────────────┐
  │  内核配置选项：                                             │
  │  CONFIG_GCOV_KERNEL=y    （启用内核gcov）                   │
  │  CONFIG_GCOV_PROFILE_ALL=y（全量分析，会产生性能影响）      │
  │                                                             │
```

### 4.4 覆盖率数据解析与分析脚本

```python
#!/usr/bin/env python3
"""
覆盖率数据分析工具
解析lcov生成的.info文件，提取关键指标
"""
import re
import sys
import json
from pathlib import Path
from collections import defaultdict


class CoverageParser:
    """解析lcov .info文件"""
    
    def __init__(self, info_file: str):
        self.info_file = Path(info_file)
        self.sf_pattern = re.compile(r'^SF:(.+)')
        self.da_pattern = re.compile(r'^DA:(\d+),(\d+)')
        self.br_pattern = re.compile(r'^BRDA:(\d+),(\d+),(\d+),([-\d]+)')
        
        self.file_data = {}
        self._parse()
    
    def _parse(self):
        current_file = None
        
        with open(self.info_file) as f:
            for line in f:
                line = line.strip()
                
                sf_match = self.sf_pattern.match(line)
                if sf_match:
                    current_file = sf_match.group(1)
                    self.file_data[current_file] = {
                        'lines': {},
                        'branches': {}
                    }
                    continue
                
                da_match = self.da_pattern.match(line)
                if da_match and current_file:
                    line_no = int(da_match.group(1))
                    count = int(da_match.group(2))
                    self.file_data[current_file]['lines'][line_no] = count
                    continue
                
                br_match = self.br_pattern.match(line)
                if br_match and current_file:
                    line_no = int(br_match.group(1))
                    block = int(br_match.group(2))
                    branch = int(br_match.group(3))
                    taken = br_match.group(4)
                    key = (line_no, block, branch)
                    self.file_data[current_file]['branches'][key] = taken
                    continue
    
    def get_file_summary(self, file_path: str) -> dict:
        """获取单个文件的覆盖率摘要"""
        data = self.file_data.get(file_path, {'lines': {}, 'branches': {}})
        
        # 行覆盖率计算
        total_lines = len(data['lines'])
        covered_lines = sum(1 for c in data['lines'].values() if c > 0)
        
        # 分支覆盖率计算
        total_branches = len(data['branches'])
        covered_branches = sum(
            1 for t in data['branches'].values()
            if t != '-' and int(t) > 0
        )
        
        return {
            'file': file_path,
            'line_coverage': {
                'total': total_lines,
                'covered': covered_lines,
                'percentage': (covered_lines / total_lines * 100)
                if total_lines > 0 else 0
            },
            'branch_coverage': {
                'total': total_branches,
                'covered': covered_branches,
                'percentage': (covered_branches / total_branches * 100)
                if total_branches > 0 else 0
            }
        }
    
    def get_module_summary(self, module_dir: str) -> dict:
        """获取模块级别的覆盖率摘要"""
        lines_total = 0
        lines_covered = 0
        branches_total = 0
        branches_covered = 0
        files = []
        
        for file_path in self.file_data:
            if module_dir in file_path:
                summary = self.get_file_summary(file_path)
                files.append(summary)
                lines_total += summary['line_coverage']['total']
                lines_covered += summary['line_coverage']['covered']
                branches_total += summary['branch_coverage']['total']
                branches_covered += summary['branch_coverage']['covered']
        
        return {
            'module': module_dir,
            'files': len(files),
            'line_coverage': {
                'total': lines_total,
                'covered': lines_covered,
                'percentage': (lines_covered / lines_total * 100)
                if lines_total > 0 else 0
            },
            'branch_coverage': {
                'total': branches_total,
                'covered': branches_covered,
                'percentage': (branches_covered / branches_total * 100)
                if branches_total > 0 else 0
            },
            'files_detail': files
        }
    
    def get_uncovered_lines(self, file_path: str) -> list:
        """获取未覆盖的行号"""
        data = self.file_data.get(file_path, {'lines': {}})
        return [
            line_no for line_no, count in data['lines'].items()
            if count == 0
        ]
    
    def get_total_summary(self) -> dict:
        """获取总覆盖率摘要"""
        total_lines = 0
        total_covered = 0
        total_branches = 0
        total_branches_covered = 0
        
        for file_path in self.file_data:
            summary = self.get_file_summary(file_path)
            total_lines += summary['line_coverage']['total']
            total_covered += summary['line_coverage']['covered']
            total_branches += summary['branch_coverage']['total']
            total_branches_covered += summary['branch_coverage']['covered']
        
        return {
            'overall_line_coverage': {
                'total_lines': total_lines,
                'covered_lines': total_covered,
                'percentage': (total_covered / total_lines * 100)
                if total_lines > 0 else 0
            },
            'overall_branch_coverage': {
                'total_branches': total_branches,
                'covered_branches': total_branches_covered,
                'percentage': (total_branches_covered / total_branches * 100)
                if total_branches > 0 else 0
            }
        }


class CoverageThresholdChecker:
    """覆盖率阈值检查"""
    
    def __init__(self, thresholds: dict):
        self.thresholds = thresholds
    
    def check_module(self, summary: dict) -> dict:
        """检查模块覆盖率是否达标"""
        results = {}
        
        if 'line' in self.thresholds:
            actual = summary['line_coverage']['percentage']
            threshold = self.thresholds['line']
            results['line_coverage'] = {
                'actual': f"{actual:.1f}%",
                'threshold': f"{threshold}%",
                'passed': actual >= threshold
            }
        
        if 'branch' in self.thresholds:
            actual = summary['branch_coverage']['percentage']
            threshold = self.thresholds['branch']
            results['branch_coverage'] = {
                'actual': f"{actual:.1f}%",
                'threshold': f"{threshold}%",
                'passed': actual >= threshold
            }
        
        if 'function' in self.thresholds:
            actual = summary.get('function_coverage', {}).get('percentage', 0)
            threshold = self.thresholds['function']
            results['function_coverage'] = {
                'actual': f"{actual:.1f}%",
                'threshold': f"{threshold}%",
                'passed': actual >= threshold
            }
        
        results['overall'] = all(r['passed'] for r in results.values())
        return results


class CoverageGapAnalyzer:
    """覆盖率缺口分析"""
    
    def __init__(self, parser: CoverageParser):
        self.parser = parser
    
    def find_cold_code(self, file_path: str, threshold: int = 3) -> list:
        """查找冷代码（执行次数少于阈值的行）"""
        data = self.parser.file_data.get(file_path, {'lines': {}})
        return [
            {'line': line_no, 'count': count}
            for line_no, count in data['lines'].items()
            if 0 < count < threshold
        ]
    
    def find_dead_code(self, file_path: str) -> list:
        """查找死代码（执行次数为0的行）"""
        uncovered = self.parser.get_uncovered_lines(file_path)
        
        # 读取源文件获取具体内容
        source_lines = {}
        if Path(file_path).exists():
            with open(file_path) as f:
                for i, line in enumerate(f, 1):
                    source_lines[i] = line.rstrip()
        
        return [
            {
                'line': line_no,
                'code': source_lines.get(line_no, '')
            }
            for line_no in uncovered
        ]
    
    def generate_improvement_report(self, module_dir: str) -> dict:
        """生成覆盖率改进建议报告"""
        module_summary = self.parser.get_module_summary(module_dir)
        
        report = {
            'module': module_dir,
            'current_coverage': {
                'line': f"{module_summary['line_coverage']['percentage']:.1f}%",
                'branch': f"{module_summary['branch_coverage']['percentage']:.1f}%"
            },
            'files_needing_improvement': [],
            'suggestions': []
        }
        
        for file_summary in module_summary['files_detail']:
            if file_summary['line_coverage']['percentage'] < 70:
                report['files_needing_improvement'].append(file_summary['file'])
                
                dead_code = self.find_dead_code(file_summary['file'])
                if dead_code:
                    report['suggestions'].append({
                        'file': file_summary['file'],
                        'issue': f"{len(dead_code)} 行代码未被覆盖",
                        'sample_lines': dead_code[:5]
                    })
        
        return report


# 使用示例
if __name__ == '__main__':
    if len(sys.argv) < 2:
        print("用法: coverage_analyzer.py <info_file>")
        sys.exit(1)
    
    parser = CoverageParser(sys.argv[1])
    
    # 生成总报告
    total = parser.get_total_summary()
    print(f"=== AMDGPU驱动覆盖率总报告 ===")
    print(f"行覆盖率: {total['overall_line_coverage']['percentage']:.1f}% "
          f"({total['overall_line_coverage']['covered_lines']}/"
          f"{total['overall_line_coverage']['total_lines']})")
    print(f"分支覆盖率: {total['overall_branch_coverage']['percentage']:.1f}% "
          f"({total['overall_branch_coverage']['covered_branches']}/"
          f"{total['overall_branch_coverage']['total_branches']})")
    
    # 生成模块报告
    module = parser.get_module_summary('drivers/gpu/drm/amd/amdgpu')
    print(f"\namdgpu模块: {module['files']} 个文件")
    print(f"  行覆盖率: {module['line_coverage']['percentage']:.1f}%")
    print(f"  分支覆盖率: {module['branch_coverage']['percentage']:.1f}%")
    
    # 缺口分析
    analyzer = CoverageGapAnalyzer(parser)
    report = analyzer.generate_improvement_report('drivers/gpu/drm/amd/amdgpu')
    if report['files_needing_improvement']:
        print(f"\n需要改进的文件 ({len(report['files_needing_improvement'])}):")
        for f in report['files_needing_improvement'][:5]:
            print(f"  - {f}")
```

```
  │    - 调试接口（sysfs、debugfs）                               │
  │                                                             │
  │  一般模块（CQ = 70%）                                        │
  │    - 配置路径（模块初始化和卸载）                              │
  │    - 辅助功能（日志、统计）                                   │
  │                                                             │
  │  低优先级（CQ = 50%）                                        │
  │    - 过时硬件路径                                            │
  │    - 调试/开发辅助功能                                        │
  └─────────────────────────────────────────────────────────────┘

原则2: 区分新旧代码
  新代码（首次合入）: 语句覆盖 ≥ 90%，分支覆盖 ≥ 80%
  旧代码（回归维护）: 保持覆盖率不下降
  重构代码: 以旧覆盖率为基准，不得低于旧值

原则3: 持续改进
  覆盖率目标 = 当前覆盖率 + 季度增长目标
  季度目标 = min(10%, 目标阈值 - 当前值)
  改进计划: 优先提升低覆盖率模块

原则4: 异常豁免
  可以豁免覆盖的情况：
    - 死代码（已确认不再使用）
    - 调试代码（仅在开发环境有意义）
    - 硬件特定代码（没有对应硬件）
  豁免需要：团队负责人审批 + 定期重评估
```

## 6. 覆盖率的局限性与陷阱

### 6.1 常见误解

```
1. 100%覆盖率 ≠ 无Bug
   语句覆盖100%只能说明所有语句都执行了
   但不能保证：
   - 所有逻辑组合被测试
   - 边界条件被覆盖
   - 并发问题被发现
   - 性能问题被检测

2. 高覆盖率 = 高质量测试？
   不一定！测试质量比覆盖率更重要
   反面示例：
   - 只测试快乐路径，不测试错误路径
   - 测试数据不具代表性
   - 断言不够严格
   - 测试本身有Bug

3. 覆盖率目标一刀切
   所有模块设定相同的覆盖率目标不合理
   错误处理代码天然覆盖率低
   但仍需要被测试（通过故障注入）

4. 只关注行覆盖，忽略分支覆盖
   ┌─────────────────────────────────────────────┐
   │  行覆盖100%但分支覆盖50%的示例：              │
   │                                              │
   │  if (a > 0) {                                │
   │      do_something();     ← 假设a总是>0      │
   │  } else {                                    │
   │      handle_error();     ← 从未执行          │
   │  }                                           │
   │                                              │
   │  行覆盖: 3/3 = 100%                          │
   │  分支覆盖: 1/2 = 50%                         │
   └─────────────────────────────────────────────┘
```

### 6.2 GPU驱动覆盖率陷阱

```
GPU驱动特有的覆盖率陷阱：

陷阱1: 硬件探测路径
  ┌─────────────────────────────────────────────────────────────┐
  │  amdgpu_device_init() 中包含了多种GPU型号的初始化路径       │
  │  测试环境只有一种GPU → 其他型号的初始化分支不覆盖            │
  │  解决方案: 使用硬件模拟器或在不同GPU上运行测试               │
  └─────────────────────────────────────────────────────────────┘

陷阱2: 错误处理路径
  ┌─────────────────────────────────────────────────────────────┐
  │  GPU驱动的错误处理（如GPU重置、固件加载失败）                │
  │  在正常测试中很少触发                                       │
  │  解决方案: 使用故障注入（Fault Injection）触发错误路径        │
  └─────────────────────────────────────────────────────────────┘

陷阱3: 并发竞态条件
  ┌─────────────────────────────────────────────────────────────┐
  │  行覆盖不能反映并发执行情况                                  │
  │  每行代码都可能被执行，但执行顺序不正确                      │
  │  解决方案: 使用KCSAN/KTSAN进行并发覆盖率分析                 │
  └─────────────────────────────────────────────────────────────┘

陷阱4: 非确定性行为
  ┌─────────────────────────────────────────────────────────────┐
  │  多次运行相同测试得到不同的覆盖率结果                         │
  │  原因: 中断时序、调度顺序、硬件响应的差异                     │
  │  解决方案: 多次运行取并集，或使用确定性模拟器                 │
  └─────────────────────────────────────────────────────────────┘

陷阱5: 覆盖率回弹
  ┌─────────────────────────────────────────────────────────────┐
  │  内核更新后，已覆盖的代码可能变成未覆盖                       │
  │  原因: 代码重构、编译优化、条件编译宏的变化                   │
  │  解决方案: 版本化覆盖率基线，对比分析覆盖率变化               │
  └─────────────────────────────────────────────────────────────┘
```

```

├──────────────────┼────────────────────────────────────────────────────┤
│ 并集合并          │ 任意GPU执行过的代码行标记为已覆盖                    │
│                  │ 优点: 反映整体测试充分性                             │
│                  │ 缺点: 可能掩盖某型号的覆盖不足                       │
├──────────────────┼────────────────────────────────────────────────────┤
│ 交集合并          │ 所有GPU都执行过的代码行才标记为已覆盖                │
│                  │ 优点: 反映跨平台通用覆盖                              │
│                  │ 缺点: 要求过高，忽略平台特定代码                      │
├──────────────────┼────────────────────────────────────────────────────┤
│ 加权合并          │ 按GPU覆盖率为每行代码分配权重                       │
│                  │ 公式: w = count_executed / total_gpus               │
│                  │ 优点: 提供覆盖率置信度                               │
│                  │ 缺点: 实现复杂，可视化困难                           │
├──────────────────┼────────────────────────────────────────────────────┤
│ 标签化合并        │ 每行代码附带覆盖它的GPU型号列表                     │
│                  │ 例如: "line 123: [RX7900, RX7800]"                 │
│                  │ 优点: 完整保留覆盖信息，支持深度分析                 │
│                  │ 缺点: 数据量大，报告冗长                             │
└──────────────────┴────────────────────────────────────────────────────┘
```

覆盖率数据聚合Python实现：

```python
#!/usr/bin/env python3
"""多GPU覆盖率数据聚合工具"""
import os
import json
import subprocess
from pathlib import Path
from collections import defaultdict


class CoverageMerger:
    def __init__(self, gpu_list, work_dir="/sys/kernel/debug/gcov"):
        self.gpu_list = gpu_list          # GPU型号列表, 如 ["RX7900", "RX7800"]
        self.work_dir = Path(work_dir)
        self.gpu_data = {}                # gpu_name -> {file -> {line -> count}}
        self.merged_data = {}             # file -> {line -> {count, gpus, merged_status}}

    def collect_gpu_coverage(self, gpu_name, gcda_path):
        """收集单GPU的覆盖率原始数据"""
        raw = subprocess.run(
            ["gcov", "-a", "-b", "-c", gcda_path],
            capture_output=True, text=True, cwd=self.work_dir
        )
        parsed = self._parse_gcov_output(raw.stdout)
        self.gpu_data[gpu_name] = parsed
        return len(parsed)

    def _parse_gcov_output(self, text):
        """解析gcov文本输出为结构化数据"""
        result = {}
        current_file = None
        for line in text.splitlines():
            if line.startswith("File "):
                current_file = line.split(" ")[1].strip("'")
                result[current_file] = {}
            elif current_file and ":" in line and line[0].isdigit():
                parts = line.split(":")
                if len(parts) >= 2 and parts[0].strip().isdigit():
                    lineno = int(parts[0].strip())
                    count_str = parts[1].strip().split()[0] if parts[1].strip() else "0"
                    count = int(count_str) if count_str.isdigit() else 0
                    result[current_file][lineno] = count
        return result

    def merge_union(self):
        """并集合并：任意GPU执行过的行标记为已覆盖"""
        self.merged_data = {}
        for gpu_name, gpu_files in self.gpu_data.items():
            for file_name, lines in gpu_files.items():
                if file_name not in self.merged_data:
                    self.merged_data[file_name] = {}
                for lineno, count in lines.items():
                    old = self.merged_data[file_name].get(lineno, 0)
                    self.merged_data[file_name][lineno] = max(old, count)
        return self.merged_data

    def merge_intersection(self):
        """交集合并：所有GPU都执行过的行才标记"""
        self.merged_data = {}
        if not self.gpu_data:
            return self.merged_data
        first_gpu = list(self.gpu_data.keys())[0]
        for file_name, lines in self.gpu_data[first_gpu].items():
            self.merged_data[file_name] = {}
            for lineno in lines:
                if all(
                    lineno in self.gpu_data[g].get(file_name, {})
                    and self.gpu_data[g][file_name][lineno] > 0
                    for g in self.gpu_data
                ):
                    self.merged_data[file_name][lineno] = min(
                        self.gpu_data[g][file_name][lineno] for g in self.gpu_data
                    )
        return self.merged_data

    def merge_tagged(self):
        """标签化合并：保留每行代码的GPU来源"""
        self.merged_data = {}
        for gpu_name, gpu_files in self.gpu_data.items():
            for file_name, lines in gpu_files.items():
                if file_name not in self.merged_data:
                    self.merged_data[file_name] = {}
                for lineno, count in lines.items():
                    if lineno not in self.merged_data[file_name]:
                        self.merged_data[file_name][lineno] = {
                            "total_count": 0, "gpu_list": []
                        }
                    self.merged_data[file_name][lineno]["total_count"] += count
                    self.merged_data[file_name][lineno]["gpu_list"].append(gpu_name)
        return self.merged_data

    def generate_diff_report(self):
        """生成GPU型号间的覆盖率差异报告"""
        report = []
        for file_name in self.merged_data:
            lines_data = self.merged_data[file_name]
            for lineno, info in lines_data.items():
                if isinstance(info, dict) and len(info.get("gpu_list", [])) < len(self.gpu_list):
                    missing_gpus = set(self.gpu_list) - set(info["gpu_list"])
                    report.append({
                        "file": file_name,
                        "line": lineno,
                        "missing_gpus": list(missing_gpus)
                    })
        report.sort(key=lambda x: len(x["missing_gpus"]), reverse=True)
        return report

    def coverage_summary(self, merged, total_lines):
        """计算合并后的覆盖率统计"""
        covered = sum(1 for f in merged.values() for l in f.values()
                     if (isinstance(l, dict) and l["total_count"] > 0) or
                        (isinstance(l, int) and l > 0))
        return {
            "covered_lines": covered,
            "total_lines": total_lines,
            "coverage_pct": round(covered / total_lines * 100, 1) if total_lines else 0
        }


if __name__ == "__main__":
    merger = CoverageMerger(["RX7900XTX", "RX7800XT", "RX7600"])
    merger.gpu_data = {
        "RX7900XTX": {"amdgpu_pm.c": {120: 5, 121: 3, 122: 0, 130: 1}},
        "RX7800XT": {"amdgpu_pm.c": {120: 3, 121: 0, 122: 0, 130: 1}},
        "RX7600": {"amdgpu_pm.c": {120: 1, 121: 1, 122: 0, 130: 0}},
    }
    union = merger.merge_union()
    tagged = merger.merge_tagged()
    diff = merger.generate_diff_report()
    print("并集合并:", merger.coverage_summary(union, 4))
    print("差异报告:", diff[:2])
```

```
  GPU阵容: RX580 (GCN) + RX6600 (RDNA2) + RX7900XTX (RDNA3)
  目标: 确保测试套件覆盖所有GPU代的执行路径
  方法: 并集合并 + 按代标记
```

### 8.3 覆盖率自动化分析工具实现

实现一个自动化的覆盖率分析框架，用于持续跟踪和改进AMDGPU驱动覆盖率。

```python
#!/usr/bin/env python3
"""AMDGPU驱动覆盖率自动化分析框架"""
import os
import re
import json
import subprocess
from pathlib import Path
from datetime import datetime, timedelta
from collections import defaultdict


class CoverageTrendTracker:
    def __init__(self, baseline_dir, report_dir):
        self.baseline_dir = Path(baseline_dir)
        self.report_dir = Path(report_dir)
        self.report_dir.mkdir(parents=True, exist_ok=True)
        self.history_file = self.report_dir / "coverage_history.json"
        self.history = self._load_history()

    def _load_history(self):
        if self.history_file.exists():
            return json.loads(self.history_file.read_text())
        return {"entries": [], "baseline": None, "thresholds": {}}

    def _save_history(self):
        self.history_file.write_text(json.dumps(self.history, indent=2))

    def set_thresholds(self, line=80, branch=70, function=85):
        self.history["thresholds"] = {
            "line": line, "branch": branch, "function": function
        }
        self._save_history()

    def collect_current_coverage(self, lcov_info_path):
        """从lcov .info文件解析当前覆盖率"""
        result = subprocess.run(
            ["lcov", "--summary", lcov_info_path],
            capture_output=True, text=True
        )
        summary = self._parse_lcov_summary(result.stdout)
        module_detail = self._parse_lcov_detail(lcov_info_path)
        entry = {
            "timestamp": datetime.now().isoformat(),
            "summary": summary,
            "modules": module_detail
        }
        self.history["entries"].append(entry)
        if self.history["baseline"] is None:
            self.history["baseline"] = entry
        self._save_history()
        return entry

    def _parse_lcov_summary(self, text):
        """解析lcov --summary输出"""
        result = {}
        patterns = {
            "lines": r"lines\.+\: (\d+\.\d%)",
            "branches": r"branches\.+\: (\d+\.\d%)",
            "functions": r"functions\.+\: (\d+\.\d%)"
        }
        for key, pat in patterns.items():
            match = re.search(pat, text)
            if match:
                result[key] = float(match.group(1).rstrip("%"))
        return result

    def _parse_lcov_detail(self, info_path):
        """解析lcov .info文件获取各模块详细覆盖率"""
        result = subprocess.run(
            ["lcov", "--list", info_path],
            capture_output=True, text=True
        )
        modules = {}
        current_module = None
        for line in result.stdout.splitlines():
            if line.startswith("["):
                current_module = line.strip("[]")
            elif current_module and ":" in line and "%" in line:
                parts = line.split()
                if len(parts) >= 4:
                    try:
                        pct = float(parts[1].rstrip("%"))
                        modules[current_module] = pct
                    except ValueError:
                        pass
        return modules

    def trend_analysis(self, window=4):
        """分析最近N次测量的趋势"""
        entries = self.history["entries"]
        if len(entries) < 2:
            return {"status": "insufficient_data", "message": "至少需要2次测量"}
        recent = entries[-window:] if len(entries) >= window else entries
        first, last = recent[0], recent[-1]
        thresholds = self.history.get("thresholds", {})
        analysis = {"period": f"{first['timestamp'][:10]} ~ {last['timestamp'][:10]}", "changes": {}}
        for metric in ["lines", "branches", "functions"]:
            if metric in first["summary"] and metric in last["summary"]:
                delta = round(last["summary"][metric] - first["summary"][metric], 1)
                threshold = thresholds.get(metric, 0)
                analysis["changes"][metric] = {
                    "from": first["summary"][metric],
                    "to": last["summary"][metric],
                    "delta": delta,
                    "trend": "up" if delta > 0 else ("down" if delta < 0 else "stable"),
                    "status": "pass" if last["summary"][metric] >= threshold else "fail"
                }
        regressed_modules = []
        improved_modules = []
        if first.get("modules") and last.get("modules"):
            all_modules = set(first["modules"]) | set(last["modules"])
            for mod in all_modules:
                old_pct = first["modules"].get(mod, 0)
                new_pct = last["modules"].get(mod, 0)
                diff = new_pct - old_pct
                if diff < -2:
                    regressed_modules.append({"module": mod, "drop": round(diff, 1)})
                elif diff > 2:
                    improved_modules.append({"module": mod, "gain": round(diff, 1)})
        analysis["regressed_modules"] = sorted(regressed_modules, key=lambda x: x["drop"])
        analysis["improved_modules"] = sorted(improved_modules, key=lambda x: x["gain"], reverse=True)
        return analysis

    def generate_report(self):
        """生成Markdown格式的覆盖率报告"""
        entries = self.history["entries"]
        if not entries:
            return "# 覆盖率报告\n\n暂无数据"
        latest = entries[-1]
        thresholds = self.history.get("thresholds", {})
        report_lines = [
            "# AMDGPU驱动覆盖率周报",
            f"报告时间: {latest['timestamp'][:10]}",
            f"测量次数: {len(entries)}",
            "",
            "## 整体指标",
        ]
        for metric, label in [("lines", "行覆盖率"), ("branches", "分支覆盖率"), ("functions", "函数覆盖率")]:
            if metric in latest["summary"]:
                val = latest["summary"][metric]
                threshold = thresholds.get(metric, 0)
                status = "✓" if val >= threshold else "✗"
                report_lines.append(f"- **{label}**: {val}% {status} (阈值: {threshold}%)")
        if len(entries) >= 2:
            analysis = self.trend_analysis()
            report_lines.extend(["", "## 趋势分析"])
            for metric, change in analysis.get("changes", {}).items():
                arrow = "↑" if change["trend"] == "up" else ("↓" if change["trend"] == "down" else "→")
                report_lines.append(f"- {metric}: {change['from']}% → {change['to']}% ({arrow}{change['delta']:+.1f}%)")
            if analysis.get("regressed_modules"):
                report_lines.extend(["", "## 覆盖率回退模块"])
                for m in analysis["regressed_modules"]:
                    report_lines.append(f"- {m['module']}: {m['drop']}%")
        report_lines.append(f"\n*报告由CoverageTrendTracker自动生成*")
        report_txt = "\n".join(report_lines)
        report_path = self.report_dir / f"coverage_report_{datetime.now().strftime('%Y%m%d')}.md"
        report_path.write_text(report_txt)
        return str(report_path)


class CoverageGapIdentifier:
    def __init__(self, coverage_info):
        self.info = coverage_info

    def find_uncovered_functions(self, min_lines=10):
        """找出未覆盖的函数（按代码行数过滤）"""
        result = subprocess.run(
            ["lcov", "--uncovered", self.info],
            capture_output=True, text=True
        )
        uncovered = []
        current_func = None
        for line in result.stdout.splitlines():
            if "FN:" in line:
                current_func = line.split(":")[-1].strip()
            elif "FNDA:" in line and "0" in line and current_func:
                uncovered.append(current_func)
        return uncovered

    def priority_suggestions(self, uncovered_list, source_tree):
        """基于未覆盖函数生成优化建议"""
        suggestions = []
        for func in uncovered_list:
            if "error" in func.lower() or "fail" in func.lower() or "err" in func.lower():
                suggestions.append(f"P0: {func} — 错误处理函数，应优先覆盖")
            elif "init" in func.lower() or "probe" in func.lower():
                suggestions.append(f"P1: {func} — 初始化函数，重要覆盖目标")
            elif "debugfs" in func.lower() or "sysfs" in func.lower():
                suggestions.append(f"P1: {func} — 调试接口，可通过sysfs触发")
            else:
                suggestions.append(f"P2: {func} — 常规函数，按优先级排序")
        return suggestions


class CoverageGateChecker:
    def __init__(self, info_path, thresholds):
        self.info_path = info_path
        self.thresholds = thresholds

    def check_gate(self):
        """检查覆盖率是否满足门禁要求"""
        result = subprocess.run(
            ["lcov", "--summary", self.info_path],
            capture_output=True, text=True, timeout=30
        )
        current = {}
        for line in result.stdout.splitlines():
            for metric in ["lines", "branches", "functions"]:
                if metric in line and "%" in line:
                    try:
                        pct = float(line.split("%")[0].split()[-1])
                        current[metric] = pct
                    except (ValueError, IndexError):
                        pass
        failed = []
        for metric, threshold in self.thresholds.items():
            if metric in current and current[metric] < threshold:
                failed.append({
                    "metric": metric,
                    "current": current[metric],
                    "threshold": threshold,
                    "gap": round(threshold - current[metric], 1)
                })
        return {
            "passed": len(failed) == 0,
            "current": current,
            "failed": failed,
            "thresholds": self.thresholds
        }


if __name__ == "__main__":
    tracker = CoverageTrendTracker("/tmp/coverage_baseline", "/tmp/coverage_reports")
```

```
覆盖率门禁验证脚本:

  #!/usr/bin/env python3
  """coverage_gate.py — 覆盖率门禁检查"""
  import sys, json, subprocess, re

  THRESHOLDS = {"lines": 80, "branches": 70}
  HARD_THRESHOLDS = {"lines": 70}
  DELTA_LIMIT = -3.0  # 覆盖率下降超过3%则阻塞

  result = subprocess.run(
      ["lcov", "--summary", sys.argv[1]],
      capture_output=True, text=True
  )
  current = {}
  for line in result.stdout.splitlines():
      for metric in ["lines", "branches", "functions"]:
          if metric in line and "%" in line:
              try:
                  pct = float(line.split("%")[0].split()[-1])
                  current[metric] = pct
              except (ValueError, IndexError):
                  pass

  passed = True
  for metric, threshold in THRESHOLDS.items():
      if metric in current and current[metric] < threshold:
          print(f"WARNING: {metric}={current[metric]}% < {threshold}%")
          if metric in HARD_THRESHOLDS and current[metric] < HARD_THRESHOLDS[metric]:
              print(f"BLOCKING: {metric}低于硬门禁!")
              passed = False

  sys.exit(0 if passed else 1)
```

## 10. 今日小结

测试覆盖率是衡量测试质量的客观指标，但它不是目的，而是手段。

**核心收获**：

1. **覆盖率分类**：语句覆盖（最基本的度量）、分支覆盖（逻辑完整性）、条件覆盖/MC/DC（安全关键）、路径覆盖（最全面但成本高）、函数覆盖（接口完整性）。不同层级的测试需要选择不同类型的覆盖率度量。

2. **工具链**：gcov（基础工具）、lcov（可视化报告）、内核gcov框架（内核覆盖率收集）。通过脚本自动化覆盖率收集、分析和报告生成。

3. **GPU驱动特有考量**：硬件探测路径差异、错误处理代码覆盖困难、并发非确定性、覆盖率回弹等都是GPU驱动覆盖率分析中的特殊挑战。

4. **阈值的设定**：不同模块应有不同的覆盖率目标（核心模块90%+，一般模块70%+）。覆盖率目标应随测试成熟度逐步提高，而非一刀切。

5. **持续改进**：覆盖率管理应是持续的过程，包括定期度量、趋势分析、缺口识别和针对性的测试改进。

**关键洞察**：覆盖率的价值不在于数字本身，而在于发现测试盲区并指导测试改进。一个80%覆盖率+有针对性分析的测试套件，远好于一个90%覆盖率但只测试快乐路径的测试套件。在GPU驱动测试中，应特别关注错误处理路径和硬件特定代码的覆盖率——这些往往是缺陷最集中的区域。
