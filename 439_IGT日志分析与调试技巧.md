# IGT 日志分析与调试技巧

## 学习目标

- 掌握 IGT 测试日志的结构和内容

## 知识详解

### 一、概念原理

#### 1.1 IGT 日志结构

```
┌─────────────────────────────────────────────────────────────────────┐
│              IGT 日志结构                                            │
│                                                                     │
│  日志层级：                                                          │
│  ──────────                                                          │
│                                                                     │
│  1. 测试框架日志 (igt_core)                                          │
│     ├── IGT-Version                  版本信息                        │
│     ├── 测试开始/结束                时间戳                          │
│     ├── 子测试开始/结束              名称和结果                      │
│     └── 夹具执行日志                 setup/teardown                  │
│                                                                     │
│  2. 测试程序日志 (tests/*.c)                                         │
│     ├── igt_info()                   信息日志                        │
│     ├── igt_debug()                  调试日志（需 --debug）          │
│     ├── igt_warn()                   警告日志                        │
│     └── 自定义输出                   printf 等                       │
│                                                                     │
│  3. 内核日志 (dmesg)                                                 │
│     ├── DRM 子系统日志               [drm]                           │
│     ├── AMDGPU 驱动日志              [drm:amdgpu]                    │
│     ├── GPU hang 检测                [drm:amdgpu] *ERROR* GPU hang   │
│     └── 内存错误                     KASAN/UBSAN                     │
│                                                                     │
│  4. 系统日志 (journalctl)                                            │
│     ├── 服务状态                     systemd                         │
│     └── 硬件事件                     udev                            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 二、实践操作

#### 2.1 日志收集脚本

```bash
#!/bin/bash
# collect_igt_logs.sh - IGT 日志收集脚本

set -e

TEST_NAME="$1"
OUTPUT_DIR="${2:-./igt_logs}"
mkdir -p "$OUTPUT_DIR"

if [ -z "$TEST_NAME" ]; then
    echo "用法: $0 <test-name> [output-dir]"
    exit 1
fi

echo "=== IGT 日志收集 ==="
echo "测试: $TEST_NAME"
echo "输出: $OUTPUT_DIR"
echo ""

# 1. 收集测试前状态
echo "【1】收集测试前状态"
dmesg -c > /dev/null 2>&1 || true  # 清空 dmesg
echo "  dmesg 已清空"

# 2. 运行测试并收集日志
echo "【2】运行测试"
LOG_FILE="$OUTPUT_DIR/${TEST_NAME}.log"
DMESG_FILE="$OUTPUT_DIR/${TEST_NAME}.dmesg"

sudo dmesg -w > "$DMESG_FILE" &
DMESG_PID=$!

# 运行测试
sudo "$TEST_NAME" --debug 2>&1 | tee "$LOG_FILE" || true

# 停止 dmesg 收集
kill $DMESG_PID 2>/dev/null || true
wait $DMESG_PID 2>/dev/null || true

echo ""

# 3. 收集系统信息
echo "【3】收集系统信息"
{
    echo "=== 系统信息 ==="
    echo "时间: $(date)"
    echo "内核: $(uname -a)"
    echo ""
    echo "=== GPU 信息 ==="
    lspci -nn | grep -E "0300|0302|0380" || true
    echo ""
    echo "=== 驱动模块 ==="
    lsmod | grep -E "amdgpu|i915|radeon|drm" || true
    echo ""
    echo "=== DRM 状态 ==="
    cat /sys/kernel/debug/dri/*/state 2>/dev/null || true
} > "$OUTPUT_DIR/${TEST_NAME}.sysinfo"

echo ""
echo "=== 日志收集完成 ==="
echo "  测试日志: $LOG_FILE"
echo "  dmesg: $DMESG_FILE"
echo "  系统信息: $OUTPUT_DIR/${TEST_NAME}.sysinfo"
```

#### 2.2 日志分析脚本

```bash
#!/bin/bash
# analyze_igt_logs.sh - IGT 日志分析脚本

set -e

LOG_DIR="${1:-./igt_logs}"

echo "=== IGT 日志分析 ==="
echo "目录: $LOG_DIR"
echo ""

# 分析所有日志文件
for log in "$LOG_DIR"/*.log; do
    if [ -f "$log" ]; then
        name=$(basename "$log" .log)
        echo "--- 分析: $name ---"
        
        # 统计结果
        PASS=$(grep -c "SUCCESS" "$log" || true)
        FAIL=$(grep -c "FAIL" "$log" || true)
        SKIP=$(grep -c "SKIP" "$log" || true)
        
        echo "  通过: $PASS, 失败: $FAIL, 跳过: $SKIP"
        
        # 检查 dmesg 错误
        dmesg_file="$LOG_DIR/${name}.dmesg"
        if [ -f "$dmesg_file" ]; then
            ERRORS=$(grep -c "ERROR\|WARN" "$dmesg_file" || true)
            echo "  dmesg 错误/警告: $ERRORS"
            
            if [ "$ERRORS" -gt 0 ]; then
                echo "  dmesg 错误摘要:"
                grep "ERROR\|WARN" "$dmesg_file" | head -5 | sed 's/^/    /'
            fi
        fi
        
        echo ""
    fi
done
```

#### 2.3 GPU Hang 调试

```bash
#!/bin/bash
# debug_gpu_hang.sh - GPU Hang 调试脚本

set -e

echo "=== GPU Hang 调试 ==="
echo ""

# 1. 检查 GPU 状态
echo "【1】GPU 状态"
cat /sys/kernel/debug/dri/*/amdgpu_gpu_recover 2>/dev/null || true
cat /sys/kernel/debug/dri/*/amdgpu_vram_mm 2>/dev/null || true
echo ""

# 2. 收集 GPU 寄存器
echo "【2】GPU 寄存器状态"
if [ -d /sys/kernel/debug/dri/0 ]; then
    for reg in amdgpu_regs amdgpu_ib amdgpu_fence; do
        if [ -f "/sys/kernel/debug/dri/0/$reg" ]; then
            echo "--- $reg ---"
            cat "/sys/kernel/debug/dri/0/$reg" 2>/dev/null | head -20 || true
        fi
    done
fi
echo ""

# 3. 检查恢复状态
echo "【3】恢复状态"
cat /sys/kernel/debug/dri/*/amdgpu_gpu_recover 2>/dev/null || true
echo ""

# 4. 收集完整 dmesg
echo "【4】收集 dmesg"
dmesg | tail -n 200 > gpu_hang_dmesg.log
echo "dmesg 已保存到 gpu_hang_dmesg.log"
echo ""
```

### 三、案例分析

#### 3.1 分析测试超时

```
症状：测试运行很长时间后报告 TIMEOUT

调试步骤：
1. 检查测试是否正在执行：
   ps aux | grep <test-name>

2. 检查 GPU 状态：
   cat /sys/kernel/debug/dri/0/amdgpu_gpu_recover

3. 检查是否有 GPU hang：
   dmesg | grep -i "hang\|timeout\|reset"

4. 如果确认 hang，收集：
   - dmesg 日志
   - GPU 寄存器状态
   - 测试输入参数

5. 尝试最小化复现：
   - 运行单个失败的子测试
   - 使用更短的超时设置
```

## 今日小结

- IGT 日志分为框架日志、测试日志、内核日志和系统日志四个层级
- 调试级别包括错误、警告、信息、调试和详细五个级别
- GPU hang 是最严重的测试失败类型，需要收集 GPU 寄存器和 dmesg
- 自动化日志收集脚本可以提高调试效率
- 日志分析应关注失败测试、dmesg 错误和 GPU 状态
