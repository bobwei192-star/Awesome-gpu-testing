### 一、概念原理

#### 1.1 持续集成与 GPU 驱动测试的特殊性

持续集成（Continuous Integration, CI）是一种软件开发实践，要求开发者频繁地将代码合并到主干，并通过自动化构建和测试来验证每次合并的正确性。GPU 驱动测试的 CI 实施面临独特挑战：

```
┌─────────────────────────────────────────────────────────────────────┐
│              GPU 驱动 CI 的特殊挑战与应对策略                        │
│                                                                     │
│  ┌──────────────────────┐    ┌──────────────────────┐              │
│  │     硬件依赖性强      │    │   测试执行时间长      │              │
│  ├──────────────────────┤    ├──────────────────────┤              │
```

#### 1.3 测试分层与门禁策略

GPU 驱动测试采用分层策略，不同层级对应不同的执行频率和通过标准：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    GPU 驱动测试分层模型                              │
│                                                                     │
│  层级          执行时机           耗时        通过标准              │
│  ─────────────────────────────────────────────────────────────      │
│                                                                     │
│  L1 编译检查   每次提交           5-10 min   编译零警告              │
│       │                                                            │
│       ▼                                                            │
│  L2 冒烟测试   每次提交           10-15 min  核心功能通过            │
│       │         (Merge Request)                                    │
│       ▼                                                            │
│  L3 快速回归   每次提交           30-60 min  回归测试集 >95% 通过   │
│       │                                                            │
│       ▼                                                            │
│  L4 全量回归   每日/每周构建      2-6 hours  全量测试集 >90% 通过   │
│       │                                                            │
│       ▼                                                            │
│  L5 性能测试   每日/每周构建      1-4 hours  性能退化 <5%           │
│       │                                                            │
│       ▼                                                            │
│  L6 压力测试   每周/发布前        24-72 hrs  无 hang/崩溃/泄露      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**质量门禁（Quality Gate）设计原则**：

| 门禁层级 | 检查内容 | 失败处理 | 参考实践 |
|---------|---------|---------|---------|
| Gate 1 | 编译通过 + 静态分析无新增警告 | 阻断合并 | Mesa CI |
| Gate 2 | L2 冒烟测试 100% 通过 | 阻断合并 | Intel Graphics CI |
| Gate 3 | L3 快速回归 >95% 通过 | 需人工审核 | AMD GPUOpen CI |
| Gate 4 | 无 S1/S2 级未修复缺陷 | 发布阻断 | 内核发布流程 |
| Gate 5 | 性能退化 <3% | 需性能团队审核 | Phoronix 跟踪 |

### 二、实践操作

#### 2.1 GitLab CI 配置示例（GPU 驱动项目）

以下是一个基于 GitLab CI 的 GPU 驱动测试流水线配置，参考了 Mesa 和 drm-misc 项目的实际配置：

```yaml
# .gitlab-ci.yml - GPU 驱动测试流水线示例
# 参考：Mesa 项目 .gitlab-ci.yml (https://gitlab.freedesktop.org/mesa/mesa)

stages:
  - build
  - smoke
  - regression
  - performance
  - stress

variables:
  # 内核源码路径
  KERNEL_REPO: "https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git"
  # 默认内核配置
  KERNEL_CONFIG: "amd_gpu_defconfig"
  # IGT 版本
  IGT_VERSION: "v1.28"

# ═══════════════════════════════════════════════════════════════
# 阶段 1：编译与静态分析
# ═══════════════════════════════════════════════════════════════

build_kernel:
  stage: build
  image: kernel-build-env:latest
  script:
    - git clone --depth 1 $KERNEL_REPO linux
    - cd linux
    - make $KERNEL_CONFIG
    # 启用 AMDGPU 相关配置
    - ./scripts/config --enable CONFIG_DRM_AMDGPU
    - ./scripts/config --enable CONFIG_DRM_AMDGPU_SI
    - ./scripts/config --enable CONFIG_DRM_AMDGPU_CIK
    - make -j$(nproc) 2>&1 | tee build.log
    # 检查编译警告
    - |
      WARNINGS=$(grep -c "warning:" build.log || true)
      echo "编译警告数量: $WARNINGS"
      if [ "$WARNINGS" -gt "10" ]; then
        echo "错误：编译警告超过阈值 (10)"
        exit 1
      fi
  artifacts:
    paths:
      - linux/arch/x86/boot/bzImage
      - linux/drivers/gpu/drm/amd/*.ko
    expire_in: 1 day
  tags:
    - docker

static_analysis:
  stage: build
  image: kernel-build-env:latest
  script:
    - cd linux
    # sparse 静态分析
    - make C=1 CHECK="sparse" drivers/gpu/drm/amd/ 2>&1 | tee sparse.log
    - |
      SPARSE_ERRORS=$(grep -c "error:" sparse.log || true)
      echo "sparse 错误: $SPARSE_ERRORS"
    # smatch 分析
    - smatch --project=kernel drivers/gpu/drm/amd/amdgpu/ 2>&1 | tee smatch.log || true
    # checkpatch 代码风格检查
    - git diff HEAD~1 | scripts/checkpatch.pl --strict 2>&1 | tee checkpatch.log
    - |
      CHECKPATCH_ERRORS=$(grep -c "ERROR:" checkpatch.log || true)
      if [ "$CHECKPATCH_ERRORS" -gt "0" ]; then
        echo "错误：代码风格检查未通过"
        exit 1
      fi
  tags:
    - docker

# ═══════════════════════════════════════════════════════════════
# 阶段 2：冒烟测试（需要真实 GPU）
# ═══════════════════════════════════════════════════════════════

smoke_test:
  stage: smoke
  needs: [build_kernel]
  # 在带有 GPU 的裸金属 runner 上执行
  tags:
    - gpu-amd-rdna3
    - baremetal
  script:
    # 部署编译好的内核模块
    - sudo insmod linux/drivers/gpu/drm/amd/amdgpu.ko
    # 验证驱动加载
    - |
      if ! lsmod | grep -q amdgpu; then
        echo "错误：amdgpu 模块加载失败"
        exit 1
      fi
    # 验证设备检测
    - |
      GPU_COUNT=$(lspci -nn | grep -c "0300: 1002:" || true)
      echo "检测到 AMD GPU 数量: $GPU_COUNT"
      if [ "$GPU_COUNT" -eq "0" ]; then
        echo "错误：未检测到 AMD GPU"
        exit 1
      fi
    # 验证 DRM 设备节点
    - ls -la /dev/dri/
    - test -c /dev/dri/card0
    # 基本渲染测试（使用 kmscube 或类似工具）
    - sudo apt-get install -y kmscube
    - timeout 30 kmscube --drm-device=/dev/dri/card0 || true
    # 收集 dmesg
    - dmesg | tail -n 100 > smoke_dmesg.log
  artifacts:
    paths:
      - smoke_dmesg.log
    expire_in: 1 week

# ═══════════════════════════════════════════════════════════════
# 阶段 3：功能回归测试
# ═══════════════════════════════════════════════════════════════

igt_regression:
  stage: regression
  needs: [smoke_test]
  tags:
    - gpu-amd-rdna3
    - baremetal
  timeout: 2 hours
  script:
    # 安装 IGT
    - git clone --branch $IGT_VERSION https://gitlab.freedesktop.org/drm/igt-gpu-tools.git
    - cd igt-gpu-tools
    - meson setup build
    - ninja -C build
    # 运行核心回归测试子集
    - |
      ./build/tests/igt-runner \
        --test-list tests/quick-tests.txt \
        --output-format junit \
        --output-file igt_results.xml \
        --timeout 300
    # 解析结果
    - |
      PASSED=$(grep -o 'failures="[0-9]*"' igt_results.xml | head -1 | cut -d'"' -f2)
      echo "失败测试数: $PASSED"
      if [ "$PASSED" -gt "5" ]; then
        echo "错误：失败测试超过阈值"
        exit 1
      fi
  artifacts:
    paths:
      - igt-gpu-tools/igt_results.xml
    reports:
      junit: igt-gpu-tools/igt_results.xml
    expire_in: 1 month

# ═══════════════════════════════════════════════════════════════
# 阶段 4：性能测试
# ═══════════════════════════════════════════════════════════════

performance_test:
  stage: performance
  needs: [igt_regression]
  tags:
    - gpu-amd-rdna3
    - baremetal
    - performance  # 专用性能测试节点
  timeout: 4 hours
  script:
    # 运行 glmark2 基准测试
    - glmark2 --off-screen --size 1920x1080 > glmark2_result.txt
    # 运行 vkmark 基准测试
    - vkmark --off-screen --size 1920x1080 > vkmark_result.txt
    # 与基线对比
    - |
      CURRENT_SCORE=$(grep "glmark2 Score" glmark2_result.txt | awk '{print $3}')
      BASELINE_SCORE=$(cat baseline/glmark2_baseline.txt)
      DEGRADATION=$(echo "scale=2; ($BASELINE_SCORE - $CURRENT_SCORE) / $BASELINE_SCORE * 100" | bc)
      echo "性能退化: ${DEGRADATION}%"
      if (( $(echo "$DEGRADATION > 5.0" | bc -l) )); then
        echo "警告：性能退化超过 5%"
        # 不阻断，但标记警告
      fi
    # 保存结果供趋势分析
    - mkdir -p perf_results/$CI_COMMIT_SHA
    - cp glmark2_result.txt vkmark_result.txt perf_results/$CI_COMMIT_SHA/
  artifacts:
    paths:
      - perf_results/
    expire_in: 6 months

# ═══════════════════════════════════════════════════════════════
# 阶段 5：压力测试
# ═══════════════════════════════════════════════════════════════

stress_test:
  stage: stress
  needs: [performance_test]
  tags:
    - gpu-amd-rdna3
    - baremetal
  timeout: 24 hours
  when: manual  # 手动触发，或每日定时执行
  script:
    # GPU 压力测试
    - gpu-burn 3600  # 运行 1 小时
    # 内存压力测试
    - ./scripts/mem_stress_test.sh 1800
    # 热插拔循环测试
    - ./scripts/hotplug_stress.sh 100
    # 收集系统日志
    - dmesg > stress_dmesg.log
    - journalctl -k > stress_journal.log
  artifacts:
    paths:
      - stress_dmesg.log
      - stress_journal.log
    expire_in: 1 month
```

**设备调度系统核心功能**（参考 Intel TPS - Test Pool Scheduler）：

```python
# device_scheduler.py - 设备调度系统概念实现
# 参考：Intel Graphics CI 设备调度方案

import json
import time
from enum import Enum
from typing import List, Optional, Dict

class DeviceStatus(Enum):
    AVAILABLE = "available"
    BUSY = "busy"
    MAINTENANCE = "maintenance"
    OFFLINE = "offline"

class GPUDevice:
    """GPU 测试设备描述"""
    def __init__(self, device_id: str, gpu_model: str, tags: List[str]):
        self.device_id = device_id
        self.gpu_model = gpu_model
        self.tags = tags
        self.status = DeviceStatus.AVAILABLE
        self.current_job = None
        self.health_score = 100
        self.last_check = time.time()
    
    def match_requirements(self, required_tags: List[str]) -> bool:
        """检查设备是否满足作业的标签要求"""
        return all(tag in self.tags for tag in required_tags)
    
    def health_check(self) -> bool:
        """执行设备健康检查"""
        # 检查 GPU 是否可访问
        # 检查温度是否在正常范围
        # 检查上次测试是否留下脏状态
        checks = [
            self._check_gpu_visible(),
            self._check_temperature(),
            self._check_clean_state()
        ]
        self.health_score = sum(checks) / len(checks) * 100
        self.last_check = time.time()
        return self.health_score >= 80
    
    def _check_gpu_visible(self) -> float:
        """检查 GPU 是否对系统可见"""
        import subprocess
        try:
            result = subprocess.run(
                ["lspci", "-nn"], 
                capture_output=True, 
                text=True, 
                timeout=10
            )
            if "1002:" in result.stdout:  # AMD vendor ID
                return 1.0
            return 0.0
        except:
            return 0.0
    
    def _check_temperature(self) -> float:
        """检查 GPU 温度"""
        try:
            with open(f"/sys/class/drm/card0/device/hwmon/hwmon0/temp1_input") as f:
                temp = int(f.read().strip()) / 1000  # millidegree to degree
                if temp < 85:
                    return 1.0
                elif temp < 95:
                    return 0.5
                else:
                    return 0.0

                capture_output=True,
                timeout=5
            )
            if result.returncode != 0:
                return 1.0  # 无进程占用
            return 0.3
        except:
            return 0.5

    def __init__(self):
```

#### 2.3 测试结果聚合与报告

CI 流水线产生的测试结果需要统一聚合和分析：

```python
# test_result_aggregator.py - 测试结果聚合器
# 参考：KernelCI 结果聚合方案 (https://kernelci.org/)

import xml.etree.ElementTree as ET
import json
from datetime import datetime
from typing import List, Dict

class TestResult:
    """单个测试结果"""
    def __init__(self, name: str, status: str, duration: float, 
                 log_path: str = "", error_msg: str = ""):
        self.name = name
        self.status = status  # pass/fail/skip/timeout
        self.duration = duration
        self.log_path = log_path
        self.error_msg = error_msg
        self.timestamp = datetime.now()

class TestRun:
    """一次测试运行"""
    def __init__(self, run_id: str, commit_sha: str, branch: str):
        self.run_id = run_id
        self.commit_sha = commit_sha
        self.branch = branch
        self.results: List[TestResult] = []
        self.start_time = datetime.now()
        self.end_time = None
        self.metadata = {
            "kernel_version": "",
            "gpu_model": "",
            "mesa_version": "",
            "firmware_version": ""
        }
    
    def add_result(self, result: TestResult):
        self.results.append(result)
    
    def get_summary(self) -> Dict:
        """生成测试摘要"""
        total = len(self.results)
        passed = sum(1 for r in self.results if r.status == "pass")
        failed = sum(1 for r in self.results if r.status == "fail")
        skipped = sum(1 for r in self.results if r.status == "skip")
        timeout = sum(1 for r in self.results if r.status == "timeout")
        
        return {
            "run_id": self.run_id,
            "commit_sha": self.commit_sha,
            "total": total,
            "passed": passed,
            "failed": failed,
            "skipped": skipped,
            "timeout": timeout,
            "pass_rate": f"{passed/total*100:.1f}%" if total > 0 else "N/A",
            "duration": str(self.end_time - self.start_time) if self.end_time else "N/A"
        }
    
    def parse_junit_xml(self, xml_path: str):
        """解析 JUnit XML 格式的测试结果"""
        tree = ET.parse(xml_path)
        root = tree.getroot()
        
        for testsuite in root.findall("testsuite"):
            for testcase in testsuite.findall("testcase"):
                name = testcase.get("name")
                duration = float(testcase.get("time", 0))
                
                # 检查失败
                failure = testcase.find("failure")
                if failure is not None:
                    result = TestResult(
                        name=name,
                        status="fail",
                        duration=duration,
                        error_msg=failure.get("message", "")
                    )
                elif testcase.find("skipped") is not None:
                    result = TestResult(name=name, status="skip", duration=duration)
                else:
                    result = TestResult(name=name, status="pass", duration=duration)
                
                self.add_result(result)

class ResultDatabase:
    """测试结果数据库"""
    def __init__(self, db_path: str):
        self.db_path = db_path
        self.runs: List[TestRun] = []
    
    def store_run(self, run: TestRun):
        """存储测试运行结果"""
        self.runs.append(run)
        # 持久化到文件
        with open(f"{self.db_path}/{run.run_id}.json", "w") as f:
            json.dump(run.get_summary(), f, indent=2, default=str)
    
    def compare_runs(self, run_id_a: str, run_id_b: str) -> Dict:
        """对比两次测试运行的结果"""
        run_a = next((r for r in self.runs if r.run_id == run_id_a), None)
        run_b = next((r for r in self.runs if r.run_id == run_id_b), None)
        
        if not run_a or not run_b:
            return {"error": "Run not found"}
        
        # 构建结果映射
        results_a = {r.name: r.status for r in run_a.results}
        results_b = {r.name: r.status for r in run_b.results}
        
        # 找出差异
        new_failures = []
        fixed_tests = []
        
        for name, status_b in results_b.items():
            status_a = results_a.get(name)
            if status_a == "pass" and status_b == "fail":
                new_failures.append(name)
            elif status_a == "fail" and status_b == "pass":
                fixed_tests.append(name)
        
        return {
            "baseline": run_id_a,
            "current": run_id_b,
            "new_failures": new_failures,
            "fixed_tests": fixed_tests,
            "regression_count": len(new_failures)
        }
```
