### 1. eDP 协议基础与演进

#### 1.1 eDP 概述

eDP（Embedded DisplayPort）是 VESA 标准化的嵌入式显示接口协议，专门针对笔记本、平板电脑、一体机等嵌入式系统设计。eDP 基于标准 DisplayPort 协议，但针对嵌入式场景进行了优化和简化。

eDP 与标准 DP 的核心差异：

```
eDP vs 标准 DisplayPort 对比
┌──────────────────────────┬─────────────────────┬────────────────────┐
│         特性             │     eDP             │  标准 DisplayPort  │
├──────────────────────────┼─────────────────────┼────────────────────┤
│ 连接器类型               │  嵌入式排线         │  物理插头/座       │
│ 最大链路长度             │  < 1m               │  3-15m             │
│ 热插拔支持               │  可选/简化          │  完整支持          │
│ 背光控制                 │  原生支持           │  无                │
│ PSR                      │  原生支持           │  可选              │
│ ALPM                     │  原生支持           │  无                │
│ 面板自刷新               │  支持               │  不支持            │
│ 链路速率                 │  HBR/HBR2/HBR3      │  同左              │
│ 最大分辨率               │  8K (eDP 1.5)       │  同左              │
│ 多流传输(MST)            │  通常不支持         │  支持              │
│ AUX 通道                 │  简化实现           │  完整实现          │
│ 功耗优化                 │  深度优化           │  一般              │
└──────────────────────────┴─────────────────────┴────────────────────┘
```

### 2. eDP 链路训练

#### 2.1 eDP 链路训练特性

eDP 的链路训练与标准 DisplayPort 有显著差异，eDP 支持更快速的链路训练机制：

```
eDP 链路训练流程对比

标准 DP 链路训练（完整 5 阶段）:
┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
│  CR  │──>│  EQ  │──>│Symbol│──>│Align │──>│Verify│
│Phase1│   │Phase2│   │ Lock │   │ment  │   │      │
└──────┘   └──────┘   └──────┘   └──────┘   └──────┘

eDP 快速链路训练（3 阶段简化）:
┌──────────┐   ┌──────────┐   ┌──────────┐
│  快速 CR │──>│  快速 EQ │──>│  链路验证 │
│  (1-2ms) │   │  (2-5ms) │   │  (1-3ms) │
└──────────┘   └──────────┘   └──────────┘

eDP No-Link-Training 模式:
┌────────────────────────────────────────┐
│  跳过完整链路训练                       │
│  直接使用预存储的链路配置               │
│  适用于 Panel Self Refresh 恢复         │
│  恢复时间 < 1ms                         │
└────────────────────────────────────────┘
```

#### 2.2 eDP DPCD 关键寄存器

eDP 定义了一系列专用 DPCD 寄存器，用于控制背光、PSR 和电源管理：

```
eDP DPCD 寄存器映射 (地址空间: 0x00700 - 0x007FF)

地址范围         寄存器名称              说明
─────────       ────────────           ────────────────────
0x00700         背光配置               背光使能、PWM 极性
0x00701         背光亮度 MSB           8 位亮度高位
0x00702         背光亮度 LSB           8 位亮度低位
0x00703         背光控制模式           PWM / DPCD / Hybrid
0x00720 - 0x0073F  PSR 配置            PSR 使能/去使能
0x00740 - 0x0075F  ALPM 配置           ALPM 参数
0x00780 - 0x0079F  Panel Replay        Panel Replay 控制
0x007A0 - 0x007AF  DSC 配置 (eDP)      eDP DSC 参数

关键 DPCD 寄存器详细说明:

DPCD 0x00700: EDP_BACKLIGHT_CONFIG
  Bit 0: BACKLIGHT_ENABLE (0=关闭, 1=使能)
  Bit 1: PWM_POLARITY (0=高有效, 1=低有效)
  Bit 2: PWM_FREQUENCY_AUTO (0=手动, 1=自动检测)
  Bits 3-7: 保留

DPCD 0x00703: EDP_BACKLIGHT_CONTROL_MODE
  0x00: PWM 直接控制
  0x01: DPCD 寄存器控制
  0x02: 混合模式 (PWM + DPCD)
  0x03-0xFF: 保留
```

#### 2.3 eDP 链路训练代码实现

Linux 内核中 eDP 链路训练的实现位于 AMDGPU DC 驱动中：

```c
// drivers/gpu/drm/amd/display/dc/core/dc_link_dp.c
// eDP 链路训练简化实现

enum dc_status dc_link_dp_perform_link_training(
    struct dc_link *link,
    const struct dc_link_settings *link_settings,
    bool skip_video_pattern)
{
    struct link_training_settings lt_settings;
    enum dc_status status = DC_OK;

    // 检查是否为 eDP 面板
    if (link->connector_signal == SIGNAL_TYPE_EDP) {
        // eDP 优先使用快速链路训练
        if (link->preferred_training_mode == LINK_TRAINING_FAST) {
            status = perform_edp_fast_link_training(link, &lt_settings);
            if (status == DC_OK)
                return DC_OK;
        }
        // 快速训练失败，回退到完整训练
        DC_LOG_WARNING("eDP fast link training failed, falling back to full LT\n");
    }

    // 标准链路训练流程
    status = perform_link_training(link, &lt_settings);
    return status;
}

// eDP 快速链路训练实现
static enum dc_status perform_edp_fast_link_training(
    struct dc_link *link,
    struct link_training_settings *lt_settings)
{
    // 1. 读取之前存储的链路配置
    // 2. 跳过 CR 阶段，直接进行 EQ 验证
    // 3. 如果 EQ 失败，回退到完整训练
    // 4. 设置链路进入正常操作状态

    uint32_t lane_count = link->cur_link_settings.lane_count;
    uint32_t link_rate = link->cur_link_settings.link_rate;

    // 直接在 AUX 上发送 EQ pattern
    dp_set_hw_training_pattern(link, DP_TRAINING_PATTERN_2);

    // 等待 EQ 完成
    if (dp_wait_for_training_eq(link, lane_count) != DC_OK) {
        // EQ 失败，需要完整重新训练
        return DC_FAIL_UNSUPPORTED;
    }

    // 链路恢复正常操作
    dp_set_hw_training_pattern(link, DP_TRAINING_PATTERN_OFF);

    return DC_OK;
}

// eDP 背光 DPCD 控制实现
bool dc_link_set_backlight_level(
    const struct dc_link *link,
    uint32_t backlight_level,
    uint32_t frame_ramp)
{
    uint8_t backlight_data[2];
    uint32_t backlight_msb, backlight_lsb;

    // 检查 eDP 面板是否支持 DPCD 背光控制
    if (!link->dpcd_caps.edp_dpcd_backlight_enabled)
        return false;

    // 转换亮度值为 16 位
    backlight_msb = (backlight_level >> 8) & 0xFF;
    backlight_lsb = backlight_level & 0xFF;

    backlight_data[0] = backlight_msb;
    backlight_data[1] = backlight_lsb;

    // 通过 AUX 写入 DPCD 背光寄存器
    dm_helpers_dp_write_dpcd(
        link->ctx,
        link->link_index,
        DP_EDP_BACKLIGHT_BRIGHTNESS_MSB,
        backlight_data,
        sizeof(backlight_data));

    return true;
}
```

#### 3.4 背光控制代码实现

AMDGPU 驱动中的背光控制实现：

```c
// drivers/gpu/drm/amd/display/amdgpu_dm/amdgpu_dm_backlight.c
// AMDGPU eDP 背光控制实现

#include <linux/backlight.h>
#include <drm/drm_probe_helper.h>

// 背光更新回调
static int amdgpu_dm_backlight_update_status(struct backlight_device *bd)
{
    struct amdgpu_display_manager *dm = bl_get_data(bd);
    struct dc_link *link = NULL;
    struct amdgpu_dm_connector *aconnector;
    int brightness = bd->props.brightness;
    int max_brightness = bd->props.max_brightness;
    uint32_t backlight_level;
    uint32_t frame_ramp = 0;

    // 查找 eDP 连接器
    list_for_each_entry(aconnector, &dm->da_list, head) {
        if (aconnector->dc_link->connector_signal == SIGNAL_TYPE_EDP) {
            link = aconnector->dc_link;
            break;
        }
    }

    if (!link)
        return -ENODEV;

    // 将用户空间亮度 (0-max_brightness) 映射到硬件范围 (0-65535)
    backlight_level = brightness * 65535 / max_brightness;

    // 通过 DC 接口设置背光
    if (link->dpcd_caps.edp_dpcd_backlight_enabled) {
        // DPCD 控制模式
        dc_link_set_backlight_level(link, backlight_level, frame_ramp);
    } else {
        // PWM 控制模式
        dc_link_set_backlight_level_nits(link, backlight_level, frame_ramp);
    }

    return 0;
}

// 背光设备结构体定义
static const struct backlight_ops amdgpu_dm_backlight_ops = {
    .get_brightness = amdgpu_dm_backlight_get_brightness,
    .update_status = amdgpu_dm_backlight_update_status,
};

// 背光设备初始化
int amdgpu_dm_backlight_register(struct amdgpu_display_manager *dm)
{
    struct backlight_properties props = {0};
    struct backlight_device *bd;
    struct dc_link *edp_link = NULL;
    struct amdgpu_dm_connector *aconnector;

    // 查找 eDP 链接
    list_for_each_entry(aconnector, &dm->da_list, head) {
        if (aconnector->dc_link->connector_signal == SIGNAL_TYPE_EDP) {
            edp_link = aconnector->dc_link;
            break;
        }
    }

    if (!edp_link)
        return 0; // 非笔记本系统，跳过

    // 配置背光属性
    props.type = BACKLIGHT_RAW;
    props.max_brightness = 255; // 用户空间可见的最大值

    // 获取当前硬件亮度
    props.brightness = amdgpu_dm_backlight_get_brightness(NULL);

    // 注册背光设备
    bd = backlight_device_register(
        "amdgpu_bl0",
        dm->adev->dev,
        dm,
        &amdgpu_dm_backlight_ops,
        &props);

    if (IS_ERR(bd)) {
        DRM_ERROR("Failed to register backlight device\n");
        return PTR_ERR(bd);
    }

    dm->backlight_dev = bd;
    return 0;
}
```

### 4. Panel Self Refresh (PSR)

#### 4.1 PSR 原理

PSR（Panel Self Refresh）是 eDP 协议的关键节电技术，允许在静态画面时将显示控制权交给面板自身：

```
PSR 工作原理

正常模式 (Non-PSR):
  GPU ──(每帧 60fps)──→ eDP 面板
  │                    │
  ├── 发送完整帧       ├── 刷新显示
  ├── 发送完整帧       ├── 刷新显示
  ├── 发送完整帧       ├── 刷新显示
  └── GPU 持续工作     └── 面板持续刷新
  功耗: GPU 全速 + 链路全速

PSR 激活 (静态画面):
  GPU ──(最后完整帧)──→ eDP 面板
  │                    │
  ├── 进入 PSR         ├── 存储帧到 SRAM
  ├── 链路关闭         ├── 自刷新显示
  ├── GPU 休眠         ├── 自刷新显示
  └── 链路关闭         └── 自刷新显示
  功耗: GPU 低功耗 + 链路关闭

PSR 退出 (画面变化):
  GPU ──(新帧)──→ eDP 面板
  │                    │
  ├── 唤醒链路         ├── 退出自刷新
  ├── 发送新帧         ├── 接收并显示新帧
  └── 重新进入 PSR     └── 判断是否再次自刷新
  功耗: 短暂峰值，然后恢复低功耗

PSR 功耗节省统计:
  ┌──────────────┬────────────┬──────────────┐
  │  使用场景     │ 非 PSR     │  PSR 启用    │
  ├──────────────┼────────────┼──────────────┤
  │  静止桌面     │  3-5W      │  0.5-1W      │
  │  文档阅读     │  3-4W      │  0.5-1W      │
  │  视频播放     │  2-3W      │  1-2W        │
  │  动态内容     │  3-5W      │  2-3W        │
  │  综合续航     │  基准      │  +20-40%     │
  └──────────────┴────────────┴──────────────┘
```

#### 4.3 PSR Linux 内核实现

```c
// drivers/gpu/drm/amd/display/modules/power/power_helpers.c
// PSR 状态管理与控制

enum psr_state {
    PSR_STATE_INACTIVE,
    PSR_STATE_ACTIVE,
    PSR_STATE_TRANSITION,
    PSR_STATE_ERROR,
};

struct psr_context {
    struct dc_link *link;
    enum psr_state state;
    uint32_t su_x_start;
    uint32_t su_y_start;
    uint32_t su_x_end;
    uint32_t su_y_end;
    bool psr2_enabled;
};

// PSR 进入函数
bool dc_link_set_psr_enable(struct dc_link *link, bool enable, bool wait)
{
    struct dmcu *dmcu = link->dc->dmcu;
    struct psr_context *psr = link->psr_context;
    uint32_t psr_power_opt = 0;

    if (!link->psr_feature_enabled)
        return false;

    if (enable) {
        // 1. 配置 PSR 参数
        psr_power_opt |= PSR_POWER_OPT_FRAME_CAPTURE;
        if (link->psr_feature_flags & PSR_FLAG2_SU_REGION_MASK)
            psr_power_opt |= PSR_POWER_OPT_SU_REGION;

        // 2. 通过 DMUB 发送 PSR 进入命令
        dmub_psr_set_state(link, psr_power_opt, true);

        // 3. 等待 PSR 激活 (可选)
        if (wait)
            psr_wait_for_active(link, 100); // 100ms 超时

        DC_LOG_PSR("PSR entered successfully\n");
    } else {
        // 退出 PSR
        dmub_psr_set_state(link, 0, false);

        if (wait)
            psr_wait_for_inactive(link, 10); // 10ms 超时

        DC_LOG_PSR("PSR exited\n");
    }

    return true;
}

// PSR2 Selective Update 配置
bool dc_link_psr2_set_su(struct dc_link *link,
                         struct dc_psr2_su_context *su_context)
{
    struct dmub_psr_psr2_su_data su_data;

    // 准备 Selective Update 数据
    su_data.su_x_start = su_context->x_start;
    su_data.su_y_start = su_context->y_start;
    su_data.su_x_end = su_context->x_end;
    su_data.su_y_end = su_context->y_end;
    su_data.su_num = su_context->su_count;

    // 通过 DMUB 固件发送 SU 区域信息
    dmub_psr_set_su(link, &su_data);

    return true;
}
```

```
│  HBR3 4-lane 带宽: 8.1 * 4 = 32.4 Gbps (勉强够用)      │

│  │     ┌──┬──┬──┬──┐                                ││
```

#### 6.2 eDP DSC 配置

```c
// eDP DSC 参数配置结构体
struct edp_dsc_config {
    uint32_t dsc_clock;           // DSC 时钟频率 (kHz)
    uint32_t dsc_slice_width;     // 每片宽度 (像素)
    uint32_t dsc_slice_height;    // 每片高度 (行)
    uint32_t dsc_bits_per_pixel;  // 目标 bpp (如 8, 10, 12)
    uint32_t dsc_bits_per_component; // 每分量位数
    uint32_t dsc_num_slices_h;    // 水平切片数
    uint32_t dsc_num_slices_v;    // 垂直切片数
    bool dsc_force_version;       // 强制 DSC 版本
    uint32_t dsc_version_major;   // DSC 主版本
    uint32_t dsc_version_minor;   // DSC 次版本
};

// eDP DSC 能力检测
bool edp_decide_dsc_config(struct dc_link *link,
                           struct edp_dsc_config *dsc_cfg)
{
    uint8_t dsc_caps[16];
    uint32_t max_slice_width, max_slice_height;

    // 读取 eDP DSC DPCD 能力
    dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                            DP_DSC_SUPPORT, dsc_caps, sizeof(dsc_caps));

    // 解析 DSC 能力
    max_slice_width = dsc_caps[0] | (dsc_caps[1] << 8);
    max_slice_height = dsc_caps[2] | (dsc_caps[3] << 8);

    // 根据链路带宽和分辨率计算所需压缩比
    // 如果带宽不足，启用 DSC
    if (link->cur_link_settings.link_rate * link->cur_link_settings.lane_count
        < required_bandwidth) {
        dsc_cfg->dsc_bits_per_pixel = 8; // 8 bpp 压缩
        return true;
    }

    return false; // 不需要 DSC
}
```

#### 8.2 PSR 闪烁问题

```c
// Bug: PSR 退出时出现短暂闪烁 (flicker)
// 修复: 增加 PSR 退出时的消隐延迟

/*
 * Bug ID: freedesktop.org bug #107599
 * 描述: 在启用 PSR 的笔记本上，画面切换时出现
 *       底部 1-2 行的水平闪烁条
 *
 * 根因分析:
 *   当 GPU 退出 PSR 发送新帧时，面板自刷新
 *   和 GPU 刷新之间存在 timing 竞争:
 *
 *   时间线:
 *   Normal frame:  |--- Frame N ---|--- Frame N+1 ---|
 *   PSR active:    |--- PSR Self Refresh ------------|
 *   PSR exit:                       |--- Frame N+1 ---|
 *                                 ^
 *                                 |
 *                     此时如果面板正在自分刷新底部行，
 *                     GPU 的新帧可能没有完全写入面板 DRAM，
 *                     导致底部出现旧数据闪烁
 *
 * 修复方案:
 *   1. PSR 退出时，GPU 发送 2-3 帧全黑画面 (消隐帧)
 *   2. 等待面板 DRAM 完全更新
 *   3. 然后恢复正常画面输出
 *
 * 代码修复:
 */
void psr_exit_with_blanking(struct dc_link *link)
{
    int i;

    // 发送 3 帧消隐画面
    for (i = 0; i < 3; i++) {
        dc_stream_set_blank(link->dc->current_state,
                           link->dc->current_state->streams[0], true);
        dc_stream_set_blank(link->dc->current_state,
                           link->dc->current_state->streams[0], false);
    }

    // 然后正常退出 PSR
    dc_link_set_psr_enable(link, false, true);
}
```

### 13. eDP 驱动开发实践

#### 13.1 驱动架构代码示例

```c
// drivers/gpu/drm/amd/display/amdgpu_dm/amdgpu_dm_edp.c
// eDP 特定功能实现

#include "amdgpu_dm_edp.h"

// eDP 面板能力结构体
struct edp_panel_caps {
    uint32_t panel_inst;           // 面板实例编号
    uint32_t backlight_max;        // 最大背光值
    uint32_t backlight_min;        // 最小背光值
    uint32_t pwm_freq;             // PWM 频率 (Hz)
    bool supports_dpcd_backlight;  // 是否支持 DPCD 背光
    bool supports_psr;             // 是否支持 PSR
    bool supports_psr2;            // 是否支持 PSR2
    bool supports_dsc;             // 是否支持 DSC
    bool supports_fald;            // 是否支持区域背光
};

// 读取 eDP 面板能力
static int edp_read_panel_caps(struct dc_link *link,
                                struct edp_panel_caps *caps)
{
    uint8_t dpcd_data[16];
    int ret;

    // 读取 eDP 背光能力
    ret = dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                                   DP_EDP_BACKLIGHT_CAPABILITY,
                                   dpcd_data, 4);
    if (ret < 0)
        return ret;

    caps->supports_dpcd_backlight =
        (dpcd_data[0] & DP_EDP_DPCD_BACKLIGHT_ENABLE) ? true : false;

    // 读取 PSR 能力
    ret = dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                                   DP_PSR_SUPPORT, dpcd_data, 2);
    if (ret >= 0) {
        caps->supports_psr =
            (dpcd_data[0] & DP_PSR_IS_SUPPORTED) ? true : false;
        caps->supports_psr2 =
            (dpcd_data[1] & DP_PSR2_IS_SUPPORTED) ? true : false;
    }

    // 读取 DSC 能力
    ret = dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                                   DP_DSC_SUPPORT, dpcd_data, 4);
    if (ret >= 0) {
        caps->supports_dsc =
            (dpcd_data[0] & DP_DSC_DECODER_IS_SUPPORTED) ? true : false;
    }

    return 0;
}

// eDP 初始化流程
int edp_panel_init(struct dc_link *link)
{
    struct edp_panel_caps caps = {0};
    int ret;

    // 1. 面板上电
    edp_panel_power_up(link);

    // 2. 等待面板就绪
    msleep(EDP_PANEL_POWERUP_DELAY_MS);

    // 3. 读取面板能力
    ret = edp_read_panel_caps(link, &caps);
    if (ret < 0) {
        DRM_ERROR("Failed to read eDP panel caps\n");
        return ret;
    }

    // 4. 配置背光控制模式
    if (caps.supports_dpcd_backlight) {
        edp_set_dpcd_backlight_mode(link);
        DRM_INFO("eDP: Using DPCD backlight control\n");
    } else {
        edp_set_pwm_backlight_mode(link);
        DRM_INFO("eDP: Using PWM backlight control\n");
    }

    // 5. 配置 PSR
    if (caps.supports_psr) {
        edp_psr_init(link);
        DRM_INFO("eDP: PSR initialized\n");
    }

    // 6. 设置初始背光
    dc_link_set_backlight_level(link, EDP_INITIAL_BACKLIGHT, 0);

    return 0;
}
```

#### 13.2 背光控制模式选择策略

```c
// 背光控制模式选择逻辑

enum backlight_control_mode {
    BACKLIGHT_MODE_PWM,        // 传统 PWM 控制
    BACKLIGHT_MODE_DPCD,       // DPCD 寄存器控制
    BACKLIGHT_MODE_HYBRID,     // 混合控制
    BACKLIGHT_MODE_AUX,        // AUX 辅助控制
};

static enum backlight_control_mode
edp_select_backlight_mode(struct dc_link *link)
{
    uint8_t caps[4];
    bool dpcd_backlight_valid = false;

    // 读取 eDP DPCD 背光能力
    dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                            DP_EDP_BACKLIGHT_CAPABILITY,
                            caps, sizeof(caps));

    // 检查 DPCD 背光是否实际可用
    if (caps[0] & DP_EDP_DPCD_BACKLIGHT_ENABLE) {
        // 写入测试值并回读验证
        uint8_t test_val[2] = {0xAA, 0xBB};
        uint8_t read_val[2] = {0};

        dm_helpers_dp_write_dpcd(link->ctx, link->link_index,
                                 DP_EDP_BACKLIGHT_BRIGHTNESS_MSB,
                                 test_val, 2);
        dm_helpers_dp_read_dpcd(link->ctx, link->link_index,
                                DP_EDP_BACKLIGHT_BRIGHTNESS_MSB,
                                read_val, 2);

        if (read_val[0] == test_val[0] && read_val[1] == test_val[1])
            dpcd_backlight_valid = true;
    }

    // 选择控制模式
    if (dpcd_backlight_valid)
        return BACKLIGHT_MODE_DPCD;

    // 检查 PWM 引脚是否连接
    if (link->dpcd_caps.edp_pwm_freq > 0)
        return BACKLIGHT_MODE_PWM;

    // 默认使用混合模式
    return BACKLIGHT_MODE_HYBRID;
}
```

#### 14.2 eDP 链路电源状态管理

```c
// eDP 链路电源状态管理

enum edp_link_power_state {
    EDP_LINK_POWER_NORMAL,      // 正常操作
    EDP_LINK_POWER_STANDBY,     // ALPM 待机
    EDP_LINK_POWER_DEEP_SLEEP,  // 深度睡眠
    EDP_LINK_POWER_OFF,         // 完全关闭
};

int edp_set_link_power_state(struct dc_link *link,
                              enum edp_link_power_state state)
{
    uint8_t dpcd_data;
    int ret;

    switch (state) {
    case EDP_LINK_POWER_NORMAL:
        // 设置链路为正常模式
        dpcd_data = DP_SET_POWER_D0;
        ret = dm_helpers_dp_write_dpcd(link->ctx, link->link_index,
                                        DP_SET_POWER, &dpcd_data, 1);
        break;

    case EDP_LINK_POWER_STANDBY:
        // 进入 ALPM 待机
        dpcd_data = DP_SET_POWER_D3;
        ret = dm_helpers_dp_write_dpcd(link->ctx, link->link_index,
                                        DP_SET_POWER, &dpcd_data, 1);
        break;

    case EDP_LINK_POWER_DEEP_SLEEP:
        // 进入深度睡眠
        dpcd_data = DP_SET_POWER_D3;
        ret = dm_helpers_dp_write_dpcd(link->ctx, link->link_index,
                                        DP_SET_POWER, &dpcd_data, 1);
        // 额外关闭 PLL
        edp_disable_pll(link);
        break;

    case EDP_LINK_POWER_OFF:
        // 完全关闭链路
        edp_panel_power_down(link);
        ret = 0;
        break;
    }

    return ret;
}
```

今天我们深入学习了 eDP（Embedded DisplayPort）显示接口协议，这是笔记本和平板等嵌入式设备中最重要的显示接口标准。我们掌握了 eDP 与标准 DisplayPort 的关键区别，包括物理连接、协议特性和电源管理方面的差异。

在背光控制方面，我们学习了 PWM 控制和 DPCD 控制两种主要方式，理解了混合调光（Hybrid Dimming）的原理和实现。PSR（Panel Self Refresh）是 eDP 的重要节电技术，我们分析了 PSR1 和 PSR2 的工作原理，以及 Selective Update 的实现机制。

我们还深入了解了 eDP 链路训练的特殊性、DSC 压缩在 eDP 中的应用、ALPM 电源管理以及 MSA 数据结构。通过实际的 Bug 案例分析（背光兼容性、PSR 闪烁、S3 恢复失败），我们理解了 eDP 驱动开发中常见的陷阱和解决方案。

明日我们将继续学习显示接口的另一个重要分支：USB-C DP Alt Mode，了解 USB-C 接口如何承载 DisplayPort 信号，以及相关的协议转换和电源管理机制。

## 扩展思考

1. 随着 OLED 和 MiniLED 面板在笔记本中的普及，eDP 背光控制协议是否需要演进以适应自发光面板的无背光特性？这些面板的亮度控制方式与 LCD 有何不同？

2. PSR 技术在减少功耗的同时引入了额外的显示延迟。在游戏场景中，PSR 的这些延迟会对玩家体验产生多大影响？是否可以设计一个智能策略，在检测到全屏游戏时自动禁用 PSR？

3. eDP 1.5 引入了 UHBR 链路速率，但笔记本内部的信号完整性受排线长度和 EMI 限制。在 20Gbps 的速率下，eDP 的信号完整性问题与标准 DP 有何不同？如何在嵌入式设计中保证信号质量？
