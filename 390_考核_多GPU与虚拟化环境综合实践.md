**参考答案与评分标准**

*第一部分：选择题*
1. B (3分) - VFIO 的作用是将物理 PCIe 设备直接分配给虚拟机
2. B (3分) - IOMMU 组内所有设备必须分配给同一个 VM
3. B (3分) - FLR 只重置单个 Function，最轻量级
4. B (3分) - PCIe Switch 上行链路是所有 GPU 共享的瓶颈
5. B (3分) - XGMI 链路训练失败后自动回退到较低速率
6. B (3分) - vCPU 和 GPU 绑定到同 NUMA 节点减少跨节点访问
7. C (3分) - 4 条链路 × 50 GB/s = 200 GB/s
8. A (3分) - `-device vfio-pci,host=03:00.0` 绑定 VFIO 设备
9. B (3分) - RCCL 提供集合通信操作
10. C (3分) - 非一致性模式下需要使用原子操作或内存屏障

*第二部分：判断题*
11. × (2分) - 需要从主机解绑 amdgpu 驱动，绑定 vfio-pci 驱动
12. √ (2分) - ACS 是 IOMMU 组隔离的关键硬件特性
13. × (2分) - XGMI 链路带宽随温度升高而降低（信号质量劣化）
14. × (2分) - All-Reduce 通信时间理论上随 GPU 数量对数增长
15. × (2分) - FLR 只重置 PCIe 功能状态，不保证清除显存

*第三部分：简答题*
16. (10分) 评分标准：
   - 启用 IOMMU (intel_iommu=on/amd_iommu=on) (1分)
   - 解绑 GPU 驱动 (1分)
   - 绑定 vfio-pci 驱动 (1分)
   - 验证 IOMMU 组完整性 (2分)
   - 配置 QEMU 参数 (-device vfio-pci) (2分)
   - 配置大页内存 (1分)
   - 配置 vCPU 亲和性和 NUMA 绑定 (1分)
   - 配置 GPU reset 机制 (1分)

17. (10分) 评分标准：
   - 协议层：一致性协议/原子操作/消息传递 (2分)
   - 链路层：流量控制/错误检测 (CRC32)/重传机制 (3分)
   - 物理层：SerDes/128b130b 编码/链路训练 (2分)
   - CRC 错误处理：重试→回退速率→链路禁用→蜂巢重配置 (3分)

18. (10分) 评分标准：
   - PCIe P2P: 较高延迟 (~1-3μs), 受 PCIe 拓扑限制 (2分)
   - XGMI P2P: 低延迟 (~500ns), 高带宽 (~200GB/s), 硬件一致性 (2分)
   - 代码路径: PCIe 通过 BAR 映射+DMA；XGMI 通过 XGMI 地址翻译+一致性引擎 (3分)
   - 驱动差异: PCIe 需要 dma_buf/prime；XGMI 使用 amdgpu_xgmi_peer (3分)

*第四部分：实践操作*
19. (15分) 评分标准：
   - (1) XGMI 链路 2 发生不可纠正错误被禁用，蜂巢拓扑从 4 节点变为 3 节点 (5分)
   - (2) 诊断脚本需要包含：xgmi_link_status 读取、xgmi_retry_count 检查、dmesg 过滤、复位操作 (5分)
   - (3) 容错设计：错误链路隔离、工作负载迁移、在线修复（热插拔）、降级模式运行 (5分)

20. (15分) 评分标准：
   - (1) 自动化脚本：sysfs 扫描、节点 ID 获取、链路状态读取、拓扑输出 (5分)
   - (2) RCCL 测试脚本：环境初始化、All-Reduce 执行、带宽记录、多 GPU 循环 (5分)
   - (3) 性能分析：跨 XGMI 组的 PCIe P2P 是瓶颈，GPU0/1↔GPU2/3 之间带宽受限 (5分)
