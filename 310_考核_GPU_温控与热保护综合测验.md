### 第 1 题
以下关于 AMD GPU 结温传感器的描述，哪项是正确的？

A. 结温传感器通常位于 GPU PCB 边缘，用于测量环境温度
B. 结温传感器基于半导体 PN 结的温度-电压特性工作
C. 结温传感器的典型精度为 ±5°C
D. 结温传感器只能在 GPU 空闲时读取

**正确答案**：B
**解析**：
- A 错误：结温传感器位于 GPU 芯片核心内部，临近计算单元
- B 正确：基于硅 PN 结的正向电压与温度的关系（约 -2 mV/°C）
- C 错误：结温传感器的典型精度为 ±1-2°C
- D 错误：结温传感器可以在任何状态下实时读取

### 第 2 题
GPU 热节流（Thermal Throttle）的主要触发条件是什么？

A. GPU 利用率达到 100%
B. GPU 温度超过设定的热节流阈值
C. GPU 功耗达到最大限制
D. GPU 显存使用达到 100%

**正确答案**：B
**解析**：
- A 错误：高利用率是发热的原因，但不是节流的直接触发条件
- B 正确：当温度达到设定阈值时，驱动会降低频率以减少发热
- C 错误：功耗限制是另一个独立的控制机制（Power Limit）
- D 错误：显存满载与热节流无直接关系

### 题目 1：温度监控脚本开发（15 分）

**任务描述**：
编写一个 Bash 脚本，实时监控 GPU 温度变化，当温度超过阈值时触发警报并记录相关信息。

**要求**：
1. 支持通过 hwmon、thermal_zone、rocm-smi 三种方式读取温度
2. 实现可配置的温度阈值（警告阈值和紧急阈值）
3. 当温度超过警告阈值时，打印警告信息并记录日志
4. 当温度超过紧急阈值时，打印紧急信息并执行清理动作
5. 输出温度历史记录到 CSV 文件

**评分标准**：
- 温度读取功能（4 分）
- 阈值检测与警报逻辑（4 分）
- 日志记录功能（3 分）
- 错误处理与鲁棒性（2 分）
- 代码注释与可读性（2 分）

**参考解答**：

```bash
#!/bin/bash
# gpu_temperature_monitor.sh
# GPU 温度监控脚本 - 考核参考实现

# ============= 配置区 =============
# 监控间隔（秒）
INTERVAL=2

# 温度阈值（°C）
WARNING_THRESHOLD=85
CRITICAL_THRESHOLD=95

# 日志文件
LOG_FILE="gpu_temp_monitor.log"
CSV_FILE="gpu_temp_history.csv"

# 警报命令（可自定义）
ALERT_CMD="echo"
# 紧急动作（如发送邮件、记录系统日志）
EMERGENCY_CMD="logger -t GPU_TEMP_ALERT"

# ============= 初始化 =============

echo "========================================"
echo "  GPU Temperature Monitor Started"
echo "========================================"
echo "Warning threshold: ${WARNING_THRESHOLD}°C"
echo "Critical threshold: ${CRITICAL_THRESHOLD}°C"
echo "Log file: $LOG_FILE"
echo "CSV file: $CSV_FILE"
echo "========================================"
echo ""

# 查找 GPU hwmon 路径
find_hwmon_path() {
    local hwmon_path=$(find /sys/class/drm/card0/device/hwmon/ -name "hwmon*" | head -1)
    if [ -n "$hwmon_path" ] && [ -d "$hwmon_path" ]; then
        echo "$hwmon_path"
    else
        echo ""
    fi
}

HWMON_PATH=$(find_hwmon_path)

# 查找 GPU thermal zone
find_thermal_zone() {
    for zone in /sys/class/thermal/thermal_zone*; do
        if [ -d "$zone" ]; then
            local zone_type=$(cat "$zone/type" 2>/dev/null)
            if echo "$zone_type" | grep -iq "amdgpu\|gpu"; then
                echo "$zone"
                return
            fi
        fi
    done
    echo ""
}

THERMAL_ZONE=$(find_thermal_zone)

# 初始化 CSV
if [ ! -f "$CSV_FILE" ]; then
    echo "Timestamp,Source,Temperature(°C),Status" > "$CSV_FILE"
fi

# ============= 温度读取函数 =============

# 方法 1: 通过 hwmon 读取
read_temp_hwmon() {
    if [ -z "$HWMON_PATH" ] || [ ! -d "$HWMON_PATH" ]; then
        echo ""
        return
    fi
    
    local temp_file=$(find "$HWMON_PATH" -name "temp1_input" 2>/dev/null | head -1)
    if [ -f "$temp_file" ]; then
        local temp_mc=$(cat "$temp_file" 2>/dev/null)
        if [ -n "$temp_mc" ] && [ "$temp_mc" -gt 0 ]; then
            echo $((temp_mc / 1000))
        else
            echo ""
        fi
    else
        echo ""
    fi
}

# 方法 2: 通过 thermal_zone 读取
read_temp_thermal() {
    if [ -z "$THERMAL_ZONE" ] || [ ! -d "$THERMAL_ZONE" ]; then
        echo ""
        return
    fi
    
    local temp=$(cat "$THERMAL_ZONE/temp" 2>/dev/null)
    if [ -n "$temp" ] && [ "$temp" -gt 0 ]; then
        echo $((temp / 1000))
    else
        echo ""
    fi
}

# 方法 3: 通过 rocm-smi 读取
read_temp_rocmsmi() {
    if ! command -v rocm-smi &> /dev/null; then
        echo ""
        return
    fi
    
    local temp_output=$(rocm-smi --showtemp 2>/dev/null)
    local temp=$(echo "$temp_output" | grep -i "temperature" | grep -v "critical" | head -1 | grep -oE '[0-9]+\.?[0-9]*' | head -1)
    if [ -n "$temp" ]; then
        # 转换为整数
        echo "$temp" | awk '{printf "%d\n", $1}'
    else
        echo ""
    fi
}

# 获取所有可用温度读数
get_all_temps() {
    local temps=()
    
    local temp_hwmon=$(read_temp_hwmon)
    if [ -n "$temp_hwmon" ]; then
        temps+=("hwmon:$temp_hwmon")
    fi
    
    local temp_thermal=$(read_temp_thermal)
    if [ -n "$temp_thermal" ]; then
        temps+=("thermal:$temp_thermal")
    fi
    
    local temp_rocm=$(read_temp_rocmsmi)
    if [ -n "$temp_rocm" ]; then
        temps+=("rocm-smi:$temp_rocm")
    fi
    
    echo "${temps[@]}"
}

# ============= 主监控循环 =============

count=0
in_warning=0
in_critical=0

trap 'echo -e "\n\n========================================"; echo "  Monitor stopped by user"; echo "========================================"; exit 0' SIGINT SIGTERM

while true; do
    timestamp=$(date "+%Y-%m-%d %H:%M:%S")
    count=$((count + 1))
    
    echo "[$timestamp] === Monitoring cycle #$count ==="
    
    # 获取所有温度读数
    temp_array=($(get_all_temps))
    
    if [ ${#temp_array[@]} -eq 0 ]; then
        echo "  ERROR: Could not read any temperature sensor"
        echo "[$timestamp] ERROR: Could not read any temperature sensor" >> "$LOG_FILE"
        sleep $INTERVAL
        continue
    fi
    
    # 检查每个温度读数
    max_temp=0
    max_source=""
    
    for temp_info in "${temp_array[@]}"; do
        source=${temp_info%%:*}
        temp=${temp_info##*:}
        
        echo "  $source: ${temp}°C"
        
        # 记录到 CSV
        echo "$timestamp,$source,$temp,Normal" >> "$CSV_FILE"
        
        # 更新最高温度
        if [ "$temp" -gt "$max_temp" ]; then
            max_temp=$temp
            max_source=$source
        fi
    done
    
    # 温度状态判断
    echo ""
    echo "  Highest: ${max_temp}°C (from $max_source)"
    echo "  Status: \c"
    
    if [ "$max_temp" -ge "$CRITICAL_THRESHOLD" ]; then
        echo -e "\033[31mCRITICAL!\033[0m"
        
        # 首次进入紧急状态
        if [ $in_critical -eq 0 ]; then
            echo "  >>> CRITICAL ALERT: Temperature reached ${max_temp}°C!"
            echo "[$timestamp] CRITICAL ALERT: Temperature ${max_temp}°C from $max_source" >> "$LOG_FILE"
            $EMERGENCY_CMD "GPU temperature critical: ${max_temp}°C"
            in_critical=1
            in_warning=1
        fi
        
    elif [ "$max_temp" -ge "$WARNING_THRESHOLD" ]; then
        echo -e "\033[33mWARNING\033[0m"
        
        # 首次进入警告状态
        if [ $in_warning -eq 0 ]; then
            echo "  >>> WARNING: Temperature reached ${max_temp}°C!"
            echo "[$timestamp] WARNING: Temperature ${max_temp}°C from $max_source" >> "$LOG_FILE"
            $ALERT_CMD "GPU temperature warning: ${max_temp}°C"
            in_warning=1
        fi
        
    else
        echo -e "\033[32mNormal\033[0m"
        
        # 从警告/紧急状态恢复
        if [ $in_critical -eq 1 ]; then
            echo "  >>> Recovered from CRITICAL state"
            echo "[$timestamp] RECOVERED: Temperature dropped to ${max_temp}°C" >> "$LOG_FILE"
            in_critical=0
        fi
        if [ $in_warning -eq 1 ]; then
            echo "  >>> Recovered from WARNING state"
            echo "[$timestamp] RECOVERED: Temperature dropped to ${max_temp}°C" >> "$LOG_FILE"
            in_warning=0
        fi
    fi
    
    echo ""
    sleep $INTERVAL
done
```

### 题目 2：温度阈值验证测试（15 分）

**评分标准**：

**参考测试方案**：

```bash
#!/bin/bash
# thermal_threshold_test.sh
# 温度阈值验证测试脚本

# 配置
TEST_DURATION=180  # 测试时长（秒）
SAMPLE_INTERVAL=1   # 采样间隔（秒）
LOG_FILE="thermal_test_report.log"
DATA_FILE="thermal_test_data.csv"

echo "========================================"
echo "  Thermal Threshold Validation Test"
echo "========================================"
echo ""

# 步骤 1: 记录初始状态
echo "[Step 1] Recording initial state..."
echo "========================================"
echo "Timestamp: $(date)"
echo ""

# 查找 hwmon 路径
HWMON_PATH=$(find /sys/class/drm/card0/device/hwmon/ -name "hwmon*" | head -1)
echo "HWMON Path: $HWMON_PATH"

# 记录当前阈值
echo ""
echo "Current temperature thresholds:"
if [ -n "$HWMON_PATH" ]; then
    for threshold in "$HWMON_PATH"/temp*_crit "$HWMON_PATH"/temp*_max; do
        if [ -f "$threshold" ]; then
            value=$(cat "$threshold" 2>/dev/null)
            if [ -n "$value" ]; then
                name=$(basename "$threshold")
                echo "  $name: $((value / 1000))°C"
            fi
        fi
    done
fi

echo ""
echo "========================================"

# 步骤 2: 记录温度阈值（通过debugfs）
echo ""
echo "[Step 2] Checking debugfs thermal info..."
if [ -d /sys/kernel/debug/dri/0/ ]; then
    cat /sys/kernel/debug/dri/0/amdgpu_pm_info 2>/dev/null | grep -i temp || echo "  (debugfs not available)"
fi

# 步骤 3: 运行负载测试
echo ""
echo "[Step 3] Running thermal load test (${TEST_DURATION}s)..."
echo ""

# 初始化CSV
echo "timestamp,temp_input,power,clock,fan_speed" > "$DATA_FILE"

# 使用glxgears作为负载（如果没有，使用stress工具）
LOAD_CMD=""
if command -v glmark2 &> /dev/null; then
    LOAD_CMD="glmark2 --run-forever"
elif command -v glxgears &> /dev/null; then
    LOAD_CMD="glxgears"
elif command -v stress-gpu &> /dev/null; then
    LOAD_CMD="stress-gpu"
else
    echo "WARNING: No GPU stress tool found. Using CPU-bound fallback."
    LOAD_CMD="stress --vm 2 --vm-bytes 512M"
fi

# 启动负载（后台）
$LOAD_CMD &
LOAD_PID=$!
echo "  Load process started (PID: $LOAD_PID)"

# 主监控循环
echo ""
echo "  Monitoring temperature/frequency/power..."
echo "  Time  |  Temp(°C) | Power(W) | Clock(MHz) | Fan(%)"
echo "  ------+----------+----------+------------+--------"

for ((i=0; i<TEST_DURATION; i+=SAMPLE_INTERVAL)); do
    timestamp=$(date +%s)
    
    # 读取温度
    temp=""
    if [ -n "$HWMON_PATH" ]; then
        temp_file=$(find "$HWMON_PATH" -name "temp1_input" 2>/dev/null | head -1)
        [ -f "$temp_file" ] && temp=$(( $(cat "$temp_file") / 1000 ))
    fi
    
    # 读取功耗
    power=""
    if [ -n "$HWMON_PATH" ]; then
        power_file=$(find "$HWMON_PATH" -name "power1_average" 2>/dev/null | head -1)
        [ -f "$power_file" ] && power=$(( $(cat "$power_file") / 1000000 ))
    fi
    
    # 读取时钟频率
    clock=""

#!/usr/bin/env python3
# gpu_multi_sensor_analysis.py
# GPU多传感器温度数据融合分析

import os
import glob
import time
import csv
import json
from datetime import datetime
from collections import defaultdict

class GPUTemperatureAnalyzer:
    AMD GPU多传感器温度分析器
    
    def __init__(self, hwmon_path=None):
        self.hwmon_path = hwmon_path or self._find_hwmon_path()
        self.sensors = {}
        self.history = []
        self._discover_sensors()
    
    def _find_hwmon_path(self):
        drm_path = "/sys/class/drm/card0/device/hwmon"
        if os.path.exists(drm_path):
            hwmons = glob.glob(os.path.join(drm_path, "hwmon*"))
            if hwmons:
                return hwmons[0]
        return None
    
    def _discover_sensors(self):
        发现所有温度传感器及其属性
        if not self.hwmon_path:
            print("ERROR: No hwmon path found")
            return
        
        temp_inputs = glob.glob(os.path.join(self.hwmon_path, "temp*_input"))
        for temp_input in sorted(temp_inputs):
            base = temp_input.replace("_input", "")
            sensor_id = os.path.basename(base)
            
            sensor_info = {
                "path": temp_input,
                "label": "unknown",
                "crit": None,
                "max": None,
                "type": "unknown",
            }
            
            读取传感器标签
            label_path = base + "_label"
            if os.path.exists(label_path):
                try:
                    with open(label_path) as f:
                        sensor_info["label"] = f.read().strip()
                except:
                    pass
            
            读取阈值
            for attr in ["crit", "max", "crit_hyst", "max_hyst"]:
                attr_path = base + "_" + attr
                if os.path.exists(attr_path):
                    try:
                        with open(attr_path) as f:
                            sensor_info[attr] = int(f.read().strip()) // 1000
                    except:
                        pass
            
            根据标签分类传感器类型
            label_lower = sensor_info["label"].lower()
            if "edge" in label_lower:
                sensor_info["type"] = "edge"
            elif "junction" in label_lower or "hot spot" in label_lower:
                sensor_info["type"] = "hotspot"
            elif "memory" in label_lower or "mem" in label_lower:
                sensor_info["type"] = "memory"
            elif "vrm" in label_lower or "vddc" in label_lower or "soc" in label_lower:
                sensor_info["type"] = "vrm"
            elif "pcie" in label_lower:
                sensor_info["type"] = "pcie"
            else:
                sensor_info["type"] = "other"
            
            self.sensors[sensor_id] = sensor_info
    
    def read_all_temperatures(self):
        读取所有传感器当前温度
        readings = {}
        for sensor_id, info in self.sensors.items():
            try:
                with open(info["path"]) as f:
                    temp_mc = int(f.read().strip())
                    readings[sensor_id] = {
                        "temp_c": temp_mc / 1000.0,
                        "label": info["label"],
                        "type": info["type"],
                        "crit": info.get("crit"),
                        "max": info.get("max"),
                    }
            except (IOError, ValueError) as e:
                readings[sensor_id] = {
                    "temp_c": None,
                    "label": info["label"],
                    "type": info["type"],
                    "error": str(e),
                }
        return readings
    
    def analyze(self, duration_sec=60, interval_sec=2):
        运行分析并生成报告
        print(f"GPU多传感器温度分析")
        print(f"=" * 50)
        print(f"hwmon路径: {self.hwmon_path}")
        print(f"发现 {len(self.sensors)} 个温度传感器")
        print()
        
        显示发现的传感器
        print("传感器清单:")
        print(f"  {'ID':<10} {'标签':<20} {'类型':<10} {'临界阈值':<12} {'最大阈值':<12}")
        print(f"  {'-'*10} {'-'*20} {'-'*10} {'-'*12} {'-'*12}")
        for sid, info in self.sensors.items():
            crit = f"{info['crit']}°C" if info.get('crit') else "N/A"
            max_t = f"{info['max']}°C" if info.get('max') else "N/A"
            print(f"  {sid:<10} {info['label']:<20} {info['type']:<10} {crit:<12} {max_t:<12}")
        
        print()
        print(f"开始采集数据（{duration_sec}秒，间隔{interval_sec}秒）...")
        
        数据采集循环
        start_time = time.time()
        while time.time() - start_time < duration_sec:
            timestamp = time.time()
            readings = self.read_all_temperatures()
            
            record = {"timestamp": timestamp}
            for sid, data in readings.items():
                if data["temp_c"] is not None:
                    record[f"{data['type']}_{sid}"] = data["temp_c"]
            
            self.history.append(record)
            time.sleep(interval_sec)
        
        self._generate_report()
    
    def _generate_report(self):
        生成分析报告
        if not self.history:
            print("No data collected")
            return
        
        按传感器类型聚合数据
        type_data = defaultdict(list)
        for record in self.history:
            for key, value in record.items():
                if key != "timestamp":
                    type_data[key].append(value)
        
        print()
        print("=" * 50)
        print("温度分析报告")
        print("=" * 50)
        print(f"采样点数: {len(self.history)}")
        print(f"采集时长: {self.history[-1]['timestamp'] - self.history[0]['timestamp']:.1f}秒")
        print()
        
        print(f"{'传感器':<20} {'平均(°C)':<12} {'最大(°C)':<12} {'最小(°C)':<12} {'方差':<12}")
        print(f"{'-'*20} {'-'*12} {'-'*12} {'-'*12} {'-'*12}")
        
        all_temps = []
        for sensor_key, temps in sorted(type_data.items()):
            if temps:
                avg = sum(temps) / len(temps)
                mx = max(temps)
                mn = min(temps)
                var = sum((t - avg) ** 2 for t in temps) / len(temps)
                all_temps.extend(temps)
                print(f"{sensor_key:<20} {avg:<12.1f} {mx:<12.1f} {mn:<12.1f} {var:<12.2f}")
        
        温差异常检测
        print()
        print("温差异常检测:")
        if len(type_data) >= 2:
            max_type = max(type_data.keys(), key=lambda k: max(type_data[k]))
            min_type = min(type_data.keys(), key=lambda k: max(type_data[k]))
            max_avg = sum(type_data[max_type]) / len(type_data[max_type])
            min_avg = sum(type_data[min_type]) / len(type_data[min_type])
            delta = max_avg - min_avg
            print(f"  最高温传感器: {max_type} ({max_avg:.1f}°C)")
            print(f"  最低温传感器: {min_type} ({min_avg:.1f}°C)")
            print(f"  温差: {delta:.1f}°C")
            if delta > 20:
                print(f"  ⚠️ 警告: 温差超过20°C，可能存在散热不均匀")
            else:
                print(f"  ✓ 温差在正常范围内")
        
        传感器健康状态
        print()
        print("传感器健康状态:")
        for sensor_key, temps in sorted(type_data.items()):
            mx = max(temps)
            mn = min(temps)
            label = sensor_key
            检查是否接近临界阈值
            for sid, info in self.sensors.items():
                if info["type"] in sensor_key:
                    if info.get("crit") and mx >= info["crit"] * 0.95:
                        print(f"  ⚠️ {label}: 温度({mx:.1f}°C)已接近临界阈值({info['crit']}°C)")
                    elif info.get("max") and mx >= info["max"] * 0.95:
                        print(f"  ⚠️ {label}: 温度({mx:.1f}°C)已接近最大阈值({info['max']}°C)")
                    else:
                        print(f"  ✓ {label}: 正常")
        
        导出数据
        csv_file = "gpu_temperature_analysis.csv"
        with open(csv_file, 'w', newline='') as f:
            if self.history:
                writer = csv.DictWriter(f, fieldnames=self.history[0].keys())
                writer.writeheader()
                writer.writerows(self.history)
        print(f"\n数据已导出至 {csv_file}")

if __name__ == "__main__":
    analyzer = GPUTemperatureAnalyzer()
    analyzer.analyze(duration_sec=30, interval_sec=1)

   - SMU 消息队列拥堵，温度查询请求被延迟处理

   - I2C/SBI 总线上的温度传感器通信超时
   - 内核驱动中的温度读取函数缺少超时处理机制

2. 修复方案：

   内核驱动修复 - 增加SMU通信超时处理
   static int amdgpu_smu_read_temp(struct smu_context *smu, int *temp)
   {
       int ret;
       uint32_t val;
       unsigned long timeout;
       
       设置超时（500ms）
       timeout = jiffies + msecs_to_jiffies(500);
       
       ret = smu_get_pptable_entry(smu, &val);
       while (ret == -EBUSY && time_before(jiffies, timeout)) {
           msleep(10);
           ret = smu_get_pptable_entry(smu, &val);
       }
       
       if (ret == -EBUSY) {
           SMU通信超时，返回上次有效值
           *temp = smu->last_valid_temp;
           dev_warn(smu->dev, "SMU temp read timeout, using cached value\n");
           return 0;
       }
       
       *temp = val;
       smu->last_valid_temp = val;
       return ret;
   }
   ```

### 案例 4：Linux Thermal 子系统与 amdgpu 驱动的温度冲突

**场景**：
在 kernel 6.5 系统上同时启用了 `thermald` 服务和 `amdgpu` 驱动的内置热管理。用户发现 GPU 性能不稳定：3D 渲染帧率在 30-60 FPS 之间大幅波动，且 GPU 利用率显示在 40%-95% 间跳动。

**问题**：
1. 分析 thermald 和 amdgpu 内置热管理可能存在的冲突（3分）
2. 提出解决方案（2分）

**参考答案**：

1. 冲突分析：
   - thermald 通过 thermal_zone 接口监控 GPU 温度，并尝试通过 intel_pstate/ACPI 方法降低温度
   - amdgpu 驱动的内置热管理（PowerPlay）独立通过 SMU 控制频率/电压
   - 两个管理实体互不知晓对方，形成调节竞争：amdgpu 试图升频提高性能，thermald 试图降频降低温度
   - 导致频率在目标值附近来回振荡，表现为帧率剧烈波动
   - PID 控制回路冲突：两个独立的 PID 控制器作用于同一被控对象（GPU频率）

2. 解决方案：
   ```bash
   方案A：禁用thermald对GPU的控制（推荐）
   # 编辑 /etc/thermald/thermal-conf.xml
   # 排除GPU thermal zone
   cat > /etc/thermald/thermal-conf.xml << 'EOF'
   <?xml version="1.0"?>
   <ThermalConfiguration>
     <Platform>
       <Exclude>thermal_zone*</Exclude>
     </Platform>
   </ThermalConfiguration>
   EOF
   systemctl restart thermald
   
   方案B：仅使用amdgpu内置热管理（禁用thermald）
   systemctl stop thermald
   systemctl disable thermald

static int smu_v13_0_get_thermal_temperature_range(struct smu_context *smu,
                                                    uint32_t *min_t,
                                                    uint32_t *max_t)
{
    struct smu_table_context *table_context = &smu->smu_table;
    struct smu_13_0_powerplay_table *powerplay_table =

    int ret = 0;

    if (!powerplay_table)
        return -EINVAL;

    thermal_table = &powerplay_table->thermal_table;

    if (min_t)
        *min_t = thermal_table->min_temperature;
    if (max_t)
        *max_t = thermal_table->max_temperature;

    if (!smu_v13_0_is_primary(smu))
        return 0;

    ret = smu_v13_0_init_thermal_temperature_table(smu);
    if (ret)
        return ret;

    if (max_t) {
        uint32_t hot_spot_limit = smu->thermal_range.hot_spot_limit;
        uint32_t mem_hot_limit = smu->thermal_range.mem_hot_limit;

        if (hot_spot_limit && (hot_spot_limit < *max_t))
            *max_t = hot_spot_limit;
        if (mem_hot_limit && (mem_hot_limit > *max_t))
            *max_t = mem_hot_limit;
        if (vrg_temp_limit && (vrg_temp_limit < *max_t))
            *max_t = vrg_temp_limit;
    }

```

**问题**：
1. 描述此函数的功能和返回值含义（1分）
2. 解释 `hot_spot_limit`、`mem_hot_limit`、`vrg_temp_limit` 三者与 `max_t` 的关系逻辑（2分）
3. 如果 `hot_spot_limit = 95000`、`mem_hot_limit = 105000`、`vrg_temp_limit = 90000`，且 PowerPlay 表中 `max_t = 100000`，最终 `max_t` 是多少？为什么？（2分）

**参考答案**：

1. 函数功能：获取 SMU v13.0 的热温度范围。`min_t` 输出最低温度（通常为 0°C 或常温），`max_t` 输出最高允许温度（热节流触发点）。返回 0 表示成功，负值表示错误。

2. 逻辑关系：三个限制值对 `*max_t` 的影响：
   - `hot_spot_limit`：热点温度限制，如果小于当前 `*max_t`，则降低 `*max_t`（收紧限制）
   - `mem_hot_limit`：显存热点限制，如果大于当前 `*max_t`，则提高 `*max_t`（放宽限制）
   - `vrg_temp_limit`：VRG（电压调节器）温度限制，如果小于当前 `*max_t`，则降低 `*max_t`
   
   最终 `*max_t` 取 PowerPlay 表初始值经过三个条件调整后的结果，实际取三者中的中间值（受 hot_spot 和 vrg 向下调整，mem_hot 向上调整）。

3. 计算过程：
   - 初始 `*max_t = 100000`（来自 PowerPlay 表）
   - hot_spot_limit=95000 < 100000，所以 `*max_t = 95000`
   - mem_hot_limit=105000 > 95000，所以 `*max_t = 105000`
   - vrg_temp_limit=90000 < 105000，所以 `*max_t = 90000`
   - 最终 `*max_t = 90000`（90°C）
   
   逻辑是先收窄再放宽，最终取最小值和最大值的综合结果，实际代码中的先后顺序导致最终受最后一步影响。

### 题目 2：节流检测与恢复逻辑分析

分析以下伪代码片段，回答问题：

```python
def thermal_throttle_manager(current_temp, current_freq, temp_history):
    THROTTLE_TEMP = 95
    THROTTLE_RECOVER = 85
    HYSTERESIS_COUNT = 3
    
    # 状态变量（全局）
    # throttle_state: 0=正常, 1=节流中
    # throttle_counter: 连续超过阈值的计数
    
    if current_temp >= THROTTLE_TEMP:
        throttle_counter += 1
        if throttle_counter >= HYSTERESIS_COUNT:
            throttle_state = 1
            # 节流：降低频率
            new_freq = current_freq * 0.8
            throttle_counter = HYSTERESIS_COUNT
    elif current_temp <= THROTTLE_RECOVER:
        if throttle_state == 1:
            throttle_counter -= 1
            if throttle_counter <= 0:
                throttle_state = 0
                # 恢复：逐步提升频率
                new_freq = min(current_freq * 1.05, MAX_FREQ)
                throttle_counter = 0
    else:
        # 处于迟滞区间
        if throttle_state == 1:
            new_freq = current_freq * 0.95
    
    temp_history.append(current_temp)
    if len(temp_history) > 100:
        temp_history.pop(0)
    
    return new_freq, throttle_state
```

**问题**：
1. 指出此算法中的迟滞设计及其作用（1分）
2. 分析在 `current_temp` 稳定在 88°C ± 2°C 范围内时，系统行为会如何？（2分）
3. 提出至少两点改进建议（2分）

**参考答案**：

1. 迟滞设计：使用两个不同的阈值入口（THROTTLE_TEMP=95°C）和出口（THROTTLE_RECOVER=85°C），形成 10°C 的迟滞窗口。同时使用 HYSTERESIS_COUNT=3 的计数迟滞，防止单次瞬时尖峰触发节流。

2. 当温度稳定在 88±2°C 范围时：
   - 温度可能在 86-90°C 范围内波动
   - 由于 THROTTLE_RECOVER=85°C，只有在温度降到 85°C 以下时才开始恢复计数
   - 当 throttle_state=1 时，处于 85-95°C 之间的迟滞区间，频率会持续以 0.95 倍递减
   - 这会导致频率不断下降，直到温度降到 85°C 以下才开始恢复
   - 恢复后温度可能再次上升，形成频率/温度振荡
   - 该温度区间落入迟滞区间的陷阱，导致持续的频率衰减

3. 改进建议：
   - 增加恢复路径的迟滞计数逻辑：当温度低于 THROTTLE_RECOVER 时，也需要连续 HYSTERESIS_COUNT 次才解除节流，与进入节流的对称设计
   - 引入 PID 控制代替阶跃响应：在迟滞区间使用比例控制，频率调整量与温度超过目标值的程度成正比
   - 添加最大节流深度限制：限制频率最低降至 INITIAL_FREQ * 0.5，避免过度降频
   - 使用指数移动平均滤波：`smooth_temp = alpha * current_temp + (1-alpha) * smooth_temp`，减少噪声影响

#### 2. PID 风扇控制器（6分）

实现基于温度的 PID 风扇控制器，目标温度 70°C，输出范围为 0-255（PWM 值）。

```python
class FanPIDController:
    PID风扇控制器
    def __init__(self, kp=1.5, ki=0.3, kd=0.1):
        self.kp = kp
        self.ki = ki
        self.kd = kd
        self.setpoint = 70.0
        self.integral = 0.0
        self.prev_error = 0.0
        self.prev_time = None
        self.output_min = 0
        self.output_max = 255
        self.windup_limit = 100
    
    def compute(self, current_temp):
        now = time.time()
        dt = (now - self.prev_time) if self.prev_time else 0.1
        self.prev_time = now
        
        error = current_temp - self.setpoint
        
        P = self.kp * error
        
        self.integral += error * dt
        self.integral = max(-self.windup_limit, min(self.windup_limit, self.integral))
        I = self.ki * self.integral
        
        derivative = (error - self.prev_error) / dt if dt > 0 else 0
        D = self.kd * derivative
        
        self.prev_error = error
        
        output = P + I + D
        output = max(self.output_min, min(self.output_max, output))
        
        return int(output), {"P": P, "I": I, "D": D, "error": error}
```

#### 3. 预判式节流预防（5分）

```python
class PredictiveThrottlePreventer:
    预判式节流预防模块
    def __init__(self, history_window=30):
        self.temp_history = []
        self.freq_history = []
        self.window = history_window
        self.throttle_temp = 95.0
        self.safety_margin = 5.0
    
    def predict_temperature(self, seconds_ahead=10):
        使用线性回归预测未来温度
        if len(self.temp_history) < 10:
            return None
        
        recent = self.temp_history[-20:]
        n = len(recent)
        x = list(range(n))
        y = recent
        
        x_mean = sum(x) / n
        y_mean = sum(y) / n
        
        numerator = sum((x[i] - x_mean) * (y[i] - y_mean) for i in range(n))
        denominator = sum((x[i] - x_mean) ** 2 for i in range(n))
        
        if denominator == 0:
            return None
        
        slope = numerator / denominator
        intercept = y_mean - slope * x_mean
        
        predicted = intercept + slope * (n + seconds_ahead)
        return predicted, slope
    
    def analyze(self, current_temp):
        self.temp_history.append(current_temp)
        if len(self.temp_history) > self.window:
            self.temp_history.pop(0)
        
        prediction = self.predict_temperature(10)
        if prediction is None:
            return 0.0
        
        predicted_temp, slope = prediction
        
        risk = 0.0
        if predicted_temp > self.throttle_temp - self.safety_margin:
            risk = min(1.0, (predicted_temp - (self.throttle_temp - self.safety_margin)) / self.safety_margin)
        
        return risk
    
    def act(self):
        执行预防动作：通知驱动降频或调整PPT
        risk = self.analyze(self.temp_history[-1] if self.temp_history else 0)
        if risk > 0.8:
            紧急：强制降频
            freq_reduction = int(risk * 200)
            with open("/sys/class/drm/card0/device/pp_od_clk_voltage", "w") as f:
                f.write(f"s 0 {CURRENT_FREQ - freq_reduction}\n")
                f.write("c\n")
            return True
        return False
```

## 附录D：答题要点与常见错误分析

### D.1 理论选择题易错题解析

**第3题（热传感器类型）常见错误**：
- 误选"VRAM温度传感器直接控制风扇" — 实际风扇策略由edge和hotspot温度共同决定，VRAM仅为监控用途
- 误选"所有GPU都有相同的传感器布局" — Navi21/RDNA2有5个传感器，Vega20不同，RDNA3增加VRM传感器

**第7题（热节流触发条件）常见错误**：
- 只答"超过温度阈值"而忽略"持续时间"因素 — 实际需要软硬件双重触发：硬件TDP限流+软件3-count滤波器
- 混淆`temp*_crit`（硬件关机）和`temp*_max`（软件节流起点）

**第9题（PID控制）常见错误**：
- 认为KP越高越好 — 实际高KP导致风扇转速剧烈波动（oscillation）
- 忽略积分限幅（windup limit）的重要性 — 无限幅时积分项无限累积导致"积分饱和"

### D.2 实践操作题常见失误

**脚本题（题目1）**：
- 未检查hwmon路径是否存在 — 不同GPU型号路径不同（hwmon0/hwmon1/hwmon2）
- 温度单位混淆 — 内核返回毫摄氏度（millidegree），需除以1000转换为摄氏度
- 采样间隔不合理 — 1秒间隔产生过多数据；5-10秒间隔足够且减少日志体积
- 忘记处理glmark2/glxgears未安装的情况

**GPUTemperatureAnalyzer（题目2）**：
- 未处理多GPU场景 — 系统中有集成显卡+独立显卡时会有多个hwmon目录
- 忽略sensor发现失败时的降级策略 — 至少应报告错误而非直接崩溃
- CSV导出未做转义 — 温度标签中可能含逗号

### D.3 案例分析的评分要点

- **案例1（数据中心温度异常）**：必须提及"机柜气流"因素，仅关注传感器本身得一半分
- **案例2（风扇噪音投诉）**：需同时给出短期（手动调曲线）和长期（RMA/更换）方案
- **案例4（thermald冲突）**：关键洞察是"双PID冲突" — thermald的PID和amdgpu风扇PID互相抵消
- **案例5（浸没式冷却）**：核心考点是"温度梯度差异" — 浸没液导热率高但传感器响应特性不同
