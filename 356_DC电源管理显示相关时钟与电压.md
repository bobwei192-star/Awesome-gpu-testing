# Day 356: DC 电源管理：显示相关时钟与电压

## 1. DC 电源管理概述

AMD Display Core (DC) 的电源管理是现代 GPU 驱动中最复杂也最关键的部分之一。显示子系统虽然有固定的刷新率（通常 60Hz），但现代 GPU 通过精细的时钟管理、动态频率调整和电源门控技术，在保持显示输出的同时最大化地节省功耗。

### 1.1 显示电源管理的挑战

显示子系统相比计算单元有独特的功耗管理需求：

```c
// 显示电源管理的核心约束
struct display_power_constraints {
    // 实时性要求：显示输出不能中断，否则用户看到闪烁或黑屏
    bool requires_continuous_display;

    // 刷新率固定：即使画面静止，也必须按 VBlank 节奏刷新
    uint32_t fixed_refresh_rate_hz;

    // 多个显示输出可能同时工作
    uint32_t active_display_count;

    // 不同显示模式对时钟频率有不同要求
    uint32_t max_resolution_width;
    uint32_t max_resolution_height;
    uint32_t color_depth_bits;

    // 节能技术如 PSR/ABM 可在保持显示的同时降低功耗
    bool psr_supported;
    bool abm_supported;
};
```

### 1.2 电源管理层次结构

DC 的电源管理从上到下分为多个层次：

```
+--------------------------------------------------+
|  用户空间 (用户设置 /sys/class/drm/ 接口)         |
+--------------------------------------------------+
|  DRM/KMS 框架 (atomic check/commit)              |
+--------------------------------------------------+
|  AMDGPU DM (Display Manager)                     |
|  - amdgpu_dm.c (atomic 入口)                     |
|  - 电源状态决策                                  |
+--------------------------------------------------+
|  DC (Display Core)                               |
|  - dc_clocks 结构 (时钟需求收集)                 |
|  - clk_mgr (时钟管理器)                          |
|  - dccg (显示时钟发生器)                         |
|  - clock_source (PLL 时钟源)                     |
+--------------------------------------------------+
|  SMU (System Management Unit)                     |
|  - 实际电压/频率控制                              |
|  - PP (PowerPlay) 接口                            |
+--------------------------------------------------+
|  硬件 (DCN / 寄存器)                             |
+--------------------------------------------------+
```

## 2. 核心数据结构：dc_clocks

`struct dc_clocks` 是整个显示子系统的时钟状态中枢，定义在 [dc.h](file:///usr/src/linux-source-5.15.0/drivers/gpu/drm/amd/display/dc/dc.h) 中：

```c
// dc.h -- 全局时钟状态结构
struct dc_clocks {
    // --- 核心显示时钟 ---

    // DISPCLK: Display Engine Clock (显示引擎时钟)
    // 驱动 DCN 中大部分显示逻辑，包括 OTG、OPTC、DSC 等模块
    // 频率范围通常为 300MHz ~ 1200MHz (取决于 DCN 代际)
    int dispclk_khz;
    int actual_dispclk_khz;  // 硬件实际设置的 DISPCLK 值

    // DPPCLK: Data Pipe Pixel Clock (数据管道像素时钟)
    // 每个 DPP 管道独立，通过 DTO 分频器从全局时钟派生
    // 频率通常 ≤ DISPCLK
    int dppclk_khz;
    int actual_dppclk_khz;

    // DISPCLK 和 DPPCLK 对应的电压等级
    int disp_dpp_voltage_level_khz;

    // --- 数据互连时钟 ---

    // DCFCLK: Data Clock Fabric Clock
    // DCN 内部 data fabric 互连的时钟频率
    int dcfclk_khz;

    // DCFCLK 深度睡眠频率 (低功耗模式)
    int dcfclk_deep_sleep_khz;

    // --- SoC 级时钟 ---

    // SOCCLK: System on Chip Clock
    int socclk_khz;

    // FCLK: Fabric Clock (与 DPPCLK 共享同一物理时钟源)
    int fclk_khz;

    // --- 物理接口时钟 ---

    // PHYCLK: PHY Clock (物理层时钟)
    int phyclk_khz;

    // --- 显存时钟 ---

    // DRAMCLK: DRAM Clock (显存时钟)
    int dramclk_khz;

    // --- 电源状态控制 ---

    // P-State 变化支持标志
    // P-State 是 CPU/GPU 的电源状态变化，显示子系统必须声明是否允许
    // 在高分辨率/多显示器场景下通常需要禁止 P-State 变化
    bool p_state_change_support;

    // Z-state 支持 (DCN 深度睡眠)
    enum dcn_zstate_support_state zstate_support;

    // DTB 时钟使能 (Display Timing Buffer)
    bool dtbclk_en;

    // 当前电源状态
    enum dcn_pwr_state pwr_state;

    // 前一次 P-State 支持状态 (用于状态变化检测)
    bool prev_p_state_change_support;

    // DTM (Display Timing Manager) P-State 等级
    enum dtm_pstate dtm_level;

    // --- 最大值记录 ---

    // 对于可能跨管道的时钟，这里存储最大值
    int max_supported_dppclk_khz;
    int max_supported_dispclk_khz;

    // 带宽计算用的时钟值副本
    int bw_dppclk_khz;
    int bw_dispclk_khz;
};
```

### 4.3 DPP DTO 分频机制

DPPCLK 通过 DTO (Digital Timing Oscillator) 从 DISPCLK 分频获得。每个 DPP 管道有独立的 DTO，使得未使用的管道可以运行在更低的频率。

```c
// dcn21_dccg.c -- DCN2.1 DPP DTO 实现
void dcn21_update_dpp_dto(struct dccg *dccg, int dpp_inst,
        int req_dppclk)
{
    struct dcn_dccg *dccg_dcn = TO_DCN_DCCG(dccg);

    // 计算 DTO 相位和模数
    // phase = ceil(dpp_pipe_clk_mhz / 10)
    // module = trunc(dpp_global_clk_mhz / 10)
    //
    // DTO 的工作原理：
    //   输出频率 = (phase / module) * 输入频率
    //   当 phase = module 时，输出 = 输入 (不分频)
    //   当 phase = module/2 时，输出 = 输入/2 (2分频)
    uint32_t phase = (req_dppclk + 5000) / 10000;  // 四舍五入
    uint32_t module = dccg_dcn->base.ref_dppclk / 10000;

    // 写入 DPPCLK_DTO_PARAM 寄存器
    REG_WRITE(DPPCLK_DTO_PARAM[dpp_inst],
            (phase << DPPCLK_DTO_PHASE_SHIFT) |
            (module << DPPCLK_DTO_MODULO_SHIFT));

    // 使能 DTO
    REG_UPDATE(DPPCLK_DTO_CTRL,
            DPPCLK_DTO_ENABLE[dpp_inst], 1);

    // 更新缓存值
    dccg->pipe_dppclk_khz[dpp_inst] = req_dppclk;
}
```

## 6. 各时钟域详解

### 6.1 DISPCLK (Display Engine Clock)

DISPCLK 是显示引擎的主时钟，驱动 DCN 中的大部分模块。它的频率直接影响显示分辨率、刷新率和色彩深度。

```
DISPCLK 驱动的模块:
+------------------+
|  OTG (输出定时器) |
+------------------+
|  OPTC (优化器)    |
+------------------+
|  DSC 编码器       |
+------------------+
|  DWB (显示回写)   |
+------------------+
|  MPC (多流水线)   |
+------------------+
```

DISPCLK 频率计算：

```c
// DISPCLK 需求计算
static int calculate_dispclk_required(
        const struct dc_stream_state *stream)
{
    // 基础需求 = 像素时钟
    int required = stream->timing.pix_clk_100hz / 100;

    // 考虑 ODM 组合 (减少所需频率)
    // ODM 2:1: 每个管道处理一半像素，所需时钟减半
    if (stream->timing.odm_combine == ODM_COMBINE_MODE_2TO1)
        required /= 2;
    if (stream->timing.odm_combine == ODM_COMBINE_MODE_4TO1)
        required /= 4;

    // 考虑 DSC (Display Stream Compression)
    // DSC 可以减少所需带宽，但增加 DISPCLK 负载
    if (stream->timing.flags.DSC) {
        required = required * 120 / 100;  // +20% 用于 DSC 处理开销
    }

    // 考虑 DWB 开销 (如果有回写流水线)
    if (stream->dwb_pipe_inst >= 0) {
        required = required * 110 / 100;  // +10%
    }

    return required;
}
```

### 6.2 DPPCLK (Data Pipe Pixel Clock)

DPPCLK 是每个 DPP (Display Pipe Processor) 管道的时钟。DPP 负责缩放、色彩校正、格式转换等处理。

```c
// DPPCLK 需求计算
static int calculate_dppclk_required(
        const struct dc_plane_state *plane)
{
    // DPPCLK 主要取决于缩放因子
    // 硬缩放 (downscale) 需要更高的 DPP 时钟
    int src_width = plane->src_rect.width;
    int dst_width = plane->dst_rect.width;

    if (src_width <= dst_width) {
        // 放大或 1:1: DPPCLK 需求较低
        return MIN_DPPCLK_KHZ;  // 例如 100 MHz
    } else {
        // 缩小: 需要更高的 DPPCLK 来处理像素过量
        return MIN_DPPCLK_KHZ +
               (src_width - dst_width) * 10000;
    }
}
```

### 6.3 DCFCLK (Data Clock Fabric Clock)

DCFCLK 驱动 DCN 内部的 data fabric 互连。它决定了显存和显示引擎之间的数据带宽。

```c
// DCFCLK 需求计算
static int calculate_dcfclk_required(
        const struct dc_state *context)
{
    int max_dcfclk = 0;

    // 遍历所有管道，计算数据带宽需求
    for (int i = 0; i < context->stream_count; i++) {
        const struct dc_stream_state *stream =
            context->streams[i];

        // 计算每个流所需的带宽
        int bpp = get_display_bpp(stream);     // bits per pixel
        int h_total = stream->timing.h_total;  // 水平总像素
        int v_total = stream->timing.v_total;  // 垂直总行
        int refresh = stream->timing.refresh_rate; // 刷新率

        // 带宽 = 分辨率 * BPP * 刷新率
        int bandwidth_bps = h_total * v_total *
                refresh * bpp;

        // data fabric 时钟需求
        // DCFCLK 必须足够快以支持所有流的总带宽
        int required_dcfclk = bandwidth_bps /
                DCFCLK_EFFICIENCY_FACTOR;

        max_dcfclk = max(max_dcfclk, required_dcfclk);
    }

    return max_dcfclk;
}
```

### 6.4 PHYCLK (Physical Layer Clock)

PHYCLK 驱动 DP 或 HDMI 的物理层（PHY）。它与链路速率直接相关。

```
PHYCLK 与链路速率的关系:
DP 1.4 HBR3: 链路速率 8.1 Gbps/通道
  PHYCLK = 8.1 GHz / 10 = 810 MHz (10-bit 编码)

DP 1.4 HBR2: 链路速率 5.4 Gbps/通道
  PHYCLK = 5.4 GHz / 10 = 540 MHz

HDMI 2.1 FRL: 每通道 12 Gbps
  PHYCLK = 12 GHz / 40 = 300 MHz (40-bit 编码)
```

### 6.5 PIXCLK (Pixel Clock)

像素时钟直接决定显示输出时序。它由 CRT 定时参数计算：

```c
static uint32_t calculate_pixel_clk(
        const struct dc_crtc_timing *timing)
{
    // 像素时钟 = h_total * v_total * refresh_rate
    uint32_t pix_clk = timing->h_total *
                       timing->v_total *
                       timing->refresh_rate;

    // 考虑隔行扫描 (实际场频减半)
    if (timing->flags.INTERLACE) {
        pix_clk /= 2;
    }

    return pix_clk;  // Hz
}
```

## 7. 时钟更新流程

### 7.1 DCN2 实现 (dcn2_update_clocks)

```c
// dcn20_clk_mgr.c -- DCN2 时钟更新核心函数
void dcn2_update_clocks(struct clk_mgr *clk_mgr_base,
        struct dc_state *context, bool safe_to_lower)
{
    struct clk_mgr_internal *clk_mgr =
        TO_CLK_MGR_INTERNAL(clk_mgr_base);
    struct dc_clocks *new_clocks = &context->bw_ctx.bw.dcn.clk;
    struct dc *dc = clk_mgr_base->ctx->dc;

    // 第1步: 计算所有所需时钟
    // 这已经由带宽计算 (bw_calc) 完成
    // new_clocks 中包含了所有需求的时钟频率

    // 第2步: 检查是否允许降低时钟
    // safe_to_lower = false 时，只能升高不能降低
    // 例如刚完成页面翻转后不允许降低
    if (!should_set_clock(safe_to_lower,
            new_clocks->dcfclk_deep_sleep_khz,
            clk_mgr_base->clks.dcfclk_deep_sleep_khz))
        goto update_dcfclk;

    // 第3步: 更新 DCFCLK 深度睡眠频率
    clk_mgr_base->clks.dcfclk_deep_sleep_khz =
        new_clocks->dcfclk_deep_sleep_khz;
    // 设置 DCFCLK 深度睡眠 (通过 SMU)
    if (clk_mgr_base->clks.dcfclk_deep_sleep_khz > 0)
        rn_vbios_smu_set_dcfclk_deep_sleep(clk_mgr,
            clk_mgr_base->clks.dcfclk_deep_sleep_khz);

update_dcfclk:
    // 第4步: 更新 DCFCLK
    if (!should_set_clock(safe_to_lower,
            new_clocks->dcfclk_khz,
            clk_mgr_base->clks.dcfclk_khz))
        goto update_soc;

    clk_mgr_base->clks.dcfclk_khz =
        new_clocks->dcfclk_khz;
    // 通过 SMU 设置 DCFCLK
    rn_vbios_smu_set_hard_min_dcfclk(clk_mgr,
        clk_mgr_base->clks.dcfclk_khz);

update_soc:
    // 第5步: 更新 SOCCLK
    if (!should_set_clock(safe_to_lower,
            new_clocks->socclk_khz,
            clk_mgr_base->clks.socclk_khz))
        goto update_dram;

    clk_mgr_base->clks.socclk_khz =
        new_clocks->socclk_khz;
    rn_vbios_smu_set_min_socclk(clk_mgr,
        clk_mgr_base->clks.socclk_khz);

update_dram:
    // 第6步: 更新 DRAMCLK
    if (!should_set_clock(safe_to_lower,
            new_clocks->dramclk_khz,
            clk_mgr_base->clks.dramclk_khz))
        goto update_dpp;

    clk_mgr_base->clks.dramclk_khz =
        new_clocks->dramclk_khz;
    rn_vbios_smu_set_min_dram_clk(clk_mgr,
        clk_mgr_base->clks.dramclk_khz);

update_dpp:
    // 第7步: 处理 DPPCLK 和 FCLK 的约束关系
    // FCLK 和 DPPCLK 运行在同一物理时钟上
    // 两者必须取较高值
    if (clk_mgr_base->clks.fclk_khz >
            clk_mgr_base->clks.dppclk_khz)
        clk_mgr_base->clks.dppclk_khz =
            clk_mgr_base->clks.fclk_khz;
    if (clk_mgr_base->clks.dppclk_khz >
            clk_mgr_base->clks.fclk_khz)
        clk_mgr_base->clks.fclk_khz =
            clk_mgr_base->clks.dppclk_khz;

    // 更新 DPPCLK
    if (!should_set_clock(safe_to_lower,
            new_clocks->dppclk_khz,
            clk_mgr_base->clks.dppclk_khz))
        goto update_fclk;

    clk_mgr_base->clks.dppclk_khz =
        new_clocks->dppclk_khz;
    rn_vbios_smu_set_min_dppclk(clk_mgr,
        clk_mgr_base->clks.dppclk_khz);

update_fclk:
    // 更新 FCLK
    if (!should_set_clock(safe_to_lower,
            new_clocks->fclk_khz,
            clk_mgr_base->clks.fclk_khz)) {
        // 即使 FCLK 没有变化，也可能因为 DPPCLK 约束
        // 需要重新同步
        goto update_disp;
    }

    clk_mgr_base->clks.fclk_khz =
        new_clocks->fclk_khz;
    rn_vbios_smu_set_fclk(clk_mgr,
        clk_mgr_base->clks.fclk_khz);

update_disp:
    // 第8步: 更新 DISPCLK (显示引擎主时钟)
    if (!should_set_clock(safe_to_lower,
            new_clocks->dispclk_khz,
            clk_mgr_base->clks.dispclk_khz))
        goto final;

    clk_mgr_base->clks.dispclk_khz =
        new_clocks->dispclk_khz;
    rn_vbios_smu_set_min_dispclk(clk_mgr,
        clk_mgr_base->clks.dispclk_khz);

final:
    // 第9步: 更新电压等级
    // 电压等级由所有时钟中的最高需求决定
    int required_voltage = max(
        clk_mgr_base->clks.dispclk_khz,
        max(clk_mgr_base->clks.dcfclk_khz,
            clk_mgr_base->clks.dppclk_khz));
    // 通过 SMU 设置电压
    rn_vbios_smu_set_voltage(clk_mgr, required_voltage);

    // 第10步: 更新 P-State 支持
    bool p_state_support =
        new_clocks->p_state_change_support;
    if (p_state_support !=
            clk_mgr_base->clks.p_state_change_support) {
        clk_mgr_base->clks.p_state_change_support =
            p_state_support;
        // 通知 SMU (允许/禁止 P-State 变化)
        rn_vbios_smu_set_pstate(clk_mgr, p_state_support);
    }
}
```

### 7.2 should_set_clock 辅助函数

```c
// 判断是否应该设置新的时钟频率
static bool should_set_clock(bool safe_to_lower,
        int new_clock, int old_clock)
{
    // 如果新频率比旧频率低，但系统不允许降低
    // (safe_to_lower = false)，则跳过
    if (!safe_to_lower && new_clock < old_clock)
        return false;

    // 如果频率相同，跳过 (避免不必要的 SMU 通信)
    if (new_clock == old_clock)
        return false;

    return true;
}
```

### 7.3 从硬件读取当前时钟

```c
// dcn20_clk_mgr.c -- 从 Dentist 读取硬件时钟值
static void dcn2_read_clocks_from_hw_dentist(
        struct clk_mgr *clk_mgr_base)
{
    struct clk_mgr_internal *clk_mgr =
        TO_CLK_MGR_INTERNAL(clk_mgr_base);

    // 读取 Dentist DISPCLK 控制寄存器
    uint32_t dentist_dispclk_cntl = REG_READ(DENTIST_DISPCLK_CNTL);

    // 提取 DISPCLK 分频比
    uint32_t dispclk_wdivider = REG_GET(DENTIST_DISPCLK_CNTL,
            DENTIST_DISPCLK_WDIVIDER, dispclk_wdivider);

    // 提取 DPPCLK 分频比
    uint32_t dppclk_wdivider = REG_GET(DENTIST_DISPCLK_CNTL,
            DENTIST_DPPCLK_WDIVIDER, dppclk_wdivider);

    // 计算实际频率
    // Dentist VCO 频率存储在 clk_mgr 中
    uint32_t vco_freq = clk_mgr_base->dentist_vco_freq_khz;

    // DISPCLK = VCO / (WDIVIDER + 1)
    clk_mgr_base->clks.dispclk_khz =
        vco_freq / (dispclk_wdivider + 1);

    // DPPCLK = VCO / (WDIVIDER + 1)
    clk_mgr_base->clks.dppclk_khz =
        vco_freq / (dppclk_wdivider + 1);
}
```

### 9.3 P-State 变化控制

P-State (Performance State) 变化是 CPU/GPU 核心电压和频率的变化。当 P-State 变化时，显存访问可能被短暂中断。显示子系统必须声明 P-State 变化是否被允许。

```c
// P-State 支持决策逻辑
static bool decide_p_state_support(
        const struct dc_state *context)
{
    // 规则1: 如果单流带宽超过阈值，禁止 P-State 变化
    for (int i = 0; i < context->stream_count; i++) {
        const struct dc_stream_state *stream =
            context->streams[i];

        uint64_t bandwidth = calculate_stream_bandwidth(
            stream);

        // 如果带宽超过 4K@60 的 15%，禁止 P-State
        if (bandwidth > BANDWIDTH_4K_60HZ_115PCT)
            return false;
    }

    // 规则2: 如果总带宽超标，禁止 P-State
    uint64_t total_bandwidth = 0;
    for (int i = 0; i < context->stream_count; i++) {
        total_bandwidth += calculate_stream_bandwidth(
            context->streams[i]);
    }
    if (total_bandwidth > MAX_BANDWIDTH_FOR_PSTATE)
        return false;

    // 规则3: 如果使用 DSC，P-State 可能更受限制
    for (int i = 0; i < context->stream_count; i++) {
        if (context->streams[i]->timing.flags.DSC) {
            // DSC 使用了额外的缓冲，可能放松限制
            // 但需要查具体硬件规格
        }
    }

    return true;  // 默认允许 P-State 变化
}
```

## 10. 时钟门控 (Clock Gating)

### 10.1 什么是时钟门控

时钟门控 (Clock Gating) 是在电路级别上，当某个模块不工作时，通过关闭其时钟来节省动态功耗的技术。

```
时钟门控示意图:

    输入时钟 ----+----[与门]----> 模块时钟
                 |      ^
                 |      |
                 +----[ 使能信号 ]

当使能信号 = 0 时，模块时钟 = 0 (静态)
当使能信号 = 1 时，模块时钟 = 输入时钟 (正常)
```

### 10.2 DCN 中的时钟门控

```c
// DCN 时钟门控控制
struct dcn_clock_gating {
    // 各个 IP 模块的时钟门控使能
    bool otg_clk_gating_enabled;     // OTG 时钟门控
    bool dpp_clk_gating_enabled;     // DPP 时钟门控
    bool mpc_clk_gating_enabled;     // MPC 时钟门控
    bool opp_clk_gating_enabled;     // OPP 时钟门控
    bool dsc_clk_gating_enabled;     // DSC 时钟门控
    bool hubp_clk_gating_enabled;    // HUBP 时钟门控
    bool dchub_clk_gating_enabled;   // DCHUB 时钟门控
};

// 设置时钟门控
void dcn_set_clock_gating(
        struct dcn_clock_gating *gating,
        struct dc_context *ctx)
{
    // OTG 时钟门控
    // 如果 OTG 空闲 (无有效时序)，关闭其时钟
    if (!is_otg_active(ctx)) {
        REG_UPDATE(OTG_CLOCK_CONTROL,
                OTG_CLOCK_GATE_DIS, 0);  // 0 = 使能门控
    }

    // DPP 时钟门控
    // 如果 DPP 管道未使用，关闭其时钟
    for (int i = 0; i < MAX_PIPES; i++) {
        if (!is_dpp_active(ctx, i)) {
            REG_UPDATE(DPP_CLOCK_CONTROL[i],
                    DPP_CLOCK_GATE_DIS, 0);
        }
    }

    // HUBP (Hub Pipe) 时钟门控
    // 如果 HUBP 没有绑定流，关闭其时钟
    for (int i = 0; i < MAX_PIPES; i++) {
        if (!is_hubp_active(ctx, i)) {
            REG_UPDATE(HUBP_CLOCK_CONTROL[i],
                    HUBP_CLOCK_GATE_DIS, 0);
        }
    }
}
```

### 10.3 DCN Z-state (深度睡眠)

Z-state 是 DCN 的深度睡眠模式，可以在显示器处于 VBlank 期间关闭大部分显示逻辑。

```c
// Z-state 控制
enum dcn_zstate_support_state zstate_support;

// 判断是否允许 Z-state
bool dcn_allow_zstate(const struct dc_state *context)
{
    // Z-state 只能在单一显示器、低分辨率的静态画面下使用
    if (context->stream_count > 1)
        return false;

    // 检查 PSR 是否激活
    if (!is_psr_active(context))
        return false;

    // 检查刷新率 (低刷新率更适合 Z-state)
    const struct dc_stream_state *stream =
        context->streams[0];
    if (stream->timing.refresh_rate > 60)
        return false;

    return true;
}
```

### 13.4 寄存器级调试方法

除了驱动层的调试接口外，直接读取硬件寄存器可以提供最底层的时钟状态信息。

```c
// 关键寄存器地址定义 (DCN3)
#define DCN3_DENTIST_DISPCLK_CNTL         0x1A
#define DCN3_DCCG_DISPCLK_RAMP_CNTL       0x1B
#define DCN3_DENTIST_DPPCLK_CNTL(p)       (0x20 + (p))
#define DCN3_DCFCLK_CNTL                  0x30
#define DCN3_SMU_MAIN_POWER_CTRL          0x40

// 寄存器级时钟调试工具
static void dump_clock_registers(struct clk_mgr *clk_mgr)
{
    uint32_t dispclk_cntl, dppclk_cntl, dcfclk_cntl;

    // 读取 DISPCLK 控制寄存器
    dispclk_cntl = REG_READ(DENTIST_DISPCLK_CNTL);
    dcfclk_cntl  = REG_READ(DCFCLK_CNTL);

    DRM_INFO("=== Clock Register Dump ===\n");
    DRM_INFO("DENTIST_DISPCLK_CNTL: 0x%08X\n", dispclk_cntl);
    DRM_INFO("  DISPCLK_FREQ[19:12]: %d MHz\n",
             REG_GET(dispclk_cntl, DISPCLK_FREQ));
    DRM_INFO("  DISPCLK_CHG_MODE[1:0]: %d\n",
             REG_GET(dispclk_cntl, DISPCLK_CHG_MODE));
    DRM_INFO("  DISPCLK_LOAD_UPDATE: %d\n",
             REG_GET(dispclk_cntl, DISPCLK_LOAD_UPDATE));

    DRM_INFO("DCFCLK_CNTL: 0x%08X\n", dcfclk_cntl);
    DRM_INFO("  DCFCLK_FREQ[23:16]: %d MHz\n",
             REG_GET(dcfclk_cntl, DCFCLK_FREQ));

    // 遍历所有管道输出 DPPCLK
    for (int i = 0; i < MAX_PIPES; i++) {
        dppclk_cntl = REG_READ(DENTIST_DPPCLK_CNTL(i));
        DRM_INFO("Pipe %d DPPCLK_CNTL: 0x%08X\n", i, dppclk_cntl);
        DRM_INFO("  DPPCLK_FREQ[23:16]: %d MHz\n",
                 REG_GET(dppclk_cntl, DPPCLK_FREQ));
        DRM_INFO("  DPP_DTO_ENABLE: %d\n",
                 REG_GET(dppclk_cntl, DPP_DTO_ENABLE));
    }
}

// 调试用：验证时钟寄存器与软件状态一致性
static int verify_clock_consistency(struct clk_mgr *clk_mgr)
{
    struct dc_clocks *clks = &clk_mgr->clks;
    uint32_t hw_dispclk, hw_dcfclk;
    int errors = 0;

    // 读取硬件实际时钟
    hw_dispclk = REG_GET(REG_READ(DENTIST_DISPCLK_CNTL), DISPCLK_FREQ);
    hw_dcfclk  = REG_GET(REG_READ(DCFCLK_CNTL), DCFCLK_FREQ);

    // 与软件状态对比
    if (abs((int)(hw_dispclk * 1000 - clks->dispclk_khz)) > 1000) {
        DRM_ERROR("DISPCLK mismatch: HW=%d MHz, SW=%d kHz\n",
                  hw_dispclk, clks->dispclk_khz);
        errors++;
    }
    if (abs((int)(hw_dcfclk * 1000 - clks->dcfclk_khz)) > 1000) {
        DRM_ERROR("DCFCLK mismatch: HW=%d MHz, SW=%d kHz\n",
                  hw_dcfclk, clks->dcfclk_khz);
        errors++;
    }

    return errors;
}
```

**关键差异详解：**

```c
// DCN2 vs DCN3 时钟更新函数对比

// DCN2: 所有管道共享 DPPCLK
void dcn2_update_clocks(struct clk_mgr *clk_mgr,
                        struct dc_state *context,
                        bool safe_to_lower)
{
    struct dc_clocks *new_clocks = &context->bw_ctx.bw.dcn.clk;
    struct dc_clocks *old_clocks = &clk_mgr->clks;

    // DCN2 取所有管道 DPPCLK 的最大值
    int new_dppclk = 0;
    for (int i = 0; i < MAX_PIPES; i++) {
        if (context->res_ctx.pipe_ctx[i].stream)
            new_dppclk = max(new_dppclk,
                context->res_ctx.pipe_ctx[i].plane_res.bw.dppclk_khz);
    }
    // 所有管道共享一个 DPPCLK 频率
    new_clocks->dppclk_khz = new_dppclk;
    // ...
}

// DCN3: 每管道独立的 DPPCLK (DPP DTO)
void dcn3_update_clocks(struct clk_mgr *clk_mgr,
                        struct dc_state *context,
                        bool safe_to_lower)
{
    struct dc_clocks *new_clocks = &context->bw_ctx.bw.dcn.clk;

    // DCN3 通过 DPP DTO 实现每管道独立 DPPCLK
    // 取 DISPCLK 和所有管道 DPPCLK 的最大值...
    // 但每管道的实际时钟通过 DTO 分频器独立设置
    for (int i = 0; i < MAX_PIPES; i++) {
        if (context->res_ctx.pipe_ctx[i].stream) {
            uint32_t pipe_dppclk =
                context->res_ctx.pipe_ctx[i].plane_res.bw.dppclk_khz;
            // DPP DTO 设置: output = (phase / modulo) * input
            dccg2_set_dppclk_dto(clk_mgr->dccg,
                                 i, pipe_dppclk);
        }
    }
    // ...
}
```

### 14.2 SMU 通信协议详解

时钟管理器与 SMU 之间通过 Mailbox 寄存器进行通信，这是驱动控制硬件的关键路径。

```c
// SMU Mailbox 命令接口 (面向显示)
enum smu_display_command {
    SMU_CMD_SET_DISPCLK      = 0x01,
    SMU_CMD_SET_DPPCLK       = 0x02,
    SMU_CMD_SET_DCFCLK       = 0x03,
    SMU_CMD_SET_PHYCLK       = 0x04,
    SMU_CMD_SET_DTBCLK       = 0x05,
    SMU_CMD_SET_MIN_DCFCLK   = 0x06,  // 设置最低 DCFCLK
    SMU_CMD_SET_HARD_MIN_DCFCLK = 0x07,
    SMU_CMD_SET_HARD_MIN_DISPCLK = 0x08,
    SMU_CMD_SET_HARD_MIN_DPPCLK  = 0x09,
    SMU_CMD_SET_PSTATE_DISPLAY = 0x0A, // P-State 显示模式
    SMU_CMD_SET_ZSTATE_DISPLAY  = 0x0B, // Z-state 显示模式
    SMU_CMD_QUERY_DISPLAY_CLKS = 0x0C, // 查询可用时钟频率
    SMU_CMD_SET_VOLTAGE      = 0x0D,  // 设置电压
};

// SMU 通信返回状态
enum smu_message_status {
    SMU_STATUS_SUCCESS         = 0,
    SMU_STATUS_FAILURE         = 1,
    SMU_STATUS_BUSY            = 2,  // SMU 正忙
    SMU_STATUS_CLOCK_NOT_SUPPORTED = 3,
    SMU_STATUS_VOLTAGE_NOT_SUPPORTED = 4,
    SMU_STATUS_PARAM_INVALID   = 5,
};

// SMU 通信超时配置
#define SMU_MESSAGE_TIMEOUT_US    100000  // 100ms
#define SMU_POLL_INTERVAL_US      10      // 10us 轮询间隔

// SMU Mailbox 寄存器地址
#define mmMP1_SMU_MAILBOX_CONTROL       0x3B10
#define mmMP1_SMU_MAILBOX_DATA          0x3B14

// 高级 SMU 通信实现
static int smu_send_message_with_data(struct clk_mgr *clk_mgr,
                                       uint32_t command,
                                       uint32_t data,
                                       uint32_t timeout_us)
{
    uint32_t status;
    int retry = 5;

    // 等待 SMU 就绪
    while (retry--) {
        status = REG_READ(MP1_SMU_MAILBOX_CONTROL);
        if ((status & 0x80000000) == 0)  // 空闲位
            break;
        udelay(SMU_POLL_INTERVAL_US);
    }

    if (retry < 0) {
        DRM_ERROR("SMU not ready before command 0x%x\n", command);
        return -EBUSY;
    }

    // 写入命令数据
    REG_WRITE(MP1_SMU_MAILBOX_DATA, data);

    // 发送命令 (写控制寄存器触发 SMU)
    REG_WRITE(MP1_SMU_MAILBOX_CONTROL,
              command | 0x80000000);  // 设置 busy 位

    // 等待 SMU 完成
    while (timeout_us > 0) {
        status = REG_READ(MP1_SMU_MAILBOX_CONTROL);
        if ((status & 0x80000000) == 0) {
            // 命令完成，检查返回值
            uint32_t result = REG_READ(MP1_SMU_MAILBOX_DATA);
            if (result == SMU_STATUS_SUCCESS)
                return 0;
            DRM_WARN("SMU command 0x%x result: %u\n",
                     command, result);
            return -EIO;
        }
        udelay(SMU_POLL_INTERVAL_US);
        timeout_us -= SMU_POLL_INTERVAL_US;
    }

    DRM_ERROR("SMU command 0x%x timeout\n", command);
    return -ETIMEDOUT;
}

// SMU 查询可用时钟频率
struct smu_clock_table {
    uint32_t num_levels;           // DPM 等级数量
    struct {
        uint32_t freq_khz;         // 频率 (kHz)
        uint32_t voltage_mv;       // 对应电压 (mV)
        bool      is_valid;        // 该等级是否可用
    } levels[8];                   // 最多 8 个 DPM 等级
};

static int smu_query_display_clocks(struct clk_mgr *clk_mgr,
                                     enum smu_display_command clk_type,
                                     struct smu_clock_table *table)
{
    int ret;

    // 发送查询命令
    ret = smu_send_message_with_data(clk_mgr,
                                      SMU_CMD_QUERY_DISPLAY_CLKS,
                                      clk_type,
                                      SMU_MESSAGE_TIMEOUT_US);
    if (ret)
        return ret;

    // 读取 SMU 返回的时钟表 (通过共享 SRAM 或专用寄存器)
    uint32_t sram_base = REG_READ(SMU_DISPLAY_CLOCK_SRAM_BASE);
    // ... 从 SMU SRAM 读取时钟表数据

    return 0;
}
```

### 14.4 Linux PM QoS 集成

DC 时钟管理与 Linux 内核 PM QoS 框架集成，确保系统电源状态满足显示需求。

```c
// PM QoS 请求类型
enum dc_pm_qos_request {
    PM_QOS_REQ_DCFCLK,        // DCFCLK 最低需求
    PM_QOS_REQ_FCLK,          // FCLK 最低需求 (APU)
    PM_QOS_REQ_CPU_LATENCY,   // CPU 响应延迟
    PM_QOS_REQ_GPU_LATENCY,   // GPU 响应延迟
    PM_QOS_REQ_MEMORY_BW,     // 内存带宽保障
    PM_QOS_REQ_MAX
};

// DC PM QoS 管理器
struct dc_pm_qos_mgr {
    // PM QoS 请求列表
    struct pm_qos_request requests[PM_QOS_REQ_MAX];

    // 当前有效请求值
    struct {
        uint32_t min_dcfclk_khz;
        uint32_t min_fclk_khz;
        uint32_t max_cpu_latency_us;
        uint32_t max_gpu_latency_us;
        uint64_t min_memory_bw_bytes;
    } current;

    // 更新 PM QoS 请求
    void (*update_qos)(struct dc_pm_qos_mgr *mgr,
                       struct dc_state *context);

    // 清理 PM QoS 请求 (系统 suspend 时)
    void (*remove_qos)(struct dc_pm_qos_mgr *mgr);
};

// PM QoS 更新实现
static void dc_pm_qos_update(struct dc_pm_qos_mgr *mgr,
                              struct dc_state *context)
{
    struct dc_clocks *clks = &context->bw_ctx.bw.dcn.clk;

    // DCFCLK 要求: 取所有管道中的最大值
    uint32_t required_dcfclk = clks->dcfclk_khz;
    if (required_dcfclk > mgr->current.min_dcfclk_khz) {
        // 提高 DCFCLK 需求
        pm_qos_update_request(&mgr->requests[PM_QOS_REQ_DCFCLK],
                              required_dcfclk);
        mgr->current.min_dcfclk_khz = required_dcfclk;
    }

    // 带宽需求: 计算总内存带宽
    uint64_t total_bw = 0;
    for (int i = 0; i < context->stream_count; i++) {
        struct dc_stream_state *stream = context->streams[i];
        uint64_t stream_bw = stream->timing.pix_clk_100hz *
                             stream->timing.h_total *
                             stream->timing.v_total /
                             stream->timing.v_total * 100;
        total_bw += stream_bw;
    }

    // 转换为字节/秒
    total_bw *= 4;  // 假设 32bpp
    if (total_bw > mgr->current.min_memory_bw_bytes) {
        pm_qos_update_request(&mgr->requests[PM_QOS_REQ_MEMORY_BW],
                              total_bw);
        mgr->current.min_memory_bw_bytes = total_bw;
    }

    DRM_DEBUG("PM QoS: DCFCLK=%u kHz, MEM BW=%llu MB/s\n",
              required_dcfclk, total_bw / 1000000);
}
```

### 14.5 自动化测试：时钟压力测试

针对时钟管理的专用压力测试框架，验证系统在各种极端场景下的稳定性。

```python
"""
时钟管理压力测试脚本
测试场景:
  1. 时钟频率快速切换
  2. 多显示器热插拔
  3. 显示模式动态切换
  4. PSR 进出与时钟联动
使用方式:
  sudo python3 clock_stress_test.py --duration 3600 --monitor 2
"""

import subprocess
import time
import random
import sys
import threading
import argparse
from dataclasses import dataclass
from typing import List, Tuple

@dataclass
class ClockTestScenario:
    """时钟测试场景定义"""
    name: str
    setup_cmd: str           # 场景设置命令
    resolution: Tuple[int, int]
    refresh_rate: int
    expected_dispclk: int    # 期望 DISPCLK (kHz)
    expected_dcfclk: int     # 期望 DCFCLK (kHz)
    duration_seconds: int
    check_psr: bool          # 是否检查 PSR 状态

class ClockStressTester:
    def __init__(self, monitor_count: int, duration: int):
        self.monitor_count = monitor_count
        self.duration = duration
        self.errors = []
        self.clock_log = []

        # 定义测试场景序列
        self.scenarios = self._build_scenarios()

    def _build_scenarios(self) -> List[ClockTestScenario]:
        """构建渐进式测试场景"""
        scenarios = []

        # 场景1: 单显示器基础模式
        scenarios.append(ClockTestScenario(
            name="single_1080p_60",
            setup_cmd="xrandr --output DP-1 --mode 1920x1080 --rate 60",
            resolution=(1920, 1080),
            refresh_rate=60,
            expected_dispclk=1200000,  # 1.2 GHz
            expected_dcfclk=800000,    # 800 MHz
            duration_seconds=60,
            check_psr=True
        ))

        # 场景2: 高分辨率单显示器
        scenarios.append(ClockTestScenario(
            name="single_4k_60",
            setup_cmd="xrandr --output DP-1 --mode 3840x2160 --rate 60",
            resolution=(3840, 2160),
            refresh_rate=60,
            expected_dispclk=1800000,  # 1.8 GHz
            expected_dcfclk=1000000,   # 1.0 GHz
            duration_seconds=60,
            check_psr=False
        ))

        # 场景3: 双显示器
        if self.monitor_count >= 2:
            scenarios.append(ClockTestScenario(
                name="dual_4k_60",
                setup_cmd=(
                    "xrandr --output DP-1 --mode 3840x2160 --rate 60 "
                    "--output DP-2 --mode 3840x2160 --rate 60"
                ),
                resolution=(3840, 2160),
                refresh_rate=60,
                expected_dispclk=2300000,  # 2.3 GHz
                expected_dcfclk=1200000,   # 1.2 GHz
                duration_seconds=120,
                check_psr=False
            ))

        # 场景4: 高刷新率
        scenarios.append(ClockTestScenario(
            name="single_1080p_144",
            setup_cmd="xrandr --output DP-1 --mode 1920x1080 --rate 144",
            resolution=(1920, 1080),
            refresh_rate=144,
            expected_dispclk=1500000,  # 1.5 GHz
            expected_dcfclk=800000,
            duration_seconds=60,
            check_psr=False
        ))

        # 场景5: PSR 测试 (如果支持)
        scenarios.append(ClockTestScenario(
            name="psr_idle_test",
            setup_cmd="xrandr --output DP-1 --mode 1920x1080 --rate 60",
            resolution=(1920, 1080),
            refresh_rate=60,
            expected_dispclk=400000,   # PSR 低时钟
            expected_dcfclk=400000,
            duration_seconds=120,
            check_psr=True
        ))

        return scenarios

    def _read_clock_debugfs(self) -> dict:
        """读取调试文件系统获取当前时钟状态"""
        clocks = {}
        try:
            # 尝试从 debugfs 读取时钟信息
            result = subprocess.run(
                ['cat', '/sys/kernel/debug/dri/0/amdgpu_clock_info'],
                capture_output=True, text=True, timeout=5
            )
            for line in result.stdout.split('\n'):
                if 'dispclk' in line.lower():
                    clocks['dispclk'] = int(
                        line.split(':')[1].strip().split()[0])
                elif 'dcfclk' in line.lower():
                    clocks['dcfclk'] = int(
                        line.split(':')[1].strip().split()[0])
        except (FileNotFoundError, ValueError, subprocess.TimeoutExpired):
            pass
        return clocks

    def _check_psr_state(self) -> bool:
        """检查 PSR 状态"""
        try:
            result = subprocess.run(
                ['cat', '/sys/kernel/debug/dri/0/amdgpu_dm_psr_state'],
                capture_output=True, text=True, timeout=5
            )
            return 'active' in result.stdout.lower()
        except (FileNotFoundError, subprocess.TimeoutExpired):
            return False

    def _set_mode(self, scenario: ClockTestScenario) -> bool:
        """设置显示模式"""
        try:
            subprocess.run(
                scenario.setup_cmd.split(),
                capture_output=True, text=True, timeout=10
            )
            time.sleep(2)  # 等待模式切换稳定
            return True
        except subprocess.TimeoutExpired:
            self.errors.append(f"Mode set timeout: {scenario.name}")
            return False

    def _verify_clocks(self, scenario: ClockTestScenario) -> bool:
        """验证时钟频率是否符合预期"""
        clocks = self._read_clock_debugfs()
        if not clocks:
            self.errors.append(f"Cannot read clocks for {scenario.name}")
            return False

        passed = True

        # 检查 DISPCLK
        dispclk = clocks.get('dispclk', 0)
        tolerance = scenario.expected_dispclk * 0.1  # 10% 容忍度
        if abs(dispclk - scenario.expected_dispclk) > tolerance:
            self.errors.append(
                f"DISPCLK mismatch in {scenario.name}: "
                f"expected {scenario.expected_dispclk}, got {dispclk}"
            )
            passed = False

        # 记录日志
        self.clock_log.append({
            'time': time.time(),
            'scenario': scenario.name,
            'dispclk': dispclk,
            'dcfclk': clocks.get('dcfclk', 0),
            'passed': passed
        })

        return passed

    def run(self):
        """执行压力测试"""
        print(f"Starting clock stress test (duration: {self.duration}s)")
        print(f"Monitor count: {self.monitor_count}")
        print(f"Total scenarios: {len(self.scenarios)}")
        print("=" * 60)

        start_time = time.time()
        elapsed = 0
        scenario_idx = 0
        total_passed = 0
        total_failed = 0

        while elapsed < self.duration:
            scenario = self.scenarios[
                scenario_idx % len(self.scenarios)]

            print(f"\n[{elapsed}s] Scenario: {scenario.name}")
            print(f"  Resolution: {scenario.resolution[0]}x"
                  f"{scenario.resolution[1]} @ {scenario.refresh_rate}Hz")

            # 设置显示模式
            if not self._set_mode(scenario):
                total_failed += 1
                scenario_idx += 1
                continue

            # 等待稳定
            time.sleep(5)

            # 验证时钟
            if self._verify_clocks(scenario):
                total_passed += 1
                print(f"  ✓ Clock verification PASSED")
            else:
                total_failed += 1
                print(f"  ✗ Clock verification FAILED")

            # PSR 检查
            if scenario.check_psr:
                time.sleep(10)  # 等待 PSR 进入
                psr_active = self._check_psr_state()
                if psr_active:
                    print(f"  ✓ PSR entry SUCCESS")
                else:
                    print(f"  ✗ PSR entry FAILED")
                    self.errors.append(
                        f"PSR entry failed in {scenario.name}")

            # 场景持续时间
            time.sleep(scenario.duration_seconds)

            elapsed = time.time() - start_time
            scenario_idx += 1

        # 报告
        duration_actual = time.time() - start_time
        print("\n" + "=" * 60)
        print(f"Test Complete: {duration_actual:.0f}s")
        print(f"  Passed: {total_passed}")
        print(f"  Failed: {total_failed}")
        print(f"  Errors: {len(self.errors)}")

        if self.errors:
            print("\nError Details:")
            for err in self.errors[:10]:  # 只显示前10个错误
                print(f"  - {err}")

        # 保存日志
        with open('clock_stress_test_log.txt', 'w') as f:
            for entry in self.clock_log:
                f.write(
                    f"{entry['time']:.0f},{entry['scenario']},"
                    f"{entry['dispclk']},{entry['dcfclk']},"
                    f"{entry['passed']}\n"
                )
        print(f"\nLog saved to clock_stress_test_log.txt")

        return len(self.errors) == 0

if __name__ == '__main__':
    parser = argparse.ArgumentParser(
        description='DC Clock Stress Test Tool')
    parser.add_argument('--duration', type=int, default=3600,
                       help='Test duration in seconds')
    parser.add_argument('--monitor', type=int, default=1,
                       help='Number of monitors to test')
    args = parser.parse_args()

    tester = ClockStressTester(args.monitor, args.duration)
    success = tester.run()
    sys.exit(0 if success else 1)
```

```bash
        local label_val=$(cat "$label" 2>/dev/null)
        if echo "$label_val" | grep -qi "$clk_name"; then
            local num=$(echo "$label" | grep -o 'freq[0-9]*_label' | \
                       grep -o '[0-9]*')
            cat "${hwmon_dir}/freq${num}_input" 2>/dev/null
            return
        fi
    done

    echo "N/A"
}

# 获取 PSR 状态
get_psr_state() {
    if [ -f /sys/kernel/debug/dri/0/amdgpu_dm_psr_state ]; then
        cat /sys/kernel/debug/dri/0/amdgpu_dm_psr_state 2>/dev/null
    else
        echo "N/A"
    fi
}

# 获取电源状态
get_power_state() {
    local power_path="/sys/class/drm/card0/device/power_state"
    if [ -f "$power_path" ]; then
        cat "$power_path" 2>/dev/null
    else
        echo "N/A"
    fi
}

# 显示信息
echo "=== DC Clock Monitor ==="
echo "Time: $(date '+%H:%M:%S')"
echo ""
echo "Clocks:"
echo "  DISPCLK: $(get_clock 'dispclk') kHz"
echo "  DCFCLK:  $(get_clock 'dcfclk') kHz"
echo "  FCLK:    $(get_clock 'fclk') kHz"
echo ""
echo "Power State: $(get_power_state)"
echo "PSR State:   $(get_psr_state)"
echo ""
echo "Active Monitors:"
for connector in /sys/class/drm/card0/*-/status; do
    status=$(cat "$connector" 2>/dev/null)
    if [ "$status" = "connected" ]; then
        connector_name=$(basename "$(dirname "$connector")")
        echo "  - $connector_name: connected"
    fi
done
```

## 15. 总结

本章深入介绍了 AMDGPU DC 子系统的电源管理，特别是显示相关的时钟与电压控制：

1. **dc_clocks 结构**：DC 中所有时钟状态的统一数据结构，包含 DISPCLK、DPPCLK、DCFCLK 等关键时钟域。

2. **clk_mgr 架构**：时钟管理器负责收集各管道的时钟需求，通过 SMU 通信设置实际时钟频率，并提供更新/查询/比较等接口。

3. **DCCG (显示时钟发生器)**：负责从 Dentist VCO 分频生成各模块所需的时钟，支持每管道独立的 DPP DTO 分频。

4. **Clock Source (PLL)**：为像素时钟和 PHY 时钟提供精确的 PLL 合成，支持多种时钟源选择。

5. **时钟更新流程**：从带宽计算到 SMU 通信的完整流程，支持 safe_to_lower 策略防止时序问题。

6. **电源状态管理**：包括 DCN 低功耗模式、P-State 控制、Z-state 深度睡眠等高级电源管理功能。

7. **时钟门控**：在模块级别关闭未使用单元的时钟，进一步降低动态功耗。

8. **DCN 生成差异**：DCN2 到 DCN3.5 在时钟管理方面的演进，包括 DPP DTO、斜坡切换等关键特性。

9. **SMU 通信协议**：通过 Mailbox 寄存器与 SMU 交互的详细实现，包括命令定义、超时处理、状态管理。

10. **PM QoS 集成**：与 Linux 内核电源管理框架的集成，确保系统级电源状态满足显示需求。
