### 概念原理

#### ECC 错误注入原理

```ascii
ECC 错误注入机制:

┌──────────────────────────────────────────────────────────────┐
│  错误注入原理                                                    │
│                                                               │
│  正常 ECC 编码写入:                                            │
│  ┌─────────┐    ┌──────────┐    ┌─────────┐                   │
│  │ 数据     │───→│ ECC 编码  │───→│ 内存写入  │                   │
│  │ 0x1234  │    │ 生成 8B   │    │ 128+8B   │                   │
│  └─────────┘    └──────────┘    └─────────┘                   │
│                                                               │
│  正常 ECC 解码读取:                                            │
│  ┌─────────┐    ┌──────────┐    ┌─────────┐                   │
│  │ 内存读取  │───→│ ECC 解码  │───→│ 数据输出  │                   │
│  │ 128+8B   │    │ 校验+纠正 │    │ 0x1234  │                   │
│  └─────────┘    └──────────┘    └─────────┘                   │
│                                                               │
│  错误注入:                                                     │
│  ┌─────────┐    ┌──────────┐    ┌─────────┐                   │
│  │ 内存写入  │───→│ 翻转特定位 │───→│ ECC 错误  │                   │
│  │ 正常数据  │    │ (注入)    │    │ 存储     │                   │
│  └─────────┘    └──────────┘    └─────────┘                   │
│                                                               │
│  错误注入方法:                                                  │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 方法 1: 硬件错误注入 (通过 UMC 寄存器)                    │  │
│  │  - UMC 内置错误注入逻辑                                  │  │
│  │  - 通过 RAS 寄存器注入单/双比特错误                        │  │
│  │  - 指定地址和数据位翻转                                   │  │
│  │                                                         │  │
│  │ 方法 2: 软件模拟错误注入 (驱动层)                          │  │
│  │  - 驱动拦截 ECC 编码/解码路径                              │  │
│  │  - 模拟 CE/UE 错误上报                                    │  │
│  │  - 不实际损坏数据                                        │  │
│  │                                                         │  │
│  │ 方法 3: debugfs 错误注入接口                              │  │
│  │  - RAS debugfs 提供的用户空间接口                          │  │
│  │  - 支持指定错误类型和地址                                  │  │
│  │  - 最常用于测试                                          │  │
│  └────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

#### RAS 错误注入寄存器

```ascii
UMC 错误注入寄存器:

┌──────────────────────────────────────────────────────────────┐
│  UMC RAS 错误注入寄存器 (Navi 31 / RDNA3)                     │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 寄存器: UMC_CH_INJECT_ADDR                             │  │
│  │ 偏移: 0x0000                                            │  │
│  │ 位域:                                                   │  │
│  │  [31:12] - 注入地址 (4KB 对齐)                           │  │
│  │  [11:0]  - 保留                                          │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 寄存器: UMC_CH_INJECT_DATA                             │  │
│  │ 偏移: 0x0004                                            │  │
│  │ 位域:                                                   │  │
│  │  [31:0] - 注入数据掩码 (每个位翻转对应数据位)              │  │
│  └────────────────────────────────────────────────────────┘  │
│                                                               │
│  ┌────────────────────────────────────────────────────────┐  │
│  │ 寄存器: UMC_CH_INJECT_CTRL                              │  │
│  │ 偏移: 0x0008                                            │  │
│  │ 位域:                                                   │  │
│  │  [0]    - ECC_EN: 1=启用注入, 0=禁用                      │  │
│  │  [1]    - ERR_TYPE: 0=CE, 1=UE                          │  │
│  │  [2]    - BIT_FLIP: 0=单比特, 1=多比特                   │  │
│  │  [3]    - TRIGGER: 写入 1 触发注入                       │  │
│  │  [7:4]  - CH_SEL: 通道选择                               │  │
│  │  [15:8] - BIT_POS: 要翻转的位位置                         │  │
│  └────────────────────────────────────────────────────────┘  │

  注入触发流程:
  1. 写入目标地址到 INJECT_ADDR
  2. 写入数据掩码到 INJECT_DATA
  3. 配置 INJECT_CTRL (类型/位置/通道)
  4. 设置 TRIGGER=1 开始注入
  5. 硬件在下一次访问该地址时翻转指定位
  6. 读取时 ECC 检测到错误 → 触发中断
```

### 实践操作

#### 启用 RAS 功能

```bash
# 1. 检查当前 RAS 状态
cat /sys/class/drm/card0/device/ras_status

# 2. 启用 RAS (如果未启用)
echo enable > /sys/class/drm/card0/device/ras_ctrl

# 3. 确认 UMC ECC 已启用
cat /sys/class/drm/card0/device/ras_status | grep UMC
```

#### 注入 CE 错误

```bash
# 1. 检查当前 CE 计数基线
cat /sys/class/drm/card0/device/ras_ecc_cnt | grep UMC

# 2. 启用注入模式
echo inject > /sys/class/drm/card0/device/ras_ctrl

# 3. 注入单比特 CE 错误
echo "umc ce single" > /sys/class/drm/card0/device/ras_inject

# 可选: 指定更详细的注入参数
echo "umc ce single addr=0x000A0000 ch=1 bit=3" > /sys/class/drm/card0/device/ras_inject

# 4. 触发内存访问 (使 ECC 检测生效)
# 使用 umr 读取目标地址
umr -r 0x000A0000

# 5. 验证 CE 计数已增加
cat /sys/class/drm/card0/device/ras_ecc_cnt | grep UMC

# 6. 查看内核日志
dmesg | tail -5

# 输出:
# [ 1234.567] [drm] RAS: UMC CE error injected at 0x000A0000
# [ 1234.567] [drm] RAS: UMC CE corrected successfully
# [ 1234.567] [drm] RAS: UMC Ch1 CE count: 1
```

#### 注入 UE 错误

```bash
# 1. 注入双比特 UE 错误
echo "umc ue double" > /sys/class/drm/card0/device/ras_inject

# 或指定精确参数
echo "umc ue double addr=0x000B0000 ch=2 bit=3,7" > /sys/class/drm/card0/device/ras_inject

# 2. 触发内存访问
umr -r 0x000B0000

# 3. 验证 UE 计数
cat /sys/class/drm/card0/device/ras_ecc_cnt | grep UMC

# 4. 查看 UE 错误日志
dmesg | tail -10

# 输出:
# [ 1235.000] [drm] RAS: UMC UE error at 0x000B0000 (uncorrectable)
# [ 1235.001] [drm] RAS: Page isolation activated for 0x000B0000
# [ 1235.002] [drm] RAS: UMC Ch2 UE count: 1

# 5. 验证页面隔离
# 尝试再次访问被隔离的地址应该返回错误
umr -r 0x000B0000

# 输出:
# umr: Error reading register at 0x000B0000 (page isolated)
```

#### 自动化注入测试

```bash
#!/bin/bash
# ras_inject_test.sh - RAS 错误注入自动化测试脚本

echo "RAS 错误注入测试"
echo "================="
echo ""

# 1. 检查环境
if [ ! -f /sys/class/drm/card0/device/ras_inject ]; then
    echo "✗ RAS injection not supported"
    exit 1
fi

# 获取初始计数
INIT_CE=$(cat /sys/class/drm/card0/device/ras_ecc_cnt | grep "CE Count" | head -1 | awk '{print $NF}')
INIT_UE=$(cat /sys/class/drm/card0/device/ras_ecc_cnt | grep "UE Count" | head -1 | awk '{print $NF}')

echo "初始 CE: $INIT_CE"
echo "初始 UE: $INIT_UE"
echo ""

# 启用注入
echo inject > /sys/class/drm/card0/device/ras_ctrl

# 2. CE 注入测试
echo "--- CE 注入测试 ---"

PASS=0
FAIL=0

for i in {1..5}; do
    echo "  注入 CE #$i..."
    echo "umc ce single addr=0x$(printf '%08x' $((0xA00000 + i * 0x1000)))" \
        > /sys/class/drm/card0/device/ras_inject
    
    # 触发内存访问
    umr -r $((0xA00000 + i * 0x1000)) > /dev/null 2>&1
    
    sleep 0.5
done

# 验证 CE 计数
CUR_CE=$(cat /sys/class/drm/card0/device/ras_ecc_cnt | grep "CE Count" | head -1 | awk '{print $NF}')
CE_DIFF=$((CUR_CE - INIT_CE))

if [ "$CE_DIFF" -ge 5 ]; then
    echo "  ✓ CE 注入成功: 计数增加 $CE_DIFF"
    PASS=$((PASS + 1))
else
    echo "  ✗ CE 注入失败: 计数增加 $CE_DIFF (期望 >= 5)"
    FAIL=$((FAIL + 1))
fi

echo ""

# 3. UE 注入测试
echo "--- UE 注入测试 ---"

echo "umc ue double addr=0x00C00000 ch=0" \
    > /sys/class/drm/card0/device/ras_inject

umr -r 0x00C00000 > /dev/null 2>&1
sleep 0.5

CUR_UE=$(cat /sys/class/drm/card0/device/ras_ecc_cnt | grep "UE Count" | head -1 | awk '{print $NF}')
UE_DIFF=$((CUR_UE - INIT_UE))

if [ "$UE_DIFF" -ge 1 ]; then
    echo "  ✓ UE 注入成功: 计数增加 $UE_DIFF"
    PASS=$((PASS + 1))
else
    echo "  ✗ UE 注入失败: 计数增加 $UE_DIFF (期望 >= 1)"
    FAIL=$((FAIL + 1))
fi

echo ""

# 4. 检查日志
echo "--- 内核日志检查 ---"
dmesg | grep "RAS" | tail -10

echo ""
echo "--- 测试结果 ---"
echo "通过: $PASS"
echo "失败: $FAIL"

if [ "$FAIL" -eq 0 ]; then
    echo "结果: 全部通过 ✓"
else
    echo "结果: 存在失败 ✗"
fi
```

#### Python 注入测试框架

```python
#!/usr/bin/env python3
# ras_inject_framework.py - RAS 错误注入测试框架

import os
import time
import json
import logging
import subprocess
from typing import List, Dict, Optional, Tuple
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum


class ErrorType(Enum):
    CE = "ce"      # Correctable Error
    UE = "ue"      # Uncorrectable Error


class BitMode(Enum):
    SINGLE = "single"  # Single bit flip = CE
    DOUBLE = "double"  # Double bit flip = UE
    MULTI = "multi"    # Multiple bit flip = UE


@dataclass
class InjectConfig:
    """注入配置"""
    error_type: ErrorType
    bit_mode: BitMode
    address: int
    channel: int = 0
    bit_positions: List[int] = field(default_factory=list)


@dataclass
class InjectResult:
    """注入结果"""
    config: InjectConfig
    success: bool
    ce_before: int
    ce_after: int
    ue_before: int
    ue_after: int
    error_message: str = ""
    dmesg_lines: List[str] = field(default_factory=list)


class RASInjectTest:
    """RAS 错误注入测试框架"""
    
    RAS_BASE = "/sys/class/drm/card0/device"
    
    def __init__(self):
        self.setup_logging()
        self.check_ras_support()
    
    def setup_logging(self):
        """配置日志"""
        logging.basicConfig(
            level=logging.INFO,
            format="%(asctime)s [%(levelname)s] %(message)s"
        )
        self.log = logging.getLogger("RASInject")
    
    def check_ras_support(self) -> bool:
        """检查 RAS 注入支持"""
        inject_path = f"{self.RAS_BASE}/ras_inject"
        ctrl_path = f"{self.RAS_BASE}/ras_ctrl"
        
        if not os.path.exists(inject_path):
            self.log.error("RAS injection not supported")
            return False
        if not os.path.exists(ctrl_path):
            self.log.error("RAS control not found")
            return False
        
        self.log.info("RAS injection support confirmed")
        return True
    
    def read_ecc_counts(self) -> Tuple[int, int, List[Dict]]:
        """读取 ECC 计数"""
        path = f"{self.RAS_BASE}/ras_ecc_cnt"
        try:
            with open(path) as f:
                content = f.read()
        except (FileNotFoundError, PermissionError) as e:
            self.log.error(f"Cannot read ECC counts: {e}")
            return 0, 0, []
        
        ce_total = 0
        ue_total = 0
        channels = []
        
        for line in content.split("\n"):
            if "│" in line and "CE Count" not in line:
                parts = [p.strip() for p in line.split("│")[1:-1]]
                if len(parts) >= 4:
                    channels.append({
                        "block": parts[0],
                        "ce": int(parts[1]) if parts[1].isdigit() else 0,
                        "ue": int(parts[2]) if parts[2].isdigit() else 0,
                    })
                    ce_total += channels[-1]["ce"]
                    ue_total += channels[-1]["ue"]
        
        return ce_total, ue_total, channels
    
    def enable_inject_mode(self):
        """启用注入模式"""
        with open(f"{self.RAS_BASE}/ras_ctrl", "w") as f:
            f.write("inject")
        self.log.info("Inject mode enabled")
    
    def inject_error(self, config: InjectConfig) -> bool:
        """注入错误"""
        inject_path = f"{self.RAS_BASE}/ras_inject"
        
        # 构建注入命令
        cmd = f"{config.error_type.value} {config.bit_mode.value}"
        cmd += f" addr=0x{config.address:08x}"
        cmd += f" ch={config.channel}"
        if config.bit_positions:
            cmd += f" bit={','.join(map(str, config.bit_positions))}"
        
        try:
            with open(inject_path, "w") as f:
                f.write(cmd)
            self.log.info(f"Injected: {cmd}")
            return True
        except (PermissionError, OSError) as e:
            self.log.error(f"Inject failed: {e}")
            return False
    
    def trigger_memory_access(self, address: int) -> bool:
        """触发出错地址的内存访问"""
        try:
            # 使用 umr 读取地址
            result = subprocess.run(
                ["umr", "-r", f"0x{address:x}"],
                capture_output=True, text=True, timeout=5
            )
            return result.returncode == 0
        except (subprocess.TimeoutError, FileNotFoundError):
            self.log.warning(f"Cannot trigger access to 0x{address:x}")
            return False
    
    def run_inject_test(self, config: InjectConfig) -> InjectResult:
        """执行单次注入测试"""
        self.log.info(f"Starting inject test: {config}")
        
        # 记录注入前计数
        ce_before, ue_before, _ = self.read_ecc_counts()
        
        # 启用注入
        self.enable_inject_mode()
        
        # 注入错误
        if not self.inject_error(config):
            return InjectResult(
                config=config, success=False,
                ce_before=ce_before, ce_after=ce_before,
                ue_before=ue_before, ue_after=ue_before,
                error_message="Inject command failed"
            )
        
        time.sleep(0.5)
        
        # 触发内存访问
        self.trigger_memory_access(config.address)
        
        time.sleep(0.5)
        
        # 读取注入后计数
        ce_after, ue_after, _ = self.read_ecc_counts()
        
        # 获取 dmesg
        try:
            result = subprocess.run(
                ["dmesg"], capture_output=True, text=True
            )
            dmesg_lines = [
                line for line in result.stdout.split("\n")
                if "RAS" in line
            ][-5:]
        except subprocess.TimeoutError:
            dmesg_lines = []
        
        # 判断成功
        if config.error_type == ErrorType.CE:
            success = (ce_after > ce_before)
        else:
            success = (ue_after > ue_before)
        
        return InjectResult(
            config=config, success=success,
            ce_before=ce_before, ce_after=ce_after,
            ue_before=ue_before, ue_after=ue_after,
            dmesg_lines=dmesg_lines
        )
    
    def run_test_suite(self) -> List[InjectResult]:
        """运行测试套件"""
        results = []
        
        test_cases = [
            # 单比特 CE 测试 (不同地址)
            InjectConfig(ErrorType.CE, BitMode.SINGLE, 0x00D00000, 0),
            InjectConfig(ErrorType.CE, BitMode.SINGLE, 0x00D01000, 1),
            InjectConfig(ErrorType.CE, BitMode.SINGLE, 0x00D02000, 2),
            
            # UE 双比特测试
            InjectConfig(ErrorType.UE, BitMode.DOUBLE, 0x00E00000, 0),
        ]
        
        for config in test_cases:
            result = self.run_inject_test(config)
            results.append(result)
            status = "✓" if result.success else "✗"
            self.log.info(f"  [{status}] {config.error_type.value} "
                          f"@ 0x{config.address:08x}: "
                          f"CE {result.ce_before}->{result.ce_after}, "
                          f"UE {result.ue_before}->{result.ue_after}")
            
            # 等待错误处理完成
            time.sleep(1)
        
        return results
    
    def generate_report(self, results: List[InjectResult]) -> str:
        """生成测试报告"""
        total = len(results)
        passed = sum(1 for r in results if r.success)
        
        report = []
        report.append("RAS 错误注入测试报告")
        report.append("=" * 60)
        report.append(f"测试时间: {datetime.now().isoformat()}")
        report.append(f"总计: {total}, 通过: {passed}, 失败: {total - passed}")
        report.append("")
        
        for i, result in enumerate(results):
            report.append(f"测试 #{i + 1}:")
            report.append(f"  类型: {result.config.error_type.value}")
            report.append(f"  模式: {result.config.bit_mode.value}")
            report.append(f"  地址: 0x{result.config.address:08x}")
            report.append(f"  通道: {result.config.channel}")
            report.append(f"  结果: {'通过' if result.success else '失败'}")
            report.append(f"  CE: {result.ce_before} -> {result.ce_after}")
            report.append(f"  UE: {result.ue_before} -> {result.ue_after}")
            if result.dmesg_lines:
                report.append(f"  日志:")
                for line in result.dmesg_lines[-3:]:

        else:
            report.append(f"结论: {total - passed} 项测试失败 ✗")

                "ue_change": r.ue_after - r.ue_before,
                "dmesg": r.dmesg_lines[-3:] if r.dmesg_lines else [],
            })
        
        with open(path, "w") as f:
            json.dump(report_data, f, indent=2)
```

### 今日小结

1. **注入原理**：ECC 错误注入通过 UMC 硬件寄存器直接在内存数据路径中翻转特定位，模拟真实的 ECC 错误。

2. **注入方法**：支持 CE（单比特）和 UE（双比特/多比特）两种错误类型注入，可指定地址、通道和位位置。

3. **测试验证**：注入后通过触发内存访问使 ECC 检测生效，验证 CE 计数增加、UE 中断触发、页面隔离等机制。

4. **测试框架**：自动化注入测试脚本和 Python 框架可以批量执行注入、验证结果、生成测试报告。

5. **应用场景**：硬件验证、驱动回归测试、RAS 功能验证、阈值告警测试等。
