### 概念原理

#### PCI 设备发现基础

GPU 作为 PCIe 设备，其发现流程从 PCI 总线枚举开始：

```
Linux PCI 设备枚举流程:

┌─────────────────────────────────────────────────────────────┐
│                系统启动 / PCI 热插拔                         │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  PCI 配置空间读取 (0xCF8/0xCFC 或 MMIO)                    │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ Vendor ID = 0x1002 (AMD)                              │  │
│  │ Device ID = 0x738C (Aldebaran/MI250)                  │  │
│  │ Class Code = 0x030000 (VGA 控制器)                    │  │
│  │ Revision ID = 0x00                                    │  │
│  │ Subsystem Vendor = 0x1002                             │  │
│  │ Subsystem ID = 0x..........                           │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  分配 PCI 总线号和设备编号                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ Bus:Device.Function = 03:00.0                        │  │
│  │ Domain:Bus:Device.Function = 0000:03:00.0            │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  初始化 PCI BAR (Base Address Registers)                    │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ BAR 0: MMIO 寄存器空间 (0xE0000000, 256KB)            │  │
│  │ BAR 1: MMIO 扩展 (0xE0040000, 256KB)                  │  │
│  │ BAR 2: GTT (Graphics Translation Table) (0x10000000)  │  │
│  │ BAR 3: VRAM (显存) (0x2000000000, 64GB)               │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  分配 IRQ 资源 (MSI/MSI-X)                                  │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ MSI-X Capability: Vector Table at BAR 0 + 0x2000     │  │

│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────┘

                          ▼
┌─────────────────────────────────────────────────────────────┐
│  注册到 PCI 子系统                                          │
│  ┌───────────────────────────────────────────────────────┐  │
│  │ pci_register_driver(&amdgpu_pci_driver)               │  │
│  │ → 匹配 pci_device_id 表                               │  │
│  │ → 调用 amdgpu_pci_probe()                             │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────┬───────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  amdgpu 驱动初始化                                         │
│  → amdgpu_pci_probe → amdgpu_device_init → ...             │
└─────────────────────────────────────────────────────────────┘
```

#### AMDGPU PCI Probe 流程

当 PCI 子系统匹配到设备后，调用 `amdgpu_pci_probe()`：

```c
// 内核源码: drivers/gpu/drm/amd/amdgpu/amdgpu_drv.c

// 主 probe 函数
static int amdgpu_pci_probe(struct pci_dev *pdev,
                              const struct pci_device_id *ent)
{
    struct drm_device *dev;
    struct amdgpu_device *adev;
    unsigned long flags;
    int ret;

    /* 阶段 1: 设备启用 */
    ret = pcim_enable_device(pdev);                    // 启用 PCI 设备
    if (ret)
        return ret;

    /* 阶段 2: BAR 资源映射 */
    ret = pcim_iomap_regions(pdev, 0xFF, "amdgpu");    // 映射所有 BAR
    if (ret) {
        DRM_ERROR("BAR mapping failed: %d\n", ret);
        return ret;
    }

    /* 阶段 3: 设置 DMA 掩码 */
    ret = dma_set_mask(&pdev->dev, DMA_BIT_MASK(64));  // 64位 DMA
    if (ret) {
        ret = dma_set_mask(&pdev->dev, DMA_BIT_MASK(40));
        if (ret) {
            DRM_ERROR("DMA mask set failed\n");
            return ret;
        }
    }

    /* 阶段 4: 启用 MSI/MSI-X */
    ret = pci_enable_msi(pdev);
    if (ret) {
        DRM_INFO("MSI not available, falling back to INTx\n");
    }
    ret = pci_enable_msix_range(pdev, msix_entries, 1, 64);
    if (ret < 0) {
        DRM_INFO("MSI-X not available\n");
    }

    /* 阶段 5: 分配 DRM 设备 */
    dev = drm_dev_alloc(&amdgpu_kms_driver, &pdev->dev);
    if (IS_ERR(dev))
        return PTR_ERR(dev);

    adev = dev->dev_private;

    /* 阶段 6: 检测 GPU 架构类型 */
    adev->family = ent->driver_data;                   // 从 PCI ID 表获取
    adev->ip_versions = amdgpu_detect_ip_versions(adev);  // 检测 IP 版本

    /* 阶段 7: 运行设备初始化 */
    ret = amdgpu_device_init(adev, flags);             // 核心初始化
    if (ret) {
        DRM_ERROR("Device init failed: %d\n", ret);
        goto err_free;
    }

    /* 阶段 8: 注册 DRM 设备 */
    ret = drm_dev_register(dev, 0);
    if (ret) {
        DRM_ERROR("DRM register failed: %d\n", ret);
        goto err_fini;
    }

    return 0;

err_fini:
    amdgpu_device_fini(adev);
err_free:
    drm_dev_put(dev);
    return ret;
}
```

```c
// IP 版本检测实现
struct amdgpu_ip_block_version {
    enum amd_ip_block_type type;  // IP 块类型
    uint32_t major;               // 主版本
    uint32_t minor;               // 次版本
    uint32_t revision;            // 修订号
};

enum amd_ip_block_type {
    AMD_IP_BLOCK_TYPE_GC,           // Graphics Compute
    AMD_IP_BLOCK_TYPE_SDMA,         // System DMA
    AMD_IP_BLOCK_TYPE_PSP,          // Platform Security Processor
    AMD_IP_BLOCK_TYPE_SMU,          // System Management Unit
    AMD_IP_BLOCK_TYPE_DCE,          // Display and Compositing Engine
    AMD_IP_BLOCK_TYPE_GMC,          // Graphics Memory Controller
    AMD_IP_BLOCK_TYPE_VCN,          // Video Core Next
    AMD_IP_BLOCK_TYPE_HDP,          // Host Data Path
    AMD_IP_BLOCK_TYPE_NBIO,         // North Bridge IO
    AMD_IP_BLOCK_TYPE_JPEG,        // JPEG Decoder
    AMD_IP_BLOCK_TYPE_IH,           // Interrupt Handler
    AMD_IP_BLOCK_TYPE_MP0,          // Multimedia Processor 0 (PSP)
    AMD_IP_BLOCK_TYPE_MP1,          // Multimedia Processor 1 (SMU)
};

// IP 版本到具体初始化函数的映射
static const struct amdgpu_ip_funcs *ip_blocks[] = {
    &gmc_v9_0_ip_funcs,       // GMC 9.x (Vega)
    &gfx_v9_0_ip_funcs,       // GFX 9.x (Vega)
    &sdma_v4_0_ip_funcs,      // SDMA 4.x (Vega)
    &psp_v3_1_ip_funcs,       // PSP 3.1
    &smu_v11_0_ip_funcs,      // SMU 11.x
    &dce_virtual_ip_funcs,    // DCE Virtual
    &vcn_v1_0_ip_funcs,       // VCN 1.0
    &jpeg_v1_0_ip_funcs,      // JPEG 1.0
    // ... 更多 IP 块
};

// IP 版本检测
static int amdgpu_detect_ip_versions(struct amdgpu_device *adev)
{
    // 从 VBIOS 或寄存器读取 IP Discovery 表
    struct ip_discovery_header *header;
    struct ip_block_entry *entries;
    int i;
    
    header = amdgpu_read_ip_discovery_table(adev);
    if (!header) {
        // 回退到基于 PCI ID 的硬编码版本
        return amdgpu_fallback_ip_versions(adev);
    }
    
    entries = (struct ip_block_entry *)(header + 1);
    
    for (i = 0; i < header->num_entries; i++) {
        switch (entries[i].hw_id) {
        case IP_HWID_GC:
            adev->ip_versions.gc_major = entries[i].major;
            adev->ip_versions.gc_minor = entries[i].minor;
            break;
        case IP_HWID_SDMA0:
            adev->ip_versions.sdma_major = entries[i].major;
            adev->ip_versions.sdma_minor = entries[i].minor;
            break;
        // ... 处理其他 IP 块
        }
    }
    
    return 0;
}
```

#### amdgpu_device_init 详细流程

```c
// 内核源码: drivers/gpu/drm/amd/amdgpu/amdgpu_device.c

int amdgpu_device_init(struct amdgpu_device *adev,
                        unsigned long flags)
{
    int r;

    /* 阶段 1: 基础初始化 */
    mutex_init(&adev->lock);
    INIT_LIST_HEAD(&adev->vm_manager.pd_bo_list);
    adev->accel_working = false;
    adev->num_rings = 0;

    /* 阶段 2: 早期硬件初始化 */
    r = amdgpu_regulator_init(adev);                // 电源调节器
    if (r)
        return r;

    r = amdgpu_atombios_init(adev);                  // ATOM BIOS 接口
    if (r)
        return r;

    // 读取 VBIOS
    r = amdgpu_get_bios(adev);
    if (r)
        return r;

    /* 阶段 3: IP 块初始化 */
    // 按顺序初始化每个 IP 块
    for (int i = 0; i < adev->num_ip_blocks; i++) {
        r = adev->ip_blocks[i]->funcs->early_init(
            (struct amdgpu_ip_block *)&adev->ip_blocks[i]);
        if (r)
            return r;
    }

    /* 阶段 4: SW 初始化 */
    for (int i = 0; i < adev->num_ip_blocks; i++) {
        r = adev->ip_blocks[i]->funcs->sw_init(
            (struct amdgpu_ip_block *)&adev->ip_blocks[i]);
        if (r)
            return r;
    }

    /* 阶段 5: HW 初始化 */
    for (int i = 0; i < adev->num_ip_blocks; i++) {
        r = adev->ip_blocks[i]->funcs->hw_init(
            (struct amdgpu_ip_block *)&adev->ip_blocks[i]);
        if (r)
            return r;
    }

    /* 阶段 6: 完成初始化 */
    adev->accel_working = true;
    amdgpu_fence_driver_init(adev);
    amdgpu_irq_init(adev);
    amdgpu_gem_init(adev);
    amdgpu_cs_init(adev);

    /* 阶段 7: 后期初始化 (需要调度的工作) */
    schedule_work(&adev->late_init_work);

    return 0;
}
```

#### GPU 设备发现调试工具

```bash
#!/bin/bash
# GPU 设备枚举检测工具

echo "=============================================="
echo "  GPU 设备发现与枚举检测"
echo "=============================================="
echo ""

detect_all_gpus() {
    echo "[1/6] 检测所有 AMD GPU"
    echo "----------------------"
    
    # 通过 lspci 检测
    echo "  lspci 检测:"
    lspci -nn | grep -i "amd\|ati\|1002" | grep -i "vga\|3d\|display" | while read line; do
        echo "    ${line}"
    done
    
    echo ""
    
    # 通过 sysfs 检测
    echo "  sysfs 检测:"
    for dir in /sys/bus/pci/devices/*; do
        vendor=$(cat ${dir}/vendor 2>/dev/null)
        if [ "${vendor}" = "0x1002" ]; then
            class=$(cat ${dir}/class 2>/dev/null)
            # VGA: 0x030000, 3D: 0x030200
            if [[ "${class}" == "0x030000" || "${class}" == "0x030200" ]]; then
                bdf=$(basename ${dir})
                device=$(cat ${dir}/device)
                echo "    ${bdf}: AMD GPU (Device ID: ${device}, Class: ${class})"
            fi
        fi
    done
}

show_pci_topology() {
    echo ""
    echo "[2/6] PCI 拓扑"
    echo "-------------"
    
    echo "  GPU 在 PCIe 树的层次:"
    for dir in /sys/bus/pci/devices/*; do
        vendor=$(cat ${dir}/vendor 2>/dev/null)
        class=$(cat ${dir}/class 2>/dev/null)
        
        if [ "${vendor}" = "0x1002" ] && [[ "${class}" == "0x030000" || "${class}" == "0x030200" ]]; then
            bdf=$(basename ${dir})
            
            # 获取 PCIe 链路信息
            link_speed=$(cat ${dir}/current_link_speed 2>/dev/null || echo "N/A")
            link_width=$(cat ${dir}/current_link_width 2>/dev/null || echo "N/A")
            
            echo "    ${bdf}: ${link_speed}, ${link_width}"
            
            # 打印父设备 (PCIe Switch 或 Root Port)
            parent=$(dirname ${dir})
            parent_bdf=$(basename $(readlink -f ${dir}/..))
            echo "       └─ 上游: ${parent_bdf}"
            
            # 打印 NUMA 节点
            numa_node=$(cat ${dir}/numa_node 2>/dev/null || echo "-1")
            echo "       NUMA 节点: ${numa_node}"
        fi
    done
}

check_drm_devices() {
    echo ""
    echo "[3/6] DRM 设备枚举"
    echo "-----------------"
    
    echo "  /dev/dri 设备:"
```

### 案例分析

#### 案例一：GPU 未正确枚举

**问题现象：**
```
系统有 2 张 GPU，但只有 1 张被识别
lspci 显示两张 GPU 都存在
但 /dev/dri/card1 不存在
```

**诊断过程：**
```bash
# 1. 检查 PCI 设备
lspci -nn | grep -i "1002"
# 03:00.0 VGA compatible controller [0300]: Advanced Micro Devices [1002] Navi 21 [73BF]
# 04:00.0 VGA compatible controller [0300]: Advanced Micro Devices [1002] Navi 21 [73BF]
# 两张 GPU 在 PCI 层面都存在

# 2. 检查 amdgpu 绑定
ls /sys/bus/pci/drivers/amdgpu/
# 0000:03:00.0  0000:04:00.0  bind  uevent  unbind
# 两张都绑定了 amdgpu

# 3. 检查 dmesg
dmesg | grep -i "amdgpu.*04:00"
# [    5.123] amdgpu 0000:04:00.0: BAR 3: can't reserve [mem 0x2000000000-0x2fffffffff]
# [    5.124] amdgpu 0000:04:00.0: device initialization failed (-16)

# 4. 检查 BAR 冲突
lspci -s 04:00.0 -vv | grep Region
# Region 0: Memory at e0000000 (64-bit, prefetchable) [size=256K]
# Region 2: Memory at e0040000 (64-bit, prefetchable) [size=2M]
# Region 3: Memory at 2000000000 (64-bit, prefetchable) [size=64G]  ← 冲突!
```

**根本原因：**

```
BAR 冲突:

┌──────────────────────────────────────────────────────┐
│  /proc/iommu 或 /proc/iomem 查看                      │
├──────────────────────────────────────────────────────┤
│                                                       │
│  2000000000-2fffffffff : PCI Bus 0000:03              │
│    2000000000-2fffffffff : 0000:03:00.0               │  ← GPU 0 占用了
│                                                       │
│  物理地址冲突!                                        │
│  两张 GPU 的 BAR 3 都映射到 2000000000 起始地址        │
│  但 PCIe 配置空间没有正确分配不同的地址范围              │
│                                                       │
│  可能原因:                                             │
│  1. BIOS 没有为第二张 GPU 分配足够的 MMIO 空间         │
│  2. Above 4G Decoding 未启用                          │
│  3. PCIe Root Port 的 MMIO 窗口不足                    │
│                                                       │
└──────────────────────────────────────────────────────┘
```

**解决方案：**
```bash
# 方案 1: 启用 Above 4G Decoding
# 在 BIOS 中启用 "Above 4G Decoding" 或 "Resizable BAR"
# 重启后验证
dmesg | grep -i "BAR"
# 检查 BAR 地址是否不重叠

# 方案 2: 调整 PCIe MMIO 大小
# 在内核命令行中添加:
# pci=realloc
# pci=assign-busses
# 或通过 BIOS 设置增大 MMIO 空间

# 方案 3: 确保两张 GPU 在不同 PCIe 域
# 某些主板需要不同的 PCIe 域来支持多 GPU

# 方案 4: 使用 pci=resource_alignment
# 内核参数确保 BAR 对齐
pci=resource_alignment=16G@0000:04:00.0
```

#### 案例二：热插拔 GPU 未被识别

**问题现象：**
```
在运行时插入 GPU，系统没有检测到新设备
需要重启才能识别
```

**诊断过程：**
```bash
# 1. 检查 PCI 热插拔支持
lspci -s 03:00.0 -vv | grep -i "hotplug\|SltCtl\|Slot"
# 检查 PCIe Root Port 的热插拔能力

# 2. 检查 GPU 的 removable 标志
cat /sys/bus/pci/devices/0000:03:00.0/removable
# removable

# 3. 重新扫描 PCI 总线
echo 1 > /sys/bus/pci/rescan
# 检查是否识别
lspci -nn | grep 1002

# 4. 如果识别了但没有绑定驱动
echo "0000:04:00.0" > /sys/bus/pci/drivers/amdgpu/bind
```

### 今日小结

GPU 设备发现与枚举是驱动加载的第一步：

1. **PCI 枚举**: PCI 子系统发现设备、分配 BAR 资源和 IRQ
2. **PCI ID 匹配**: 通过 pci_device_id 表匹配支持的 GPU
3. **IP 版本检测**: 使用 IP Discovery 表识别 GPU 内部 IP 块
4. **IP 块初始化**: 按顺序执行 early_init → sw_init → hw_init
5. **多 GPU 拓扑**: 总线号决定枚举顺序，NUMA 节点影响性能
6. **调试工具**: lspci、sysfs、dmesg 是设备发现调试的主要手段
7. **常见问题**: BAR 冲突、IOMMU 组隔离、热插拔支持
