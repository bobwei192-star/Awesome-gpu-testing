## 参考答案

### 第一部分：选择题

1. **B** - igt_runner 是 IGT 测试执行器
2. **B** - JUnit XML 是 CI 系统广泛支持的格式
3. **B** - gem_basic 验证缓冲区对象操作
4. **B** - CRC 校验验证显示输出正确性
5. **C** - SKIP 表示条件不满足，测试未执行
6. **B** - IGT_DEBUG=1 启用调试输出
7. **B** - dmesg 是首先检查的日志
8. **B** - 超时默认单位是秒
9. **B** - piglit 用于 Mesa 3D 驱动测试
10. **B** - --list-subtests 列出子测试

### 第二部分：判断题

11. **错误**。IGT 支持多种 GPU，包括 AMD、Intel、ARM。

12. **正确**。igt_runner 支持 --parallel 参数并行执行测试。

13. **错误**。TIMEOUT 可能由多种原因引起，包括测试逻辑问题、资源竞争等。

14. **正确**。dmesg 包含内核驱动日志，对分析测试失败至关重要。

15. **正确**。JSON 输出包含完整的测试结果信息。

### 第三部分：简答题

**16. igt_runner 功能：**

- 测试发现：扫描测试目录
- 测试执行：顺序或并行执行
- 结果收集：收集 pass/fail/skip/timeout
- 报告生成：支持 text/json/junit 格式

常用参数：
- --list-tests：列出测试
- --test-list：指定测试列表
- --output-format：输出格式
- --timeout：超时设置
- --parallel：并行数量

**17. gem_basic 子测试：**

- create-close：验证 BO 创建和关闭的基本功能
- create-large：验证大 BO（256MB）创建，测试显存分配

**18. kms_pipe_crc_basic 原理：**

- 设置显示模式
- 绘制已知图案
- 读取显示管线的 CRC32 值
- 验证 CRC 与预期匹配

CRC 作用：验证显示输出正确性，检测像素损坏。

**19. 五种结果状态：**

| 状态 | 含义 | 处理 |
|------|------|------|
| PASS | 通过 | 正常 |
| FAIL | 失败 | 调查根因 |
| SKIP | 跳过 | 检查条件 |
| TIMEOUT | 超时 | 检查 hang |
| CRASH | 崩溃 | 立即调查 |

**20. 失败分析方法：**

1. 收集测试日志
2. 收集 dmesg 日志
3. 检查 GPU 状态
4. 分析调用栈
5. 最小化复现
6. 定位代码问题

### 第四部分：综合实践题

**综合实践题一参考答案：**

运行脚本：
```bash
#!/bin/bash
# 清空 dmesg
sudo dmesg -c

# 运行测试
sudo ./gem_basic --output-format json --output-file gem_basic.json 2>&1 | tee gem_basic.log

# 收集 dmesg
sudo dmesg > gem_basic.dmesg
```

解析脚本：
```bash
#!/bin/bash
# 解析 JSON
jq -r '
    "通过: \([.tests[] | select(.result=="pass")] | length)",
    "失败: \([.tests[] | select(.result=="fail")] | length)",
    "跳过: \([.tests[] | select(.result=="skip")] | length)"
' gem_basic.json
```

调试流程：
1. 检查 dmesg 是否有内存分配错误
2. 检查 GPU 显存使用情况
3. 尝试手动分配大 BO 验证
4. 检查是否有其他应用占用显存

**综合实践题二参考答案：**

CI 配置：
```yaml
igt_regression:
  stage: test
  tags:
    - gpu-amd
  timeout: 1h
  script:
    - sudo dmesg -c
    - sudo igt-runner --test-list quick-tests.txt
          --output-format junit --output-file igt.xml
          --timeout 300 ./tests
    - sudo dmesg > igt.dmesg
  artifacts:
    reports:
      junit: igt.xml
```
