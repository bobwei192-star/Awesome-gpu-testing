## 参考答案

### 第一部分：选择题

1. **B** - IGT 最初由 Intel 开发
2. **C** - 高性能计算不是 IGT 的设计哲学
3. **B** - 测试程序位于 tests/
4. **B** - AMDGPU 测试位于 tests/amdgpu/
5. **C** - IGT 使用 Meson 构建系统
6. **D** - igt_cuda 不是 IGT 组件
7. **B** - igt_main() 是测试入口
8. **B** - modetest 用于显示模式测试
9. **B** - video 组是 IGT 运行所需权限
10. **B** - 测试执行器位于 runner/

### 第二部分：判断题

11. **错误**。IGT 最初为 Intel GPU 设计，但现已支持 AMD、ARM 等多种 GPU。

12. **正确**。lib/ 目录包含 igt_core、igt_kms、igt_gem 等核心组件。

13. **错误**。libdrm 是 IGT 的必需依赖，用于 DRM 接口操作。

14. **正确**。IGT 直接操作 DRM/KMS 接口，绕过显示服务器，这是其贴近硬件的设计哲学。

15. **正确**。runner/ 目录包含 igt_runner、resultgen 等执行器组件。

### 第三部分：简答题

**16. IGT 发展历程：**

- 2008 年：Intel 内部开发图形驱动测试工具
- 2010 年：开源发布为 intel-gpu-tools
- 2012 年：更名为 igt-gpu-tools，开始支持非 Intel GPU
- 2015 年：成为 DRM 子系统官方测试框架
- 2018 年：支持 AMDGPU 专用测试
- 现在：成为 GPU 驱动测试的事实标准

**17. IGT 设计哲学：**

- 贴近硬件：直接操作 DRM/KMS 接口，绕过显示服务器
- 可重复性：固定随机种子，确定性时序控制
- 可移植性：抽象硬件差异，支持 Intel/AMD/ARM
- 可扩展性：子测试机制，动态测试生成
- 可调试性：详细日志，失败自动截图

**18. IGT 目录结构：**

| 目录 | 功能 |
|------|------|
| lib/ | 核心库（libigt） |
| tests/ | 测试程序 |
| tests/amdgpu/ | AMDGPU 专用测试 |
| tools/ | 辅助工具 |
| runner/ | 测试执行器 |
| docs/ | 文档 |
| include/ | 头文件 |
| scripts/ | 辅助脚本 |

**19. IGT 核心库组件：**

- igt_core：测试框架核心，提供 igt_main、igt_subtest 等
- igt_aux：辅助函数（时间、随机数、文件操作）
- igt_log：日志系统（igt_info、igt_debug、igt_warn）
- ioctl_wrappers：DRM ioctl 安全包装
- igt_fb：帧缓冲区辅助
- igt_kms：KMS 辅助（CRTC、Connector、Mode）
- igt_gem：GEM 辅助（BO、MMAP、执行列表）

**20. AMDGPU 测试价值：**

- GEM 测试：验证 BO 生命周期、内存管理
- KMS 测试：验证显示模式、页面翻转
- PM 测试：验证电源管理、运行时 PM
- AMDGPU 专用：验证 AMD 特有功能（如 Freesync）

### 第四部分：综合实践题

**综合实践题一参考答案：**

必需依赖：
```bash
sudo apt install build-essential meson ninja-build pkg-config \
    libdrm-dev libpciaccess-dev
```

可选依赖：
```bash
sudo apt install libkmod-dev libprocps-dev libcairo2-dev \
    libwayland-dev wayland-protocols libx11-dev libxrandr-dev
```

编译脚本：
```bash
#!/bin/bash
set -e
git clone https://gitlab.freedesktop.org/drm/igt-gpu-tools.git
cd igt-gpu-tools
meson setup build --prefix=/usr/local
ninja -C build
sudo ninja -C build install
```

验证：
```bash
igt-runner --list-tests
modetest --help
```

**综合实践题二参考答案：**

目录结构图：
```
igt-gpu-tools/
├── lib/          # 核心库
│   ├── igt_core.c    # 测试框架
│   ├── igt_kms.c     # KMS 辅助
│   └── igt_gem.c     # GEM 辅助
├── tests/        # 测试程序
│   ├── kms_*.c       # KMS 测试
│   ├── gem_*.c       # GEM 测试
│   └── amdgpu/       # AMDGPU 测试
├── runner/       # 测试执行器
└── tools/        # 辅助工具
```

设计模式：
- 分层架构：lib/ 提供基础服务，tests/ 使用服务
- 模块化：每个组件职责单一
- 可扩展：通过子测试机制动态添加测试
