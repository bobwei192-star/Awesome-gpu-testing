## 知识详解

### 一、概念原理

#### 1.1 测试计划的核心要素

测试计划（Test Plan）是指导测试活动的核心文档。根据 IEEE 829-2008 标准，测试计划应包含以下核心要素：

```
┌─────────────────────────────────────────────────────────────┐
│                    测试计划核心要素                           │
│                                                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │  测试范围  │  │  测试策略  │  │  测试资源  │  │  测试进度  │   │
│  │ (Scope)  │  │(Strategy)│  │(Resource)│  │(Schedule)│   │
│  ├──────────┤  ├──────────┤  ├──────────┤  ├──────────┤   │
│  │● 测试对象  │  │● 测试层次  │  │● 人员配置  │  │● 里程碑  │   │
│  │● 不测内容  │  │● 测试类型  │  │● 硬件设备  │  │● 交付物  │   │
│  │● 假设条件  │  │● 通过标准  │  │● 预算估算  │  │● 依赖关系 │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

在 GPU 驱动测试领域，测试计划还需要额外考虑：

1. **硬件矩阵覆盖**：需要明确覆盖哪些 GPU 型号、显存配置、PCIe 版本
2. **软件栈组合**：需要覆盖哪些内核版本、Mesa 版本、固件版本组合
3. **工作负载类型**：需要覆盖图形、计算、编解码、显示输出等工作负载
4. **非确定性因素**：如何处理 GPU 测试中特有的时序相关问题

测试范围界定是测试计划的第一步，需要明确"测什么"和"不测什么"。

**边界界定方法：**

```
// test_scope_analyzer.c (概念代码)
// 功能影响范围分析算法

enum ScopeCategory {
    SCOPE_IN,     // 在范围内
    SCOPE_OUT,    // 不在范围内
    SCOPE_TBD,    // 待讨论确定
    SCOPE_COND    // 条件包含
};

struct ScopeCriteria {
    const char *module;         // 模块名称
    bool kernel_driver;         // 是否内核驱动相关
    bool amd_owned;             // 是否 AMD 维护
    bool testing_needed;        // 是否需要专项测试
    bool regression_risk;       // 是否有回归风险
};

ScopeCategory evaluate_scope(struct ScopeCriteria *criteria) {
    if (!criteria->kernel_driver) {
        return SCOPE_OUT;
    }
    if (!criteria->amd_owned) {
        return SCOPE_COND;  // 第三方代码需评估
    }
    if (criteria->regression_risk && criteria->testing_needed) {
        return SCOPE_IN;
    }
    if (criteria->testing_needed) {
        return SCOPE_IN;
    }
    return SCOPE_TBD;
}
```

#### 1.3 测试策略制定

测试策略是测试计划的核心，定义了测试的深度、广度和方法。

**GPU 驱动测试的层次化策略：**

```
测试策略层次结构
Level 0: 烟雾测试 (Smoke Test)
├── 驱动加载/卸载基本功能
├── 基本设备检测
└── 快速通过/失败判定 (~5 min)

Level 1: 基本功能测试 (Basic Functional)
├── 核心 API 功能验证
├── 基本渲染输出
├── 基本编解码功能
└── ~30 min

Level 2: 完整功能测试 (Full Functional)
├── 所有 IP 块功能验证
├── 多配置组合测试
├── 边界条件测试
└── ~4 hours

Level 3: 压力与稳定性测试 (Stress & Stability)
├── 长时间压力测试
├── 热循环测试
├── 并发场景测试
└── ~24 hours

Level 4: 专项测试 (Specialized)
├── 性能基准测试
├── 功耗分析
├── 兼容性矩阵测试
└── ~1 week
```

**硬件资源估算：**

GPU 驱动测试的硬件需求远高于普通软件测试：

```bash
#!/bin/bash
# hardware_resource_estimator.sh
# GPU 驱动测试硬件资源估算工具

# 配置区：需要覆盖的硬件矩阵
GPU_MODELS=(
    "Radeon RX 7900 XTX"     # Navi 31, DCN 3.2
    "Radeon RX 7800 XT"      # Navi 32, DCN 3.1.4
    "Radeon RX 7600"         # Navi 33, DCN 3.1.4
    "Radeon PRO W7900"       # Navi 31, workstation
    "Radeon PRO W5700"       # Navi 10, workstation
    "Ryzen 7 7840HS APU"     # Phoenix, integrated
)

OS_CONFIGS=(
    "Ubuntu 22.04 LTS / kernel 6.5"
    "Ubuntu 24.04 LTS / kernel 6.8"
    "RHEL 9.3 / kernel 5.14"
    "Fedora 40 / kernel 6.9"
)

ESTIMATE_TEST_TYPES=("smoke" "functional" "stress" "perf" "compat")

# 计算最小硬件需求
calculate_min_hardware() {
    local gpu_count=${#GPU_MODELS[@]}
    local os_count=${#OS_CONFIGS[@]}
    
    echo "GPU 驱动测试硬件资源估算"
    echo "=========================================="
    echo "需要覆盖的 GPU 型号: ${gpu_count}"
    echo "需要覆盖的操作系统: ${os_count}"
    echo ""
    
    # 策略 1: 完全物理机部署
    local physical_nodes=$((gpu_count * 2))  # 每个 GPU 型号至少 2 块
    echo "方案 A: 完全物理机"
    echo "  物理节点: ${physical_nodes}"
    echo "  每节点 GPU: 1 块"
    echo "  总计 GPU: ${physical_nodes} 块"
    echo "  预估成本: $((physical_nodes * 5000)) USD"
    echo "  优点: 测试结果最可靠"
    echo "  缺点: 成本高、维护复杂"
    echo ""
    
    # 策略 2: 虚拟化 + 物理混合
    local physical_with_gpu=4  # 关键 GPU 型号用物理机
    local ci_runners=$((gpu_count * 3))
    echo "方案 B: 混合部署"
    echo "  物理机 (含 GPU): ${physical_with_gpu}"
    echo "  CI Runner 节点: ${ci_runners}"
    echo "  虚拟化无 GPU 节点: 4"
    echo "  预估成本: $((physical_with_gpu * 8000 + ci_runners * 2000)) USD"
    echo "  优点: 性价比高"
    echo "  缺点: 需要虚拟化 GPU 透传配置"
    echo ""
    
    # 策略 3: CI Runner Farm
    local runner_count=$((gpu_count * os_count))
    echo "方案 C: CI Runner Farm"
    echo "  GitLab Runner (含 GPU): ${runner_count}"
    echo "  预估成本: $((runner_count * 3000)) USD"
    echo "  优点: 弹性扩展、易于管理"
    echo "  缺点: 初期配置复杂"
}

calculate_min_hardware
```

#### 2.2 测试计划生成脚本

以下脚本帮助自动生成测试计划草稿，基于项目的 Git 变更分析：

```bash
#!/bin/bash
# test_plan_generator.sh
# 基于 Git 变更自动生成测试计划

REPO_DIR="${1:-.}"
OUTPUT_FILE="${2:-test_plan_generated.md}"

cd "${REPO_DIR}" || exit 1

echo "正在分析 Git 仓库变更..."

# 获取当前版本与上一版本的差异
CURRENT_TAG=$(git describe --tags --abbrev=0 2>/dev/null || echo "HEAD")
PREVIOUS_TAG=$(git tag --sort=-creatordate | head -2 | tail -1 2>/dev/null || echo "HEAD~1")

echo "当前版本: ${CURRENT_TAG}"
echo "对比版本: ${PREVIOUS_TAG}"

# 分析变更的文件
CHANGED_FILES=$(git diff --name-only "${PREVIOUS_TAG}".."${CURRENT_TAG}" 2>/dev/null)

# 模块分类统计
declare -A MODULE_CHANGES
for file in $CHANGED_FILES; do
    case "$file" in
        drivers/gpu/drm/amd/amdgpu/*)
            MODULE_CHANGES["amdgpu"]=$((MODULE_CHANGES["amdgpu"] + 1))
            ;;
        drivers/gpu/drm/amd/display/*)
            MODULE_CHANGES["display"]=$((MODULE_CHANGES["display"] + 1))
            ;;
        drivers/gpu/drm/amd/amdkfd/*)
            MODULE_CHANGES["kfd"]=$((MODULE_CHANGES["kfd"] + 1))
            ;;
        drivers/gpu/drm/amd/pm/*)
```

#### 2.3 测试工作量估算模型

准确的测试工作量估算是测试计划的关键部分：

```python
#!/usr/bin/env python3
"""
test_effort_estimator.py
GPU 驱动测试工作量估算工具
基于功能点分析（FPA）和测试点分析（TPA）方法
"""

import math
from enum import Enum
from dataclasses import dataclass, field
from typing import Dict, List, Optional, Tuple


class Complexity(Enum):
    SIMPLE = 1
    MEDIUM = 2
    COMPLEX = 3
    VERY_COMPLEX = 4


@dataclass
class FeatureModule:
    """功能模块定义"""
    name: str
    interfaces: int           # 对外接口数量
    internal_states: int      # 内部状态数量
    config_combinations: int  # 配置组合数量
    complexity: Complexity
    test_priority: int        # 1-5, 1 最高


@dataclass
class TestEffortEstimate:
    """测试工作量估算结果"""
    total_hours: float = 0.0
    breakdown: Dict[str, float] = field(default_factory=dict)
    test_cases: int = 0
    exec_hours: float = 0.0
    analysis_hours: float = 0.0
    report_hours: float = 0.0
    automation_hours: float = 0.0


class GPUTestEffortEstimator:
    """GPU 驱动测试工作量估算器"""
    
    # 基础测试系数（基于历史数据校准）
    BASE_FACTORS = {
        "test_design_per_interface": 2.0,     # 每个接口的测试设计小时数
        "test_execution_per_case": 0.5,       # 每个用例的执行小时数
        "test_analysis_per_case": 0.25,       # 每个用例的结果分析小时数
        "automation_per_case": 1.5,           # 每个用例的自动化小时数
        "environment_setup": 40.0,            # 环境搭建小时数
        "regression_factor": 0.3,             # 回归测试系数
        "documentation_factor": 0.15,         # 文档工作量系数
    }
    
    # GPU 驱动特有系数
    GPU_SPECIFIC_FACTORS = {
        "hardware_variety": 0.2,       # 硬件多样性附加
        "non_determinism": 0.3,        # 非确定性附加
        "cross_module_testing": 0.25,  # 跨模块测试附加
        "kernel_compat": 0.15,         # 内核兼容性附加
    }
    
    def __init__(self):
        self.modules: List[FeatureModule] = []
        self.factors = dict(self.BASE_FACTORS)
    
    def add_module(self, module: FeatureModule):
        """添加功能模块"""
        self.modules.append(module)
    
    def estimate_module_effort(self, module: FeatureModule) -> TestEffortEstimate:
        """估算单个模块的测试工作量"""
        est = TestEffortEstimate()
        
        # 1. 估算测试用例数
        base_cases = (
            module.interfaces * 3 +           # 每个接口至少 3 个用例
            module.internal_states * 2 +      # 每个状态至少 2 个用例
            min(module.config_combinations, 10)  # 配置组合（上限 10）
        )
        
        # 复杂度调整
        complexity_multiplier = {
            Complexity.SIMPLE: 1.0,
            Complexity.MEDIUM: 1.5,
            Complexity.COMPLEX: 2.5,
            Complexity.VERY_COMPLEX: 4.0,
        }
        
        adjusted_cases = int(base_cases * complexity_multiplier[module.complexity])
        est.test_cases = adjusted_cases
        
        # 2. 估算设计时间
        design_hours = (
            module.interfaces * self.factors["test_design_per_interface"] *
            complexity_multiplier[module.complexity]
        )
        
        # 3. 估算执行时间
        exec_hours = adjusted_cases * self.factors["test_execution_per_case"]
        
        # 4. GPU 特有附加
        gpu_overhead = 1.0
        for factor_name, factor_value in self.GPU_SPECIFIC_FACTORS.items():
            gpu_overhead += factor_value
        
        exec_hours *= gpu_overhead
        
        # 5. 分析时间
        analysis_hours = adjusted_cases * self.factors["test_analysis_per_case"]
        
        # 6. 文档时间
        doc_hours = (design_hours + exec_hours + analysis_hours) * \
                    self.factors["documentation_factor"]
        
        # 7. 自动化时间（一次性投入，均摊到模块）
        automation_hours = (
            adjusted_cases * self.factors["automation_per_case"] *
            (1.0 if module.test_priority <= 2 else 0.5)
        )
        
        est.design_hours = design_hours
        est.exec_hours = exec_hours
        est.analysis_hours = analysis_hours
        est.report_hours = doc_hours
        est.automation_hours = automation_hours / max(len(self.modules), 1)
        est.total_hours = design_hours + exec_hours + analysis_hours + \
                         est.report_hours + est.automation_hours
        
        return est
    
    def estimate_total_effort(self) -> Dict[str, TestEffortEstimate]:
        """估算所有模块的总工作量"""
        results = {}
        total_estimate = TestEffortEstimate()
        
        for module in self.modules:
            est = self.estimate_module_effort(module)
            results[module.name] = est
            
            total_estimate.total_hours += est.total_hours
            total_estimate.test_cases += est.test_cases
            total_estimate.exec_hours += est.exec_hours
            total_estimate.analysis_hours += est.analysis_hours
            total_estimate.report_hours += est.report_hours
            total_estimate.automation_hours += est.automation_hours
        
        # 添加环境搭建时间
        total_estimate.total_hours += self.factors["environment_setup"]
        
        # 添加回归测试时间
        regression_hours = total_estimate.exec_hours * self.factors["regression_factor"]
        total_estimate.total_hours += regression_hours
        
        results["TOTAL"] = total_estimate
        results["regression_hours"] = TestEffortEstimate(total_hours=regression_hours)
        
        return results
    
    def generate_report(self, estimates: Dict[str, TestEffortEstimate]) -> str:
        """生成估算报告"""
        lines = [
            "# GPU 驱动测试工作量估算报告",
            "",
            f"**估算日期**: {__import__('datetime').datetime.now().strftime('%Y-%m-%d')}",
            f"**模块数量**: {len(self.modules)}",
            "",
            "---",
            "",
            "## 模块级估算",
            "",
            "| 模块 | 用例数 | 设计(h) | 执行(h) | 分析(h) | 报告(h) | 自动化(h) | 总计(h) |",
            "|------|--------|---------|---------|---------|---------|-----------|---------|",
        ]
        
        for module_name, est in estimates.items():
            if module_name == "TOTAL" or module_name == "regression_hours":
                continue
            lines.append(
                f"| {module_name} | {est.test_cases} | "
                f"{est.design_hours:.1f} | {est.exec_hours:.1f} | "
                f"{est.analysis_hours:.1f} | {est.report_hours:.1f} | "
                f"{est.automation_hours:.1f} | {est.total_hours:.1f} |"
            )
        
        total = estimates.get("TOTAL", TestEffortEstimate())
        reg = estimates.get("regression_hours", TestEffortEstimate())
        
        lines.extend([
            "",
            "## 汇总",
            "",
            f"| 项目 | 小时数 | 人天 (8h) |",
            f"|------|--------|-----------|",
            f"| 测试设计 | {total.design_hours:.1f} | {total.design_hours/8:.1f} |",
            f"| 测试执行 | {total.exec_hours:.1f} | {total.exec_hours/8:.1f} |",
            f"| 结果分析 | {total.analysis_hours:.1f} | {total.analysis_hours/8:.1f} |",
            f"| 测试报告 | {total.report_hours:.1f} | {total.report_hours/8:.1f} |",
            f"| 测试自动化 | {total.automation_hours:.1f} | {total.automation_hours/8:.1f} |",
            f"| 回归测试 | {reg.total_hours:.1f} | {reg.total_hours/8:.1f} |",
            f"| 环境搭建 | {self.factors['environment_setup']:.1f} | {self.factors['environment_setup']/8:.1f} |",
            f"| **总计** | **{total.total_hours:.1f}** | **{total.total_hours/8:.1f}** |",
            "",
            f"**总测试用例数**: {total.test_cases}",
            f"**建议团队规模**: {max(1, round(total.total_hours / 160))} 人（月）",
        ])
        
        return '\n'.join(lines)


if __name__ == "__main__":
    estimator = GPUTestEffortEstimator()
    
    # 示例模块定义
    estimator.add_module(FeatureModule(
        name="amdgpu_core",
        interfaces=15,
        internal_states=8,
        config_combinations=24,
        complexity=Complexity.VERY_COMPLEX,
        test_priority=1,
    ))
    estimator.add_module(FeatureModule(
        name="amdgpu_display",
        interfaces=12,
        internal_states=10,
        config_combinations=48,
        complexity=Complexity.COMPLEX,
        test_priority=1,
    ))
    estimator.add_module(FeatureModule(
        name="amdgpu_power",
        interfaces=8,
        internal_states=6,
        config_combinations=12,
        complexity=Complexity.MEDIUM,
        test_priority=2,
    ))
    
    estimates = estimator.estimate_total_effort()
    report = estimator.generate_report(estimates)
    print(report)
```

- 第 5 周验证通过
- 整体发布仅延迟 3 天

**经验教训：**

1. **预留缓冲时间**：测试计划应包含 15-20% 的缓冲时间用于处理意外问题
2. **动态资源重分配**：建立资源重分配机制，快速响应发现的严重问题
3. **计划版本控制**：每次计划调整都应记录变更原因和影响
